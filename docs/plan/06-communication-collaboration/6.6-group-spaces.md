## 1. Problem Statement

Enrollment groups were introduced (`server/migrations/066_enrollment_groups.sql`) but have no dedicated workspace. Groups cannot have a private feed, cannot share files with group members only, and cannot work on group-only assignments within the LMS. Instructors who assign group projects must route everything through general course channels, creating visibility confusion: students see other groups' work and there is no clear group home. This is a table-stakes collaborative LMS feature referenced in the FEATURES.md backlog (§5) and still absent.

## 2. Goals

- Provide each enrollment group with a private workspace: a group-only discussion/feed channel, a group file storage area, and group assignment support.
- Surface group spaces in the course navigation as a dedicated "Groups" tab.
- Allow group members to see only their own group's content.
- Allow instructors to monitor all groups' spaces from the instructor view.
- Integrate with collaborative documents (6.5) so groups can co-author documents in their space.

## 3. Non-Goals

- Group video calls — use the virtual classroom feature (6.4) with group-scoped meeting.
- Cross-course groups — groups are scoped to a single course.
- Group leader/role differentiation within a group — V1 all members are equal.
- Automated group formation (random, diversity-balanced) — deferred.

## 4. Personas & User Stories

- **As a student in Group 3**, I want a private feed channel for my group so that we can coordinate without the whole class seeing our messages.
- **As a group member**, I want to upload files to a group-only folder so that only my teammates can access them.
- **As a student**, I want a group-only assignment submission area so that only one submission is needed from the whole group and all members receive the grade.
- **As an instructor**, I want to view all groups' workspaces from a unified panel so that I can check progress without joining each group individually.
- **As a student**, I want to see my group's collaborative document alongside our group feed so that everything is in one place.

## 5. Functional Requirements

- **FR-1.** The system MUST create a `group_spaces` record for each enrollment group, with a private discussion feed channel (`group_feed`), a file area, and an assignments list.
- **FR-2.** The system MUST enforce that only members of a group (or the course instructor) can read or post in a group feed channel.
- **FR-3.** The system MUST support group file upload: files uploaded to a group space are stored in `course_files` with a `group_id` owner and restricted access.
- **FR-4.** The system MUST support group assignments: a group assignment has one submission shared by all group members; all members receive the same grade.
- **FR-5.** The system MUST provide an instructor "all groups" dashboard showing each group's activity count and last post timestamp.
- **FR-6.** The system SHOULD surface group-scoped collaborative documents (6.5) within the group workspace.
- **FR-7.** The system SHOULD notify group members (via 6.2/6.3) when a new message is posted in their group feed.
- **FR-8.** The system MAY allow groups to name themselves (instructor approval optional).

## 6. Non-Functional Requirements

- **Performance** — group feed loads in ≤200 ms for a group with 50 messages; instructor "all groups" dashboard ≤500 ms for 30 groups.
- **Security** — group membership enforced on every group-space request in `authz.rs`; instructors have read access to all groups in their course; org admins have read access to all groups in the org.
- **Privacy & Compliance** — group feed posts are educational records; instructor can see all group content (FERPA school official exception).
- **Accessibility** — group feed uses the same accessible feed component as the course feed; group tabs keyboard navigable.
- **Scalability** — a course may have ≤100 groups; group feed paginated; file list paginated.
- **Reliability** — group assignment submission is transactional — all group member submission records updated atomically.
- **Observability** — log `group_id`, `course_id` on all group-space requests.
- **Maintainability** — group space routes in `server/src/routes/course_feed.rs` (extend existing feed) and `server/src/routes/courses.rs`.
- **Internationalization** — UI labels externalized.
- **Backward compatibility** — existing enrollment_groups rows get a `group_spaces` record created by backfill migration; no breaking changes.

## 7. Acceptance Criteria

- **AC-1.** *Given* a student in Group 3, *When* they navigate to the Groups tab and open Group 3, *Then* they see only Group 3's feed, files, and assignments.
- **AC-2.** *Given* a student in Group 3, *When* they attempt to access Group 5's feed via direct URL, *Then* HTTP 403 is returned.
- **AC-3.** *Given* a group assignment, *When* one group member submits, *Then* all group members have a submission record created and all receive the same grade when graded.
- **AC-4.** *Given* an instructor, *When* they view the "All Groups" panel, *Then* all 10 groups are listed with message count and last-activity timestamp.
- **AC-5.** *Given* a group member posts a message, *When* another group member views the feed, *Then* the message is visible; when a non-member views the course, the message is not visible.
- **AC-6.** *Given* a group workspace, *When* a group member uploads a file, *Then* it appears in the group files area accessible only to group members and the instructor.

## 8. Data Model

```sql
-- server/migrations/088_group_spaces.sql
CREATE TABLE group_spaces (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  group_id     UUID NOT NULL UNIQUE REFERENCES enrollment_groups(id) ON DELETE CASCADE,
  course_id    UUID NOT NULL REFERENCES courses(id),
  display_name TEXT,
  created_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Extend existing course_files to support group ownership
ALTER TABLE course_files ADD COLUMN group_id UUID REFERENCES enrollment_groups(id);
CREATE INDEX idx_course_files_group ON course_files(group_id);

-- Group assignments: link assignment to group space
ALTER TABLE module_assignments ADD COLUMN is_group_assignment BOOLEAN NOT NULL DEFAULT false;

-- Group feed posts extend the existing feed model (course_feed table)
-- Add group_id filter column
ALTER TABLE course_feed ADD COLUMN group_id UUID REFERENCES enrollment_groups(id);
CREATE INDEX idx_course_feed_group ON course_feed(group_id);

-- Group submission (one canonical submission per group)
CREATE TABLE group_submissions (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  assignment_id UUID NOT NULL REFERENCES module_assignments(id),
  group_id      UUID NOT NULL REFERENCES enrollment_groups(id),
  submitted_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (assignment_id, group_id)
);
```

## 10. UI / UX

New tab in course navigation: "Groups"
- Student view: shows their group(s) only; opens group workspace with tabs: Feed, Files, Assignments, Docs.
- Instructor view: dropdown to select group OR "All Groups" overview table.

Group workspace layout mirrors course layout at small scale.
Group assignment submit page: shows group members, single submit button, "Submitting on behalf of your group."
Empty state: "Your group hasn't posted anything yet. Start the conversation!"
Mobile: group tabs become a bottom sheet; files list is card view.
Accessibility: group tab panel uses `role="tabpanel"`; group feed reuses accessible feed component.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Group membership check adds query per request | M | M | Cache group membership in request context |
| Group assignment grade sync to all members fails partway | M | H | Transactional grade creation for all group members |
| Instructor accidentally posts in one group feed instead of course feed | M | M | Clear "Group X" label on compose area; confirmation for group-scoped post |

## 15. Rollout Plan

- Feature flag: `group_spaces_enabled` (default: `false`).
- Sequence: DB migration + backfill → group-scoped feed API → files → group assignments → UI → flag.
- Dogfood: create 3 groups in internal test course; test feed and file sharing.
- GA criteria: group isolation test passes; group assignment submission grades all members.
- Rollback: disable flag; schema additions are non-breaking.

## 16. Test Plan
