## 1. Problem Statement

The RAG-based student notebook (`server/src/services/student_notebook_rag_ai.rs`) is a reactive Q&A tool — the student asks a question and gets an answer. There is no persistent conversational AI tutor that remembers the student's concept mastery, knows the current assignment context, and proactively surfaces hints or explanations. For Lextures to credibly own the "AI-native LMS" positioning, the AI tutor must be aware of what the student is working on, what they already know, and where they are confused — not just the contents of a document.

## 2. Goals

- Build a persistent, context-aware AI tutor accessible from any course page via a side-panel.
- Ground the tutor's responses in the current course materials, the learner's concept mastery state (1.1), and recent activity.
- Persist multi-turn conversation history per student per course.
- Enforce PII redaction before any prompt leaves the system (19.11).
- Provide per-tenant AI provider configuration and opt-out controls (19.10).

## 3. Non-Goals

- Voice interaction with the tutor — deferred to 19.6.
- Tutor grading or grade-modification — the tutor suggests only; grading is human-only.
- Public/marketing-facing AI chat — tutor is authenticated users only.
- Training or fine-tuning a model on student data — using pre-trained LLMs via OpenRouter only.
- Unlimited-token conversations at no cost — token budgets per student enforced (19.14).

## 4. Personas & User Stories

- **As a student working on a quiz**, I want to ask "I'm stuck on Question 3 — what concept does this question test?" and receive a hint that guides without giving the answer away.
- **As a self-learner**, I want the AI tutor to remember that I struggled with derivatives last session and proactively offer a review exercise when I start the integration module.
- **As an instructor**, I want to see an aggregate view of what questions students most frequently asked the AI tutor so that I can adjust my lesson plan.
- **As a district admin**, I want to confirm which AI model is used for student tutoring and opt out specific courses from AI interaction so that I meet our district's AI governance policy.

## 5. Functional Requirements

- **FR-1.** The system MUST provide a persistent `tutor_conversations` table: `id`, `student_id`, `course_id`, `context_item_id` (nullable — current module item), `messages` (JSONB array of role/content pairs), `created_at`, `updated_at`.
- **FR-2.** The system MUST construct each tutor prompt with: (a) system context (course name, current item, student mastery summary from 1.1), (b) conversation history (last 20 turns), (c) retrieved course material chunks (RAG from `student_notebook_rag_ai.rs`), (d) PII-redacted student message.
- **FR-3.** The system MUST enforce PII redaction (19.11) on every user message before it is sent to the LLM provider.
- **FR-4.** The system MUST respect per-org AI governance settings (19.10): if AI tutoring is disabled for a course or org, the side-panel must not be rendered.
- **FR-5.** The system MUST enforce a per-student monthly token budget (configurable, default 50,000 tokens) and display remaining budget; reject requests when budget exhausted (19.14).
- **FR-6.** The system MUST stream the tutor response (Server-Sent Events) so students see text appear in real time.
- **FR-7.** The system SHOULD surface a "Tutor Insights" panel for instructors showing the top 10 most-asked questions in their course over the past week.
- **FR-8.** The system SHOULD include citation links in tutor responses pointing to the specific course material chunk used (19.12).
- **FR-9.** The system MAY proactively suggest a review exercise (linked to concept graph) when the student has been idle on a low-mastery concept module.

## 6. Non-Functional Requirements

- **Performance** — first token streaming must begin within 2 s of request; full response for typical tutor message ≤15 s.
- **Security** — tutor endpoint requires authenticated student; conversation history is private to the student and their instructors; no cross-student data leakage in prompts.
- **Privacy & Compliance** — FERPA: tutor conversation is educational record; exportable. COPPA: under-13 students in K-12 require additional consent before AI interaction; check `account_type` and org setting. PII redaction applied to all outbound prompts. AI usage disclosure shown to students.
- **Accessibility** — tutor side-panel keyboard accessible; ARIA live region announces new streamed text; close button accessible.
- **Scalability** — LLM calls are slow and expensive; use background streaming with token budget enforcement; do not block other requests.
- **Reliability** — LLM provider timeout (30 s) triggers graceful fallback message; conversation history is preserved even if a request fails.
- **Observability** — log `conversation_id`, `tokens_used`, `provider`, `latency_ms`; alert on token budget exhaustion rate; track tutor usage by course.
- **Maintainability** — tutor service in `server/src/services/ai/tutor.rs`; reuse RAG retrieval from `student_notebook_rag_ai.rs`; PII redaction in `server/src/services/ai/pii_redactor.rs`.
- **Internationalization** — tutor system prompt can be provided in multiple languages (default English); student messages in any language supported by the LLM.
- **Backward compatibility** — RAG notebook remains available unchanged; tutor is an additive side-panel.

## 7. Acceptance Criteria

- **AC-1.** *Given* a student is on a quiz page, *When* they open the tutor and ask "What is this question testing?", *Then* the tutor responds within 15 s with a hint grounded in course materials (citing the source section).
- **AC-2.** *Given* a student with low mastery on "derivatives" (from 1.1), *When* they open the tutor on the integration module, *Then* the system prompt includes a mastery context note about derivatives, and the tutor proactively acknowledges the prerequisite gap.
- **AC-3.** *Given* a student's monthly token budget is exhausted, *When* they send a message, *Then* the tutor responds with "You have reached your monthly AI interaction limit. Your budget resets on [date]." and no LLM call is made.
- **AC-4.** *Given* a message containing a student's full name in the text, *When* the PII redactor processes it, *Then* the name is replaced with `[REDACTED]` in the prompt sent to the provider.
- **AC-5.** *Given* AI tutoring is disabled for a course by the org admin, *When* a student in that course opens the course page, *Then* the tutor side-panel icon is not displayed.
- **AC-6.** *Given* an instructor views the Tutor Insights panel, *When* 50 students have asked questions this week, *Then* the top 10 unique question themes are shown with frequency counts.

## 8. Data Model

```sql
-- server/migrations/090_tutor_conversations.sql
CREATE TABLE tutor_conversations (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id      UUID NOT NULL REFERENCES users(id),
  course_id       UUID NOT NULL REFERENCES courses(id),
  context_item_id UUID REFERENCES course_structure_items(id),
  messages        JSONB NOT NULL DEFAULT '[]',
  tokens_used     INTEGER NOT NULL DEFAULT 0,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (student_id, course_id, context_item_id)
);

CREATE INDEX idx_tutor_conv_student ON tutor_conversations(student_id, course_id);

CREATE TABLE student_token_budgets (
  student_id    UUID NOT NULL REFERENCES users(id),
  org_id        UUID NOT NULL REFERENCES organizations(id),
  period_month  DATE NOT NULL, -- first day of month
  tokens_used   INTEGER NOT NULL DEFAULT 0,
  token_limit   INTEGER NOT NULL DEFAULT 50000,
  PRIMARY KEY (student_id, period_month)
);
```

## 11. AI / ML Considerations

- **Models used:** OpenRouter-routed LLM (GPT-4o or Claude 3.5 Sonnet); per-tenant model choice via 19.10.
- **Prompt design:** system prompt includes course context, learner mastery summary (from 1.1 when available), RAG-retrieved chunks (up to 2,000 tokens). Conversation history truncated to last 20 turns.
- **Eval metric:** citation accuracy (% of responses with valid source citations); helpfulness rating (thumbs up/down from students).
- **Fallback:** on LLM timeout or error, return "I'm having trouble right now. Please try again in a moment." — conversation history preserved.
- **PII redaction:** `pii_redactor.rs` strips emails, names, phone numbers from student messages before sending.
- **Cost budget:** default 50,000 tokens/student/month; configurable per org; tracked in `student_token_budgets`.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| LLM gives incorrect answer, student is misled | H | M | Add disclaimer; cite sources; instructor Insights reveals common errors |
| Token budget exceeded causing poor UX | M | M | Warn at 80% usage; allow org admin to increase limit |
| PII redaction misses a name pattern | M | H | Conservative regex + NER model; log all redacted fields for audit |
| COPPA violation: AI processes under-13 student data | M | H | Check `account_type` and org COPPA flag before enabling tutor |

## 15. Rollout Plan

- Feature flag: `ai_tutor_enabled` (default: `false`).
- Sequence: PII redactor → conversation persistence → RAG integration → SSE streaming → token budgets → UI panel → insights → flag.
- Dogfood: enable for internal test students in dev org.
- GA criteria: citation accuracy >80% on test prompt set; PII redaction test passes; streaming confirmed.
- Rollback: disable flag; conversation data preserved.

## 16. Test Plan

- **Unit** — prompt construction; PII redaction patterns; token budget enforcement.
- **Integration** — SSE stream from LLM mock; conversation history persistence; RAG retrieval.
- **End-to-end** — Playwright: student opens tutor, asks question, receives streamed response with citation.
- **Security** — cross-student conversation isolation; COPPA flag blocks tutor for under-13.
- **Accessibility** — tutor panel keyboard accessible; live region announces new messages.
- **Performance / load** — first token ≤2 s under 50 concurrent sessions.
- **Manual exploratory** — budget exhaustion fallback; LLM timeout fallback; AI governance disable.
