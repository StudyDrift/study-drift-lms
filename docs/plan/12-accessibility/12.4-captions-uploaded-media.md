## 1. Problem Statement

Any video content uploaded to Lextures — lecture recordings, assignment responses, instructional clips — is served as raw MP4 with no captions, no transcript, and no audio description. This violates WCAG 2.1 AA SC 1.2.2 (Captions Pre-recorded) and SC 1.2.3 (Audio Description), which are absolute requirements under ADA Title II (US K-12 public schools) and Section 504. The US Department of Education's Office for Civil Rights has issued consent decrees against multiple ed-tech platforms for exactly this gap. Until auto-captioning and manual-caption editing exist, Lextures cannot legally be deployed in any US public school or federally-funded university. This plan is the companion accessibility deliverable to §8.4 (Auto-captioning & transcripts) and focuses on the UI, workflow, and WCAG conformance layer on top of the captioning pipeline.

## 2. Goals

- All video content uploaded to Lextures must have captions available before students can view it.
- Provide AI-generated captions (via the §8.4 transcription pipeline) with an instructor review-and-edit UI.
- Enable students to toggle captions on/off and control caption styling (size, color, position).
- Support WebVTT and SRT caption file import/export.
- Emit WCAG 2.1 AA SC 1.2.2 conformance for all platform-hosted video.

## 3. Non-Goals

- Audio descriptions (SC 1.2.5) — deferred to a follow-on sprint.
- Live captioning for real-time video (§6.4 virtual classroom handles that separately).
- Caption generation for audio-only content (§12.8 text-to-speech covers read-aloud; audio files without video are a separate case).
- Automatic captioning for third-party embedded YouTube/Vimeo links (those services provide their own captions).

## 4. Personas & User Stories

- **As a Deaf student**, I want captions to appear automatically on all lecture videos so that I can access the same content as hearing peers.
- **As an instructor**, I want to upload a video and have AI-generated captions ready within minutes, with a simple editor to correct errors, so that I do not need specialized captioning software.
- **As a student with auditory processing difficulty**, I want to download the transcript as a text file so that I can read the material at my own pace.
- **As a district accessibility coordinator**, I want to generate a report showing which courses have uncaptioned videos so that I can enforce our ADA compliance policy.
- **As an instructor**, I want to import an SRT file I received from a captioning service so that I can use professional captions on my videos.

## 5. Functional Requirements

- **FR-1.** The system MUST automatically trigger a captioning job when a video file is successfully uploaded and transcoded (cross-ref §8.3, §8.4), using a background job queue (cross-ref §17.3) (WCAG SC 1.2.2).
- **FR-2.** Instructors MUST be able to edit AI-generated captions via an inline WebVTT editor in the course content editor (`clients/web/src/components/editor/block-editor/`), with cue-level timestamp and text editing.
- **FR-3.** Instructors MUST be able to import captions by uploading a `.vtt` or `.srt` file; the system converts SRT to WebVTT internally.
- **FR-4.** Instructors MUST be able to export captions as `.vtt` or `.srt` from the video media settings panel.
- **FR-5.** The video player MUST display captions using the HTML5 `<track kind="captions">` element, ensuring native browser and AT integration.
- **FR-6.** Students MUST be able to toggle captions on/off per video without page reload, and their preference MUST be persisted per user session.
- **FR-7.** Students MUST be able to adjust caption font size (3 levels), foreground/background color (minimum 3 presets including high-contrast), and position (top/bottom) via a caption settings panel.
- **FR-8.** The system MUST block video publication if no captions are present and the course is configured for mandatory captioning (`courses.require_captions = true`). An instructor override with "caption pending" state SHOULD be available.
- **FR-9.** An admin dashboard MUST list all uncaptioned videos across courses (compliance report) with links to each video's caption editor.

## 6. Non-Functional Requirements

- **Performance** — AI caption generation must complete within 5 minutes for a 60-minute video (p95). Caption track files must be served from CDN (cross-ref §8.1) with < 200 ms TTFB.
- **Security** — Caption files inherit the same access control as their parent video; unauthorized users cannot download raw VTT. Caption edit is restricted to the course instructor or admin (authz in `server/src/authz.rs`).
- **Privacy & Compliance** — WCAG 2.1 AA SC 1.2.2 (Captions Pre-recorded), SC 1.2.3 (Audio Description or Media Alternative). ADA Title II. Section 504. FERPA: caption content may contain PII if the video is a student response; caption files must be subject to the same FERPA access controls as the video (cross-ref §10.1). AI transcription must not send student video frames to third-party services without COPPA/FERPA consent (cross-ref §19.11 PII redaction proxy).
- **Accessibility** — The caption editor UI itself must be keyboard-navigable (SC 2.1.1) and screen-reader accessible (SC 4.1.2). Caption text must meet contrast requirements when rendered over video (SC 1.4.3 — minimum 4.5:1 against video background; high-contrast preset required). Caption settings panel must be operable by keyboard and announce setting changes via live region.
- **Scalability** — Captioning job queue must handle concurrent submissions; max queue depth alarm at 100 jobs.
- **Reliability** — If AI captioning fails, the video remains accessible with an instructor notification; the system must not block content delivery on caption failure. Retry logic: 3 attempts with exponential backoff.
- **Observability** — Metrics: `captions_generated_total`, `captions_edit_rate`, `captioning_job_duration_seconds`. Alerts on job failure rate > 5%.
- **Maintainability** — WebVTT parsing/serialization in a dedicated `server/src/services/captions.rs` module.
- **Internationalization** — Caption text is language-specific; the `track` element's `srclang` attribute must be set to the detected or declared language. UI strings for caption controls externalized via §11.1 i18n.
- **Backward compatibility** — Existing videos without captions are not deleted; they enter a "caption pending" state and are flagged in the compliance report.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor uploads a 10-minute MP4 lecture, *When* the transcoding pipeline completes (cross-ref §8.3), *Then* a captioning job is automatically queued and AI-generated captions appear in the caption editor within 5 minutes.
- **AC-2.** *Given* an instructor edits a caption cue's text and saves, *When* a student opens the video, *Then* the corrected caption text appears in the video player at the correct timestamp.
- **AC-3.** *Given* a VoiceOver user on Safari opens the video player, *When* captions are enabled, *Then* VoiceOver announces caption state changes and the caption settings button is reachable and labeled "Caption settings".
- **AC-4.** *Given* a student enables high-contrast captions (white text on black background), *When* the video plays, *Then* the caption text achieves at least 7:1 contrast ratio (AAA) against the caption background, and the setting persists for subsequent videos.
- **AC-5.** *Given* an admin views the compliance report, *When* there are 3 uncaptioned videos across courses, *Then* those 3 videos are listed with links to their caption editors, and the count matches the server-side query.
- **AC-6.** *Given* a JAWS user on the caption editor, *When* they navigate caption cues via Tab, *Then* JAWS reads timestamp and text for each cue, and editing the text field announces character count changes.
- **AC-7.** *Given* a course has `require_captions = true`, *When* an instructor attempts to publish a video module item without captions, *Then* the system shows an error "This course requires captions. Please add captions before publishing." and prevents publication.

## 8. Data Model

```
-- server/migrations/073_video_captions.sql
CREATE TABLE video_captions (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    media_id    UUID NOT NULL REFERENCES course_files(id) ON DELETE CASCADE,
    language    TEXT NOT NULL DEFAULT 'en',
    status      TEXT NOT NULL DEFAULT 'pending', -- pending | processing | ready | failed
    source      TEXT NOT NULL DEFAULT 'ai',      -- ai | import | manual
    vtt_url     TEXT,                            -- CDN path to WebVTT file
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX video_captions_media_id_idx ON video_captions(media_id);

ALTER TABLE courses ADD COLUMN require_captions BOOLEAN NOT NULL DEFAULT false;
```

Migration: `server/migrations/073_video_captions.sql`. Backfill: all existing `course_files` rows with `content_type` like `video/*` get a `video_captions` row with `status = 'pending'`.

## 9. API Surface

- `GET /api/media/:id/captions` — list captions for a media item; returns `[{id, language, status, vtt_url}]`.
- `POST /api/media/:id/captions` — trigger captioning job or import caption file (multipart); instructor auth.
- `PATCH /api/captions/:id` — update VTT content (save edited cues); instructor auth.
- `DELETE /api/captions/:id` — delete caption track; instructor auth.
- `GET /api/admin/captions/compliance` — list uncaptioned videos (admin auth).
- WebSocket event: `captions.ready { media_id, caption_id }` — pushed to instructor when job completes.

## 10. UI / UX

- **Modified components:** `clients/web/src/components/editor/block-editor/CourseAwareTipTapImage.tsx` — extend to video blocks with caption badge indicator.
- **New components:** `clients/web/src/components/media/CaptionEditor.tsx`, `clients/web/src/components/media/VideoPlayer.tsx` (HTML5-based with `<track>` support), `clients/web/src/components/media/CaptionSettings.tsx`.

- **User flows:** 1. Upload video → captioning auto-starts → instructor sees "Captions processing" badge → receives notification → opens caption editor → edits cues → saves → publishes. 2. Student opens video → captions auto-on if user preference set → adjusts style → downloads transcript.
- **Accessibility annotations:** Caption toggle button has `aria-pressed`, caption settings panel is `role="dialog"`.

## 11. AI / ML Considerations

- Model: Whisper (OpenAI) or equivalent via the §8.4 captioning pipeline; invoked asynchronously via background job queue (§17.3).
- PII concern: Student video responses may contain PII; invoke §19.11 PII redaction proxy before sending to AI service, or use on-premise Whisper model.
- Fallback: If AI fails after 3 retries, mark status `failed` and notify instructor with manual upload option.
- Cost budget: Whisper API costs approximately $0.006/minute; budget alert at $50/month per tenant.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| §8.3 transcoding not ready in time | H | H | Build caption import (SRT/VTT upload) as independent path that does not depend on transcoding |
| Whisper accuracy too low for technical/STEM content | M | M | Allow instructor to upload professional captions as override; track edit rate as quality signal |
| Large caption files slow video load time | L | L | Serve VTT from CDN; compress with gzip |
| FERPA violation if AI service stores student video | H | H | Use ephemeral API with no data retention; add to DPA template (§10.5) |

## 15. Rollout Plan

- Feature flag: `ff_video_captions` (default `false`).
- Sequencing: DB migration → storage → job queue wiring → AI captioning service → caption editor UI → player integration → flip flag.
- Dogfood: Internal team uploads test videos; captions reviewed on staging.
- GA: All 7 ACs pass; compliance report functional; legal review of AI service DPA.
- Rollback: Flag off; existing videos unaffected; caption rows remain but UI hidden.
