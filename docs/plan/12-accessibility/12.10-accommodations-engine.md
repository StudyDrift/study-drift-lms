## 1. Problem Statement

The Individuals with Disabilities Education Act (IDEA) and Section 504 of the Rehabilitation Act legally require K-12 schools to provide students with disabilities equal access to educational materials through documented accommodation plans (IEPs and 504 plans). Lextures has no mechanism to store, enforce, or track student accommodations: no extended-time overrides on quizzes (§2.11), no automatic activation of text-to-speech or dyslexia fonts for individual students, no alternate-format content delivery, and no compliance audit trail. Every K-12 district evaluating Lextures will cite this as a hard blocker. This plan delivers a server-enforced accommodations engine that links student profiles to a structured accommodation set, automatically applies those accommodations on quiz delivery, content presentation, and assignment workflow, and generates a per-student accommodation audit log required for IEP compliance documentation.

## 2. Goals

- Allow special education coordinators and instructors to define structured accommodation plans per student.
- Automatically enforce time extensions on quizzes for students with documented time accommodation.
- Automatically activate reading preferences (font, TTS, STT, high-contrast) for students with documented display accommodations.
- Allow instructors to assign extra quiz attempts for students with documented attempts accommodations.
- Generate a compliance audit log of every accommodation applied per student per assessment.
- Respect FERPA access controls: accommodation data visible only to the student, their instructors, and admin/SPED coordinators.

## 3. Non-Goals

- IEP document storage (Medicaid-billing IEP management software) — Lextures stores the accommodation settings derived from an IEP, not the IEP document itself.
- Automatic IEP import from SIS (§13.7 SIS integration owns that data pipeline).
- 504 hearing workflow or legal dispute management.
- Accommodations for higher-ed disability services (§14.16 owns that).

## 4. Personas & User Stories

- **As a special education coordinator**, I want to enter a student's documented accommodations (extended time 1.5×, text-to-speech, large print) so that those supports are automatically applied whenever the student takes an assessment.
- **As an instructor**, I want to see which of my students have active accommodations so that I can be aware of their needs without seeing protected IEP details.
- **As a student with a 504 plan**, I want my extended-time accommodation applied automatically on every quiz without having to request it each time so that I can focus on the assessment itself.
- **As a parent**, I want to confirm my child's accommodations are active in the LMS so that I can advocate for them effectively in IEP meetings.
- **As a district compliance officer**, I want an audit report showing which accommodations were applied on which assessments so that I can demonstrate IDEA compliance during monitoring.

## 5. Functional Requirements

- **FR-1.** The system MUST provide a per-student accommodations profile, editable only by users with `sped_coordinator` or `admin` role, containing a structured set of accommodation types (IDEA/504 requirement).
- **FR-2.** The accommodations profile MUST support at minimum: (a) extended time multiplier (e.g., 1.5×, 2×) applied to all quiz time limits, (b) extra attempts (override max-attempts on all quizzes), (c) text-to-speech auto-enabled, (d) speech-to-text auto-enabled, (e) dyslexia font + spacing preset, (f) high-contrast theme, (g) reduced-motion, (h) separate testing environment flag (informational, no technical enforcement in v1).
- **FR-3.** When a student with an active time-extension accommodation begins a quiz attempt, the system MUST multiply the base `time_limit_seconds` (§2.7) by the student's time multiplier BEFORE the attempt timer is initialized; the multiplied limit MUST be stored on the attempt record and server-enforced.
- **FR-4.** When a student with an active reading-display accommodation loads a content page or quiz, the system MUST apply the accommodation preferences BEFORE first paint, overriding the student's personal preferences if the accommodation specifies a value.
- **FR-5.** Accommodation data MUST be scoped by organization (§5.1) and school (§5.2); SPED coordinators can only manage accommodations for students in their school.
- **FR-6.** The system MUST generate an immutable accommodation application event in the audit log (`accommodation_audit_log` table) each time an accommodation is applied to a quiz attempt or content view, recording student ID (no name), accommodation type, value applied, and timestamp.
- **FR-7.** Instructors MUST be able to view a non-PII summary of accommodations for students in their courses (e.g., "3 students have extended time"), but MUST NOT see IEP details or the specific accommodation configuration.
- **FR-8.** The system SHOULD allow bulk upload of accommodation profiles via CSV (student external ID + accommodation type + value) for district-level provisioning.
- **FR-9.** FERPA-protected accommodation records MUST only be accessible to: the student themselves, their instructors (summary view), SPED coordinators, and admins. No parent access to raw accommodation configuration (parents access via §5.10 parent portal with FERPA-compliant summary).

## 6. Non-Functional Requirements

- **Performance** — Accommodation lookup on quiz-start must add < 5 ms to p99 latency (indexed by `(user_id, course_id)`). Preference injection at page load must complete before first paint; loaded with existing reading-preferences API call.
- **Security** — Accommodation data is FERPA-protected PII. Access enforced in `server/src/authz.rs` with explicit role checks (`sped_coordinator`, `admin`). All accommodation endpoints require HTTPS. Audit log is append-only (no UPDATE/DELETE on `accommodation_audit_log`).
- **Privacy & Compliance** — IDEA (20 U.S.C. §1400): accommodations are legally required educational services. Section 504 of the Rehabilitation Act. FERPA (20 U.S.C. §1232g): accommodation plan data is an education record; access controlled to school officials with legitimate educational interest. COPPA: student accommodation data for under-13 protected as sensitive PII. ADA Title II: platform must make accommodations technically enforceable, not just documented. The accommodation audit log satisfies IDEA monitoring requirements.
- **Accessibility** — The accommodations management UI must itself be WCAG 2.1 AA compliant (accessible by SPED coordinators who may also have disabilities). All form inputs labeled, error messages announced, keyboard-navigable. WCAG SC 2.1.1, 4.1.2, 1.3.1.
- **Scalability** — A district with 10,000 students and 15% SPED population has ~1,500 accommodation profiles; accommodation lookup is a simple indexed read.
- **Reliability** — If accommodation lookup fails (DB timeout), the system MUST fall back to the student's manual preferences rather than blocking the quiz; log the failure. Accommodations must not be a single point of failure.
- **Observability** — Metrics: `accommodations_applied_per_day` by type, `accommodation_audit_log_size`. Alerts: any `accommodation_lookup_failure`.
- **Maintainability** — Accommodation type registry as a server-side enum + migration; new accommodation types added without schema changes (use `accommodation_type TEXT NOT NULL` + `value JSONB`).
- **Internationalization** — Accommodation type names and descriptions externalized; coordinator UI supports translated labels.
- **Backward compatibility** — Students without accommodations experience zero change. Existing quiz attempt schema extended only (§2.7 `time_limit_seconds` already a dependency).

## 7. Acceptance Criteria

- **AC-1.** *Given* a SPED coordinator creates an accommodation profile for a student with `time_multiplier = 1.5`, *When* that student begins a 40-minute quiz, *Then* the quiz timer shows 60 minutes and the attempt record stores `effective_time_limit_seconds = 3600`.
- **AC-2.** *Given* a student has `tts_enabled = true` in their accommodation profile, *When* they open a content page, *Then* the read-aloud controls auto-activate before the student takes any action and the panel shows "Enabled by your accommodation plan."
- **AC-3.** *Given* an instructor views their course roster, *When* they see a student with accommodations, *Then* they see "Has active accommodations" indicator but cannot see the accommodation details (privacy enforced).
- **AC-4.** *Given* a student without a `sped_coordinator` role calls `PATCH /api/accommodations/:userId`, *When* the request is received, *Then* the server returns HTTP 403 and no change is made.
- **AC-5.** *Given* a quiz attempt is started with time extension accommodation applied, *When* the audit log is queried, *Then* a record exists with `student_id`, `accommodation_type = 'time_extension'`, `value = '1.5'`, `applied_at = <timestamp>`, and `quiz_attempt_id`.
- **AC-6.** *Given* a SPED coordinator uploads a CSV with 50 student accommodation profiles, *When* the import completes, *Then* all 50 profiles are created with correct values and an import summary report is returned.
- **AC-7.** *Given* axe-core runs on the accommodations management page, *When* the check executes, *Then* zero Critical or Serious violations are reported.

## 8. Data Model

```sql
-- server/migrations/079_accommodations_engine.sql

CREATE TABLE student_accommodations (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    org_id          UUID NOT NULL REFERENCES organizations(id),  -- §5.1
    school_id       UUID,                                         -- §5.2
    accommodation_type TEXT NOT NULL,  -- time_extension | extra_attempts | tts | stt |
                                       -- dyslexia_display | high_contrast | reduced_motion |
                                       -- separate_setting
    value           JSONB NOT NULL,    -- e.g. {"multiplier": 1.5} or {"enabled": true}
    notes           TEXT,              -- FERPA-protected; coordinator only
    active          BOOLEAN NOT NULL DEFAULT true,
    created_by      UUID NOT NULL REFERENCES users(id),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX student_accommodations_user_type_idx
    ON student_accommodations(user_id, accommodation_type)
    WHERE active = true;

CREATE TABLE accommodation_audit_log (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id          UUID NOT NULL REFERENCES users(id),
    accommodation_type  TEXT NOT NULL,
    value_applied       JSONB NOT NULL,
    context             TEXT NOT NULL,  -- quiz_attempt | content_view | assignment
    context_id          UUID,
    applied_at          TIMESTAMPTZ NOT NULL DEFAULT now()
    -- No UPDATE/DELETE; append-only enforced by row-level policy
);
CREATE INDEX aal_student_idx ON accommodation_audit_log(student_id, applied_at DESC);
```

Backfill: none (new table). Existing quiz attempts unaffected.

## 9. API Surface

- `GET /api/students/:userId/accommodations` — returns accommodation list; `sped_coordinator` or `admin` or self.
- `POST /api/students/:userId/accommodations` — create accommodation; `sped_coordinator` or `admin`.
- `PATCH /api/accommodations/:id` — update; `sped_coordinator` or `admin`.
- `DELETE /api/accommodations/:id` — soft-delete (sets `active = false`); `sped_coordinator` or `admin`.
- `POST /api/admin/accommodations/import` — CSV bulk import; `admin`.
- `GET /api/admin/accommodations/audit` — compliance audit log; `admin` or `sped_coordinator`.
- Internal: accommodation lookup called from quiz-start handler in `server/src/routes/courses.rs`.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| FERPA violation if instructor accidentally sees IEP accommodation details | H | H | Role-based API enforcement + UI shows only non-PII indicator; external legal review |
| Time multiplier arithmetic error results in shorter quiz time for student | M | H | Unit test all multiplier calculations; server-authoritative timer |
| §5.1 org entity not yet available when this ships | M | H | Stub `org_id` with a default value; refactor when §5.1 ships |
| CSV import creates duplicate accommodation records | M | M | Upsert on `(user_id, accommodation_type)` unique index |

## 15. Rollout Plan

- Feature flag: `ff_accommodations_engine` (default `false`).
- Sequencing: DB migration → authz role → API → admin UI → quiz integration → reading-prefs integration → flip flag.
- Legal review: FERPA access model reviewed by legal counsel before GA.
- Dogfood: Pilot with one school district (1–2 SPED coordinators, 10 students with accommodations).
- GA: All 7 ACs pass; legal review signed off; audit log validated by compliance officer.
- Rollback: Feature flag off; quiz time limits revert to base values; reading preferences revert to manual.

## 16. Test Plan

- **Unit:** Time multiplier calculation; accommodation lookup with/without active flag; audit log append-only enforcement.
- **Integration:** Quiz-start with time extension; content-load with TTS auto-activation; FERPA access matrix (student, instructor, coordinator, admin, parent role).
- **End-to-end:** Playwright — SPED coordinator creates accommodation → student starts quiz → verify multiplied timer → audit log entry.
- **Security:** Role-matrix tests for all accommodation endpoints; student cannot access another student's accommodations.
- **Accessibility:** axe-core on accommodations management page; keyboard navigation of all form controls.
- **Performance:** Accommodation lookup adds < 5 ms to quiz-start.
- **Manual:** QA verifies TTS auto-activation, extended timer display, accommodation indicator on roster.

## 18. Open Questions

1. Should `sped_coordinator` be a system-level role or a course-level role? System-level makes more sense for IEP management.
2. Does the platform need to surface a parent-accessible (§5.10) view of "which accommodations are active for my child" without exposing IEP document details?
3. Should accommodations have an expiration date (IEPs are reviewed annually)?
4. Is a "separate testing environment" flag (informational) the right v1 scope, or should it trigger a locked-browser mode?
