## 1. Problem Statement

`clients/web/src/components/editor/EditorWithPrompt.tsx` exposes AI-assisted content generation inside the TipTap editor, but only as a locked, inline-only interaction. The capability is not available outside the editor (assignment description page, rubric page, course settings, etc.) and is not a system-wide contextual assistant. Instructors spend significant time writing and refining content across many surfaces; a persistent sidebar copilot available on every instructor-facing page would dramatically accelerate course authoring.

## 2. Goals

- Elevate the existing `EditorWithPrompt` AI capability to a system-wide instructor sidebar that is context-aware (knows what page/content the instructor is looking at).
- Allow the copilot to suggest edits, generate new sections, rephrase, simplify, or translate content on any instructor-editable surface.
- Keep the copilot's suggestions as non-destructive drafts that the instructor explicitly applies.
- Apply PII redaction and model governance.

## 3. Non-Goals

- Student-facing copilot (§19.1 tutor handles that).
- Copilot on grading/feedback forms (§19.3).
- Voice-activated copilot (§19.6).
- Auto-applying suggestions without instructor intent.

## 4. Personas & User Stories

- **As a teacher creating a quiz**, I want to ask the copilot "Make these questions harder" and see revised question drafts so that I can pick the best ones.
- **As a teacher writing a lesson description**, I want to ask "Simplify this for 4th graders" and have the copilot rewrite the selected text in place.
- **As a teacher on the rubric page**, I want the copilot to see the current rubric and suggest improvements based on best practices.
- **As an instructor**, I want to dismiss the copilot sidebar when I do not need it so that it does not distract me.

## 5. Functional Requirements

- **FR-1.** The system MUST render a collapsible AI copilot sidebar panel available on all instructor-facing pages; default state: collapsed.
- **FR-2.** The copilot MUST receive context about the current page: page type (course editor, quiz editor, assignment, rubric, grading), and the current content (title, description, selected text).
- **FR-3.** The instructor MUST be able to type free-form instructions to the copilot ("make this more engaging", "generate 3 more questions like question 2").
- **FR-4.** The copilot MUST return suggestions as formatted diffs or replacement text, never auto-applying them.
- **FR-5.** The instructor MUST be able to accept (apply to editor), reject, or request a revision of each suggestion.
- **FR-6.** The copilot MUST include the current editor selection or document content in the LLM context; content MUST pass through PII proxy (§19.11) first.
- **FR-7.** The system MUST limit copilot context to content the instructor has access to (their own courses only).
- **FR-8.** The copilot MUST display a prompt character/token counter so instructors understand cost implications (cross-ref §19.14).
- **FR-9.** The system MUST log copilot interactions (instructor_id, page_context_type, input_token_count, output_token_count) for cost monitoring (§19.14).
- **FR-10.** The system MUST respect model governance settings (§19.10) for model selection.

## 6. Non-Functional Requirements

- **Performance** — First token response ≤ 1.5 s p95; full suggestion ≤ 8 s p95.
- **Security** — Copilot cannot access other instructors' content or student submissions; context injected server-side to prevent client-side prompt injection.
- **Privacy & Compliance** — Instructor-created content (not student data) is the primary input; PII proxy applied; instructor AI usage logged for billing (§19.14).
- **Accessibility** — Copilot sidebar fully keyboard navigable; suggestion diff announced by screen reader; focus trap when sidebar is open.
- **Scalability** — SSE streaming for suggestions; stateless server component.
- **Reliability** — Copilot failure shows inline error; editor state unaffected.
- **Observability** — `copilot_requests_total`, `copilot_latency_ms`, `copilot_acceptance_rate` (accepted vs. rejected suggestions).
- **Maintainability** — Copilot server route in `server/src/routes/copilot.rs`; React component `clients/web/src/components/CopilotSidebar.tsx`; extend `EditorWithPrompt.tsx` to share the sidebar.
- **Internationalization** — Copilot responses in instructor's configured locale.
- **Backward compatibility** — `EditorWithPrompt.tsx` inline prompt preserved; copilot is a superset.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor is on the quiz editor page with 3 questions visible, *When* they open the copilot and type "Make question 2 harder", *Then* a revised question 2 draft appears in the copilot panel without auto-applying.
- **AC-2.** *Given* a copilot suggestion is shown, *When* the instructor clicks "Apply", *Then* the relevant editor field is updated with the suggestion and the copilot shows a success state.
- **AC-3.** *Given* the instructor selects a paragraph of text before opening the copilot, *When* they type "Simplify this", *Then* the LLM receives the selected text as context and returns a simplified version.
- **AC-4.** *Given* the copilot sidebar is open, *When* the instructor presses Escape, *Then* the sidebar collapses and focus returns to the editor.
- **AC-5.** *Given* the LLM provider returns an error, *When* the copilot displays the error, *Then* the error message is visible in the copilot panel, the editor is unchanged, and a retry button is available.

## 8. Data Model

```sql
-- 0XX_copilot_interactions.sql
CREATE TABLE copilot_interactions (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  instructor_id    UUID NOT NULL REFERENCES users(id),
  org_id           UUID NOT NULL REFERENCES organizations(id),
  page_context     TEXT NOT NULL,     -- 'quiz_editor' | 'assignment' | 'rubric' | ...
  input_tokens     INT,
  output_tokens    INT,
  accepted         BOOLEAN,
  created_at       TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX copilot_instructor ON copilot_interactions(instructor_id, created_at);
```

## 9. API Surface

```
POST /api/copilot/suggest    { context_type, context_content, instruction, selected_text }
                              -> SSE stream of CopilotSuggestionTokens
POST /api/copilot/apply      { interaction_id, accepted: bool }   -> 204
```

Context content passes through PII proxy server-side before LLM call.

## 10. UI / UX

- `clients/web/src/components/CopilotSidebar.tsx` — right-side collapsible panel; toggle button in instructor layout header.
- Panel sections: (1) current context summary (what page/content), (2) free-text instruction input, (3) suggestion display with syntax-highlighted diff, (4) Accept / Reject / Revise buttons.
- Keyboard shortcut: `Cmd+Shift+K` to open/close copilot.
- Suggestions stream in progressively via SSE.
- Token counter: subtle char count under instruction input.

Extend `clients/web/src/components/editor/EditorWithPrompt.tsx` to dispatch selected text to the sidebar.

## 11. AI / ML Considerations

- **Model**: Claude-3.5-Sonnet (good at instruction-following and prose editing); configurable per §19.10.
- **Prompt source-of-truth**: System prompt for teacher copilot in DB (`copilot_system_prompts` table); version-controlled per §19.10.
- **Context injection**: Page type and content assembled server-side; selected_text from client.
- **Eval**: Acceptance rate as primary metric (target ≥ 40%); blind comparison study with instructors.
- **Fallback**: Provider error → clear error message; no editor mutation.
- **Model card**: `docs/ai/copilot-model-card.md`.
- **Cost**: ~1 000–3 000 tokens per copilot turn; §19.14 budget.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Instructors over-rely on copilot, reducing content quality | M | M | Acceptance-rate analytics; content review workflow unchanged |
| Prompt injection via crafted course content | M | H | Server-side context assembly; content sanitization before injection |
| Copilot sidebar distracts UX on small screens | M | L | Default collapsed; responsive: drawer overlay on mobile |

## 15. Rollout Plan

- Feature flag: `teacher_copilot_enabled` (OpenFeature, default `false`).
- Phase 1: Internal instructors; acceptance-rate baseline.
- Phase 2: Opt-in for all instructors.
- Phase 3: GA.

## 16. Test Plan

- **Unit** — Context assembly; PII proxy call; SSE streaming response parsing; apply/reject action.
- **Integration** — Full round-trip: context → suggestion → accept → editor updated.
- **End-to-end** — Playwright: instructor opens copilot → types instruction → receives suggestion → accepts → verifies editor updated.
- **Security** — Cross-instructor context leakage; prompt injection via adversarial context content.
- **Accessibility** — axe-core on copilot sidebar; keyboard trap test; SSE announcement via aria-live.

## 17. Documentation & Training

## 18. Open Questions

1. Should the copilot have access to the instructor's full course library as additional context, or only the current page?
2. Should copilot suggestions be saved as drafts that can be revisited, or ephemeral per session?
3. Should we allow multi-turn conversation with the copilot, or single-shot suggestions only?
