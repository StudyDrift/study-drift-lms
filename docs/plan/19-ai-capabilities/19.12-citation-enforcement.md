## 1. Problem Statement

The RAG notebook in `server/src/services/student_notebook_rag_ai.rs` retrieves course content but does not enforce that responses cite their sources. An AI tutor that produces plausible-sounding answers without grounding creates a hallucination risk and erodes trust. K-12 and HE institutions cite "AI hallucinations in academic content" as a top concern when evaluating AI-enhanced LMSs. RAGAS and ARES are the leading evaluation frameworks for RAG grounding quality.

## 2. Goals

- Enforce that every RAG-backed AI response includes structured citations referencing the specific course content chunks used.
- Validate citation validity (referenced chunk must exist in the course's RAG index) before responding.
- Make citations navigable (click citation → opens source content).
- Measure and monitor citation coverage and grounding accuracy via the eval harness (§19.13).

## 3. Non-Goals

- Citations for non-RAG AI features (lesson plan generation, rubric generation — these draw from the prompt, not a retrieval index).
- Blocking all responses with zero retrieved chunks (some questions are legitimately general and need a fallback).
- Full bibliographic citation formatting (APA/MLA) — source chunk + title is sufficient for this plan.

## 4. Personas & User Stories

- **As a student**, I want to see "Based on: Unit 3, Lesson 2 – Cell Division" after a tutor response so that I can verify the answer against the original material.
- **As an instructor**, I want confidence that the tutor is not inventing answers and is grounding responses in my course content.
- **As a district administrator**, I want to tell parents that the AI tutor cites its sources and does not fabricate information.
- **As an AI auditor**, I want to measure the citation grounding rate across tutor sessions to include in a compliance report.

## 5. Functional Requirements

- **FR-1.** The system MUST include a `citations` field in every RAG-backed API response containing an array of `{ chunk_id, source_title, source_type, excerpt_text }`.
- **FR-2.** The LLM system prompt for all RAG features MUST instruct the model to end each factual claim with a `[CIT:<chunk_id>]` marker; the server MUST parse these markers and return structured citations.
- **FR-3.** The system MUST validate that each cited `chunk_id` exists in the current course's RAG index; invalid citations MUST be stripped and logged.
- **FR-4.** If a RAG response contains zero citations, the system MUST append a fallback disclaimer: "This response was not grounded in your course materials. Please verify with your instructor."
- **FR-5.** Citations MUST be rendered as clickable links in the student UI navigating to the cited content section.
- **FR-6.** Grounding metrics (citations per response, grounding rate, hallucination-suspect responses) MUST be emitted to the eval harness (§19.13).
- **FR-7.** The system SHOULD apply a confidence threshold: if the cosine similarity of the top retrieved chunk is below 0.5, treat the response as ungrounded and apply the fallback disclaimer even if citations are present.

## 6. Non-Functional Requirements

- **Performance** — Citation parsing adds < 20 ms to response post-processing.
- **Security** — Cited chunk content must not expose content from other courses (org + course scoped RAG index).
- **Privacy & Compliance** — Citation content may contain student-generated materials in some edge cases; display excerpt from course materials only (not student submissions).
- **Accessibility** — Citation links rendered as proper `<a>` elements with descriptive text; citation list keyboard navigable.
- **Scalability** — Citation validation is a hash-map lookup against a small in-memory chunk index per request.
- **Reliability** — Citation parsing failure (malformed LLM output) MUST NOT block the response; strip citations and add disclaimer.
- **Observability** — `citations_per_response_mean`, `ungrounded_responses_total`, `invalid_citation_strip_total` (§19.13 metrics).
- **Maintainability** — Citation parsing in `server/src/services/citation_parser.rs`; extend RAG pipeline in `student_notebook_rag_ai.rs`.
- **Internationalization** — Disclaimer text in i18n keys.
- **Backward compatibility** — `citations` field additive to existing RAG response; existing clients that ignore the field are unaffected.

## 7. Acceptance Criteria

- **AC-1.** *Given* a student asks the tutor a question answered in Unit 3 Lesson 2, *When* the response is returned, *Then* `citations[0].source_title` is "Unit 3 – Lesson 2" and `citations[0].chunk_id` resolves to a real chunk in the RAG index.
- **AC-2.** *Given* the LLM includes a `[CIT:nonexistent_chunk_id]` marker, *When* citation validation runs, *Then* that citation is stripped from the response and a `citation.invalid_stripped` event is logged.
- **AC-3.** *Given* a response with zero retrieved chunks, *When* the response is returned, *Then* the disclaimer "This response was not grounded in your course materials." is appended.
- **AC-4.** *Given* a citation is present in the student UI, *When* the student clicks it, *Then* the browser navigates to the cited lesson section and highlights the relevant excerpt.
- **AC-5.** *Given* the RAGAS eval pipeline runs on 50 golden Q&A pairs, *When* grounding evaluation completes, *Then* faithfulness score ≥ 0.85 and context precision ≥ 0.80.

## 8. Data Model

No new tables. Extend `tutor_messages` (§19.1):
```sql
-- citations column already defined as JSONB in 19.1 data model:
-- citations JSONB:  [{ chunk_id, source_title, source_type, excerpt, cosine_similarity }]
```

Add `grounding_score NUMERIC` column to `tutor_messages` for the mean cosine similarity of retrieved chunks.

## 9. API Surface

Citation data included in existing endpoints (§19.1):
```
POST /api/courses/:id/tutor/sessions/:sid/messages
-> TutorMessage {
     content: string,
     citations: CitationDto[],
     grounding_score: number | null,
     ungrounded_disclaimer: boolean
   }
```

## 10. UI / UX

- Citations rendered as footnote-style numbered chips `[1]`, `[2]` inline with response text.
- Hovering/focusing a chip shows: source title, excerpt preview.
- Clicking navigates to source section.
- Ungrounded disclaimer rendered in amber italic text below the response.
- Empty citations gracefully handled — no broken chip links.

## 11. AI / ML Considerations

- **RAG eval**: RAGAS framework — faithfulness, answer relevance, context precision, context recall.
- **ARES**: Alternative RAG Evaluation using Statistical and model-based scoring.
- **Grounding threshold**: Cosine similarity ≥ 0.5 for retrieved chunks; configurable in org AI settings.
- **Prompt engineering**: System prompt instructs model to use `[CIT:<id>]` syntax; few-shot examples of well-cited responses.
- **Eval metric targets**: Faithfulness ≥ 0.85, answer relevance ≥ 0.80, hallucination rate ≤ 5%.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| LLM ignores citation instruction and produces no markers | M | M | Post-processing fallback disclaimer; eval harness alert |
| Retrieved chunks from wrong course shown in citations | L | H | RAG index scoped by course_id; chunk lookup validates course ownership |
| High false-positive "ungrounded" rate for general knowledge questions | M | L | Configurable cosine threshold; general-knowledge fallback prompt path |

## 15. Rollout Plan

- Feature flag: `citation_enforcement_enabled` (OpenFeature, default `false`).
- Phase 1: Citation parsing and validation enabled; no UI yet; measure in eval harness.
- Phase 2: Citation UI chips enabled.
- Phase 3: Ungrounded disclaimer enabled.
- Phase 4: GA.

## 16. Test Plan

- **Unit** — Citation marker parsing; chunk ID validation; fallback disclaimer trigger.
- **Integration** — RAG query → response with citations → chunk IDs resolve → UI receives structured citations.
- **AI eval** — RAGAS pipeline: 50 golden Q&A pairs; faithfulness ≥ 0.85.
- **Accessibility** — Citation chips keyboard accessible; excerpt popup screen-reader accessible.
