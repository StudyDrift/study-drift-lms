## 1. Problem Statement

Language learning, oral presentation skills, and speech-based assessments are completely unsupported in Lextures. Students cannot submit audio/video recordings for graded feedback, and there is no pronunciation or fluency evaluation. For K-12 language arts and foreign-language courses, and for self-learners studying a new language, this is a significant gap. OpenAI Whisper and speech-analysis LLMs now make automated transcription and pronunciation scoring feasible as a background processing pipeline.

## 2. Goals

- Allow students to record or upload audio/video responses to designated oral-response question types.
- Transcribe submissions using Whisper and score them against a rubric using an LLM.
- Provide pronunciation feedback for foreign-language submissions using phoneme-level analysis.
- Feed transcripts into the instructor approval queue (§19.3 pattern) before scores reach students.
- Comply with FERPA/COPPA for audio/video data of minors.

## 3. Non-Goals

- Real-time live speech coaching during in-class activities (requires WebRTC, out of scope).
- Full proctoring via video (§14.9).
- Instructor audio/video feedback on submissions (§3.2 — separate plan).
- Support for video content creation by instructors (§8.3).

## 4. Personas & User Stories

- **As a Spanish teacher**, I want students to submit 60-second audio recordings saying a prompt so that I get automated transcription and fluency scoring.
- **As a student learning Mandarin**, I want AI pronunciation feedback comparing my tones to a native speaker baseline so that I can correct my mistakes.
- **As a student**, I want to record directly in the browser without installing software so that the workflow is frictionless.
- **As an instructor**, I want to review the AI transcript and score before it reaches the student so that I can catch transcription errors.

## 5. Functional Requirements

- **FR-1.** The system MUST support a new question/assignment type: `oral_response` with a configurable recording duration limit (30 s – 10 min).
- **FR-2.** The system MUST allow recording via browser MediaRecorder API (WebM/Opus) or file upload (MP3, M4A, WAV, MP4).
- **FR-3.** The system MUST transcribe submitted audio using OpenAI Whisper (via API or self-hosted) asynchronously and store the transcript alongside the recording.
- **FR-4.** The system MUST score transcripts against the assignment rubric using an LLM (same approval-queue pattern as §19.3).
- **FR-5.** For foreign-language assignments with `pronunciation_grading: true`, the system SHOULD analyze phoneme-level accuracy using a pronunciation scoring API (e.g., Azure Cognitive Services Speech, or an open-source alternative) and return a per-phoneme or per-word confidence score.
- **FR-6.** All audio/video files MUST be stored in object storage (§8.1) with org-scoped access control; raw recordings MUST NOT be accessible via public URL.
- **FR-7.** The system MUST apply PII redaction to transcripts before any LLM calls (§19.11).
- **FR-8.** The system MUST comply with COPPA: for under-13 users, audio recording requires verified parental consent flag on the user account; if absent, the oral_response question type MUST be hidden.
- **FR-9.** The transcript and AI feedback MUST be displayed to the student after instructor approval, alongside the audio playback.
- **FR-10.** The system SHOULD auto-generate captions for video submissions (§8.4 dependency).

## 6. Non-Functional Requirements

- **Performance** — Transcription of a 2-minute audio MUST complete within 60 s; pronunciation scoring within 30 s additional.
- **Security** — Audio files accessible only to the submitting student and their course instructor; signed short-lived URLs for playback; files encrypted at rest (§10.13).
- **Privacy & Compliance** — Audio recordings of students are FERPA-protected; biometric voice data may trigger BIPA (Illinois), TDPSA (Texas) — legal review required before K-12 rollout; COPPA consent gate for under-13.
- **Accessibility** — Transcripts provided as text alternative to all audio submissions (WCAG 1.2.1 / 1.2.3); recording controls keyboard accessible.
- **Scalability** — Transcription jobs in background queue; Whisper can run on CPU with acceptable latency for short clips; GPU inference for scale.
- **Reliability** — Transcription failure falls back to manual grading queue; recording preserved regardless of transcription status.
- **Observability** — `audio_transcription_jobs_total{status}`, `transcription_latency_ms`, `pronunciation_score_mean{language}`.
- **Maintainability** — Media pipeline in `server/src/services/audio_transcription.rs`; pronunciation scoring adapter pattern for provider swap.
- **Internationalization** — Whisper is multilingual; language code on assignment triggers appropriate model mode.
- **Backward compatibility** — Existing assignment/quiz types unchanged.

## 7. Acceptance Criteria

- **AC-1.** *Given* a student records a 45-second audio response in the browser, *When* they submit, *Then* the file is stored in object storage and a transcription job is queued; the student sees "Submission received — being transcribed."
- **AC-2.** *Given* a transcription completes, *When* the instructor views the approval queue, *Then* they see the audio player, transcript, and AI-suggested rubric scores.
- **AC-3.** *Given* a foreign-language assignment with pronunciation grading, *When* the student submits audio, *Then* the feedback includes per-word pronunciation confidence scores.
- **AC-4.** *Given* an under-13 user without parental consent, *When* they view an assignment with an oral_response question, *Then* the recording interface is replaced with a notice: "Audio recording requires parental consent."
- **AC-5.** *Given* a transcript is generated, *When* it is sent to the LLM for rubric scoring, *Then* any student names present in the transcript have been replaced by the PII proxy.
- **AC-6.** *Given* a student requests their audio recording playback, *When* the URL is fetched, *Then* it is a signed URL expiring in 1 hour, not a publicly accessible permanent URL.

## 8. Data Model

```sql
-- 0XX_oral_responses.sql
CREATE TABLE oral_response_submissions (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  submission_id    UUID NOT NULL REFERENCES assignment_submissions(id),
  storage_key      TEXT NOT NULL,          -- object storage key
  duration_seconds INT,
  mime_type        TEXT NOT NULL,
  transcript       TEXT,
  transcript_lang  TEXT,
  pronunciation_scores JSONB,
  whisper_model    TEXT,
  transcription_status TEXT NOT NULL DEFAULT 'pending',
  created_at       TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

## 11. AI / ML Considerations

- **Transcription**: OpenAI Whisper `whisper-1` API (Anthropic and Vertex do not offer ASR natively); fallback to self-hosted `whisper-medium` for data-residency orgs.
- **Rubric scoring**: Same LLM as §19.3 (Claude-3.5-Sonnet); transcript passed as submission text.
- **Pronunciation**: Azure Cognitive Services Speech Assessment or CMU Pronouncing Dictionary + Montreal Forced Aligner (open-source) for phoneme alignment.
- **Eval metric**: Word Error Rate (WER) ≤ 10% on English test set; pronunciation scoring correlation with human rater ≥ 0.7.
- **Cost**: Whisper API ~$0.006/min; budget tracked per org (§19.14).
- **Fallback**: If Whisper fails, flag for manual review; preserve audio recording.
- **Model card**: `docs/ai/voice-ai-model-card.md`.
- **Opt-out**: `ai_oral_grading_opt_out` flag on user; falls back to instructor manual grading.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Biometric voice data laws (BIPA, TDPSA) | H | H | Legal review; store transcripts not biometric voiceprints; opt-out path |
| COPPA under-13 audio recording without consent | M | H | Hard gate on recording UI; parental consent flow prerequisite |
| Whisper WER too high for non-native English speakers | H | M | Language-specific Whisper model; instructor review gate |
| Audio storage costs escalate | M | M | Per-org storage quota (§8.5); auto-expire recordings after N months |

## 15. Rollout Plan

- Feature flag: `oral_response_enabled` (OpenFeature, default `false`).
- Phase 1: HE foreign-language courses (no COPPA concern); English only.
- Phase 2: Multi-language support; pronunciation scoring.
- Phase 3: K-12 pilot with legal review complete and parental consent flow in place.

## 16. Test Plan

- **Accessibility** — Keyboard access to recorder; transcript text alternative present.
