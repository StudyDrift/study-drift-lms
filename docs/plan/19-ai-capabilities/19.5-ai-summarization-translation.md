## 1. Problem Statement

Instructors upload long lecture slides, documents, and videos; students with limited time, language barriers, or reading difficulties cannot efficiently access this content. Lextures has no summarization or translation capability. Competitor platforms (Coursera, Canvas with third-party plugins) already offer AI-generated notes and machine translation. This is a meaningful differentiator gap for multilingual K-12 districts and international HE students.

## 2. Goals

- Allow students and instructors to request AI-generated summaries of any text-based course content (lessons, PDFs, transcripts).
- Allow on-demand translation of content into any supported language.
- Tag all summarized/translated content with provenance metadata (§19.9) and display a disclosure.
- Apply PII redaction before content leaves the system (§19.11).
- Implement cost controls (§19.14) since summarization + translation can be token-heavy.

## 3. Non-Goals

- Real-time live transcription/translation of video streams (§19.6).
- Machine translation as a replacement for human-quality localization (§11 i18n framework).
- Translation of UI strings (§11.1).
- Auto-translating all content by default — user-triggered only in this plan.

## 4. Personas & User Stories

- **As a Spanish-speaking student**, I want to request a Spanish translation of the week's lesson notes so that I can understand them better than the English original.
- **As a student preparing for an exam**, I want a 3-bullet summary of each module so that I can review key points quickly.
- **As an instructor**, I want to generate a student-facing summary of my uploaded lecture slide deck so that students have a reading-level-appropriate overview.
- **As an admin**, I want summaries flagged as AI-generated so that content quality reviews can distinguish human-authored from AI-generated text.

## 5. Functional Requirements

- **FR-1.** The system MUST provide a "Summarize" action on any text-based content item (lesson, module description, uploaded PDF, quiz explanation) that returns a structured summary (key points list + paragraph form).
- **FR-2.** The system MUST provide a "Translate" action on any content item that accepts a target language (BCP 47 code) and returns a translated version.
- **FR-3.** Summaries and translations MUST be cached per (content_item_id, language, summary_type) to avoid re-generating for subsequent requests.
- **FR-4.** All input content MUST pass through the PII redaction proxy (§19.11) before LLM calls.
- **FR-5.** Generated outputs MUST carry provenance metadata `{ generated_by: "lextures-ai", model_id, language }` (§19.9).
- **FR-6.** The system MUST display a disclosure "AI-generated content — may contain errors" adjacent to any summary or translation.
- **FR-7.** The system SHOULD allow instructors to approve, reject, or edit a generated translation/summary before it appears to students.
- **FR-8.** The system MUST respect per-tenant model and prompt governance (§19.10).
- **FR-9.** The system SHOULD support batched summarization of all content items in a module in a single admin action.

## 6. Non-Functional Requirements

- **Performance** — Summarization of a 2 000-word document MUST complete in ≤ 15 s; cached responses MUST return in ≤ 100 ms.
- **Security** — Content access permissions enforced before summarization (enrolled students only for student-requested summaries).
- **Privacy & Compliance** — Lesson content may contain student work examples (FERPA); PII proxy applied; caching must be org-isolated.
- **Accessibility** — Summary output displayed as structured HTML with headings; translation toggle keyboard accessible.
- **Scalability** — Cache keyed by (item_id + lang + hash of content) in Postgres with TTL; Redis for hot cache.
- **Reliability** — Cache-miss falls back to live generation; generation failure shows inline error with retry.
- **Observability** — `summarization_requests_total{type,lang}`, cache hit rate, token costs per request.
- **Maintainability** — Service: `server/src/services/content_ai.rs`.
- **Internationalization** — Translation supports all BCP 47 languages supported by the LLM provider.
- **Backward compatibility** — Original content unchanged; summaries/translations are additive overlays.

## 7. Acceptance Criteria

- **AC-1.** *Given* a student clicks "Summarize" on a 1 500-word lesson, *When* the summary is generated, *Then* a 5-bullet key points list and a 150-word paragraph summary are returned within 15 s with a provenance badge.
- **AC-2.** *Given* the same student requests the same summary again, *When* the request is processed, *Then* the cached result is returned in ≤ 100 ms (no LLM call made).
- **AC-3.** *Given* a student requests translation to Spanish (`es`), *When* the translation is returned, *Then* the content is rendered in Spanish with the disclosure "AI-generated content — may contain errors" visible.
- **AC-4.** *Given* a lesson contains a student name in an example, *When* the PII proxy processes the content, *Then* the name is replaced with a token in the prompt sent to the LLM.
- **AC-5.** *Given* a non-enrolled user attempts to summarize a private course lesson, *When* the request is made, *Then* HTTP 403 is returned.

## 8. Data Model

```sql
-- 0XX_content_ai_cache.sql
CREATE TABLE content_ai_cache (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id        UUID NOT NULL REFERENCES organizations(id),
  content_ref   TEXT NOT NULL,    -- e.g. 'lesson:<uuid>' or 'file:<key>'
  content_hash  TEXT NOT NULL,    -- SHA256 of source content for invalidation
  ai_type       TEXT NOT NULL,    -- 'summary' | 'translation'
  language      TEXT,             -- BCP 47, NULL for summaries in source language
  output        TEXT NOT NULL,
  provenance    JSONB NOT NULL,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  expires_at    TIMESTAMPTZ NOT NULL DEFAULT now() + interval '30 days'
);
CREATE UNIQUE INDEX content_ai_cache_key ON content_ai_cache(content_ref, content_hash, ai_type, language);
```

## 9. API Surface

```
POST /api/content/summarize  { content_ref, summary_type: 'bullets'|'paragraph'|'both' }
                              -> { summary, provenance }
POST /api/content/translate   { content_ref, target_lang }
                              -> { translated_text, provenance }
```

Both endpoints check enrollment ACL before proceeding. Rate limit: 10 requests per student per hour.

## 10. UI / UX

- "Summarize" and "Translate" buttons in lesson/module header action bar.
- Translation: language picker dropdown (shows org-supported languages); output replaces or is displayed beside original.
- Summary: collapsible panel below content with key-points list and paragraph summary.
- Provenance badge and disclosure text on all generated outputs.
- Instructor approval gate (optional): generated content shows as "Pending instructor review" to students if approval required per org settings.

## 11. AI / ML Considerations

- **Model**: Claude-3.5-Haiku for summaries (cost-efficient); Claude-3.5-Sonnet for translation quality; configurable per §19.10.
- **Prompt**: Source-of-truth in DB; summarization prompt instructs output format (bullets, key concepts, Bloom's level).
- **Cost**: ~1 000–4 000 tokens per summarization request; monthly per-org budget tracked (§19.14).
- **Eval**: ROUGE-L score for summaries vs. human-authored summaries; BLEU for translations vs. professional translations on a test set.
- **Fallback**: Provider failure → inline error + manual link to original content.
- **Model card**: `docs/ai/content-ai-model-card.md`.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Translation quality unacceptable for academic content | M | M | Quality disclosure; instructor approval gate option |
| Cache invalidation miss (stale summary after content edit) | M | M | Invalidate cache on content_hash change |
| Token costs spike for large modules | M | M | Batch-summarization requires org_admin; per-org monthly budget (§19.14) |

## 15. Rollout Plan

- Feature flag: `content_summarization_enabled`, `content_translation_enabled` (separate flags, OpenFeature, both default `false`).
- Phase 1: Summarization only, for instructors only; eval ROUGE-L.
- Phase 2: Summarization for students.
- Phase 3: Translation feature GA.

## 16. Test Plan

- **Unit** — Cache key construction; content hash computation; PII proxy call.
- **Integration** — Summarize lesson → cache → re-request returns cached → content edit invalidates cache.
- **Security** — Non-enrolled user blocked; cross-org cache isolation.
- **AI eval** — ROUGE-L ≥ 0.4 for summaries; BLEU ≥ 0.5 for translations on test corpus.
