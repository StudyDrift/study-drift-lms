## 1. Problem Statement

Instructors frequently need to export gradebook data for offline analysis, SIS upload, or regulatory reporting — and to import scores that were generated outside Lextures (e.g., from a paper-based exam scored by an external service). Neither capability exists today. Teaching assistants also use bulk import to enter large batches of scores efficiently. Without CSV import/export, Lextures cannot participate in any institutional workflow that involves grade passback to a registrar, transfer to an external SIS, or accreditation reporting.

## 2. Goals

- Allow instructors to export the full gradebook for a course as a CSV file including student info, per-assignment raw scores, display grades, and final course grade.
- Allow instructors to upload a CSV file to update grades in bulk, with validation and a preview-before-confirm workflow.
- Log every bulk import as a series of grade-change audit events (3.10).
- Support round-trip fidelity: export → edit in spreadsheet → import back without data loss.
- Optionally export in a format compatible with common SIS systems (Canvas CSV format as baseline).

## 3. Non-Goals

- LTI-based grade passback (deferred to 14.5, 2.12).
- Direct SIS API integration (deferred to 14.1).
- Import of submission files (only grade values imported).
- Importing standards-based proficiency scores (CSV export of SBG deferred to 3.7 follow-on).

## 4. Personas & User Stories

- **As an instructor**, I want to download a CSV of all grades so that I can share them with my department or upload to the registrar portal.
- **As a TA**, I want to upload a CSV with scores I entered in a spreadsheet for a 200-student exam so that I do not need to enter each grade individually.
- **As an admin**, I want all bulk import operations logged so that I can audit who changed which grades in a grade dispute.
- **As an instructor**, I want to preview the changes a CSV import will make before confirming so that accidental overwrites do not happen silently.

## 5. Functional Requirements

- **FR-1.** The system MUST export the gradebook as a CSV with columns: `student_id`, `student_name`, `student_email`, per-assignment columns (score + display_grade), `final_score`, `final_grade`.
- **FR-2.** The CSV MUST include a header row with assignment names and a metadata row indicating the assignment ID for round-trip matching.
- **FR-3.** The system MUST accept a CSV upload with the same column structure; validate rows for unknown student IDs, out-of-range scores, and unknown assignment IDs before processing.
- **FR-4.** The system MUST present a diff preview (added, updated, unchanged cells) before the instructor confirms the import.
- **FR-5.** The system MUST apply imported scores as grade updates, each emitting a `grade_audit_events` record with `action = 'updated'`, `changed_by` = importing user, and `reason = 'bulk_import'`.
- **FR-6.** The system MUST reject any row where the assignment is in manual-hold posting policy (3.8) and student identity is actively blind (3.3) without explicit confirmation.
- **FR-7.** The system SHOULD support an optional `excuse` column in the import CSV to mark a student's score as excused (3.12).
- **FR-8.** The system MAY support exporting in a Canvas-compatible gradebook CSV format as an alternative template for institutions migrating from Canvas.

## 6. Non-Functional Requirements

- **Performance** — Export of 500 students × 50 assignments generated in ≤ 5 s. Import of 500 rows validated and previewed in ≤ 3 s.
- **Security** — Export and import restricted to instructor/owner role. Export served as a file download (not stored in object storage). Uploaded CSV processed server-side; XSS via CSV formula injection mitigated by prefixing any cell value starting with `=`, `+`, `-`, `@` with a single quote.
- **Privacy & Compliance** — Exported CSV contains student PII (name, email) — FERPA-protected. File delivered directly to browser as `Content-Disposition: attachment`; not cached on server. Audit events on import are part of the grade record.
- **Accessibility** — Import/export buttons keyboard accessible; validation error messages presented in a screen-reader-friendly list. WCAG 2.1 AA.
- **Scalability** — CSV generation streams rows; no full in-memory materialisation for large courses.
- **Reliability** — Import is transactional: all rows succeed or none; no partial update on validation failure.
- **Observability** — Log `gradebook_export` and `gradebook_import` events with `course_id`, row_count. Alert on import errors > 5% of rows.
- **Maintainability** — CSV service in `server/src/services/grading/csv.rs`.
- **Internationalization** — CSV header column names configurable (default English); date/number format locale-aware in export.
- **Backward compatibility** — New endpoints; no changes to existing gradebook API.

## 7. Acceptance Criteria

- **AC-1.** *Given* a course with 3 students and 2 assignments, *when* the instructor downloads the gradebook CSV, *then* the file has 3 data rows and columns for both assignments plus final grade.
- **AC-2.** *Given* a valid CSV with 2 updated scores, *when* the instructor uploads it and confirms, *then* both grades are updated and 2 `grade_audit_events` rows with `reason = 'bulk_import'` are created.
- **AC-3.** *Given* a CSV with a student_id that does not exist in the course, *when* validation runs, *then* the preview shows an error for that row and the instructor cannot confirm the import.
- **AC-4.** *Given* a CSV cell with value `=SUM(A1:A10)` (formula injection attempt), *when* the value is exported back, *then* the raw score is exported, not the formula; the import path treats it as a string/error.
- **AC-5.** *Given* a 500-student course, *when* the export is requested, *then* the response begins streaming within 2 s and completes within 7 s.
- **AC-6.** *Given* an import CSV where one row has score 150 but max_points = 100, *when* validation runs, *then* that row is flagged as invalid and the instructor is warned (not auto-rejected — out-of-range scores are permitted with confirmation).

## 8. Data Model

No new tables. Reads from `course_grades`, `module_assignments`, `assignment_groups`, `users`, and `course_enrollments`. Writes to `course_grades` and `grade_audit_events`.

Temporary staging table (optional, for large imports):
```sql
-- Transient; created per import job, dropped after confirm/cancel
-- No persistent migration needed; handled in-memory for ≤500 rows.
```

## 9. API Surface

| Verb | Path | Auth | Description |
|---|---|---|---|
| GET | `/api/courses/:cid/gradebook.csv` | instructor | Export gradebook as CSV download |
| POST | `/api/courses/:cid/gradebook/import/validate` | instructor | Validate CSV, return diff preview |
| POST | `/api/courses/:cid/gradebook/import/confirm` | instructor | Apply validated import |
| DELETE | `/api/courses/:cid/gradebook/import/:token` | instructor | Cancel pending import |

Import validation returns:
```ts
{ token: string; stats: { unchanged, updated, added, errors }; rows: ImportPreviewRow[] }
```

## 10. UI / UX

Modified: `clients/web/src/pages/lms/CourseGradebook.tsx` — add "Export CSV" button and "Import CSV" button in toolbar.
New modal: `GradebookImportModal.tsx` — file picker, validation progress, diff preview table, confirm/cancel.

Key flows:
1. Instructor clicks "Export" → browser downloads CSV immediately.
2. Instructor clicks "Import" → file picker → upload → validation spinner → diff table shown → "Confirm" or "Cancel."
3. Error rows highlighted in red; instructor can correct file and re-upload.

ARIA: diff preview table with `role="grid"`, status cells `aria-label="Score changed from {old} to {new}"`.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Formula injection in exported CSV | H | M | Prefix cells with `'` if starting with formula characters |
| Import overwrites manually adjusted grades silently | H | H | Diff preview required before confirm; audit trail captures change |
| Encoding issues for international character sets | M | M | Export as UTF-8 with BOM for Excel compatibility |

## 15. Rollout Plan

- Feature flag: `GRADEBOOK_CSV_ENABLED` (default: `false`).
- Phase 1: export only.
- Phase 2: import with validation preview.
- GA criteria: round-trip test (export → modify → import → verify) passes for 100-student course.

## 16. Test Plan

- **Unit** — CSV generation for all grade types; formula injection sanitisation; row validation logic.
- **Integration** — Export endpoint: verify CSV columns and row count; import: valid + invalid rows; audit events created.
- **End-to-end** — Playwright: export CSV → edit in memory → import → verify gradebook updated.
- **Security** — Formula injection test; unauthorised user gets 403; student cannot access export.
- **Accessibility** — Import modal axe-core; keyboard navigation of diff table.
- **Performance** — Export 500×50 matrix ≤ 5 s.
