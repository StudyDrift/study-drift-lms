## 1. Problem Statement

Positive Behavioral Interventions and Supports (PBIS) is a federal initiative (IDEA) used in over 26,000 US schools to create consistent, data-driven behavioral frameworks. Schools implementing PBIS award positive recognition points to students for meeting behavioral expectations and document negative referrals for incidents. Lextures has no behavior module, no referral workflow, and no PBIS point ledger. Competing LMS products (Schoology, PowerSchool Behavior) include this, and it is a frequent RFP requirement from elementary and middle schools. Without it, Lextures cannot serve as the behavioral management layer alongside its academic tools.

## 2. Goals

- Allow teachers to award PBIS positive points to individual students or whole classes.
- Allow teachers and administrators to file positive and negative behavior referrals.
- Provide a per-student behavior summary accessible to administrators and parents (read-only via §13.1).
- Generate a school-level PBIS data report showing aggregate behavior trends.
- Keep the system lightweight — this is not a full student information system; the focus is on classroom-level logging.

## 3. Non-Goals

- Full PBIS tier-2/tier-3 intervention tracking (multi-disciplinary team workflow).
- Student discipline records, suspension/expulsion workflow (SIS territory).
- Behavior goal tracking tied to IEPs (§12.10 accommodations engine is adjacent; full behavior IEP is out of scope).
- Integration with Panorama, PBIS Rewards, or ClassDojo in v1 (future §16 integrations).

## 4. Personas & User Stories

- **As a teacher**, I want to award a "Wildcat Point" (school-specific positive behavior recognition) to a student in 5 seconds so that I can reinforce good behavior without disrupting instruction.
- **As an administrator**, I want to file a behavior referral documenting the incident, category, and response so that I have a record for pattern analysis.
- **As a principal**, I want to see a weekly PBIS dashboard showing which classrooms are awarding the most points and which behavior categories are most frequent so that I can allocate support resources.
- **As a parent**, I want to see my child's behavior recognitions and referrals in the parent portal so that I stay informed about their behavioral experience at school.

## 5. Functional Requirements

- **FR-1.** The system MUST allow teachers to award positive PBIS points to one or more students in their sections with a behavior category (e.g., "Respect", "Responsibility", "Safety") and optional note.
- **FR-2.** The system MUST allow teachers and office staff to file negative behavior referrals with: student, date/time, location, behavior category, incident description, and initial response taken.
- **FR-3.** Behavior categories MUST be configurable per school (§5.2) by administrators, with a default PBIS set (Respect, Responsibility, Safety, Kindness).
- **FR-4.** Each student MUST have a cumulative PBIS points balance and a referral history, viewable by the student (read-only), instructors, administrators, and parents (via §13.1).
- **FR-5.** The system MUST provide a school-level PBIS analytics dashboard: daily/weekly points awarded, referrals by category, top/bottom referring classes, and per-student point leaderboards (optional).
- **FR-6.** Behavior data (especially negative referrals) MUST be FERPA-protected; access limited to school officials with legitimate educational interest.
- **FR-7.** The system SHOULD allow bulk positive-point awards to an entire class ("Everyone in 3rd period gets a safety point today").
- **FR-8.** Negative referrals MUST trigger a configurable notification to parents (via §6.2 email and §13.1 parent portal) if school policy enables it.

## 6. Non-Functional Requirements

- **Performance** — PBIS point award must save in < 200 ms. Dashboard loads in < 1 s p95.
- **Security** — Negative referral incident descriptions are FERPA-protected; access enforced in `server/src/authz.rs`. Students cannot view each other's referral records.
- **Privacy & Compliance** — FERPA: behavior referrals are education records (§99.3 definition). COPPA: behavior data for under-13 subject to data minimization. IDEA PBIS framework: positive behavioral supports are legally required for students with IEPs. State mandates vary (CA, NY have specific discipline record requirements).
- **Accessibility** — Referral form and PBIS award UI must be WCAG 2.1 AA compliant; screen-reader accessible; keyboard-navigable.
- **Scalability** — A school with 800 students awarding 10 points/student/day = 8,000 records/day; 1.4M/school year. Indexed by `(student_id)` and `(school_id, date)`.
- **Reliability** — Point awards are idempotent (duplicate award prevention within 1-second window). Referrals use UUID; no duplicates.
- **Observability** — Metrics: `pbis_points_awarded_per_day`, `referrals_filed_per_day`. Dashboard query < 1 s.
- **Maintainability** — Behavior categories as configurable data; new categories via admin UI without code changes.
- **Internationalization** — Category labels and referral form fields i18n-ready.
- **Backward compatibility** — New tables only; no existing schema affected.

## 7. Acceptance Criteria

- **AC-1.** *Given* a teacher opens the PBIS award panel for their class of 28 students, *When* they select "Responsibility" and click "Award to all", *Then* 28 point records are saved in < 500 ms and the UI shows a success confirmation.
- **AC-2.** *Given* an administrator files a negative referral for a student with category "Disruptive behavior" and a description, *When* saved, *Then* the referral appears in the student's behavior history and triggers a parent notification email if the school policy enables it.
- **AC-3.** *Given* a parent views the behavior summary for their child in the parent portal, *When* there are 15 positive points and 1 referral, *Then* the summary shows the total points, point category breakdown, and referral date/category (not the incident description).
- **AC-4.** *Given* a student attempts to view another student's behavior referrals via the API, *When* the request is received, *Then* the server returns HTTP 403.
- **AC-5.** *Given* a principal views the PBIS dashboard for the current week, *When* rendered, *Then* it shows top-5 point categories, total points across school, and referral count by category.
- **AC-6.** *Given* axe-core runs on the PBIS award page and referral form, *When* the check executes, *Then* zero Critical or Serious violations are reported.

## 8. Data Model

```sql
-- server/migrations/082_behavior_pbis.sql
CREATE TABLE behavior_categories (
    id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    school_id UUID NOT NULL REFERENCES schools(id),
    name      TEXT NOT NULL,
    type      TEXT NOT NULL,  -- positive | negative
    color     TEXT,
    active    BOOLEAN NOT NULL DEFAULT true,
    UNIQUE(school_id, name)
);

CREATE TABLE pbis_awards (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id   UUID NOT NULL REFERENCES users(id),
    awarded_by   UUID NOT NULL REFERENCES users(id),
    category_id  UUID NOT NULL REFERENCES behavior_categories(id),
    points       INTEGER NOT NULL DEFAULT 1,
    note         TEXT,
    awarded_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX pbis_student_idx ON pbis_awards(student_id, awarded_at DESC);

CREATE TABLE behavior_referrals (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id    UUID NOT NULL REFERENCES users(id),
    filed_by      UUID NOT NULL REFERENCES users(id),
    school_id     UUID NOT NULL REFERENCES schools(id),
    category_id   UUID NOT NULL REFERENCES behavior_categories(id),
    incident_at   TIMESTAMPTZ NOT NULL,
    location      TEXT,
    description   TEXT NOT NULL,  -- FERPA-protected
    response      TEXT,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX referral_student_idx ON behavior_referrals(student_id, incident_at DESC);
```

## 9. API Surface

- `POST /api/pbis/awards` — award points; instructor/admin.
- `GET /api/students/:id/behavior` — student behavior summary; instructor/admin/parent (limited)/self.
- `POST /api/behavior/referrals` — file referral; instructor/admin.
- `GET /api/schools/:id/behavior/dashboard` — school-level PBIS analytics; admin.
- `GET /api/admin/behavior/categories` — manage behavior categories.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Negative referral data exposed to unauthorized users | L | H | Strict FERPA authz; security test matrix |
| PBIS point inflation (gaming by teachers) | M | L | Audit log of awards; admin can review per-teacher award rates |
| Parent notification fatigue for minor incidents | M | M | Configurable notification threshold per school; default OFF for parent referral emails |
