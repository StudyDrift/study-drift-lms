## 1. Problem Statement

Parent-teacher conferences occur 2–4 times per year in most K-12 schools and require a scheduling system that allows parents to book 10–20 minute slots with each of their child's teachers. Schools currently use standalone tools (SignUpGenius, Calendly, paper sign-up sheets) because their LMS does not offer scheduling. Lextures has no scheduling module, no availability management for teachers, and no booking interface for parents. Integrating conference scheduling into the parent portal (§13.1) and teacher dashboard makes Lextures the primary communication hub for parent-school interaction, reducing the need for fragmented third-party tools and increasing daily-active use of the platform.

## 2. Goals

- Allow teachers to define available time slots for parent-teacher conferences.
- Allow parents to book a slot for each of their linked children's teachers from the parent portal.
- Send automated email confirmations, reminders (24h before), and calendar invites (ICS) to both parties.
- Provide school-level scheduling coordination (block-out all-school conference days, set universal slot durations).
- Support virtual (video link) and in-person (room location) conference modes.

## 3. Non-Goals

- General staff scheduling (substitute coordination, PD day scheduling).
- Integration with Google Calendar, Outlook Calendar (ICS download covers basic need; deep sync deferred to §16.5).
- Multi-teacher or IEP team conferences in v1 (single teacher per slot).
- Student-led conferences (different workflow; deferred).

## 4. Personas & User Stories

- **As a teacher**, I want to set my available slots for the November conference week (e.g., Tuesday 3–7 PM in 15-minute increments) so that parents can book online without emailing me.
- **As a parent**, I want to see all my children's teachers' available slots and book one for each teacher in a single session so that I can schedule the whole conference night efficiently.
- **As a school secretary**, I want to see a room assignment grid for conference night so that I can ensure teachers are not double-booked in the same room.
- **As a parent**, I want to receive a calendar invite for my booked conference so that it is in my calendar automatically.

## 5. Functional Requirements

- **FR-1.** Teachers MUST be able to define conference availability windows: date range, daily start/end times, and slot duration (5 / 10 / 15 / 20 / 30 minutes).
- **FR-2.** The system MUST auto-generate individual time slots from the availability window, with configurable gaps between slots (e.g., 5-minute buffer).
- **FR-3.** Parents MUST be able to browse available slots per teacher (for each of their linked children) and book a slot from the parent portal (§13.1).
- **FR-4.** A slot MUST be lockable to prevent double-booking (optimistic lock at booking time; last-writer-wins protected by DB row lock).
- **FR-5.** The system MUST send email confirmations (§6.2) to both parent and teacher upon booking, modification, or cancellation, including an ICS calendar attachment.
- **FR-6.** The system MUST send a reminder email 24 hours before each booked conference.
- **FR-7.** Conferences can be flagged as virtual (include a video-conference link, e.g., Google Meet URL or Jitsi — teacher-supplied) or in-person (room number).
- **FR-8.** School admins MUST be able to view a conference schedule grid showing all bookings across all teachers for the conference day, including room utilization.

## 6. Non-Functional Requirements

- **Performance** — Slot availability query for one teacher (100 slots max): < 100 ms. Booking confirmation: < 500 ms.
- **Security** — Parents can only book slots with teachers of their linked children; authz enforced. Teacher's personal contact details not exposed to parents beyond what is configured. Booking records are FERPA-protected.
- **Privacy & Compliance** — FERPA: conference records (who met with whom, notes) are education records. ICS calendar invite includes child's name; treat as PII. COPPA: conference involves minor's educational data. Email addresses of parents and teachers must not be shared cross-party in the ICS header.
- **Accessibility** — Slot calendar grid must be keyboard-navigable (WCAG SC 2.1.1); available/booked states must not be color-alone (icons required, WCAG SC 1.4.1); booking confirmation dialog accessible (§12.1 dialog pattern).
- **Scalability** — A school with 50 teachers and 200 conference slots each = 10,000 slots per conference event. Row-level locking for concurrent booking.
- **Reliability** — Double-booking prevented by DB unique constraint on `(slot_id, booked_by_parent_id)`. ICS generation is synchronous and not dependent on external services.
- **Observability** — Metrics: `conferences_booked_per_day`, `slot_fill_rate_pct`, `reminder_email_send_rate`.
- **Maintainability** — Conference scheduling in `server/src/routes/conferences.rs`; ICS generation in `server/src/services/ics_generator.rs`.
- **Internationalization** — Date/time formatting locale-aware; email templates i18n-ready.
- **Backward compatibility** — New feature; no existing tables affected.

## 7. Acceptance Criteria

- **AC-1.** *Given* a teacher sets availability "Nov 18, 4–6 PM" with 15-minute slots and 5-minute gaps, *When* availability is saved, *Then* 6 bookable slots are generated (4:00, 4:20, 4:40, 5:00, 5:20, 5:40).
- **AC-2.** *Given* a parent books the 4:00 PM slot with Teacher A, *When* another parent simultaneously tries to book the same slot, *Then* only one booking succeeds; the second parent sees "This slot is no longer available."
- **AC-3.** *Given* a booking is confirmed, *When* the parent checks their email, *Then* they receive a confirmation with teacher name, date/time, location/video link, and an ICS attachment parseable by Apple Calendar.
- **AC-4.** *Given* a conference is booked for tomorrow, *When* the 24-hour reminder job runs, *Then* both the parent and teacher receive a reminder email.
- **AC-5.** *Given* a school admin views the conference schedule grid, *When* 40 of 50 teachers have booked slots, *Then* the grid shows each teacher's bookings color-coded by booked/available/cancelled, with room assignments.
- **AC-6.** *Given* a keyboard-only parent navigates the slot picker, *When* they tab through available slots, *Then* each slot button has focus indicator and screen reader announces "Nov 18, 4:00 PM with Ms. Smith — available, press Enter to book."

## 8. Data Model

```sql
-- server/migrations/091_conference_scheduling.sql
CREATE TABLE conference_availability (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    teacher_id      UUID NOT NULL REFERENCES users(id),
    school_id       UUID NOT NULL REFERENCES schools(id),
    date            DATE NOT NULL,
    slot_duration   INTEGER NOT NULL,   -- minutes
    gap_duration    INTEGER NOT NULL DEFAULT 0,
    window_start    TIME NOT NULL,
    window_end      TIME NOT NULL,
    location        TEXT,
    video_link      TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE TABLE conference_slots (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    availability_id   UUID NOT NULL REFERENCES conference_availability(id),
    start_at          TIMESTAMPTZ NOT NULL,
    end_at            TIMESTAMPTZ NOT NULL,
    status            TEXT NOT NULL DEFAULT 'open',  -- open | booked | cancelled
    booked_by_parent  UUID REFERENCES users(id),
    booked_for_child  UUID REFERENCES users(id),
    booked_at         TIMESTAMPTZ,
    UNIQUE(availability_id, start_at)
);
```

## 9. API Surface

- `POST /api/teachers/:id/conference-availability` — teacher sets availability.
- `GET /api/teachers/:id/conference-slots?date=YYYY-MM-DD` — list available slots.
- `POST /api/conference-slots/:id/book` — parent books slot.
- `DELETE /api/conference-slots/:id/book` — parent cancels booking.
- `GET /api/admin/schools/:id/conference-schedule` — school-level grid.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Race condition on slot booking | M | H | DB `SELECT ... FOR UPDATE` on slot row during booking |
| ICS format incompatibility with Outlook | M | M | Test ICS with Apple Calendar, Google Calendar, Outlook; validate RFC 5545 compliance |
| Teacher forgetting to set availability before conference season | H | M | Admin can set availability on behalf of teachers; reminder email 2 weeks before |

## 15. Rollout Plan

- Feature flag: `ff_conference_scheduling` (default `false`).
- Pilot: One school conference night with 10 teachers.
- GA: All 6 ACs pass; ICS tested with top 3 calendar clients.

## 16. Test Plan

- **Unit:** Slot generation from availability window; ICS formatting (RFC 5545).
- **Integration:** Concurrent booking race condition; reminder job.
- **Security:** Parent can only book with their child's teachers; teacher cannot see parent email in API response.
- **Accessibility:** SlotPicker keyboard navigation; color-not-alone for slot states.
