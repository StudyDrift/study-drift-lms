## 1. Problem Statement

Every K-12 district runs a Student Information System (SIS) as the authoritative source of truth for student enrollment, sections, bell schedules, grade passback, and state reporting. Lextures must integrate with the four most prevalent US K-12 SIS platforms — PowerSchool (46% market share), Infinite Campus (24%), Skyward (14%), and Aeries (California-dominant) — to participate in the K-12 market. Without SIS integration, districts must maintain dual rosters and manually re-enter grades, which is an absolute non-starter. The §4.3 OneRoster 1.2 plan delivers the standard protocol layer; this plan delivers the vendor-specific connectors and grade-passback workflow on top of that foundation. Cross-reference: §4.3 OneRoster 1.2.

## 2. Goals

- Implement OneRoster 1.2 REST pull from PowerSchool, Infinite Campus, Skyward, and Aeries endpoints.
- Sync students, teachers, sections, enrollments, and academic sessions (terms) from SIS to Lextures on a configurable schedule.
- Pass back final grades from Lextures to the SIS at grading-period close.
- Provide a sync audit log showing what was created/updated/deleted on each sync run.
- Handle incremental delta syncs (not full replacement each time).

## 3. Non-Goals

- Real-time SIS sync (event-driven from SIS) — polling is sufficient for v1.
- Behavior referral sync to SIS (bidirectional; complex; deferred to v2).
- SIS for higher-ed (Banner, Workday — §14.1 owns that).
- Financial data (SIS may contain lunch status, fee records — those are handled in §13.13).

## 4. Personas & User Stories

- **As a district IT director**, I want Lextures to pull our PowerSchool roster every night so that all class lists in Lextures are always current with section changes made in the SIS.
- **As a registrar**, I want to push final Q1 grades from Lextures back to Infinite Campus in one click so that I don't have to re-enter 2,000 grades manually.
- **As a district data coordinator**, I want a sync audit log showing exactly which records were created, updated, or skipped on each sync run so that I can troubleshoot discrepancies.
- **As an IT director**, I want to configure the SIS integration once with our OAuth credentials and then have syncs run automatically every night at 2 AM.

## 5. Functional Requirements

- **FR-1.** The system MUST implement an OneRoster 1.2 REST consumer (cross-ref §4.3) with endpoints for `orgs`, `academicSessions`, `courses`, `classes`, `users`, and `enrollments`.
- **FR-2.** The system MUST support the following SIS vendor connections via their OneRoster 1.2 or native REST API: PowerSchool (OneRoster + native PS API v2), Infinite Campus (OneRoster), Skyward (OneRoster), Aeries (native Aeries API v3 + OneRoster).
- **FR-3.** Each SIS connector MUST support OAuth 2.0 client credentials authentication and token refresh.
- **FR-4.** Sync MUST run on a configurable schedule (default: nightly at 2 AM, configurable per org) via the background job queue (§17.3).
- **FR-5.** Sync MUST be incremental where the SIS supports delta APIs; full-replacement sync MUST be available as a fallback with a district-configurable "sync mode".
- **FR-6.** The system MUST support grade passback: at grading-period close, an authorized user sends final course grades from Lextures to the SIS via OneRoster `results` resource or native SIS grade-passback API.
- **FR-7.** A sync audit log MUST record: sync start/end timestamps, records synced by type (created/updated/skipped/errored), error messages with record IDs, and the SIS vendor/connection name.
- **FR-8.** The system MUST handle SIS data conflicts gracefully: if a student exists in Lextures but is dropped from the SIS, mark enrollment as `inactive` (not delete) and notify the admin.

## 6. Non-Functional Requirements

- **Performance** — Incremental sync for a 1,000-student school must complete in < 5 minutes. Grade passback for 500 students must complete in < 2 minutes.
- **Security** — SIS OAuth credentials stored in secrets manager (§17.17), never in DB plain text. SIS data in transit encrypted (HTTPS). Sync service uses a dedicated service account, not a user account. FERPA: SIS data treated as education records.
- **Privacy & Compliance** — FERPA: SIS data is student PII; access restricted to admins and authorized sync service. OneRoster 1.2: https://www.imsglobal.org/activity/oneroster. CALPADS: attendance and demographic data synced to state-reportable format. COPPA: under-13 student data handled with data-minimization.
- **Accessibility** — Sync configuration UI must be WCAG 2.1 AA. Audit log table must be keyboard-navigable (§12.1 ARIA grid).
- **Scalability** — Multi-org; each org has independent SIS connection config. Background job parallelization for large districts (shard by school).
- **Reliability** — Sync failures log errors and alert admin but do not roll back partial syncs; idempotent upsert on `external_id`. Retry failed records up to 3 times with exponential backoff.
- **Observability** — Metrics: `sis_sync_duration_seconds`, `sis_sync_records_created`, `sis_sync_errors_total` by vendor. Alert: sync failure rate > 5% or duration > 10 min.
- **Maintainability** — Vendor connectors implemented as Rust traits in `server/src/services/sis/`: `SISConnector` trait + `PowerSchoolConnector`, `InfiniteCampusConnector`, `SkywardConnector`, `AeriesConnector` implementations.
- **Internationalization** — SIS data arrives in English; localization of Lextures display is separate.
- **Backward compatibility** — Manually-created Lextures accounts gain an `external_id` on first sync match; no accounts deleted.

## 7. Acceptance Criteria

- **AC-1.** *Given* a district configures a PowerSchool OneRoster connection with valid OAuth credentials, *When* they trigger a manual sync, *Then* all students, sections, and enrollments for the current term are created in Lextures and the sync log reports zero errors.
- **AC-2.** *Given* a student is dropped from a section in the SIS between syncs, *When* the nightly sync runs, *Then* the student's enrollment in Lextures is marked `inactive` and the audit log records "1 enrollment deactivated."
- **AC-3.** *Given* a registrar triggers grade passback for Q1 in Lextures, *When* the passback completes, *Then* final grades for 500 students are reflected in the Infinite Campus gradebook and the audit log shows 500 records sent with 0 errors.
- **AC-4.** *Given* the SIS API is temporarily unavailable during a scheduled sync, *When* the retry logic exhausts 3 attempts, *Then* the admin receives an alert email "SIS sync failed: PowerSchool API unreachable" and the previous sync state is preserved.
- **AC-5.** *Given* a district admin views the sync audit log, *When* the last sync created 50 users and updated 10 sections, *Then* the log shows itemized counts by type (users_created: 50, sections_updated: 10) and a total duration.
- **AC-6.** *Given* the OneRoster `grade` field is "07" for imported students, *When* the sync completes, *Then* `users.grade_level = '7'` for all imported students (cross-ref §13.6).

## 8. Data Model

```sql
-- server/migrations/086_sis_integration.sql
CREATE TABLE sis_connections (
    id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id         UUID NOT NULL REFERENCES organizations(id),
    vendor         TEXT NOT NULL,  -- powerschool | infinite_campus | skyward | aeries
    base_url       TEXT NOT NULL,
    client_id_ref  TEXT NOT NULL,  -- reference to secrets manager key
    client_secret_ref TEXT NOT NULL,
    sync_schedule  TEXT NOT NULL DEFAULT '0 2 * * *',  -- cron
    sync_mode      TEXT NOT NULL DEFAULT 'incremental',  -- incremental | full
    active         BOOLEAN NOT NULL DEFAULT true,
    last_sync_at   TIMESTAMPTZ,
    created_at     TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE sis_sync_logs (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    connection_id UUID NOT NULL REFERENCES sis_connections(id),
    started_at   TIMESTAMPTZ NOT NULL,
    finished_at  TIMESTAMPTZ,
    status       TEXT NOT NULL,  -- running | success | partial | failed
    summary      JSONB,          -- {users_created: N, enrollments_updated: N, ...}
    errors       JSONB           -- [{record_id, message}, ...]
);

ALTER TABLE users        ADD COLUMN external_sis_id TEXT;
ALTER TABLE courses      ADD COLUMN external_sis_id TEXT;
ALTER TABLE enrollments  ADD COLUMN external_sis_id TEXT;
CREATE INDEX users_sis_id ON users(external_sis_id) WHERE external_sis_id IS NOT NULL;
```

## 9. API Surface

- `GET /api/admin/sis/connections` — list SIS connections; admin.
- `POST /api/admin/sis/connections` — create SIS connection.
- `PATCH /api/admin/sis/connections/:id` — update connection config.
- `POST /api/admin/sis/connections/:id/sync` — trigger manual sync.
- `GET /api/admin/sis/sync-logs` — paginated sync audit log.
- `POST /api/admin/sis/grade-passback` — initiate grade passback for a grading period.

## 10. UI / UX

- **New pages:** `clients/web/src/pages/admin/SISIntegration.tsx` — connection setup wizard (vendor select, OAuth config, schedule, test connection button). `SISSyncLogs.tsx` — audit log table with filters.
- **User flows:** 1. Admin adds connection → test → save → first sync runs. 2. Nightly sync runs automatically → admin reviews log next morning. 3. End of Q1 → registrar triggers grade passback → confirms in SIS.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| SIS vendor API changes break connector | H | H | Version pin; vendor change log monitoring; integration tests against sandbox |
| Full sync re-creates all users, wiping manual changes | H | H | Match on `external_sis_id`; upsert, never delete; soft-deactivation only |
| District has mixed SIS (elementary on Aeries, secondary on Infinite Campus) | M | M | Support multiple SIS connections per org |
| OAuth token expiry during long sync | M | M | Refresh token before each batch; handle 401 with re-auth |

## 15. Rollout Plan

- Feature flag: `ff_sis_integration` (default `false`).
- Sequencing: PowerSchool connector first (largest market) → Infinite Campus → Skyward → Aeries.
- Pilot: One PowerSchool district (test sandbox) → Infinite Campus district.
- GA: All 6 ACs pass per vendor; grade passback validated with district registrar.
