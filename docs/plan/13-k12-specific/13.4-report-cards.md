## 1. Problem Statement

Report cards are one of the most legally and operationally significant outputs of K-12 education. Every school district must produce formatted grade reports at the end of each grading period (quarters, trimesters, semesters) that include letter grades, attendance summaries, teacher comments, and sometimes behavioral ratings. Lextures has no grading periods (§5.3), no letter-grade scales (§3.6), no comment banks, no narrative writing tools, and no PDF rendering for printed or distributed report cards. Without report cards, Lextures cannot serve as the authoritative gradebook in any K-12 school, and the gradebook data collected all semester has no formal output artifact. This is a BLOCKER.

## 2. Goals

- Implement grading-period-scoped report cards with district-configurable templates.
- Support letter-grade and percentage final grades per course.
- Include attendance summary (§13.2) and optional behavior summary (§13.3).
- Provide a comment bank from which teachers select pre-written comments, with a free-text narrative option.
- Generate PDF report cards formatted per district template, distributable via parent portal (§13.1).
- AI-assisted comment drafting that suggests a comment based on the student's grade and engagement metrics.

## 3. Non-Goals

- Standards-based report cards (§13.5 handles that as a separate plan).
- College transcript or official academic record generation (HE scope).
- Printing workflow / physical mailing (administrative process; Lextures generates the PDF).
- Multiple report card templates per course within one district (v1: one template per org).

## 4. Personas & User Stories

- **As a teacher**, I want to write report card comments for 30 students using a comment bank and free-text narrative so that I can complete report cards in under 2 hours rather than a full day.
- **As a principal**, I want to review and approve all report card comments before they are released to parents so that the district maintains communication quality standards.
- **As a parent**, I want to receive my child's report card as a PDF in the parent portal at the end of each quarter so that I have an official academic record.
- **As a district registrar**, I want to export all report cards for the semester as a ZIP of PDFs so that we can distribute them district-wide in one operation.
- **As an instructor**, I want AI to suggest a report card comment based on my student's grade and quiz performance so that I have a starting point I can personalize.

## 5. Functional Requirements

- **FR-1.** The system MUST support configurable grading periods per organization (§5.3): quarters, trimesters, semesters, with defined start/end dates.
- **FR-2.** At grading-period close, the system MUST compute a final course grade for each enrolled student (based on the grade weighting/categories configured for the course, per §3.6) and present it for instructor review.
- **FR-3.** Instructors MUST be able to write a report card comment per student per course, using: (a) a comment bank (admin-curated set of pre-written phrases); (b) free-text narrative input; (c) AI-suggested comment (see §11).
- **FR-4.** Comment banks MUST be organized by category (e.g., "Academic Effort", "Social Skills", "Areas for Growth") and searchable.
- **FR-5.** Completed report cards MUST be reviewable and approvable by a principal or admin before release.
- **FR-6.** Approved report cards MUST be renderable as PDF (district-branded with logo, term, student name, ID, grades, attendance totals, comments) accessible to parents in the portal (§13.1) and as a ZIP export for district admins.
- **FR-7.** The report card template MUST be configurable per organization: columns displayed, grade scale labels, attendance row inclusion, behavior row inclusion, signature lines.
- **FR-8.** The system SHOULD allow AI-assisted comment generation (see §11) that suggests a personalized comment draft based on the student's grade, quiz performance, and attendance in the course.

## 6. Non-Functional Requirements

- **Performance** — Report card PDF generation for one student: < 3 s. District-wide ZIP export for 1,000 students: < 5 minutes (background job, §17.3).
- **Security** — Report card PDFs are FERPA-protected; URLs must be signed/tokenized, not guessable. Only authorized parents and school officials can download a student's report card.
- **Privacy & Compliance** — FERPA (20 U.S.C. §1232g): report cards are education records; parent access rights apply. COPPA: student data in PDFs for under-13 subject to access controls. Grade records must be audited per §3.10 grade-change audit trail.
- **Accessibility** — Report card authoring UI must be WCAG 2.1 AA (§12 series). PDFs must be tagged PDFs (accessible PDF/UA) so screen readers can navigate them.
- **Scalability** — 1,000-student district × 4 grading periods = 4,000 PDFs/year. Background job queue (§17.3) handles bulk generation.
- **Reliability** — If PDF generation fails for a single student, continue with others; report failures in the job summary. Store generated PDFs in object storage (§8.1).
- **Observability** — Metrics: `report_cards_generated_per_period`, `comment_bank_usage_rate`, `ai_comment_acceptance_rate`. Alert on generation failure rate > 1%.
- **Maintainability** — PDF template engine isolated in `server/src/services/report_card_pdf.rs` (Typst or Headless Chrome/Puppeteer); templates stored as versioned files.
- **Internationalization** — Comment bank entries and UI labels i18n-ready; PDF content in district's configured language.
- **Backward compatibility** — Course grade calculation uses existing `client/web/src/pages/lms/gradebook/computeCourseFinalPercent.ts` logic.

## 7. Acceptance Criteria

- **AC-1.** *Given* a teacher has 30 students and it is the end of Q1, *When* they open the report card authoring view for their course, *Then* each student's computed final grade (%) and letter grade are displayed, and the teacher can select or type a comment for each.
- **AC-2.** *Given* a teacher selects a comment bank phrase "Shows strong effort and improvement throughout the quarter", *When* the report card is saved, *Then* the phrase appears verbatim in the generated PDF.
- **AC-3.** *Given* a principal approves a set of 30 report cards, *When* the release action is taken, *Then* the parent portal (§13.1) for each student's parent shows the report card PDF download link within 5 minutes.
- **AC-4.** *Given* a parent downloads a student's Q1 report card PDF, *When* the PDF is opened in Adobe Acrobat's accessibility checker, *Then* no accessibility errors are reported (tagged PDF/UA).
- **AC-5.** *Given* a district admin triggers a ZIP export for all 800 students for Q1, *When* the job completes, *Then* the ZIP file contains 800 PDFs named `{studentId}_Q1_2026.pdf` and the job summary shows 0 failures.
- **AC-6.** *Given* the AI comment suggestion is requested for a student with a 94% grade and 2 absences, *When* the AI responds, *Then* a suggested comment references the student's strong performance and mentions attendance, and the instructor can accept or edit it.
- **AC-7.** *Given* a parent's link to a report card PDF is shared with another user who is not linked to the student, *When* the other user opens the URL, *Then* they receive a 403 response (signed URL expired or not authorized).

## 8. Data Model

```sql
-- server/migrations/083_report_cards.sql
CREATE TABLE report_card_comments_bank (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id     UUID NOT NULL REFERENCES organizations(id),
    category   TEXT NOT NULL,
    text       TEXT NOT NULL,
    active     BOOLEAN NOT NULL DEFAULT true
);

CREATE TABLE report_cards (
    id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id       UUID NOT NULL REFERENCES users(id),
    course_id        UUID NOT NULL REFERENCES courses(id),
    grading_period   TEXT NOT NULL,          -- e.g. "Q1-2026"
    final_grade_pct  NUMERIC(5,2),
    letter_grade     TEXT,
    comment          TEXT,
    status           TEXT NOT NULL DEFAULT 'draft',  -- draft | submitted | approved | released
    pdf_url          TEXT,
    generated_at     TIMESTAMPTZ,
    released_at      TIMESTAMPTZ,
    created_at       TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE(student_id, course_id, grading_period)
);
```

## 9. API Surface

- `GET /api/courses/:id/report-cards/:period` — list all student report cards for a course/period.
- `PATCH /api/report-cards/:id` — update comment/grade; instructor.
- `POST /api/admin/report-cards/generate` — batch-generate PDFs for a period; background job.
- `POST /api/admin/report-cards/release` — release all approved cards for a period.
- `GET /api/report-cards/:id/pdf` — download signed PDF; parent/admin/student.
- `POST /api/ai/report-card-comment` — AI comment suggestion; instructor.

## 11. AI / ML Considerations

- AI comment suggestion via OpenRouter; prompt: "Write a 2-sentence report card comment for a student with a {grade}% average in {course}, {absences} absences, and {behavior_note}. Be encouraging and specific."
- Fallback: If AI unavailable, comment field is blank; instructor enters manually.
- PII: Student name passed to AI prompt; apply §19.11 PII proxy or substitute "the student" in prompt.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| PDF generation library choice (Headless Chrome vs. Typst) has complex deployment | M | M | Evaluate Typst (pure Rust) for server-side; Puppeteer as fallback |
| District template requirements vary wildly; one template won't fit all | H | M | Modular template engine with configurable sections; accept custom CSS per org |
| Comment bank AI suggestions contain inappropriate content | L | H | Content moderation filter on AI output; instructor must explicitly accept |

## 15. Rollout Plan
