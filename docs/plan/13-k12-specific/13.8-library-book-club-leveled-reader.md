## 1. Problem Statement

Elementary and middle schools heavily use independent reading programs, book clubs, and leveled-reader libraries as core instructional strategies. Teachers assign books from leveled libraries (e.g., Fountas & Pinnell bands A–Z, Lexile levels) and students track their reading in digital reading logs. Lextures has no library resource catalog, no reading-log widget, no book-club group structure, and no Lexile/F&P level metadata on content items. Platforms like Seesaw and Google Classroom (with integrated Newsela or Epic! links) provide these affordances. Without them, K-12 elementary teachers cannot use Lextures as their reading-program hub, limiting adoption in the highest-volume K-5 market segment.

## 2. Goals

- Provide a school-level digital library catalog where admins add books/resources with Lexile and Fountas & Pinnell level metadata.
- Allow teachers to assign specific books or reading levels to individual students or groups.
- Provide students with a reading log to record books read, pages completed, and a brief reflection.
- Enable book-club groups where students in a shared reading group post discussion questions and responses.
- Filter library materials by student grade level (§13.6) and reading level.

## 3. Non-Goals

- Full e-book reader / DRM-protected content delivery (§8.10 owns DRM).
- Integration with Epic!, Raz-Kids, Newsela, or other reading platforms in v1 (future §16 integrations).
- AI-generated leveled-text rewrites (§11.6 handles that separately).
- Public library catalog (this is a school/district-curated catalog).

## 4. Personas & User Stories

- **As a 2nd grade teacher**, I want to assign a Level H book from the school's digital library to my guided reading group so that students know which book to bring to our next session.
- **As a student**, I want to log the books I read each week and see my total pages read so that I can track my reading goal for the month.
- **As a teacher**, I want to see which students are on track with their reading goals so that I can follow up with struggling readers.
- **As a student in a book club**, I want to post my discussion question for this week's reading and respond to my classmates so that I can participate in our group conversation asynchronously.

## 5. Functional Requirements

- **FR-1.** The system MUST support a school-level library catalog where admins and teachers add books with: title, author, ISBN, cover image, Lexile level (numeric), Fountas & Pinnell band (A–Z), grade-level band, and summary.
- **FR-2.** Teachers MUST be able to assign books from the library to individual students, small groups, or the whole class with a target completion date.
- **FR-3.** Students MUST be able to log a reading entry: book (from library or free-text), date, pages read, and a brief reflection (1–3 sentences).
- **FR-4.** A reading-progress dashboard MUST show each student's weekly pages read vs. their goal, and the teacher can view an aggregated class reading log.
- **FR-5.** The system MUST support book-club course groups where students post discussion prompts and responses, using the existing course feed/discussion infrastructure (cross-ref §6.1 threaded discussions when available).
- **FR-6.** The library catalog MUST be filterable by Lexile range, F&P band, and grade level (§13.6).
- **FR-7.** The system SHOULD display a "recommended for you" book list based on the student's current reading level (from §11.6 or teacher-set Lexile target).

## 6. Non-Functional Requirements

- **Performance** — Library catalog search with filters: < 300 ms. Reading log save: < 200 ms.
- **Security** — Library catalog is school-scoped; students cannot access another school's catalog. Reading logs are student-private (visible to teacher and parent).
- **Privacy & Compliance** — FERPA: reading logs are education records. COPPA: reading data for under-13 students; parental consent for cloud storage. Book catalog data (titles, authors) is not PII.
- **Accessibility** — WCAG 2.1 AA: library grid filterable by keyboard; reading log form keyboard-accessible; book covers have alt text (§12.5).
- **Scalability** — A school library catalog of 1,000 books; 30 students × 200 log entries/year = 6,000 entries/class.
- **Reliability** — Reading log entries saved locally and synced (future PWA §7.3 use case).
- **Observability** — Metrics: `reading_log_entries_per_week`, `library_catalog_size`.
- **Maintainability** — Library catalog and reading log in separate `server/src/routes/library.rs` module.
- **Internationalization** — Book catalog multilingual; Lexile and F&P levels are numeric/alphabetic (universal).
- **Backward compatibility** — New feature; no existing tables affected.

## 7. Acceptance Criteria

- **AC-1.** *Given* an admin adds a book "Charlotte's Web" with Lexile 680, F&P level P, Grade 4, *When* a teacher filters the library by "Grade 3–5" and "Lexile 500–800", *Then* "Charlotte's Web" appears in results.
- **AC-2.** *Given* a student logs "Read pages 1–25 of Charlotte's Web on April 17", *When* their teacher views the class reading dashboard, *Then* the student's entry shows date, title, pages, and the weekly total updates.
- **AC-3.** *Given* a book club group has 5 students, *When* one student posts a discussion question, *Then* the other 4 group members can see and respond to it via the group feed.
- **AC-4.** *Given* a student's Lexile target is 700, *When* they view recommended books, *Then* the list shows books with Lexile range 650–750.
- **AC-5.** *Given* axe-core runs on the library catalog page, *When* the check executes, *Then* zero Critical or Serious violations are reported and all book cover images have alt text.

## 8. Data Model

```sql
-- server/migrations/087_library.sql
CREATE TABLE library_books (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    school_id    UUID NOT NULL REFERENCES schools(id),
    title        TEXT NOT NULL,
    author       TEXT,
    isbn         TEXT,
    cover_url    TEXT,
    lexile_level INTEGER,
    fp_band      TEXT,         -- A-Z
    grade_band   TEXT,         -- K-2, 3-5, etc.
    summary      TEXT
);
CREATE TABLE reading_log_entries (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id  UUID NOT NULL REFERENCES users(id),
    book_id     UUID REFERENCES library_books(id),
    book_title  TEXT,          -- free-text if not from catalog
    date        DATE NOT NULL,
    pages_read  INTEGER,
    reflection  TEXT,
    logged_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

## 9. API Surface

- `GET /api/schools/:id/library?lexile_min=X&lexile_max=Y&grade_band=Z` — catalog search.
- `POST /api/schools/:id/library` — add book; teacher/admin.
- `POST /api/students/:id/reading-log` — log entry; student.
- `GET /api/courses/:id/reading-dashboard` — class reading progress; teacher.

## 10. UI / UX

- **New pages:** `clients/web/src/pages/lms/LibraryCatalog.tsx`, `ReadingLog.tsx`, `ReadingDashboard.tsx`.
- **User flows:** 1. Teacher opens library → assigns book to group. 2. Student opens reading log → adds entry. 3. Teacher reviews class dashboard.
- **K-2 mode:** Large book cover images, minimal text, emoji ratings (cross-ref §13.11).

## 11. AI / ML Considerations

Book recommendation engine (FR-7): simple Lexile-range filter in v1; future: collaborative filtering or AI content matching.
