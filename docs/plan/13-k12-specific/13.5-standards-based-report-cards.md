## 1. Problem Statement

Standards-based grading (SBG) — also called competency-based or mastery-based grading — is used by a significant and growing portion of K-12 schools, particularly at the elementary level. Instead of a single letter grade per course, SBG reports a student's mastery level (e.g., "Exceeds Standard", "Meets Standard", "Approaching Standard", "Below Standard") for each specific learning standard (e.g., "CCSS.MATH.3.OA.A.1: Interpret products of whole numbers"). Lextures has no standards schema (§1.2, §1.3 are missing), no standard-level mastery score, and no SBG report card format. This is a BLOCKER for every elementary school that uses SBG, and it is an increasingly common requirement at the middle school level. Cross-reference: §3.7 (standards-based grading mastery transcripts).

## 2. Goals

- Implement a standards/learning outcomes hierarchy (domain → standard → sub-standard) per organization.
- Allow instructors to assess each standard independently on assignments and quizzes.
- Compute a grading-period mastery level per standard per student, based on aggregated evidence.
- Generate a standards-based report card showing mastery levels per standard, with configurable mastery scales.
- Produce PDFs and parent-portal views of SBG report cards (extends §13.4 infrastructure).

## 3. Non-Goals

- State-standard content catalog import (Common Core/NGSS bulk import deferred to §1.3).
- Competency-based progression (unlocking next unit only when mastery achieved) — that is §1.4 adaptive paths.
- Standards alignment for all existing quiz questions (retrofitting requires §1.2 concept graph).
- Higher-ed competency transcripts (§14.12 ePortfolio).

## 4. Personas & User Stories

- **As a 3rd grade teacher**, I want to record whether each student "Meets Standard" or "Approaching Standard" on CCSS.MATH.3.OA.A.1 after their unit assessment so that the report card reflects standards mastery rather than a percentage.
- **As an elementary school principal**, I want to see a school-wide snapshot of which standards have the highest proportion of students "Below Standard" so that I can focus professional development.
- **As a parent**, I want the report card to show my child's mastery level for each reading and math standard so that I understand what they specifically know and don't know.
- **As a district curriculum director**, I want to import our district's SBG standards list and mastery scale so that all schools use consistent definitions.

## 5. Functional Requirements

- **FR-1.** The system MUST support a hierarchical standards taxonomy: `standard_domains` → `standards` → `sub_standards`, configurable per organization (CCSS, NGSS, or custom district standards) (cross-ref §1.2, §1.3).
- **FR-2.** Instructors MUST be able to link individual assignments and quiz questions to one or more standards, and record a mastery-level score per standard per student (e.g., 4 = Exceeds, 3 = Meets, 2 = Approaching, 1 = Below).
- **FR-3.** The mastery scale (labels, numeric values, colors) MUST be configurable per organization with a default 4-level scale.
- **FR-4.** At grading-period close, the system MUST compute a final mastery level per standard per student using a configurable aggregation method: most-recent, highest, mode, or trend (most recent 3 assessments).
- **FR-5.** The standards-based report card MUST display: student name and ID, grading period, per-standard mastery level (color-coded), teacher comments per domain, attendance summary (§13.2), and the mastery scale legend.
- **FR-6.** SBG report cards MUST be generated as accessible PDFs (PDF/UA) and made available in the parent portal (§13.1) at grading-period release.
- **FR-7.** A district admin MUST be able to import standards from a CSV file containing code, description, domain, and grade level.
- **FR-8.** The system SHOULD provide an instructor view showing all students' mastery levels per standard in a heat-map grid, to identify whole-class gaps (cross-ref §9.3 mastery heatmap).

## 6. Non-Functional Requirements

- **Performance** — SBG report card PDF generation < 4 s per student. Standards heat-map grid for 30 students × 20 standards renders in < 1 s.
- **Security** — Standards mastery scores are FERPA-protected education records; access controlled same as grades.
- **Privacy & Compliance** — FERPA (20 U.S.C. §1232g). IDEA: IEP progress reporting can reference standard-level mastery. Standards are aligned to Common Core State Standards (CCSS), NGSS, or state-specific standards — all public frameworks.
- **Accessibility** — Heat-map grid must be keyboard-navigable with ARIA grid pattern (§12.1); mastery levels must not be distinguished by color alone (icon + text label required, WCAG SC 1.4.1).
- **Scalability** — 30 standards × 30 students × 4 assessments = 3,600 mastery score records per course per period. Indexed by `(student_id, standard_id, grading_period)`.
- **Reliability** — Mastery aggregation computed on demand (not pre-materialized in v1); caching with 5-minute TTL.
- **Observability** — Metrics: `sbg_report_cards_generated`, `standards_assessed_per_course`. Alert on aggregation query > 2 s.
- **Maintainability** — Aggregation methods implemented as strategy pattern in `server/src/services/sbg_aggregation.rs`.
- **Internationalization** — Mastery level labels externalized; standard descriptions stored in source language.
- **Backward compatibility** — SBG is additive to existing percentage-based grading; a course uses one mode or the other, not both.

## 7. Acceptance Criteria

- **AC-1.** *Given* a teacher records mastery assessments for 20 students on standard CCSS.MATH.3.OA.A.1 over 3 assignments, *When* the SBG report card is generated using "most recent" aggregation, *Then* each student's mastery level matches the score from their most recent assessment of that standard.
- **AC-2.** *Given* an organization uses a 4-level mastery scale (4=Exceeds, 3=Meets, 2=Approaching, 1=Below), *When* a parent downloads the SBG report card PDF, *Then* every standard row shows the correct mastery label, a colored indicator, and a text label (not color alone).
- **AC-3.** *Given* a district admin uploads a CSV of 35 CCSS Math Grade 3 standards, *When* the import completes, *Then* all 35 standards are available for teacher use and grouped by domain.
- **AC-4.** *Given* an instructor views the mastery heat-map for their class, *When* rendered, *Then* cells below "Meets Standard" are visually highlighted and the keyboard-only user can tab through every cell and hear "Standard X: Approaching Standard" via screen reader.
- **AC-5.** *Given* a parent portal displays the SBG report card, *When* there are 12 standards with mastery levels, *Then* all 12 are listed with labels and the accessibility PDF/UA check passes.
- **AC-6.** *Given* a course uses SBG mode, *When* an instructor tries to enter a numeric percentage grade for a student, *Then* the UI shows "This course uses standards-based grading" and prevents percentage entry.

## 8. Data Model

```sql
-- server/migrations/084_standards_sbg.sql
CREATE TABLE standard_domains (
    id      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id  UUID NOT NULL REFERENCES organizations(id),
    code    TEXT NOT NULL,
    name    TEXT NOT NULL,
    grade_level TEXT   -- "K" "1" "2" ... "12" or "K-2"
);
CREATE TABLE standards (
    id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    domain_id UUID NOT NULL REFERENCES standard_domains(id),
    code      TEXT NOT NULL,
    description TEXT NOT NULL
);
CREATE TABLE mastery_scales (
    id      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id  UUID NOT NULL REFERENCES organizations(id),
    label   TEXT NOT NULL,
    value   INTEGER NOT NULL,  -- 1..4
    color   TEXT
);
CREATE TABLE mastery_scores (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id      UUID NOT NULL REFERENCES users(id),
    standard_id     UUID NOT NULL REFERENCES standards(id),
    course_id       UUID NOT NULL REFERENCES courses(id),
    grading_period  TEXT NOT NULL,
    score_value     INTEGER NOT NULL,
    assessed_by     UUID REFERENCES users(id),
    source          TEXT NOT NULL,  -- assignment | quiz | observation
    source_id       UUID,
    assessed_at     TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX mastery_scores_student_std ON mastery_scores(student_id, standard_id, grading_period);
```

## 9. API Surface

- `GET /api/courses/:id/sbg/standards` — standards linked to course.
- `POST /api/sbg/mastery-scores` — record mastery score; instructor.
- `GET /api/courses/:id/sbg/heatmap` — mastery heat-map for class; instructor.
- `GET /api/students/:id/sbg/:period` — per-student SBG report; instructor/admin/parent.
- `POST /api/admin/standards/import` — CSV import; admin.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Standards import format inconsistency across districts | H | M | Accept both CASE format and simple CSV; document expected format |
| Teachers find mastery-level entry slower than percentage grading | M | M | Provide quick-entry keyboard shortcuts; bulk class entry |
| SBG and traditional grading in same district confuses parents | M | M | Report card clearly labeled "Standards-Based" vs. "Traditional" |

## 15. Rollout Plan

- Feature flag: `ff_sbg_report_cards` (default `false`).
- Pilot: One K-5 elementary school with existing SBG practice.
- GA: All 6 ACs pass; CCSS import validated; PDF accessibility check passes.

## 16. Test Plan

- **Unit:** Mastery aggregation strategies (most-recent, mode, trend); grade scale conversion.
- **Integration:** Mastery score recording; heatmap query; SBG PDF generation.
- **Accessibility:** Heatmap keyboard navigation; PDF/UA check; color-alone test.
- **Performance:** Heatmap 30×20 < 1 s; PDF generation < 4 s.
