## 1. Problem Statement

Higher-ed and self-learner communities often organize on Slack, Discord, or Microsoft Teams rather than within the LMS. Instructors want to announce new assignments, post due-date reminders, and surface grade notifications in channels where students are already active. Without bot integrations, instructors must manually copy-paste announcements between Lextures and their community platforms, and students must navigate back to the LMS to discover new content. A set of first-party bots converts Lextures event streams (via webhooks) into actionable messages in these platforms.

## 2. Goals

- Deliver a Slack bot that posts configurable event notifications (new assignments, due-date reminders, grade releases) to designated Slack channels.
- Deliver an equivalent Microsoft Teams bot for Teams Education workspaces.
- Deliver a Discord bot for self-learner and course community servers.
- Provide a slash-command interface (`/lextures status`) allowing students to check their upcoming due dates without leaving their messaging platform.
- Wire bot event delivery through the existing outbound webhook infrastructure (16.3) to avoid duplicating delivery logic.

## 3. Non-Goals

- Full conversational AI tutor via Slack/Teams/Discord (that is 6.9 / 19.1).
- Grade submission or assignment submission through bots (security and identity implications defer to later phase).
- Native Slack Canvas or Teams wiki integration (content sync is a longer-term roadmap item).
- WhatsApp or SMS channels (out of scope for this iteration).

## 4. Personas & User Stories

- **As a university instructor**, I want new assignment announcements automatically posted to my course Slack channel so that students see them in the tool they check most often.
- **As a self-learner on a Discord community server**, I want to type `/lextures upcoming` and see my next three due dates so that I can plan my study session without switching apps.
- **As an admin**, I want to connect our Microsoft Teams Education workspace to Lextures once and have all course channels receive relevant notifications automatically.
- **As a student**, I want to receive a direct message from the Lextures Slack bot 24 hours before an assignment is due so that I am reminded without checking my email.

## 5. Functional Requirements

- **FR-1.** The system MUST provide an OAuth 2.0 app installation flow for Slack ("Add to Slack" button), Teams (app manifest + admin consent), and Discord (bot invite URL).
- **FR-2.** The system MUST allow admins/instructors to map Lextures course events to specific workspace channels or direct-message the bot for each platform.
- **FR-3.** The system MUST deliver the following event types to connected channels: `assignment.created`, `assignment.due_soon` (configurable advance notice, default 24 h), `grade.released`, `announcement.created`.
- **FR-4.** The system MUST use the outbound webhook infrastructure (16.3) as the internal event delivery mechanism, with each bot acting as a webhook subscriber.
- **FR-5.** The system MUST support Slack slash command `/lextures upcoming` returning the user's next 5 upcoming items (if the user has linked their Lextures account via OAuth).
- **FR-6.** The system SHOULD support Discord slash command `/lextures upcoming` with the same behavior.
- **FR-7.** The system SHOULD support Slack Block Kit rich messages (due date, course name, direct link button) rather than plain text.
- **FR-8.** The system MAY support Teams Adaptive Cards for rich message rendering.
- **FR-9.** The system MUST allow users to link/unlink their Lextures account to their Slack/Discord user via an OAuth flow, enabling personal due-date reminders.

## 6. Non-Functional Requirements

- **Performance** — Bot notification delivery latency (event trigger → message posted) ≤ 5 seconds under normal load. Slash command responses must return within Slack's 3-second interactive timeout; use async response URL for anything slower.
- **Security** — Slack/Discord/Teams request signing verified using respective platform signing secrets on inbound slash commands. Bot tokens stored encrypted (17.17). User identity linking uses OAuth 2.0 authorization-code; never requires users to share their LMS password with the bot.
- **Privacy & Compliance** — Grade notifications sent to channels must be configurable to "release/no-release"; FERPA prohibits posting individual student grades to group channels. Personal DMs only for individual grade notifications.
- **Accessibility** — Slack Block Kit and Adaptive Cards must include alt text for any images; message structure should be parseable by screen readers used with Slack's desktop app.
- **Scalability** — Bot delivery handled by background job workers (17.3); platform API rate limits per-workspace respected via per-workspace token bucket.
- **Reliability** — Platform API failures (Slack outage) must not fail the Lextures event pipeline; bot delivery job retries independently per webhook retry policy (16.3).
- **Observability** — Counter `bot_messages_total{platform,event_type,status}`; alert on delivery failure rate > 5%.
- **Maintainability** — Each platform client in `server/src/services/bots/{slack,teams,discord}.rs`; shared message template logic in `server/src/services/bots/templates.rs`.
- **Internationalization** — Bot message templates externalized in i18n keys; language determined by workspace locale setting (with Lextures org locale as fallback).
- **Backward compatibility** — Adding a new event type to bot notifications is additive; existing connected workspaces must explicitly opt in to new event types.

## 7. Acceptance Criteria

- **AC-1.** *Given* a Slack workspace is connected and the `assignment.created` event is enabled, *When* an instructor creates a new assignment, *Then* a Slack message appears in the configured channel within 5 seconds containing the assignment title, due date, and a "View Assignment" link.
- **AC-2.** *Given* a student has linked their Lextures account to their Slack user, *When* they type `/lextures upcoming`, *Then* the bot replies with their next 5 due dates in a private response visible only to them.
- **AC-3.** *Given* a Slack slash command with an invalid or tampered signature, *When* the bot receives the request, *Then* it returns HTTP 401 and no response is posted.
- **AC-4.** *Given* the Slack API returns a rate-limit error, *When* the bot attempts message delivery, *Then* the job is retried after the `Retry-After` duration from Slack's response.
- **AC-5.** *Given* a grade is released for an individual student, *When* the notification fires, *Then* the grade is sent as a direct message to the student's linked Slack account, NOT posted to the course channel.
- **AC-6.** *Given* a Discord bot is invited to a server and a course is linked, *When* an announcement is created in Lextures, *Then* the Discord channel receives a rich embed with course name, announcement title, and a URL.

## 8. Data Model

```sql
-- server/migrations/077_bot_connections.sql
CREATE TABLE integrations.bot_connections (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id          UUID NOT NULL,
    platform        TEXT NOT NULL,  -- 'slack', 'teams', 'discord'
    workspace_id    TEXT NOT NULL,
    bot_token_enc   TEXT NOT NULL,
    signing_secret_enc TEXT NOT NULL,
    settings        JSONB NOT NULL DEFAULT '{}',
    connected_by    UUID REFERENCES user.users(id),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE (org_id, platform, workspace_id)
);

CREATE TABLE integrations.bot_channel_mappings (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    connection_id   UUID NOT NULL REFERENCES integrations.bot_connections(id),
    course_id       UUID REFERENCES course.courses(id),
    channel_id      TEXT NOT NULL,
    event_types     TEXT[] NOT NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE integrations.bot_user_links (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID NOT NULL REFERENCES user.users(id) ON DELETE CASCADE,
    platform        TEXT NOT NULL,
    platform_user_id TEXT NOT NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE (platform, platform_user_id)
);
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Slack/Discord API changes breaking message format | M | M | Version-pin Slack SDK; automated message format integration tests |
| FERPA violation: grade posted to group channel | M | H | Grade notifications channel-blocked by default; DM only |
| Bot token leaked | L | H | Encrypted storage (17.17); rotate on compromise; no token in logs |
| Slash command latency > 3 s triggers Slack timeout | M | L | Return immediate ACK; send actual data via response_url |

## 15. Rollout Plan

- **Feature flag:** `bot_slack`, `bot_teams`, `bot_discord` (separate flags, default `false`).
- **Phase 1:** Slack bot — channel notifications; pilot with HE beta users.
- **Phase 2:** Slash commands + user account linking; Discord bot.
- **Phase 3:** Teams bot; GA.
- **Rollback:** Disable feature flags; webhook subscriptions deactivated.

## 16. Test Plan

- **Unit** — Slack/Discord signing secret verification; message template rendering; token encryption.
- **Integration** — Mock Slack API; verify message delivery job triggered on assignment creation.
- **End-to-end** — Playwright + Slack test workspace: create assignment; verify Slack message posted.
- **Security** — Tampered signing secret returns 401; grade-to-channel blocking verified.
- **Accessibility** — Slack Block Kit alt text present; Teams Adaptive Card screen-reader review.
- **Performance / load** — 1,000 concurrent assignment creation events; bot messages enqueued within 500 ms.
- **Manual exploratory** — QA: slash command response; user account link/unlink; disconnect workspace.
