## 1. Problem Statement

Educators in K-12 and HE already live in Google Classroom, Microsoft Teams, Canva, and a constellation of EdTech tools. Lextures requires instructors to re-create content that already exists in these platforms, creating a high-friction adoption barrier. Without inbound integration connectors, Lextures cannot be positioned as a complementary layer over an existing tech stack — it only competes head-on for complete replacement, which dramatically lengthens sales cycles and shrinks the addressable market.

## 2. Goals

- Provide one-time or recurring import from Google Classroom (course roster, assignments, grades).
- Support Google Workspace Drive file embeds and OneDrive/SharePoint file pickers in the content editor.
- Enable Microsoft Teams Education class sync (roster, assignments, meeting links).
- Integrate Canva for Education as an embedded content creator (OAuth + embed URL).
- Support Edpuzzle, Khan Academy, and Quizlet via LTI 1.1 / iframe embed as a bridge until LTI 1.3 (2.12) ships.
- Provide a consistent "Connect an app" flow in the admin UI backed by the OAuth 2.0 authorization-code pattern.

## 3. Non-Goals

- Full LTI 1.3 Advantage implementation (2.12, tracked separately).
- SIS-grade roster sync (OneRoster, Clever — tracked in 4.3/4.4).
- Outbound push back to Google/Microsoft (bi-directional sync out of scope for this phase).
- SCORM/xAPI package ingestion from these platforms (8.x).

## 4. Personas & User Stories

- **As a K-12 teacher who already uses Google Classroom**, I want to import my class roster and assignments into Lextures in one click so that I do not have to re-enter everything manually.
- **As an instructor**, I want to embed a Canva presentation I designed directly into a Lextures module so that students see the interactive design without leaving the LMS.
- **As a student**, I want my Khan Academy exercises linked from a Lextures assignment so that I can complete them without logging in to a separate service.
- **As an admin**, I want to manage all connected third-party apps from one "Integrations" page so that I have visibility into what data flows are active.
- **As a teacher using Microsoft Teams Education**, I want my Teams class to stay in sync with my Lextures course roster so that adding a student in Teams reflects in Lextures within an hour.

## 5. Functional Requirements

- **FR-1.** The system MUST provide an OAuth 2.1 authorization-code flow for connecting Google Workspace and Microsoft 365 accounts, storing refresh tokens encrypted at rest (17.17).
- **FR-2.** The system MUST support one-time Google Classroom course import: roster (students + co-teachers), assignments (title, due date, point value, attachments), and previously posted grades.
- **FR-3.** The system MUST support recurring roster sync from Google Classroom (poll interval configurable, default 6 h) via background job (17.3).
- **FR-4.** The system MUST support Google Drive and OneDrive file picker embeds in the TipTap content editor, linking (not copying) the file with permission-proxy rendering.
- **FR-5.** The system MUST provide a Canva for Education OAuth embed flow: instructor authenticates with Canva, selects a design, and the design is embedded as an iframe in a module item.
- **FR-6.** The system SHOULD support Edpuzzle, Khan Academy, and Quizlet via a configurable LTI 1.1 iframe embed module item type, with instructor-supplied launch URL and consumer key/secret.
- **FR-7.** The system SHOULD support Microsoft Teams Education roster sync using the Microsoft Graph API, reading `educationClass` membership.
- **FR-8.** The system MAY provide a diff view before applying an import, showing new/changed/removed users so instructors can review before committing.
- **FR-9.** The system MUST log every import operation to the audit log: actor, source platform, records affected, timestamp.

## 6. Non-Functional Requirements

- **Performance** — Import jobs run asynchronously in the background; UI shows progress via polling endpoint. Single-class import (up to 500 students) completes within 60 seconds.
- **Security** — Refresh tokens stored with AES-256 encryption (17.17). Scopes requested from Google/Microsoft are minimal (read-only classroom data). No raw Drive file content stored — only embed URLs served through a permission-checking proxy.
- **Privacy & Compliance** — FERPA implications: imported student data is governed by Lextures' DPA with the institution. Google Classroom API Terms require "educational use" compliance. COPPA: imported student data from K-12 must follow existing minor-account flag (10.2).
- **Accessibility** — OAuth consent and import wizard pages must be WCAG 2.1 AA. Embedded iframes from third-party tools must provide a fallback link for users who cannot load the frame.
- **Scalability** — Sync jobs horizontally scalable through the job queue (17.3). OAuth token refresh handled idempotently (multiple workers will not double-refresh the same token).
- **Reliability** — Google/Microsoft API downtime must not fail the Lextures UI: degrade gracefully with a "sync unavailable, last synced X" message.
- **Observability** — Counter `integration_sync_total{source,status}`, gauge `integration_last_sync_age_seconds{source}`. Alert if any active sync is >12 h stale.
- **Maintainability** — Each integration isolated in `server/src/services/integrations/{google,microsoft,canva}.rs`; shared OAuth token management in `server/src/services/integrations/oauth.rs`.
- **Internationalization** — All wizard copy in i18n keys; date/time fields locale-aware.
- **Backward compatibility** — If a connected integration is later disconnected, imported content remains; no destructive cascade.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor authenticates with Google Classroom OAuth, *When* they select a class and click "Import," *Then* all enrolled students and assignments appear in Lextures within 60 seconds.
- **AC-2.** *Given* a recurring Google Classroom sync is active, *When* a new student is added to the Classroom, *Then* they are enrolled in the corresponding Lextures course within the next sync cycle (≤6 h).
- **AC-3.** *Given* an instructor embeds a Canva design in a module, *When* a student views the module, *Then* the Canva embed renders in an accessible iframe with a fallback "Open in Canva" link.
- **AC-4.** *Given* an LTI 1.1 Quizlet embed is configured, *When* a student clicks the module item, *Then* the LTI launch URL opens in a same-page iframe with the correct launch parameters.
- **AC-5.** *Given* the Google Classroom API returns a 503 error during a scheduled sync, *When* the error is received, *Then* the sync job is retried (17.3 backoff) and the UI displays "Sync unavailable — last synced [timestamp]."
- **AC-6.** *Given* an import operation completes, *When* the audit log is queried, *Then* a record exists with `event_type=integration_import`, `source=google_classroom`, `records_imported=N`.

## 8. Data Model

```sql
-- server/migrations/075_integration_connections.sql
CREATE TABLE integrations.oauth_connections (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id          UUID NOT NULL,
    provider        TEXT NOT NULL,    -- 'google_classroom', 'microsoft_teams', 'canva'
    external_id     TEXT NOT NULL,    -- provider's account/tenant ID
    access_token_enc TEXT NOT NULL,
    refresh_token_enc TEXT NOT NULL,
    token_expires_at TIMESTAMPTZ,
    scopes          TEXT[] NOT NULL,
    connected_by    UUID REFERENCES user.users(id),
    last_synced_at  TIMESTAMPTZ,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE (org_id, provider, external_id)
);

CREATE TABLE integrations.external_course_links (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    lextures_course_id UUID NOT NULL REFERENCES course.courses(id),
    connection_id   UUID NOT NULL REFERENCES integrations.oauth_connections(id),
    external_course_id TEXT NOT NULL,
    sync_roster     BOOLEAN NOT NULL DEFAULT true,
    sync_interval_hours SMALLINT NOT NULL DEFAULT 6,
    last_synced_at  TIMESTAMPTZ,
    UNIQUE (lextures_course_id, connection_id)
);
```

## 10. UI / UX

- **Admin → Integrations** page: grid of integration cards (Google Classroom, Teams, Canva, etc.) with "Connect" / "Disconnect" / status chip.
- OAuth connect flow: redirect to provider → return to callback → confirmation screen.
- Google Classroom import wizard: step 1 — select classes; step 2 — diff preview; step 3 — confirm import.
- Canva embed flow: "Add Canva design" button in TipTap toolbar → OAuth → design picker → iframe in module.
- LTI embed: module item type "External Tool" with URL, key, secret fields.
- Mobile: import wizard is full-screen step flow on small screens.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Google/Microsoft API quota exhaustion | M | H | Per-org rate tracking; exponential backoff; cache responses |
| OAuth token refresh race condition under horizontal scaling | M | M | Distributed lock (Redis SETNX) on token refresh |
| Provider API schema changes breaking import | M | M | Versioned API client with schema validation; alert on parse errors |
| FERPA violation if data shared back to Google beyond import | L | H | Read-only scopes; no write-back in this phase; legal review of DPA |

## 15. Rollout Plan

- **Feature flag:** `integrations_google_classroom`, `integrations_microsoft_teams`, `integrations_canva` (separate flags, default `false`).
- **Phase 1:** Google Classroom import (one-time) for pilot schools.
- **Phase 2:** Recurring sync; Teams Education; Canva embed.
- **Phase 3:** Edpuzzle/Khan/Quizlet LTI embeds; GA.
- **Rollback:** Disable feature flags; connected apps remain but sync stops.

## 16. Test Plan

- **Unit** — OAuth token encryption/decryption; import diff computation; LTI launch parameter generation.
- **Integration** — Mock Google Classroom API; full import pipeline; recurring sync job.
- **End-to-end** — Playwright: connect Google account (mock OAuth); import class; verify enrollments appear.
- **Security** — Verify read-only OAuth scopes; attempt write-back to provider API (expect 403); token storage encryption verified.
- **Accessibility** — axe-core on OAuth redirect page, import wizard steps, Canva embed fallback link.
- **Performance / load** — Import 500-student class; verify completion within 60 s.
- **Manual exploratory** — QA: provider API downtime simulation; partial import; re-import deduplication.

## 18. Open Questions

1. Should bi-directional grade sync (push grades back to Google Classroom) ship in this phase or a later one?
2. Do we support connecting multiple Google accounts per org (e.g., multiple teacher Gmail accounts)?
3. For Khan Academy and Quizlet LTI embeds — do we need a content security policy exemption, and how do we handle those vendors' iframe `X-Frame-Options` policies?
4. Should the Canva design be snapshotted (stored in our object storage) or always loaded live from Canva?
