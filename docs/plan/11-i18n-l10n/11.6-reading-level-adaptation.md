## 1. Problem Statement

Reading level adaptation — the ability to automatically rewrite content at a different reading-difficulty level — is a natural differentiator for an AI-native LMS targeting K-12 students. Lextures currently has no reading-level measurement, no Lexile or Flesch-Kincaid scoring of content, and no AI-driven content rewriting capability. K-12 educators routinely differentiate instruction by reading level (below grade, on grade, above grade), and IEP/504 plans often specify a reading-level accommodation. Without this feature, Lextures cannot serve differentiated instruction workflows or position itself as an accessibility and adaptive learning leader. The Flesch-Kincaid Grade Level formula is free, simple, and widely understood; Lexile is proprietary but the most cited in US K-12 procurement.

## 2. Goals

- Implement Flesch-Kincaid Grade Level and Flesch Reading Ease scoring for any text content in the system.
- Provide AI-powered content rewriting to target a specified reading level (e.g., "simplify this assignment description to a 4th-grade reading level").
- Allow instructors to tag content with a target reading level or let the system measure existing content.
- Allow students with accommodations to request simplified versions of assignment descriptions and module content.
- Surface reading-level metadata in the content editor to help instructors author appropriate-level content.

## 3. Non-Goals

- Lexile score computation (proprietary MetaMetrics® formula) — Flesch-Kincaid is the v1 target; Lexile integration requires a commercial license and is a v2 consideration.
- Reading-level adaptation for AI-generated quiz questions — covered by plan 19.2 when AI generates differentiated content.
- Text-to-speech / read-aloud features (plan 12.8) — reading level and TTS are complementary but separate features.
- Automatic adaptation of student-authored content (assignments, responses) — only instructor-authored content is adapted.

## 4. Personas & User Stories

- **As a K-12 reading teacher**, I want to see the Flesch-Kincaid grade level of an assignment description so that I know if it's appropriate for my 5th graders.
- **As an instructor creating a module item**, I want the editor to warn me if my explanation text reads at a 12th-grade level so that I can simplify it.
- **As a student with a reading disability (IEP accommodation)**, I want to click "Simplify this assignment" and receive a version written at my IEP-specified reading level.
- **As a district curriculum coordinator**, I want to bulk-score all course items in a course for reading level so that I can identify content that needs revision before deployment.
- **As an accessibility specialist**, I want reading-level data attached to content items so that I can generate reports showing compliance with IEP accommodation requirements.

## 5. Functional Requirements

- **FR-1.** The system MUST compute Flesch-Kincaid Grade Level (FKGL) and Flesch Reading Ease (FRE) scores for any text content field ≥ 50 words; scores MUST be stored alongside the content item and re-computed on each content save.
- **FR-2.** The system MUST display the FKGL score in the TipTap content editor toolbar as a live indicator (updated on save, not on keystroke, to avoid performance impact).
- **FR-3.** The system MUST warn instructors when content reads above a configurable grade-level threshold (default: 2 grade levels above the course's configured grade level).
- **FR-4.** The system MUST provide an AI-powered "Simplify" action in the editor that rewrites selected text to a target FKGL (e.g., "Simplify to Grade 4"); the rewritten text MUST be presented as a suggestion requiring explicit instructor acceptance.
- **FR-5.** The system MUST support an accommodation flag on student enrollments: `reading_level_override` (target FKGL integer, e.g., `4`). When set, the student's view of assignment descriptions and module text MUST show the AI-simplified version at that reading level (if a cached version exists or on-demand generation if the student has AI opt-in).
- **FR-6.** The system MUST cache AI-simplified versions of content items per (item_id, target_grade_level) to avoid re-generating on every page load.
- **FR-7.** The system SHOULD display a "reading level" badge on content items in the instructor view showing the computed FKGL.

## 6. Non-Functional Requirements

- **Performance** — FKGL computation for up to 5,000 words MUST complete in < 50 ms (pure algorithmic; no ML required). AI simplification (on-demand) MUST complete within 10 seconds; cached versions MUST serve in < 200 ms.
- **Security** — `reading_level_override` on enrollment can only be set by instructors or admins with `enrollment:manage` permission. AI simplification routes through AI gateway (10.17); requires student AI opt-in (no COPPA minor simplification without parental AI consent). See `server/src/authz.rs`.
- **Privacy & Compliance** — FKGL scoring is computed locally (no PII sent to AI for scoring). AI simplification sends content text (not student data) to the AI provider; instructor-authored text is not PII. WCAG 2.1 SC 3.1.5 (Reading Level — AAA, but relevant for IEP compliance); Section 508 Functional Performance Criteria §302.9 (with limited cognition). IEP/504 legal basis: IDEA 34 CFR Part 300 (individualized education) and ADA Section 504. AI simplification invocations logged to `compliance.ai_inference_log` (10.17). OWASP ASVS 4.0 §13 indirectly relevant for AI content injection.
- **Accessibility** — Reading-level badges in the editor MUST have accessible names (`aria-label="Flesch-Kincaid Grade Level: 7.2"`). Simplified content offered to students with reading accommodations MUST be functionally equivalent (not degraded) — WCAG 2.1 SC 1.3.3 (Sensory Characteristics). Cross-link: 10.7, 12.10 (Accommodations engine).
- **Scalability** — Bulk scoring job (FR-7 instructor view) runs asynchronously for courses with > 50 items.
- **Reliability** — If AI simplification fails, fall back to original content with a "Simplified version unavailable" notice; never hide the original.
- **Observability** — Metrics: `content_fkgl_scores_total{grade_level_bucket}`, `ai_simplifications_total`, `ai_simplification_cache_hits`. Cross-link: 17.7.
- **Maintainability** — FKGL algorithm in `server/src/services/reading_level.rs`; AI simplification in `server/src/services/content_simplification_ai.rs`.
- **Internationalization** — FKGL is an English-language formula; for non-English content, surface a "reading difficulty" metric using the appropriate locale formula (Dale-Chall for Spanish, Flesch adaptation for German). v1 ships English only; locale-specific formulas are v2.
- **Backward compatibility** — `reading_level_score` is a new nullable column on content tables; existing content shows no score until re-saved.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor saves a module item with 200-word body text, *when* the save completes, *then* a `reading_level_score` FKGL value is stored and displayed in the editor as a grade-level badge (e.g., "Grade 8.2").
- **AC-2.** *Given* an instructor selects a paragraph in the TipTap editor and clicks "Simplify to Grade 4," *when* the AI returns the simplified text, *then* it is displayed as a suggestion in a diff view with the original and simplified text side by side; the instructor clicks "Accept" to replace.
- **AC-3.** *Given* a student has `reading_level_override = 4` on their enrollment, *when* they open an assignment description for which a Grade 4 simplified version is cached, *then* the simplified version is shown automatically with a "Simplified for your reading level" notice and a link to view the original.
- **AC-4.** *Given* content is simplified by AI to FKGL 4, *when* the FKGL of the simplified output is computed, *then* the score is ≤ 5.5 (within 1.5 grade levels of the target) in at least 80% of test cases.
- **AC-5.** *Given* an instructor views the course content list, *when* reading-level badges are displayed, *then* any item with FKGL > (course_grade_level + 2) is highlighted with a warning indicator.
- **AC-6.** *Given* AI simplification is requested for a COPPA minor without parental AI opt-in, *when* the request is processed by the AI gateway, *then* the request is blocked and the original content is shown.

## 8. Data Model

```sql
-- Migration: 090_reading_level.sql

ALTER TABLE course.module_items
  ADD COLUMN reading_level_fkgl NUMERIC(4,1),
  ADD COLUMN reading_level_fre  NUMERIC(4,1);

ALTER TABLE course.module_assignments
  ADD COLUMN reading_level_fkgl NUMERIC(4,1),
  ADD COLUMN reading_level_fre  NUMERIC(4,1);

CREATE TABLE IF NOT EXISTS i18n.simplified_content_cache (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source_item_id   UUID NOT NULL,
  source_item_type TEXT NOT NULL,
  target_fkgl      INTEGER NOT NULL,   -- target grade level (e.g., 4)
  simplified_text  TEXT NOT NULL,
  computed_fkgl    NUMERIC(4,1),       -- actual FKGL of simplified output
  generated_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(source_item_id, source_item_type, target_fkgl)
);

ALTER TABLE course.enrollments
  ADD COLUMN reading_level_override INTEGER;  -- target FKGL for student, NULL = no override

CREATE INDEX idx_simplified_cache ON i18n.simplified_content_cache(source_item_id, target_fkgl);
```

## 10. UI / UX

- **Editor toolbar** — FKGL badge (e.g., "Grade 8.2") next to word count; clicking opens FKGL detail panel with FRE score and simplification action.
- **Simplify action** — "Simplify to grade level" dropdown (K, 1–12); AI generates suggestion in split view; accept/reject buttons.
- **Student view** — "Simplified version" banner with reading level and "view original" link (only shown if `reading_level_override` is set and simplified version exists).
- **Course content list** — optional column showing FKGL per item; warning icon for above-threshold items.
- WCAG 2.1 AA; `aria-label` for reading-level badges; i18n keys prefixed `reading_level.*`.

## 11. AI / ML Considerations

AI simplification uses the configured LLM via OpenRouter with a system prompt: "Rewrite the following text to a Flesch-Kincaid Grade Level of {n}. Preserve all factual content. Return only the rewritten text." The AI gateway (10.17) checks consent before any prompt is sent. The FKGL of the AI output is computed server-side after generation and stored in `simplified_content_cache.computed_fkgl` for quality monitoring. Model: `openai/gpt-4o` or equivalent capable of controlled rewriting. Eval metric: mean FKGL deviation from target < 1.5 grade levels across a test set of 100 passages.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| AI simplification changes factual content or introduces errors | H | H | Human review required before publishing simplified version; diff view makes changes visible |
| FKGL algorithm inaccurate for short text (< 50 words) | H | L | Minimum word threshold; show "insufficient text for scoring" for short content |
| Students without accommodation view simplified content only | L | M | Original always accessible via "view original" link; simplified is supplemental |
| Proprietary Lexile scoring required by district RFP | M | M | Disclose FKGL as methodology; document that Lexile license is purchasable for v2 |

## 15. Rollout Plan

- Feature flag: `reading_level_enabled` — default `false`.
- Phase 1: FKGL/FRE computation library + schema migration + editor badge.
- Phase 2: Bulk scoring for existing course content.
- Phase 3: AI simplification + instructor simplify action.
- Phase 4: Student reading-level override + simplified content serving.
- GA: Available for K12 tenants with `reading_level_enabled = true`.

## 16. Test Plan

- **Unit** — FKGL formula: known passages with reference scores (e.g., Hemingway sentences → grade 4); FRE formula.
- **Integration** — Save module item with 200 words → FKGL stored in DB.
- **End-to-end** — Playwright: instructor simplifies to Grade 4 → simplified text shows in diff → instructor accepts → student with override sees simplified version.
- **AI eval** — 100-passage test set; mean FKGL deviation from target < 1.5.
- **Security** — Student without accommodation cannot access simplified content endpoint; COPPA minor blocked from AI gateway.
- **Accessibility** — axe-core on editor badge and student simplified-content banner.
