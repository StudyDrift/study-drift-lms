## 1. Problem Statement

Lextures courses are authored in a single language. There is no mechanism for translating course content (module items, assignment descriptions, quiz questions) into other languages, no glossary or terminology consistency tooling, and no per-course translation workflow. International HE institutions that offer bilingual or multilingual courses (common in Canada, Switzerland, Belgium, and many APAC institutions) cannot use the platform. K-12 dual-language programs in the US similarly require translated content. The absence of a translation workflow also means that compliance notices added by plan 10.x (which must be available in multiple languages) have no systematic management.

## 2. Goals

- Implement a per-course translation workflow allowing instructors or designated translators to create locale variants of course content (module items, assignment descriptions, quiz question text).
- Build a translation memory (TM) that stores previously translated segments and suggests them when similar text appears in a new course.
- Implement a per-course glossary for domain-specific term consistency across translations.
- Provide an AI-assisted first-draft translation using the existing AI infrastructure (plan 19.5), with human review required before publishing.
- Store translated content as versioned locale variants alongside the source content.

## 3. Non-Goals

- Machine translation without human review (fully automated translation is prohibited for legally significant content such as assignment rubrics without instructor sign-off).
- Translation of AI-generated dynamic content (adaptive quiz questions) — static course content only for v1.
- Building a full TMS (Translation Management System) — integration with an existing TMS (Crowdin, Phrase) is the preferred path for managing external translators; this plan covers the in-system data model and instructor-facing workflow.

## 4. Personas & User Stories

- **As a bilingual instructor at a Canadian university**, I want to author my course in English and French simultaneously, with the platform helping me maintain terminology consistency.
- **As a K-12 curriculum coordinator**, I want to create Spanish translations of assignment descriptions so that ELL students can read instructions in their native language.
- **As a translator**, I want to see a translation memory suggestion when I encounter a sentence I've translated before so that I don't re-translate from scratch.
- **As a student**, I want to switch the course content language between English and Spanish so that I can read in my preferred language.

## 5. Functional Requirements

- **FR-1.** The system MUST support multiple locale variants per course content item (`module_item`, `assignment`, `quiz`); each variant stores the translated title, body, and instructions linked to the source item.
- **FR-2.** The system MUST provide a translator UI (accessible to users with `course:translate` permission) showing source text alongside translation input fields.
- **FR-3.** The system MUST implement a translation memory: when a translator submits a translated segment, it is stored in `translation_memory` with the source locale, target locale, source text hash, and translated text; future translations of similar or identical source text MUST surface TM matches sorted by similarity score.
- **FR-4.** The system MUST support a per-course glossary: a list of source-term → target-term pairs that override TM suggestions and are highlighted in translation editor.
- **FR-5.** The system MUST integrate with the AI translation service (plan 19.5) to generate a first-draft translation on demand; drafts MUST be flagged as `machine_translation_draft = true` and MUST require instructor or translator review before being marked as `published`.
- **FR-6.** Students MUST be able to switch the content language for a course from a course-level locale selector; the switch persists in their user session.
- **FR-7.** The system SHOULD track translation coverage per course (% of content items translated for each target locale) and display it in the course settings.

## 6. Non-Functional Requirements

- **Performance** — TM fuzzy-match lookup MUST complete in < 200 ms for a segment up to 500 characters against a TM of 1M segments; use trigram index (`pg_trgm`) on source text hash.
- **Security** — `course:translate` permission required for translation editing; students can only view published translations. AI-generated draft translations route through AI gateway (10.17) and check AI opt-out.
- **Privacy & Compliance** — Translated content may include student PII (e.g., sample student work in assignment examples); GDPR erasure (10.3) MUST propagate to translated variants. FERPA disclosure log (10.1) MUST record if student-identifying content is included in a translation workflow. No student-submitted work MUST be sent to AI for translation without explicit opt-in.
- **Accessibility** — Translation editor MUST meet WCAG 2.1 AA (10.7); RTL target languages supported in translation input (11.2).
- **Scalability** — TM grows over time; partitioned by `(source_locale, target_locale)` with pg_trgm index; query plans verified for 1M rows.
- **Reliability** — Translation saves are idempotent; concurrent edits by two translators resolved by last-write-wins with optimistic lock.
- **Observability** — Metrics: `translation_coverage_pct{course_id, locale}`, `tm_cache_hit_ratio`. Cross-link: 17.7.
- **Maintainability** — `server/src/services/translation_memory.rs`; `server/src/routes/translation.rs`.
- **Internationalization** — Translation editor UI itself must be i18n-enabled (11.1); target locales can be RTL (11.2).
- **Backward compatibility** — Existing course content unaffected; translation is additive.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor opens the translation editor for a course item, *when* they type the first 10 words of a Spanish translation for a previously translated English sentence, *then* the TM suggests the prior translation with a similarity score.
- **AC-2.** *Given* an AI-generated draft translation is created for a quiz question, *when* a student views the course in Spanish before the translation is reviewed, *then* the draft is NOT shown (only published translations are student-visible) and the source language is displayed instead.
- **AC-3.** *Given* a student selects Spanish as their course language, *when* they navigate to a module item that has a published Spanish translation, *then* the Spanish content is rendered and their preference is remembered for the course.
- **AC-4.** *Given* the translation coverage page is viewed, *when* a course has 10 items with 6 translated to French, *then* the coverage shows "60% French" and lists the 4 untranslated items.
- **AC-5.** *Given* a glossary term "assignment" → "tarea" is defined for a course, *when* the translation editor processes a segment containing "assignment," *then* the glossary term is highlighted and the suggested translation pre-fills "tarea" for that segment.

## 8. Data Model

```sql
-- Migration: 089_translation_memory.sql

CREATE TABLE IF NOT EXISTS i18n.content_translations (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source_item_id  UUID NOT NULL,         -- FK to module item, assignment, quiz, etc.
  source_item_type TEXT NOT NULL,        -- 'module_item', 'assignment', 'quiz_question'
  source_locale   TEXT NOT NULL DEFAULT 'en',
  target_locale   TEXT NOT NULL,         -- BCP 47
  translated_title TEXT,
  translated_body  TEXT,
  is_draft         BOOLEAN NOT NULL DEFAULT TRUE,
  machine_translation_draft BOOLEAN NOT NULL DEFAULT FALSE,
  reviewed_by     UUID REFERENCES user.users(id),
  published_at    TIMESTAMPTZ,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(source_item_id, source_item_type, target_locale)
);

CREATE TABLE IF NOT EXISTS i18n.translation_memory (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source_locale   TEXT NOT NULL,
  target_locale   TEXT NOT NULL,
  source_text     TEXT NOT NULL,
  source_hash     TEXT NOT NULL,          -- SHA-256 for exact match
  translated_text TEXT NOT NULL,
  quality_score   NUMERIC(3,2) DEFAULT 1.0,  -- 0.0–1.0
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(source_locale, target_locale, source_hash)
);

CREATE TABLE IF NOT EXISTS i18n.course_glossaries (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  course_id       UUID NOT NULL,          -- FK to course
  source_locale   TEXT NOT NULL,
  target_locale   TEXT NOT NULL,
  source_term     TEXT NOT NULL,
  target_term     TEXT NOT NULL,
  UNIQUE(course_id, source_locale, target_locale, source_term)
);

CREATE EXTENSION IF NOT EXISTS pg_trgm;
CREATE INDEX idx_tm_source_trgm ON i18n.translation_memory USING GIN (source_text gin_trgm_ops);
```

## 10. UI / UX

- **Translation editor** — split-pane view: source text on left, translation input on right; TM suggestions panel; glossary highlights; publish/draft toggle.
- **Course content language selector** — dropdown in course header for students (only shows locales with > 0% coverage).
- **Translation coverage widget** — in course settings: bar chart per locale.
- WCAG 2.1 AA; RTL input supported for Arabic/Hebrew translations.
- i18n keys prefixed `translation.*`.

## 11. AI / ML Considerations

AI-assisted first-draft translation uses plan 19.5 (AI summarization/translation). The AI request routes through the AI gateway (10.17) with `feature_name = 'content_translation'`. Course content used in AI translation prompts may contain student-authored examples; the PII redaction proxy (19.11) MUST strip any identifiable student content before the prompt is sent.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| TM grows very large; fuzzy-match becomes slow | M | M | pg_trgm GIN index; limit TM query to same (source, target) locale pair |
| AI draft translation quality is poor for specialized content | H | M | Always require human review; label drafts prominently as AI-generated |
| Translator and instructor edit same item simultaneously | L | M | Optimistic locking; conflict notification |

## 15. Rollout Plan

- Feature flag: `translation_memory_enabled` — default `false`.
- Phase 1: Data model + translation editor (no TM, no AI).
- Phase 2: TM integration + fuzzy match.
- Phase 3: AI draft generation + course glossary.
- Phase 4: Student language selector.
- GA: When 2+ institutional customers request multilingual courses.

## 16. Test Plan

- **Unit** — TM fuzzy-match ranking; glossary highlight logic.
- **Integration** — Save translation → appears in `content_translations`; TM updated.
- **End-to-end** — Playwright: instructor translates module item → publishes → student sees Spanish content.
- **Security** — Student cannot access unpublished draft translations via API.
- **Accessibility** — Translation editor RTL input for Arabic; axe-core.
- **Performance** — TM fuzzy-match < 200 ms for 1M rows.
