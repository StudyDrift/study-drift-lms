## 1. Problem Statement

Quiz and exam data accumulates in `quiz_attempts_responses` but there are no psychometric statistics derived from it. Instructors cannot determine whether a question is too easy, too hard, poorly discriminating, or whether a distractor is never chosen. Without these statistics, the quiz bank grows unchecked with poor-quality items, adaptive algorithms (§1.6) have no difficulty parameter to calibrate against, and instructors lack the evidence needed to revise assessments between semesters.

## 2. Goals

- Compute classical test theory (CTT) item statistics for every quiz question: item difficulty (p-value), point-biserial discrimination index, distractor frequency distribution.
- Compute test-level reliability coefficients: KR-20 (for dichotomous items) and Cronbach's alpha (for polytomous).
- Present these statistics in an instructor-facing item analysis report accessible from the quiz analytics page.
- Flag items that fall outside recommended psychometric quality thresholds.

## 3. Non-Goals

- Full IRT (Item Response Theory) parameter estimation — that is §1.6 (IRT/difficulty calibration).
- Adaptive item selection based on item statistics — that is §1.6.
- Real-time item statistics during a live exam window.

## 4. Personas & User Stories

- **As an exam coordinator**, I want to see the discrimination index for each quiz question so that I can identify items that do not differentiate high performers from low performers.
- **As an instructor**, I want to see which distractors are never chosen in a multiple-choice question so that I can replace weak options before the next administration.
- **As an instructor**, I want KR-20 for my midterm to verify that it is internally consistent before returning grades.
- **As a department assessment committee**, I want a downloadable item-analysis report so that I can include it in our program review documentation.

## 5. Functional Requirements

- **FR-1.** The system MUST compute for each quiz question, after a quiz closes: item difficulty p-value (`p = correct_count / attempt_count`), point-biserial correlation `r_pb` between item score and total test score, and mean score of students selecting each distractor.
- **FR-2.** The system MUST compute test-level **KR-20** for quizzes composed entirely of dichotomous (right/wrong) items.
- **FR-3.** The system MUST compute **Cronbach's alpha** for quizzes with polytomous or partial-credit items.
- **FR-4.** The system MUST flag items with `p < 0.20` (very hard), `p > 0.90` (very easy), or `r_pb < 0.15` (poor discriminator) with a visual warning in the report.
- **FR-5.** The system MUST store computed statistics in an `analytics.item_stats` table keyed by `(quiz_id, question_index, computed_at)` for auditability and trend tracking.
- **FR-6.** Instructors MUST be able to export the item analysis report as CSV.
- **FR-7.** The system SHOULD compute statistics automatically when a quiz attempt window closes (triggered by a job event), not only on manual request.
- **FR-8.** The system MUST require a minimum of 10 responses per item before computing statistics; below 10, display "Insufficient data."

## 6. Non-Functional Requirements

- **Performance** — Statistics computed for a 50-question quiz with 200 responses in < 5 s server-side. Report page loads < 1 s (pre-computed).
- **Security** — Item analysis accessible to the course instructor and admin only; student responses are aggregated, not individually identifiable in the public report.
- **Privacy & Compliance** — FERPA: individual response data never exposed; only aggregated statistics shown. Statistics are an instructor-use education record.
- **Accessibility** — Statistics table accessible by keyboard; flagged items indicated by both color and icon. Charts have text alternative. WCAG 2.1 AA.
- **Scalability** — Computation runs as a background job; results pre-stored. Does not block quiz close action.
- **Reliability** — If computation fails, retry up to 3 times; show "Statistics pending" in UI.
- **Observability** — Metrics: `item_analysis_computed_total`, `item_analysis_duration_ms`. Log quiz_id, question_count, response_count.
- **Maintainability** — Statistics engine in `server/src/services/item_analysis.rs` using **jStat** (via WASM) or pure Rust math; test plan validates against known hand-calculated values.
- **Internationalization** — Statistical terms translatable; numerical formats locale-aware.
- **Backward compatibility** — No changes to existing quiz attempt or grading tables.

## 7. Acceptance Criteria

- **AC-1.** *Given* a 20-question quiz has 50 completed attempts *When* the quiz closes *Then* the item analysis job computes difficulty, discrimination, and distractor stats for all 20 questions and stores them within 60 seconds.
- **AC-2.** *Given* a question where 40 of 50 students answered correctly *When* the item analysis is viewed *Then* the difficulty p-value displayed is 0.80 (±0.01).
- **AC-3.** *Given* a question with a point-biserial `r_pb = 0.12` *When* the report renders *Then* the row is flagged with a "Poor discriminator" warning icon with tooltip "r_pb < 0.15 — consider revising."
- **AC-4.** *Given* a dichotomous 15-item quiz *When* KR-20 is computed *Then* the value matches a hand-calculated reference to within 0.001.
- **AC-5.** *Given* a quiz has only 8 responses *When* item analysis is requested *Then* each item shows "Insufficient data (n=8, minimum 10)" rather than computed statistics.
- **AC-6.** *Given* an instructor clicks "Export CSV" *When* the download completes *Then* the CSV contains columns: question_text, n, p_value, r_pb, distractor_a_pct, distractor_b_pct, distractor_c_pct, distractor_d_pct, flag.

## 8. Data Model

```sql
-- server/migrations/087_item_stats.sql
CREATE TABLE analytics.item_stats (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  quiz_id         UUID NOT NULL,
  question_index  INTEGER NOT NULL,
  question_text   TEXT,
  n_responses     INTEGER NOT NULL,
  p_value         REAL,
  r_pb            REAL,
  distractor_freqs JSONB,   -- { "A": 0.12, "B": 0.67, "C": 0.08, "D": 0.13 }
  flag            TEXT,     -- 'easy' | 'hard' | 'poor_discriminator' | NULL
  computed_at     TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (quiz_id, question_index, computed_at)
);

CREATE TABLE analytics.test_stats (
  quiz_id         UUID PRIMARY KEY,
  n_responses     INTEGER NOT NULL,
  kr20            REAL,
  cronbach_alpha  REAL,
  mean_score      REAL,
  std_dev         REAL,
  computed_at     TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

## 10. UI / UX

- New "Item Analysis" tab on the quiz detail page (`CourseModuleQuizPage.tsx` instructor view).
- Test-level summary card: KR-20 / Cronbach α badge, mean score, std dev, N responses.
- Item table: columns Question (truncated with tooltip), N, Difficulty (p-value bar + value), Discrimination (r_pb), Distractor Breakdown (mini bar chart per option), Flags.
- Flags shown as icon + tooltip: flame icon for very hard, sun icon for very easy, warning triangle for poor discriminator.
- Distractor breakdown: horizontal stacked bar (color per option A–D) with hover-tooltip showing %.
- Export CSV button.
- Empty state: "Not enough responses yet — at least 10 are required per question."
- Accessibility: all tables have `<caption>` and `scope` headers; flag icons have `aria-label`; stacked bar charts have text table alternatives.
- i18n keys: `itemAnalysis.difficulty`, `itemAnalysis.discrimination`, `itemAnalysis.distractor`, `itemAnalysis.kr20`, `itemAnalysis.alpha`.

## 11. AI / ML Considerations

Not applicable for CTT statistics. jStat provides the statistical functions. Future: IRT parameter estimation (§1.6) will consume these CTT estimates as starting values for EM algorithm.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Statistical formula implementation error | M | H | Validate against hand-calculated test cases and Apache Commons reference values |
| Point-biserial requires score variance; zero-variance quiz causes NaN | M | M | Detect degenerate case (all same score) and display "Unable to compute" |
| Small N produces unstable estimates | H | M | Enforce N ≥ 10 gate; display confidence intervals for N < 30 |

## 15. Rollout Plan

- **Flag:** `FEATURE_ITEM_ANALYSIS=true` (default false).
- **Sequencing:** implement stats engine → unit test against references → compute job → API → UI tab → enable flag.
- **GA criteria:** KR-20 and α correct to 3 decimal places on reference datasets; export CSV functional.
- **Rollback:** disable flag; quiz detail page reverts to score-only view.

## 16. Test Plan

- **Unit** — p-value, r_pb, KR-20, Cronbach α: each tested against hand-calculated values from psychometric textbook examples.
- **Integration** — Quiz with seeded responses: compute job produces expected stats in DB.
- **End-to-end** — Playwright: open quiz item analysis tab, verify column values match expected, export CSV and verify content.
- **Security** — Student cannot access item analysis endpoint (403).
- **Accessibility** — axe scan; stacked bar chart text alternative present.
- **Performance** — 50-item quiz, 500 responses: computation < 5 s.
