## 1. Problem Statement

The `course.learning_outcomes` table (migration 072) exists, but there are no reports aggregating how a cohort of students performed against each outcome. Accreditation bodies (HLC, SACSCOC, regional accreditors) require documentation of student achievement on course learning outcomes (CLOs). K-12 districts require standards mastery roll-up reports. Without aggregated outcomes data, Lextures is not viable for any accreditation workflow and cannot support standards-based grading (§3.7) or competency-based education programs.

## 2. Goals

- Aggregate per-outcome achievement rates across all students in a course and section.
- Surface which outcomes are met, partially met, or not met at the cohort level.
- Allow instructors to link assignments and quiz items to specific learning outcomes.
- Enable export of outcomes reports in CSV and PDF (via 9.8).

## 3. Non-Goals

- Program-level or curriculum-level outcomes mapping across courses (deferred to a future curricular analytics feature).
- Automated standards alignment to Common Core / NGSS (deferred to §1.3).
- Grade passback to SIS based on outcomes (deferred to §14.5).

## 4. Personas & User Stories

- **As an instructor completing an accreditation report**, I want to see what percentage of students met each course learning outcome so that I can complete the required Assurance of Learning form.
- **As a department chair**, I want to compare outcome achievement across sections of the same course so that I can identify which instructors may need pedagogical support.
- **As a K-12 curriculum coordinator**, I want to see mastery rates for each state standard addressed in a course so that I can report to the district assessment office.
- **As an instructor**, I want to map my assignments to learning outcomes so that the outcomes report reflects the full assessment picture.

## 5. Functional Requirements

- **FR-1.** The system MUST provide a `GET /courses/:id/analytics/outcomes` endpoint returning each learning outcome with: outcome_id, title, n_students, n_assessed, mean_score, pct_met (≥ threshold), pct_not_met, threshold (configurable, default 70 %).
- **FR-2.** Instructors MUST be able to link course assignments and quiz questions to outcomes via an "Align to outcome" UI in the assignment settings.
- **FR-3.** An outcome is "met" for a student when their average score on all aligned assessments ≥ the configured mastery threshold.
- **FR-4.** The system MUST display an outcome achievement summary chart (horizontal bar per outcome) on the outcomes report page.
- **FR-5.** The outcomes report MUST be filterable by section (when sections exist, §5.4) and by student group.
- **FR-6.** The system MUST support cross-outcome weighting (some outcomes may count more heavily for accreditation purposes).
- **FR-7.** Outcomes data MUST be refreshed within 1 hour of a grade change for any aligned assessment.
- **FR-8.** Instructors MUST be able to add qualitative "improvement notes" per outcome (narrative for accreditation portfolio).

## 6. Non-Functional Requirements

- **Performance** — Outcomes report for a 200-student course loads < 800 ms p95 (pre-aggregated via materialized view).
- **Security** — Instructor and admin access only. Student individual performance tied to outcomes is accessible to the student via their own progress page (9.1).
- **Privacy & Compliance** — FERPA: aggregated outcome data is not individually identifiable in the public report; individual linkage protected. GDPR: included in account erasure scope.
- **Accessibility** — Horizontal bar chart has `role="img"` with `aria-label` summarizing the data. Table view available as fallback. WCAG 2.1 AA.
- **Scalability** — Materialized view refreshed on grade change event via Postgres LISTEN/NOTIFY or scheduled job.
- **Reliability** — If refresh fails, stale data served with timestamp; alert if staleness > 2 h.
- **Observability** — Metrics: `outcomes_report_views_total`, `outcomes_refresh_duration_ms`. Log course_id, outcome_count, refresh_trigger on each refresh.
- **Maintainability** — Outcomes aggregation in `server/src/repos/outcomes_report.rs` (new); linked from `server/src/routes/reports.rs`.
- **Internationalization** — Outcome titles stored in course language; mastery threshold labels externalised.
- **Backward compatibility** — `course.learning_outcomes` table pre-exists (migration 072); this feature adds assessment linkage and aggregation on top.

## 7. Acceptance Criteria

- **AC-1.** *Given* a course has 3 learning outcomes and assignments mapped to each *When* all students have been graded *Then* the outcomes report shows n_assessed, pct_met, and pct_not_met for all 3 outcomes.
- **AC-2.** *Given* an instructor sets the mastery threshold to 80 % *When* the report recomputes *Then* pct_met reflects students scoring ≥ 80 % on aligned assessments.
- **AC-3.** *Given* an outcome has no aligned assessments *When* the report renders *Then* that outcome shows "No assessments aligned — align assignments to see data."
- **AC-4.** *Given* a grade is changed for a student in a course *When* 60 minutes have elapsed *Then* the outcomes report data reflects the updated grade.
- **AC-5.** *Given* an instructor adds a qualitative improvement note "Strengthen lab report instructions for Outcome 2" *When* the note is saved *Then* it appears under that outcome in the report view and in the PDF export (9.8).
- **AC-6.** *Given* the outcomes report is exported via 9.8 *When* the PDF is generated *Then* all three outcomes with their statistics and improvement notes are included in the report.

## 8. Data Model

```sql
-- server/migrations/088_outcomes_alignment.sql
-- course.learning_outcomes already exists (migration 072)

CREATE TABLE course.outcome_alignments (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  outcome_id      UUID NOT NULL REFERENCES course.learning_outcomes(id) ON DELETE CASCADE,
  assignment_id   UUID REFERENCES course.module_assignments(id) ON DELETE CASCADE,
  quiz_id         UUID REFERENCES course.module_quizzes(id) ON DELETE CASCADE,
  weight          REAL NOT NULL DEFAULT 1.0,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  CHECK (assignment_id IS NOT NULL OR quiz_id IS NOT NULL)
);

CREATE MATERIALIZED VIEW analytics.outcomes_report AS
SELECT
  lo.id AS outcome_id,
  lo.course_id,
  lo.title,
  COUNT(DISTINCT e.user_id) AS n_students,
  COUNT(DISTINCT g.user_id) AS n_assessed,
  AVG(g.score_pct) AS mean_score,
  COUNT(*) FILTER (WHERE g.score_pct >= lo.mastery_threshold) AS n_met
FROM course.learning_outcomes lo
JOIN course.enrollments e ON e.course_id = lo.course_id
JOIN course.outcome_alignments oa ON oa.outcome_id = lo.id
LEFT JOIN course_grades g ON g.course_id = lo.course_id AND g.user_id = e.user_id
GROUP BY lo.id, lo.course_id, lo.title;

CREATE UNIQUE INDEX ON analytics.outcomes_report (outcome_id);
```

## 9. API Surface

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/courses/:id/analytics/outcomes` | Instructor JWT | Outcomes report with achievement rates |
| `POST` | `/courses/:id/outcomes/:oid/alignments` | Instructor JWT | Align assignment or quiz to outcome |
| `DELETE` | `/courses/:id/outcomes/:oid/alignments/:aid` | Instructor JWT | Remove alignment |
| `PUT` | `/courses/:id/outcomes/:oid/notes` | Instructor JWT | Save improvement notes |
| `POST` | `/courses/:id/analytics/outcomes/refresh` | Instructor JWT | Manual refresh |

## 10. UI / UX

- New "Outcomes" tab in the course analytics reports page (`Reports.tsx`).
- Mastery threshold control: numeric input (default 70 %).
- Outcome rows: outcome title, n assessed / n enrolled, horizontal achievement bar (green = met, red = not met), % met label, "Align assessments" link, "Add note" expander.
- Improvement notes: expandable textarea per outcome; saved on blur.
- Filter bar: section selector, date range.
- Export button: triggers PDF/CSV export (9.8).
- Empty state: "Align your assignments to outcomes to see this report."
- Accessibility: outcome list is a `<table>` with proper `<caption>` and `scope`. Progress bar `role="progressbar"` with `aria-valuenow`. Notes textarea labeled.
- i18n keys: `outcomes.report`, `outcomes.met`, `outcomes.notMet`, `outcomes.threshold`, `outcomes.noAlignments`, `outcomes.note`.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Instructors don't align assessments, leaving report empty | H | M | Onboarding prompt on first visit; auto-suggest alignment based on outcome wording |
| Accreditation body requires specific aggregation formula | M | M | Configurable threshold; exportable raw data for custom calculation |
| Refresh triggered too frequently by grade changes | M | M | Debounce refresh: batch grade change events within a 5-min window |

## 15. Rollout Plan

- **Flag:** `FEATURE_OUTCOMES_REPORT=true` (default false).
- **Sequencing:** alignment table migration → aggregation view → API → UI tab → export integration (9.8) → enable flag.
- **GA criteria:** outcomes with aligned assessments show correct pct_met; export functional; improvement notes persist.
- **Rollback:** disable flag; learning outcomes table unaffected.

- **Security** — Student cannot access outcomes report (403).
