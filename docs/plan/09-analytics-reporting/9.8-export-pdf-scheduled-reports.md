## 1. Problem Statement

All existing and planned analytics reports (`Reports.tsx`, outcomes, progress dashboard, gradebook) are screen-only: there is no way to export them as a PDF or schedule a recurring email delivery. Instructors preparing accreditation portfolios must take screenshots. Administrators cannot schedule a weekly grade summary to department chairs. District reporting often requires printable formatted documents. Without export and scheduling, analytics are inaccessible to stakeholders who are not logged into the LMS.

## 2. Goals

- Enable PDF export for: gradebook, student progress report, outcomes report, at-risk report, item analysis.
- Enable scheduled email delivery of any report on a daily / weekly / monthly cadence.
- Generate PDFs server-side to ensure consistent formatting regardless of user's browser or OS.
- Provide a report scheduling UI in the admin and instructor dashboards.

## 3. Non-Goals

- XLSX/Word export (CSV export for gradebook is a separate feature, §3.11).
- Real-time report streaming (PDF generation is async, delivered via download or email).
- Building a custom PDF rendering engine (use Headless Chrome or a Rust PDF library).

## 4. Personas & User Stories

- **As an instructor**, I want to download a PDF of my student progress report so that I can include it in my annual teaching portfolio.
- **As a department chair**, I want a weekly email with the outcomes report for each section I supervise so that I can monitor cohort performance without logging in daily.
- **As an admin**, I want to schedule a monthly at-risk summary to be emailed to all instructors in a school so that the report is automatic.
- **As an accreditation coordinator**, I want a formatted PDF of course outcomes data to attach to our assurance of learning documentation.

## 5. Functional Requirements

- **FR-1.** The system MUST generate PDFs for the following report types: gradebook summary, per-student progress report, course outcomes report, at-risk report, item analysis report.
- **FR-2.** PDF generation MUST be performed server-side using Headless Chrome (via `chromiumoxide` crate or `puppeteer` microservice) rendering the existing React report pages with print CSS.
- **FR-3.** Generated PDFs MUST be stored temporarily in object storage (8.1) with a 24-hour expiry pre-signed URL returned to the requester.
- **FR-4.** The system MUST provide a "Schedule this report" UI that accepts: recipient email(s), cadence (daily / weekly / monthly), day of week/month, delivery time (timezone-aware).
- **FR-5.** Scheduled reports MUST be executed as background jobs (17.3) triggered by the scheduler (17.4) and delivered via email (6.2) with the PDF attached or a download link.
- **FR-6.** The system MUST allow instructors and admins to manage their scheduled reports (list, edit, delete) via a `GET/PUT/DELETE /reports/schedules` API.
- **FR-7.** PDF reports MUST include the institution name, course name, generation timestamp, and a page footer with "Confidential — FERPA protected" for reports containing student data.
- **FR-8.** The system SHOULD support per-report parameters (date range, section filter) saved as part of the schedule configuration.

## 6. Non-Functional Requirements

- **Performance** — PDF generation for a 30-student progress report completes within 30 s. Scheduled report email delivered within 5 min of scheduled time.
- **Security** — PDF pre-signed URLs expire in 24 h. Scheduled report recipients must be authenticated users or explicitly whitelisted admin emails. PDFs do not contain raw student IDs (use names only unless `REPORT_INCLUDE_IDS=true`).
- **Privacy & Compliance** — FERPA: PDFs containing student data must be delivered only to authorized personnel; recipient list validated against course roles. GDPR: scheduled reports referencing a deleted user omit that user's data. Footer disclosure required.
- **Accessibility** — PDF accessibility: Headless Chrome generates tagged PDFs when print CSS includes `@page` with proper heading tags. Reports should include alt text for charts.
- **Scalability** — PDF generation is CPU-intensive; isolated worker pool with max 5 concurrent generation jobs. Queue depth monitored.
- **Reliability** — If PDF generation fails, retry once; notify requester by email with error if second attempt fails. Scheduled report retried next interval.
- **Observability** — Metrics: `pdf_generation_jobs_total`, `pdf_generation_duration_s`, `scheduled_report_deliveries_total`, `scheduled_report_failures_total`. Log report_type, course_id, recipient_count.
- **Maintainability** — PDF worker in `server/src/workers/pdf_generator.rs`; schedule management in `server/src/routes/reports.rs`.
- **Internationalization** — Report content localized per course language. Date/time in scheduled report delivery formatted per recipient locale.
- **Backward compatibility** — Reports UI unchanged; export button is additive.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor clicks "Export PDF" on the outcomes report *When* generation completes *Then* a download link appears within 30 s; the downloaded PDF contains the outcomes table, mastery threshold, and improvement notes.
- **AC-2.** *Given* a scheduled weekly report is configured for every Monday at 8:00 AM *When* Monday 8:00 AM passes *Then* the PDF is generated and an email is delivered to the configured recipient with the PDF attached or a 24-h download link.
- **AC-3.** *Given* a PDF generation job fails twice *When* the second failure is recorded *Then* an error email is sent to the requester with "Report generation failed — please try again" and the error reason.
- **AC-4.** *Given* the PDF pre-signed URL expires after 24 hours *When* a user tries to download using the expired URL *Then* they receive a 403 and the UI shows "Download link expired — regenerate report."
- **AC-5.** *Given* a scheduled report includes a student who has been GDPR-erased *When* the PDF is generated *Then* that student's name is replaced with "Deleted User" and their individual data row is omitted.
- **AC-6.** *Given* a report PDF is opened in Adobe Acrobat *When* accessibility check runs *Then* the document has at least document title, language tag, and body headings tagged (PDF/UA basic compliance).

## 8. Data Model

```sql
-- server/migrations/091_report_schedules.sql
CREATE TABLE analytics.report_schedules (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  owner_id        UUID NOT NULL REFERENCES user_account(id),
  course_id       UUID REFERENCES course.courses(id),
  report_type     TEXT NOT NULL,
  parameters      JSONB NOT NULL DEFAULT '{}',
  recipients      TEXT[] NOT NULL,
  cadence         TEXT NOT NULL,  -- 'daily' | 'weekly' | 'monthly'
  cadence_detail  JSONB,          -- { day_of_week: 1, hour: 8, tz: 'America/Chicago' }
  enabled         BOOLEAN NOT NULL DEFAULT true,
  last_run_at     TIMESTAMPTZ,
  next_run_at     TIMESTAMPTZ NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX ON analytics.report_schedules (next_run_at) WHERE enabled = true;
```

## 9. API Surface

| Method | Path | Auth | Description |
|---|---|---|---|
| `POST` | `/courses/:id/reports/:type/export` | Instructor JWT | Request PDF export; returns `{ job_id, status_url }` |
| `GET` | `/reports/export-jobs/:job_id` | JWT | Poll job status; returns download URL when done |
| `GET` | `/reports/schedules` | JWT | List own report schedules |
| `POST` | `/reports/schedules` | JWT | Create schedule |
| `PUT` | `/reports/schedules/:id` | JWT | Update schedule |
| `DELETE` | `/reports/schedules/:id` | JWT | Delete schedule |

## 10. UI / UX

- All report pages: "Export PDF" button in the header. On click: loading spinner → "Generating report…" → download link or error.
- "Schedule this report" link beside Export button: opens a modal with recipient email chips, cadence selector, day/time inputs, timezone picker.
- "My Scheduled Reports" page (linked from user settings): table of active schedules with columns Report, Course, Cadence, Recipients, Next Run, Actions (Edit, Disable, Delete).
- Scheduled delivery email: subject "Weekly Outcomes Report — [Course Name]"; body: brief summary table + "Download full PDF" button (links to pre-signed URL).
- Empty state: "No scheduled reports. Click 'Schedule this report' on any analytics page."
- Accessibility: modal `role="dialog"`; cadence selector labeled; timezone picker has `aria-label`. PDF download link has descriptive text.
- i18n keys: `report.exportPdf`, `report.schedule`, `report.cadence`, `report.recipients`, `report.nextRun`.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Headless Chrome OOM on large reports | M | M | Limit concurrent PDF jobs; set Chrome `--max-old-space-size`; page-size limit |
| Scheduled report delivery silently fails | M | M | Dead-letter with alert; next-run retry logic |
| FERPA: PDF emailed to wrong recipient | L | H | Validate recipients against course enrollment; admin must approve external addresses |
| Headless Chrome binary licensing / patching | M | M | Use chromium-headless from apt/brew; pin version; automate security updates |

## 15. Rollout Plan

- **Flag:** `FEATURE_REPORT_EXPORT=true` (default false).
- **Sequencing:** PDF worker → manual export API → UI button → schedule API → schedule UI → email delivery → enable flag.
- **GA criteria:** PDF export works for gradebook, outcomes, progress; scheduled weekly delivery tested end-to-end.
- **Rollback:** disable flag; export buttons hidden.

## 16. Test Plan

- **Unit** — Schedule next-run calculation; parameter serialization; PDF job status state machine.
- **Integration** — PDF worker generates non-zero PDF for each report type; stored in object storage; pre-signed URL accessible.
- **End-to-end** — Playwright: trigger PDF export from outcomes page, poll for completion, download and verify PDF page count > 0.
- **Security** — Confirm recipient validation against course roles; confirm cross-user schedule access blocked.
- **Accessibility** — Verify generated PDFs have document title tag; confirm export modal accessible.
- **Performance** — 30-student progress PDF generated < 30 s; 5 concurrent jobs complete without OOM.
