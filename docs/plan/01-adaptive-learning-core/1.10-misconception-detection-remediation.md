## 1. Problem Statement

When a student selects a wrong answer in Lextures, nothing happens: no feedback, no explanation, no link to remediation content. Research in cognitive science consistently identifies specific, common misconceptions per concept domain (e.g., "multiplying by a fraction makes numbers bigger", "the equals sign means 'put the answer here'"). These are predictable, well-documented errors that instructional designers can map in advance. Currently Lextures has no mechanism to detect, label, or respond to these errors. Building a misconception detection and remediation library would transform wrong-answer feedback from absent to pedagogically meaningful, directly differentiating Lextures from systems that only give right/wrong signals.

## 2. Goals

- Allow course authors to define misconceptions: a named misconception, its targeted incorrect-answer pattern, and a remediation response (text, link to resource, or AI explanation).
- Tag distractor options in multiple-choice questions to specific misconceptions.
- Detect when a learner selects a misconception-tagged wrong answer and surface targeted remediation immediately.
- Record misconception events in the learner model to inform future hint selection and concept mastery updates.
- Provide an instructor-facing misconception frequency report per question and per concept.

## 3. Non-Goals

- Open-ended (free-text) misconception detection via NLP — pattern-matched distractors only in v1; AI misconception detection for free-text is covered in 19.4.
- Automatic misconception remediation generation without author input in v1 (AI-suggested remediation content is a v2 enhancement).
- Learning pathways triggered automatically by misconception (adaptive path rules in 1.4 handle the routing; this plan covers detection and inline feedback).
- Cross-learner misconception analytics beyond per-course reports (covered in 9.4 item analysis).

## 4. Personas & User Stories

- **As a math instructor**, I want to tag the distractor "4/8 = 4 ÷ 8 = 0.5, therefore 4/8 > 1/2" to the common misconception "students believe fractions with larger numerators are always larger" so students selecting it receive targeted feedback.
- **As a K-12 student**, I want to see an explanation of why my wrong answer reflects a specific misunderstanding, not just "Incorrect — the answer is B."
- **As a course author**, I want a library of pre-authored misconception templates for common concepts (math, grammar, science) that I can attach to my questions without writing them from scratch.
- **As an instructor**, I want a report showing which misconceptions are most commonly triggered in my class this week so I can address them in the next lesson.
- **As a self-learner**, I want the system to identify a recurring misconception across multiple questions and suggest a targeted remediation resource.

## 5. Functional Requirements

- **FR-1.** The system MUST allow authors to create `misconception` records: `name`, `description`, `concept_id`, `remediation_body` (rich text), and optional `remediation_url`.
- **FR-2.** The system MUST allow authors to tag individual distractor answer options in MC questions to a misconception_id, via the question bank editor (2.1).
- **FR-3.** The system MUST detect misconception-tagged distractor selection during quiz grading and record a `misconception_events` row (attempt_id, question_id, misconception_id, selected_option_id).
- **FR-4.** The system MUST return the matched misconception's `remediation_body` in the per-question feedback payload after submission (or immediately on selection if `show_feedback = 'immediate'`).
- **FR-5.** The system MUST incorporate misconception frequency into the mastery update (1.1): a wrong answer matching a known misconception reduces mastery more than a random wrong answer, reflecting a structural knowledge gap.
- **FR-6.** The system MUST expose `GET /api/v1/courses/{id}/misconception-report` returning per-question and per-misconception trigger counts, accessible to instructors and admins.
- **FR-7.** The system SHOULD ship a seed remediation library for common K-12 math, grammar, and science misconceptions (50–100 items) that course authors can adopt or customise.
- **FR-8.** The system SHOULD surface a recurring-misconception alert to students: if the same misconception is triggered 3+ times across attempts, show a dedicated remediation prompt ("You've made this mistake before — here's a resource to help").
- **FR-9.** The system MAY allow AI to generate remediation content for a misconception when none is authored, using the misconception name and concept tags as context.

## 6. Non-Functional Requirements

- **Performance** — Misconception lookup at grading time MUST complete in ≤ 10 ms (simple join, single row fetch). Remediation body delivered in the same grading response payload with no extra round trip.
- **Security** — Misconception records are course-level and author-editable. Student-facing remediation content is read-only for students. Misconception event log is instructor/admin-only for reporting.
- **Privacy & Compliance** — `misconception_events` contains (user_id, attempt_id) — FERPA educational record. Included in DSAR export and erasure. AI remediation generation MUST redact PII before calling LLM.
- **Accessibility** — Remediation feedback panel MUST use ARIA live region (`aria-live="polite"`) so screen readers announce it when it appears. All remediation content MUST have keyboard-accessible dismiss.
- **Scalability** — Misconception events table partitioned by month. At peak (50 k learners each triggering 5 events/day) = 250 k rows/day — manageable in Postgres with monthly partitions.
- **Reliability** — If misconception lookup fails, quiz grading proceeds without it; remediation is a "best effort" enhancement, not blocking.
- **Observability** — Prometheus counter `misconceptions.triggered` (labeled by misconception_id), `misconceptions.remediation_shown`. Alert if misconception lookup error rate > 0.5 %.
- **Maintainability** — Misconception detection in `server/src/services/misconception.rs`. No inline logic in grading handler. Seed data in `server/seeds/misconceptions/`.
- **Internationalization** — Misconception remediation_body supports locale variants (same pattern as hints in 1.9). Seed library ships English-only; locale variants added by community.
- **Backward compatibility** — All misconception features are additive. Distractors not tagged to misconceptions continue to work exactly as today.

## 7. Acceptance Criteria

- **AC-1.** *Given* a MC question has distractor D tagged to misconception M, *when* a student selects D and submits the question, *then* the feedback payload contains M's `remediation_body` and `misconception_id`.
- **AC-2.** *Given* a student triggers the same misconception 3 times across separate attempts, *when* they see the quiz feedback again, *then* a "recurring misconception" alert is shown with a remediation link.
- **AC-3.** *Given* a student answers a misconception-tagged distractor, *when* mastery is updated in 1.1, *then* the mastery delta is 20 % more negative than for an untagged wrong answer (configurable weight).
- **AC-4.** *Given* an instructor opens the misconception report for a course, *when* the report loads, *then* they see a table with misconception name, question, trigger count, and affected student count.
- **AC-5.** *Given* the misconception lookup service fails at grading time, *when* the grade is submitted, *then* the grade is recorded normally and no error is returned to the student.
- **AC-6.** *Given* a distractor is not tagged to any misconception, *when* a student selects it, *then* no remediation feedback is shown and no misconception event is recorded.

## 8. Data Model

```sql
-- Migration: server/migrations/0039_misconceptions.sql

CREATE TABLE course.misconceptions (
    id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    concept_id       UUID REFERENCES course.concepts(id) ON DELETE SET NULL,
    name             TEXT NOT NULL,
    description      TEXT,
    remediation_body TEXT,
    remediation_url  TEXT,
    locale           TEXT NOT NULL DEFAULT 'en',
    is_seed          BOOLEAN NOT NULL DEFAULT FALSE,
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_misconceptions_concept ON course.misconceptions (concept_id);

-- Tag distractor options to misconceptions (stored in question bank JSONB options)
-- For structured MC questions: option-level tag stored in question_options table (2.1)
CREATE TABLE course.question_option_misconception_tags (
    question_id      UUID NOT NULL,
    option_id        UUID NOT NULL,   -- references option within question (from 2.1)
    misconception_id UUID NOT NULL REFERENCES course.misconceptions(id) ON DELETE CASCADE,
    PRIMARY KEY (question_id, option_id)
);

CREATE TABLE course.misconception_events (
    id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id          UUID NOT NULL,
    attempt_id       UUID NOT NULL,
    question_id      UUID NOT NULL,
    misconception_id UUID NOT NULL,
    remediation_shown BOOLEAN NOT NULL DEFAULT FALSE,
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
) PARTITION BY RANGE (created_at);

CREATE INDEX idx_me_user ON course.misconception_events (user_id, misconception_id);
```

DELETE /api/v1/misconceptions/{id}              Auth: admin

-- Tag distractor to misconception (inline in question editor, 2.1)

  Body: { misconception_id }

-- Reports
GET  /api/v1/courses/{id}/misconception-report   Auth: instructor|admin
  Response: { misconceptions: MisconceptionReportRow[] }

GET  /api/v1/learners/{user_id}/misconception-summary  Auth: own | instructor | admin

## 10. UI / UX

**Post-question feedback** (in `clients/web/src/components/quiz/`):
- After submission, wrong answers show a "Understanding check" panel below the question.
- Misconception-matched wrong answers show: misconception name badge + remediation text + optional link.
- Non-misconception wrong answers show standard "Incorrect — correct answer is X" feedback (unchanged).
- Recurring alert: yellow banner "You've seen this pattern before. Here's a resource to help." with dismiss.

**Question editor — distractor tagging** (`clients/web/src/components/editor/`):
- Each MC distractor row gains a "Tag misconception" dropdown.
- Misconception picker: search by name or concept.
- "Create new misconception" inline creation flow.

**Misconception report page** (`/courses/{id}/misconception-report`):
- Bar chart: top 10 misconceptions by trigger count.
- Table: misconception, question, student count, first seen, last seen.
- Export to CSV.

Mobile: feedback panel is full-width below question; misconception badge shown as a chip with expand button.

## 11. AI / ML Considerations

V1 is purely pattern-matched (distractor tag → misconception). V2 AI misconception detection for free-text and short-answer responses is covered in 19.4.

AI-generated remediation (FR-9): prompt sends `{ misconception_name, concept_name, bloom_level }` to LLM, requests a ≤ 100-word pedagogical explanation. PII redaction applied. Cost budget: $0.003 per generation. Generated remediation marked `ai_generated: true` for instructor review before publishing.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Authors don't tag distractors, making feature invisible | H | M | Auto-suggest misconception tags at question-creation time using AI; seed library gives authors a starting point |
| Remediation body reveals the correct answer | M | M | Author review required before publishing; AI-generated content requires instructor approval |
| Misconception events table grows very large | L | M | Monthly partitioning; archive events older than 2 years to cold storage |
| False positives: correct answer confused with misconception tag | L | H | Validate at save time that misconception-tagged options are not the correct answer |

## 15. Rollout Plan

- **Feature flag:** `misconception_detection_enabled` (default: `false`).
- **Migration sequencing:** deploy migration → seed library → ship editor tagging UI → ship student feedback UI → enable for pilot → GA.
- **Pilot cohort:** two K-12 math courses; instructors tag 30 questions.
- **GA criteria:** misconception event log populated; remediation shown to > 50 students; report page accessible to instructors.
- **Rollback:** disable flag; grading returns standard feedback; misconception tables are additive.

## 16. Test Plan

- **Unit** — Misconception detector (given a tagged option_id, returns misconception record); recurring alert logic (trigger count threshold).
- **Integration** — Seed misconception → tag distractor → student selects tagged distractor → assert `misconception_events` row; assert feedback payload contains remediation.
- **End-to-end** — Playwright: student selects misconception-tagged wrong answer, sees remediation panel; instructor views misconception report with correct counts.
- **Security** — Student cannot view misconception event log for others; instructor can view course report; misconception CRUD restricted to instructor/admin.
- **Accessibility** — ARIA live region fires when remediation panel appears; keyboard dismiss works; no color-only status indicators.
- **Performance / load** — 500 concurrent quiz submissions with misconception lookup; grading p95 unchanged (within ± 5 ms of baseline).
- **Manual exploratory** — Verify non-tagged wrong answer shows no remediation; verify recurring alert appears on 3rd occurrence; verify AI-generated remediation requires instructor approval before showing.

## 18. Open Questions

1. Should misconception tagging be per option (distractor-level) or per question (if the student gets the question wrong for any reason)? Per-option is more precise but requires structured MC options.
2. Should the remediation body support embedded video or only text + URL?
3. Should recurring-misconception alerts be shown during the attempt (after each trigger) or only at the end of the attempt?
4. How should misconception detection interact with partial-credit questions — should partial credit suppress the misconception flag?
