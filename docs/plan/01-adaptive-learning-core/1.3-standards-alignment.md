## 1. Problem Statement

K-12 districts in the United States are legally required to align instruction and assessment to adopted academic standards (Common Core State Standards for ELA/Math, NGSS for science, and dozens of state-specific variants). Lextures has no metadata model for standards, no way to tag questions or course content to a standard code, and no reporting surface showing standards mastery. Without this, the product cannot enter district procurement conversations: every RFP, every curriculum director review, and every state-level digital-instructional-materials vetting process requires standards-alignment evidence.

## 2. Goals

- Model external academic standards (framework, domain, cluster, standard code) in a normalized schema that can hold Common Core, NGSS, state standards, and Bloom's.
- Allow concepts (1.2) and individual questions to be aligned to one or more standard codes.
- Expose standards-coverage reports showing which standards a course addresses and at what mastery level.
- Provide a seed data pipeline for importing publicly available standard sets (CCSS-M, CCSS-ELA, NGSS) from machine-readable sources.
- Maintain alignment links immutably when standard codes are superseded, using a versioned reference.

## 3. Non-Goals

- Standards-based grading report cards (covered in 3.7).
- Automatic AI-assisted alignment (future; not in this plan).
- International standards frameworks beyond US K-12 initial scope (e.g., UK National Curriculum, IB) — reserved for v2.
- Per-student standards-mastery transcript (depends on 1.1 and 3.7).

## 4. Personas & User Stories

- **As a curriculum director**, I want to filter course content by standard code (e.g., CCSS.MATH.CONTENT.6.EE.A.1) to verify curriculum coverage before district adoption.
- **As an instructor**, I want to tag each quiz question to one or more standard codes so the gradebook can roll up scores to standards mastery.
- **As an admin**, I want to import a complete state standards framework once and have all instructors reference it rather than re-entering codes manually.
- **As a K-12 student**, I want to see which standards I am making progress on alongside my grades, presented in language appropriate to my grade level.
- **As a state education agency auditor**, I want a coverage report showing which CCSS standards are addressed in a course and at what Bloom's cognitive level.

## 5. Functional Requirements

- **FR-1.** The system MUST store academic standards in a hierarchical table (`standard_frameworks` → `standard_domains` → `standard_clusters` → `standard_codes`) supporting at least four nesting levels.
- **FR-2.** The system MUST allow many-to-many alignment between `course.concepts` and `standard_codes`, and between individual questions (via `concept_question_tags`) and `standard_codes`.
- **FR-3.** The system MUST version standard frameworks: when a framework is updated (e.g., a state revises standards), existing alignment links reference the prior version and a migration path is offered.
- **FR-4.** The system MUST provide an admin-only import endpoint (`POST /api/v1/admin/standards/import`) accepting a JSON document conforming to the Case Network / IMS Global CASE 1.0 format.
- **FR-5.** The system MUST provide `GET /api/v1/standards?framework=ccss-math&grade=6` returning all standard codes for a given framework and grade band.
- **FR-6.** The system MUST provide a course-level standards-coverage report endpoint `GET /api/v1/courses/{id}/standards-coverage` returning, for each aligned standard: question count, average mastery (from 1.1), and coverage status (none/partial/covered).
- **FR-7.** The system SHOULD ship seed data for CCSS Math (K–8 and High School), CCSS ELA (K–12), and NGSS (K–12) derived from publicly available JSON exports.
- **FR-8.** The system MAY provide a `GET /api/v1/standards/search?q=` endpoint for fuzzy search across standard descriptions, to assist instructors in finding the right code.

## 6. Non-Functional Requirements

- **Performance** — Standards search response p95 ≤ 150 ms. Coverage report for a course with 500 questions and 200 standards MUST compute in ≤ 500 ms (use materialized view or pre-computed aggregates from 1.1).
- **Security** — Standard framework import is admin-only. Alignment read endpoints require authentication. Standard data itself is public-domain (CCSS, NGSS are openly published).
- **Privacy & Compliance** — Standards coverage aggregates may contain class-level mastery data. Must respect FERPA when reporting at student level. State-level reporting exports must be encrypted in transit (TLS 1.3).
- **Accessibility** — Standards browsing and tagging UI MUST be keyboard navigable. Bloom's level badges MUST have sufficient color contrast (WCAG 2.1 AA ≥ 4.5:1 ratio).
- **Scalability** — CCSS + NGSS combined: ~5 000 standard codes. State variants may add another 3 000. Index on `framework_id` + `grade_band` + `code` covers standard query patterns.
- **Reliability** — Import is idempotent (upsert on `code` + `framework_version`); re-running import does not duplicate rows or break alignment links.
- **Observability** — Log each import with framework name, version, and record count. Expose `standards_alignment.coverage_pct` gauge per course updated after question tagging.
- **Maintainability** — Standards service in `server/src/services/standards.rs`; seed data scripts in `server/seeds/standards/`. No hardcoded standard codes in application logic.
- **Internationalization** — Standard descriptions stored as UTF-8; `description` column supports full-text in any language. State standards for non-English-medium states (e.g., Puerto Rico Spanish) modeled identically.
- **Backward compatibility** — Alignment tables are additive. Removing a standard code is a soft delete (`archived_at`); existing alignment links preserved.

## 7. Acceptance Criteria

- **AC-1.** *Given* a CCSS Math JSON file is imported via the admin endpoint, *when* the import completes, *then* all standard codes for grades K–8 and High School are queryable via `GET /api/v1/standards?framework=ccss-math`.
- **AC-2.** *Given* a question is tagged to concept X which is aligned to standard `CCSS.MATH.CONTENT.6.EE.A.1`, *when* a student answers that question correctly, *then* the standards-coverage report for the course reflects updated mastery on that standard.
- **AC-3.** *Given* a standards framework is versioned and a new version imported, *when* the coverage report is requested, *then* existing alignment links display the prior version code with a "superseded" indicator rather than breaking.
- **AC-4.** *Given* an instructor searches for "ratio", *when* `GET /api/v1/standards/search?q=ratio&framework=ccss-math` is called, *then* results include `6.RP.A.1` and related ratio-and-proportion standards.
- **AC-5.** *Given* a course covers 40 of 80 Grade 6 Math CCSS standards, *when* `GET /api/v1/courses/{id}/standards-coverage` is called, *then* the response shows 40 standards as "covered" and 40 as "none."
- **AC-6.** *Given* a student accesses their standards progress (UI), *when* the page loads, *then* the standards list is navigable by keyboard and each standard chip has an accessible label.

## 8. Data Model

```sql
-- Migration: server/migrations/0031_standards_alignment.sql

CREATE TABLE course.standard_frameworks (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    code       TEXT NOT NULL,           -- e.g. 'ccss-math'
    name       TEXT NOT NULL,           -- 'Common Core State Standards — Mathematics'
    version    TEXT NOT NULL,           -- '2010', '2023-revision'
    publisher  TEXT,
    archived_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (code, version)
);

CREATE TABLE course.standard_codes (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    framework_id    UUID NOT NULL REFERENCES course.standard_frameworks(id),
    parent_id       UUID REFERENCES course.standard_codes(id),
    code            TEXT NOT NULL,      -- 'CCSS.MATH.CONTENT.6.EE.A.1'
    short_code      TEXT,               -- '6.EE.A.1'
    description     TEXT NOT NULL,
    grade_band      TEXT,               -- 'K', '1', '6', '9-12'
    depth_level     SMALLINT NOT NULL DEFAULT 1,  -- 1=framework..4=standard
    archived_at     TIMESTAMPTZ,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (framework_id, code)
);

CREATE INDEX idx_sc_framework ON course.standard_codes (framework_id);
CREATE INDEX idx_sc_grade_band ON course.standard_codes (framework_id, grade_band);
CREATE INDEX idx_sc_fts ON course.standard_codes USING gin(
    to_tsvector('english', code || ' ' || description)
);

-- Concept <-> Standard alignment (many-to-many)
CREATE TABLE course.concept_standard_alignments (
    concept_id       UUID NOT NULL REFERENCES course.concepts(id) ON DELETE CASCADE,
    standard_code_id UUID NOT NULL REFERENCES course.standard_codes(id) ON DELETE RESTRICT,
    alignment_type   TEXT NOT NULL DEFAULT 'primary',  -- 'primary' | 'supplementary'
    created_by       UUID REFERENCES "user".users(id),
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (concept_id, standard_code_id)
);
```

Seed data: `server/seeds/standards/ccss_math.json`, `server/seeds/standards/ccss_ela.json`, `server/seeds/standards/ngss.json` (derived from publicly licensed CASE JSON exports).

Backfill: no backfill needed; alignment is additive.

## 9. API Surface

```
POST /api/v1/admin/standards/import
  Auth: admin
  Body: CASE 1.0 JSON document
  Response 202: { job_id, framework_code, record_count }

GET  /api/v1/standards?framework=&grade=&q=   Auth: authenticated
GET  /api/v1/standards/{id}                   Auth: authenticated
GET  /api/v1/standards/search?q=&framework=   Auth: authenticated

POST /api/v1/concepts/{id}/standards          Auth: instructor|admin
  Body: { standard_code_id, alignment_type }
DELETE /api/v1/concepts/{id}/standards/{sid}  Auth: instructor|admin

GET  /api/v1/courses/{id}/standards-coverage  Auth: instructor|admin
  Response: { standards: StandardCoverageItem[] }
```

## 10. UI / UX

**Standard alignment picker** (in concept editor and question editor):
- Hierarchical tree picker filtered by framework + grade band.
- Search box backed by `/api/v1/standards/search`.
- Selected alignments shown as chips: `[6.EE.A.1 — Write and evaluate expressions…]`.

**Course standards-coverage page** (`/courses/{id}/standards-coverage`):
- Table grouped by framework domain, showing per-standard: question count, average mastery gauge, coverage badge.
- Export to CSV button.
- Accessible: full keyboard navigation, sortable columns with `aria-sort`.

**Empty state:** "No standards aligned yet. Tag your concepts to standards to enable coverage reporting."

## 11. AI / ML Considerations

Not applicable for v1. Future: LLM-assisted auto-alignment — given a question stem, suggest top-3 standard alignments based on semantic similarity to standard descriptions. Would call an endpoint alongside `quiz_generation_ai.rs`.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| CCSS/NGSS license restrictions on redistribution | L | H | Verify CC BY 4.0 license on source data; only seed from explicitly licensed exports |
| State-specific standards proliferate into unmaintainable seed files | M | M | Build admin import tool so states load their own standards rather than shipping all 50 states' variants |
| Standard code changes between framework versions break coverage reports | M | H | Soft-delete with `archived_at`, store `framework_version` on alignment link, show "superseded" badge |
| Import of large CASE document times out HTTP request | M | M | Process import asynchronously via job queue; return 202 with job_id |

## 15. Rollout Plan

- **Feature flag:** `standards_alignment_enabled` (default: `false`).
- **Migration sequencing:** `0031_standards_alignment.sql` → seed CCSS + NGSS → deploy import endpoint → enable for pilot district.
- **Pilot cohort:** one K-8 pilot district; curriculum director aligns a Grade 6 Math unit.
- **GA criteria:** full CCSS Math import successful; coverage report computes correctly for a 100-question course.
- **Rollback:** disable flag; alignment tables are additive and harmless if flag is off.

## 16. Test Plan

- **Unit** — CASE JSON parser; framework version comparison; coverage percentage calculation.
- **Integration** — Import CCSS Math seed → query by grade band → verify record count matches source; align concept → standard → fetch coverage report.
- **End-to-end** — Playwright: instructor aligns a question to a standard, navigates to coverage report, verifies standard appears as "covered."
- **Security** — Non-admin cannot POST import (403); non-authenticated cannot read standards (401).
- **Accessibility** — axe scan on standards picker tree; keyboard navigation to select alignment.
- **Performance / load** — Import of 1 200-record CCSS Math file completes in ≤ 30 s; coverage report for 500-question course in ≤ 500 ms.
- **Manual exploratory** — Verify Bloom's level badges render correctly; verify "superseded" indicator appears after re-importing updated framework version.

## 18. Open Questions

1. Should we pre-ship all 50 US state standards sets or build a community-maintained standards registry that admins can pull from?

3. Should Bloom's taxonomy be treated as a standard framework or remain a first-class field on the `concepts` table (current plan)?
