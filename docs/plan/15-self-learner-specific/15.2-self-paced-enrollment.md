## 1. Problem Statement

Every Lextures course today assumes an instructor is present as the owner and requires enrollment by that instructor. Self-learner platforms (Coursera, Udemy, Khan Academy) allow learners to enroll in a click with no instructor involvement after enrollment opens. The current schema creates course ownership through the user who creates the course; there is no "open enrollment" mode, no instructor-free experience, and no self-directed progress model. This thin implementation blocks the entire self-learner market segment.

## 2. Goals

- Introduce a `self_paced` course mode where learners enroll themselves without instructor intervention.
- Remove instructor-presence assumptions from the learner UI for self-paced courses (no "waiting for grading," no "instructor feedback").
- Support automated completion tracking and progress gates without human review.
- Allow creators to set a course as self-paced with open enrollment (free or paid via 15.3).
- Provide learner-facing progress tracking, module gating, and "continue where you left off" functionality.

## 3. Non-Goals

- Instructor-led course mode changes — existing instructor-led behavior untouched.
- Live cohort scheduling for self-paced courses (deferred).
- Social learning features in self-paced courses (15.13 community Q&A is separate).

## 4. Personas & User Stories

- **As a self-learner**, I want to enroll in a course with one click so that I can start learning immediately without waiting for an instructor.
- **As a self-learner**, I want to pick up exactly where I left off when I return to a course so that I do not waste time re-finding my place.
- **As a course creator**, I want to publish my course in self-paced mode so that it runs without my daily involvement after launch.
- **As a self-learner**, I want to see my overall progress percentage and module completion status so that I know how far along I am.

## 5. Functional Requirements

- **FR-1.** The system MUST support a `course_mode` enum on courses: `instructor_led` (default) and `self_paced`.
- **FR-2.** In `self_paced` mode, the system MUST allow any authenticated user (or unauthenticated user who completes checkout in 15.3) to enroll without instructor approval.
- **FR-3.** The system MUST track per-enrollment module and item completion status automatically (module item viewed = visited; quiz passed = passed threshold).
- **FR-4.** The system MUST surface "Resume Course" on the learner Dashboard linking to the last-visited module item.
- **FR-5.** The system MUST compute and display a course progress percentage (completed items / total items) on the course page and Dashboard card.
- **FR-6.** The system SHOULD support configurable module gating: learner must complete module N before accessing module N+1.
- **FR-7.** The system MUST hide instructor-specific UI elements (instructor feedback queue, grade posting, etc.) from the learner view in self-paced courses.
- **FR-8.** The system SHOULD trigger the certificate flow (15.5) automatically when course progress reaches 100%.

## 6. Non-Functional Requirements

- **Performance** — Enrollment action <300 ms; progress percentage computed in <200 ms.
- **Security** — Enrollment in a paid self-paced course requires a valid entitlement (15.3); unauthorized enrollment returns 402.
- **Privacy & Compliance** — Progress data is personal data under GDPR; retained per privacy policy.
- **Accessibility** — Progress bar WCAG 2.1 AA with text percentage; "Resume Course" button has descriptive aria-label.
- **Scalability** — Enrollment and progress writes at 1,000 concurrent enrollments; item-completion events batched.
- **Reliability** — Progress state is durable; browser close does not lose last-visited position.
- **Observability** — Metric `self_paced_enrollments_total`, `course_completions_total`.
- **Maintainability** — Course mode as enum; self-paced business logic isolated in `server/src/services/self_paced.rs`.
- **Internationalization** — Progress labels externalised; "X% complete" formatted with locale number.
- **Backward compatibility** — All existing courses default to `instructor_led`; no behavior change.

## 7. Acceptance Criteria

- **AC-1.** *Given* a course is in `self_paced` mode with open enrollment, *when* an authenticated user clicks "Enroll," *then* they are immediately enrolled and redirected to the first module without any instructor action.
- **AC-2.** *Given* a learner has completed 3 of 10 items, *when* they visit the course page, *then* the progress bar shows "30% complete" and a "Resume" button links to the 4th item.
- **AC-3.** *Given* module gating is enabled and module 1 is incomplete, *when* a learner attempts to access module 2, *then* the system shows a "Complete Module 1 to unlock" message.
- **AC-4.** *Given* a self-paced course, *when* a learner views it, *then* no instructor feedback queue, grade-posting controls, or "awaiting review" states are visible.
- **AC-5.** *Given* a learner completes all items in a self-paced course, *when* the last item is marked complete, *then* the certificate flow (15.5) is triggered and the learner sees a completion celebration screen.

## 8. Data Model

Migration `093_self_paced_mode.sql`:
```sql
CREATE TYPE course_mode AS ENUM ('instructor_led', 'self_paced');
ALTER TABLE courses ADD COLUMN course_mode course_mode NOT NULL DEFAULT 'instructor_led';
ALTER TABLE courses ADD COLUMN open_enrollment BOOLEAN NOT NULL DEFAULT FALSE;
ALTER TABLE courses ADD COLUMN module_gating_enabled BOOLEAN NOT NULL DEFAULT FALSE;

CREATE TABLE learner_item_progress (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  enrollment_id   UUID NOT NULL REFERENCES course_enrollments(id),
  item_id         UUID NOT NULL REFERENCES course_structure_items(id),
  status          TEXT NOT NULL DEFAULT 'not_started', -- 'not_started', 'in_progress', 'completed'
  last_visited_at TIMESTAMPTZ,
  completed_at    TIMESTAMPTZ,
  UNIQUE(enrollment_id, item_id)
);
CREATE INDEX idx_learner_progress_enrollment ON learner_item_progress(enrollment_id, status);
```

## 9. API Surface

```
POST /api/v1/courses/:id/self-enroll              -- open enrollment (no instructor)
GET  /api/v1/courses/:id/my-progress              -- enrollment progress + last item
POST /api/v1/courses/:id/items/:item_id/complete  -- mark item complete
GET  /api/v1/me/enrollments?mode=self_paced        -- list self-paced enrollments
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Learners enroll in paid courses without paying | M | H | Entitlement check is blocking; 402 response if no active subscription/purchase |
| Progress state de-sync on multi-device use | M | M | Server-authoritative progress; client shows latest server state on load |
| Module gating locks learners out of content they paid for | L | H | Creator can disable gating per course; admin bypass for support |

## 15. Rollout Plan

- Feature flag: `self_paced_mode` (default: off; SL tier).
- Pilot with a creator who has existing course content; validate completion flow.
- GA alongside 15.3 (billing) and 15.5 (certificates).
- Rollback: disable flag; courses revert to instructor_led mode.

## 16. Test Plan

- **Unit** — Open enrollment guard; module gating logic; progress percentage computation; completion trigger.
- **Integration** — Self-enroll API; item complete API; progress query.
- **End-to-end** — Playwright: learner enrolls, progresses through modules, completes course, certificate triggered.
- **Security** — Assert paid course rejects enrollment without entitlement; assert learner A cannot mark items complete for learner B.
- **Accessibility** — axe on course page with progress bar and gated modules.
- **Performance / load** — 1,000 concurrent item-complete events; p95 <200 ms.
- **Manual exploratory** — QA: gated module access attempt, multi-device progress sync, completion on last item.
