## 1. Problem Statement

Lextures has no payment infrastructure. The self-learner market model — where creators sell individual courses or subscriptions — cannot function without billing. Every commercial LMS and online learning platform (Udemy, Teachable, Coursera, Thinkific) has a Stripe integration as table-stakes infrastructure. Without payment processing, course creators cannot monetize their content, the business cannot generate SL revenue, and paid course listings in the catalog are meaningless. This is a BLOCKER for the entire SL segment. Cross-reference: 16.8 payment provider abstraction covers PayPal/iDEAL for international expansion; this plan focuses on Stripe as the GA provider.

## 2. Goals

- Implement Stripe Billing API for three purchase models: one-time course purchase, subscription (monthly/annual platform access), and freemium (free tier with locked premium content).
- Store entitlement records that the self-enroll flow (15.2) checks before granting access.
- Handle Stripe webhooks for payment confirmation, subscription renewal, cancellation, and failed payments.
- Provide learner-facing billing management (view subscription, cancel, update payment method).
- Implement coupon/discount code support via Stripe Promotions.
- Comply with Apple App Store and Google Play subscription rules if a mobile app is built (7.1).

## 3. Non-Goals

- PayPal, iDEAL, or other payment providers (covered by 16.8 abstraction layer in a future plan).
- Instructor payout processing (covered by 15.8 revenue share).
- Institutional invoice billing for HE or K-12 (different commercial model).
- Tax calculation beyond Stripe Tax integration.

## 4. Personas & User Stories

- **As a self-learner**, I want to purchase a course with my credit card in under 60 seconds so that I can start learning immediately.
- **As a self-learner**, I want to subscribe to Lextures for a monthly fee to access the full course library so that I get better value than buying individual courses.
- **As a self-learner**, I want to apply a discount code at checkout so that I can use a promotional offer.
- **As a course creator**, I want to set a price for my course and receive a percentage of every sale so that I earn revenue from my content.
- **As a learner**, I want to manage my subscription and cancel it from my account settings without calling support.

## 5. Functional Requirements

- **FR-1.** The system MUST integrate with Stripe Billing API to process one-time payments (course purchase) and recurring subscriptions (monthly/annual).
- **FR-2.** The system MUST create a Stripe Customer record for each user who makes a purchase and store the `stripe_customer_id` on the `users` table.
- **FR-3.** The system MUST store entitlement records (`user_entitlements`) linking a user to a specific course or subscription tier, derived from confirmed Stripe payments.
- **FR-4.** The system MUST handle Stripe webhook events: `checkout.session.completed`, `invoice.payment_succeeded`, `invoice.payment_failed`, `customer.subscription.deleted`.
- **FR-5.** The system MUST support Stripe Promotion Codes (coupon/discount) at checkout.
- **FR-6.** The system MUST provide a learner billing portal via Stripe Customer Portal for subscription management (cancel, update card, view invoices).
- **FR-7.** The system SHOULD implement a freemium model where the first N items of a course are free and the rest require payment/subscription.
- **FR-8.** The system MUST NOT store full card numbers; payment data is tokenized by Stripe.js on the client.
- **FR-9.** The system SHOULD display pricing in the user's currency using Stripe's multi-currency support.

## 6. Non-Functional Requirements

- **Performance** — Checkout redirect to Stripe <300 ms; webhook processing <500 ms.
- **Security** — Stripe webhook signature verified (HMAC-SHA256) before processing; Stripe secret key stored in secrets manager, not env vars; PCI DSS SAQ A compliance through Stripe-hosted checkout.
- **Privacy & Compliance** — Stripe processes cardholder data; Lextures is PCI SAQ A (no card data touches server); GDPR: Stripe is a sub-processor listed in the privacy policy; Apple/Google subscription rules apply for in-app purchases.
- **Accessibility** — Checkout flow uses Stripe-hosted UI (inherits Stripe's WCAG compliance); pre-checkout course price display WCAG 2.1 AA.
- **Scalability** — Stripe handles payment scale; entitlement table indexed on (user_id, course_id) for fast entitlement checks.
- **Reliability** — Idempotent webhook handler (Stripe can resend); entitlement creation is idempotent using Stripe event ID as idempotency key.
- **Observability** — Metric `payments_total{type, currency}`; `subscription_mrr_cents` gauge; alert on >5% payment failure rate.
- **Maintainability** — Stripe service in `server/src/services/billing/stripe.rs`; entitlement service in `server/src/services/billing/entitlements.rs`.
- **Internationalization** — Price formatted in user's locale currency using Intl.NumberFormat; "free" label externalised.
- **Backward compatibility** — Free courses remain accessible without any entitlement check; only paid courses are gated.

## 7. Acceptance Criteria

- **AC-1.** *Given* a learner clicks "Buy for $29" on a course landing page, *when* they complete the Stripe Checkout flow, *then* within 10 seconds their entitlement record is created and they are redirected to the course.
- **AC-2.** *Given* a Stripe `invoice.payment_failed` webhook arrives for an active subscriber, *when* processed, *then* the subscriber's tier is downgraded to free and they receive an email notification.
- **AC-3.** *Given* a learner enters a valid promo code at checkout, *when* they complete payment, *then* the Stripe invoice reflects the discounted amount and the entitlement is created at the discounted price.
- **AC-4.** *Given* a subscribed learner opens their billing settings, *when* they click "Manage Subscription," *then* Stripe Customer Portal opens in a new tab where they can cancel or update their payment method.
- **AC-5.** *Given* a duplicate Stripe webhook event arrives, *when* processed, *then* no duplicate entitlement record is created (idempotent).

## 8. Data Model

Migration `094_billing.sql`:
```sql
ALTER TABLE users ADD COLUMN stripe_customer_id TEXT UNIQUE;

CREATE TABLE user_entitlements (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id          UUID NOT NULL REFERENCES users(id),
  entitlement_type TEXT NOT NULL, -- 'course_purchase', 'subscription_monthly', 'subscription_annual'
  course_id        UUID REFERENCES courses(id),  -- NULL for subscription
  stripe_event_id  TEXT NOT NULL UNIQUE,         -- idempotency key
  stripe_invoice_id TEXT,
  amount_paid_cents INT NOT NULL DEFAULT 0,
  currency         TEXT NOT NULL DEFAULT 'usd',
  valid_from       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  valid_until      TIMESTAMPTZ,                  -- NULL for lifetime purchases
  status           TEXT NOT NULL DEFAULT 'active', -- 'active', 'expired', 'refunded'
  created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_entitlements_user_course ON user_entitlements(user_id, course_id, status);
CREATE INDEX idx_entitlements_user_subscription ON user_entitlements(user_id, entitlement_type, valid_until)
  WHERE entitlement_type LIKE 'subscription%';
```

## 9. API Surface

```
POST /api/v1/billing/checkout          -- create Stripe Checkout Session
     Body: { course_id?, plan?: 'monthly'|'annual', promo_code?, success_url, cancel_url }

GET  /api/v1/billing/portal            -- redirect to Stripe Customer Portal
GET  /api/v1/me/entitlements           -- list active entitlements
POST /api/v1/webhooks/stripe           -- Stripe webhook receiver (HMAC-verified)

-- Internal (used by self-enroll guard)
GET  /api/v1/internal/entitlements/check?user_id=&course_id=  -- Boolean entitlement check
```

Rate limit: checkout endpoint 10 req/min per user.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Webhook delivery failure leaves user without entitlement | M | H | Customer support unlock endpoint; webhook replay via Stripe dashboard |
| Subscription cancels mid-course — learner loses access | M | M | Configurable grace period (30 days) on cancellation before entitlement expires |
| Stripe API key compromised | L | H | Secrets manager, not env vars; restricted key permissions; monitor for unexpected charges |
| Apple/Google 30% cut on mobile IAP | H | M | Web-first billing; mobile app points to web checkout; App Store policy research before 7.1 ships |
