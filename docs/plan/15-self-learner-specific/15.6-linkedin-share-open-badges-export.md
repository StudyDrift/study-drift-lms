## 1. Problem Statement

Self-learners invest time and money in courses primarily to signal competency to employers. Without the ability to share credentials to LinkedIn or export them as Open Badges JSON, the credential has no distribution mechanism beyond a PDF. LinkedIn's "Add Certification" feature is now the de-facto signal channel for professional learners. Competitors (Coursera, LinkedIn Learning, edX) all provide one-click LinkedIn share. This is a relatively small feature that has outsized impact on learner acquisition (social proof) and motivation (visible reward).

## 2. Goals

- Provide a one-click "Add to LinkedIn" flow using LinkedIn's certification deep link.
- Allow learners to export their Open Badges 3.0 credential as a backpack-compatible JSON file.
- Surface the sharing CTA prominently on the certificate completion screen, credential card, and email notification.
- Track LinkedIn shares and Open Badges exports as marketing signals.

## 3. Non-Goals

- Auto-posting to LinkedIn on the learner's behalf without explicit action.
- Integration with third-party badge platforms (Badgr, Credly) at GA.
- Twitter/X or other social media shares (LinkedIn only for professional credentials).

## 4. Personas & User Stories

- **As a self-learner**, I want to add my Lextures certificate to my LinkedIn profile in one click so that my network sees my achievement.
- **As a learner**, I want to export my Open Badge JSON so that I can host it in a badge backpack or share it with my university.
- **As a hiring manager**, I want the LinkedIn certification link to show Lextures as the issuing organization so that the credential is clearly attributed.

## 5. Functional Requirements

- **FR-1.** The system MUST provide a "Add to LinkedIn" button that opens LinkedIn's certification add URL pre-populated with credential name, issuing organization ("Lextures"), issue date, and credential URL.
- **FR-2.** The system MUST provide a "Download Badge JSON" option that downloads the raw Open Badges 3.0 JSON-LD credential.
- **FR-3.** The system MUST surface these sharing options on: the completion celebration screen, the credential detail page, and the certificate email.
- **FR-4.** The system SHOULD track LinkedIn share clicks and badge JSON downloads as analytics events.
- **FR-5.** The system MAY provide social preview meta tags on the public verification URL so that when the URL is shared on social media, it generates a rich preview card.

## 6. Non-Functional Requirements

- **Performance** — LinkedIn share button opens LinkedIn in <100 ms (client-side URL construction).
- **Security** — Badge JSON download URL is signed and expires in 24 hours; verification URL is public but does not expose learner PII beyond what the credential contains.
- **Privacy & Compliance** — Sharing is always learner-initiated; no automatic social posting; GDPR: learner controls which data is in the public credential.
- **Accessibility** — Share buttons have descriptive `aria-label`; LinkedIn icon has alt text.
- **Scalability** — Badge download is a simple file download from object storage or signed URL; no server load.
- **Reliability** — Badge JSON download is a static file; LinkedIn URL construction is client-side.
- **Observability** — Event `credential_shared{channel: 'linkedin'|'badge_export'}`.
- **Maintainability** — LinkedIn share is a URL construction utility in frontend; badge export calls existing credential JSON endpoint.
- **Internationalization** — Share button labels externalised; LinkedIn URL parameters are English (LinkedIn API requirement).
- **Backward compatibility** — No impact on existing credential records; adds sharing layer on top.

## 7. Acceptance Criteria

- **AC-1.** *Given* a learner clicks "Add to LinkedIn," *when* the LinkedIn URL opens, *then* LinkedIn's "Add Certification" form is pre-filled with course name, "Lextures" as issuer, and the completion date.
- **AC-2.** *Given* a learner clicks "Download Badge JSON," *when* the download completes, *then* the file is a valid Open Badges 3.0 JSON-LD document with the credential's proof intact.
- **AC-3.** *Given* the public verification URL is shared on Twitter/LinkedIn, *when* the social platform previews the link, *then* a rich preview card shows the credential title, learner name, and Lextures logo.
- **AC-4.** *Given* a learner clicks "Add to LinkedIn," *when* the click is recorded, *then* the analytics event `credential_shared{channel: linkedin}` is emitted.

## 8. Data Model

No new tables required. Sharing events are written to `user_audit` with a `credential_share` event type.

## 9. API Surface

```
GET /api/v1/credentials/:id/badge-export   -- signed URL for Open Badges JSON download
GET /api/v1/credentials/:id/linkedin-params -- returns pre-filled LinkedIn URL params (JSON)
```

LinkedIn certification URL format:
`https://www.linkedin.com/profile/add?startTask=CERTIFICATION_NAME&name={name}&organizationId={org_id}&issueYear={year}&issueMonth={month}&certUrl={url}&certId={id}`
