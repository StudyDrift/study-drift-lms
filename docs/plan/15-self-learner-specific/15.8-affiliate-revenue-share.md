## 1. Problem Statement

Creator-economy platforms live or die by their revenue sharing model. If Lextures wants instructors to publish courses on the platform, it must offer competitive revenue share (Udemy: 37%, Teachable: up to 95%, Coursera: 25%). Currently there is no revenue share system — creators have no financial incentive to publish on Lextures versus other platforms. Similarly, affiliate referral programs are a major customer acquisition driver: Udemy's affiliate network drives ~30% of new learner enrollments. Without affiliate tracking, Lextures cannot build a referral channel or incentivize influencer promotion.

## 2. Goals

- Track instructor revenue share on each course purchase and compute creator earnings.
- Support configurable platform fee percentage (admin-set) with creator receiving the remainder.
- Implement an affiliate link system with UTM-style tracking code that attributes enrollments and triggers referral commissions.
- Deliver creator earnings payouts via Stripe Connect.
- Provide a creator earnings dashboard (feeds into 15.14).

## 3. Non-Goals

- Multi-level marketing or pyramid referral chains.
- Off-platform affiliate tracking (cookie-based web affiliate network) at GA — URL-code-based only.
- Tax form generation (1099-K) — platform generates reports; tax filing is creator's responsibility.
- Course co-authorship revenue splits in v1.

## 4. Personas & User Stories

- **As a course creator**, I want to know exactly how much I earn from each sale so that I can evaluate whether publishing on Lextures is worth my time.
- **As a creator**, I want to receive my monthly earnings payout via direct deposit without manual invoicing.
- **As a marketer/affiliate**, I want a referral link I can share so that I earn a commission when someone buys through my link.
- **As an admin**, I want to set the platform's revenue share policy globally and override it for individual creators.

## 5. Functional Requirements

- **FR-1.** The system MUST compute creator earnings on each confirmed course purchase as: `sale_price × (1 - platform_fee_pct)`, where `platform_fee_pct` is configurable per creator (default 30%).
- **FR-2.** The system MUST record an `earnings_ledger` entry for every purchase, subscription proration, and refund that affects creator revenue.
- **FR-3.** The system MUST support Stripe Connect (Express accounts) for creator payouts, initiating transfers on a monthly schedule.
- **FR-4.** The system MUST support affiliate/referral tracking codes: unique per-user URLs (`/courses/:slug?ref=XXXXX`) that set a 30-day cookie; purchases within the cookie window credit the affiliate.
- **FR-5.** The system MUST record affiliate commission earnings (configurable percentage, default 10% of sale) as a separate `earnings_ledger` entry.
- **FR-6.** The system SHOULD allow creators to generate referral links for their own courses to earn affiliate commission on top of creator share when they drive the sale.
- **FR-7.** The system MUST prevent self-referral (creator buying their own course via their referral link earning commission).

## 6. Non-Functional Requirements

- **Performance** — Earnings ledger write <200 ms; monthly payout batch job for up to 10,000 creators runs in <30 min.
- **Security** — Stripe Connect keys scoped per creator; platform account has no write access to creator's bank account beyond transfers; affiliate codes are UUIDs, not guessable sequences.
- **Privacy & Compliance** — Earnings data is financial PII; GDPR: creator may export their earnings data; SOC 2 financial controls documentation required.
- **Accessibility** — Earnings dashboard WCAG 2.1 AA; charts have text alternatives.
- **Scalability** — Earnings ledger table partitioned by `created_at` month.
- **Reliability** — Payout batch is idempotent per Stripe transfer idempotency key; failed payouts retry and alert creator.
- **Observability** — Metric `creator_payouts_total{status}`; `platform_revenue_cents_total`; alert on payout failure rate >1%.
- **Maintainability** — Revenue share service in `server/src/services/billing/revenue_share.rs`.
- **Internationalization** — Earnings displayed in creator's currency; payout amounts formatted with locale.
- **Backward compatibility** — Free courses have no earnings records; existing paid courses retroactively generate earnings records from historical Stripe events.

## 7. Acceptance Criteria

- **AC-1.** *Given* a learner purchases a $49 course with a 30% platform fee, *when* the Stripe webhook confirms payment, *then* an earnings_ledger entry of $34.30 is created for the creator.
- **AC-2.** *Given* a learner purchases via an affiliate link, *when* the month-end payout batch runs, *then* the affiliate receives a Stripe Connect transfer for their commission.
- **AC-3.** *Given* a creator attempts to buy their own course via their referral link, *when* the entitlement is created, *then* no affiliate commission entry is created (self-referral blocked).
- **AC-4.** *Given* a creator has $500 in pending earnings, *when* the monthly payout runs, *then* a Stripe Connect transfer of $500 is initiated and the earnings_ledger entries are marked `paid`.
- **AC-5.** *Given* a refund occurs on a purchase with creator earnings, *when* the Stripe refund webhook arrives, *then* a negative earnings_ledger entry offsets the original creator earnings.

## 8. Data Model

Migration `098_revenue_share.sql`:
```sql
ALTER TABLE users ADD COLUMN stripe_connect_id TEXT UNIQUE;

CREATE TABLE creator_revenue_configs (
  user_id             UUID PRIMARY KEY REFERENCES users(id),
  platform_fee_pct    NUMERIC(5,4) NOT NULL DEFAULT 0.30,  -- 0.30 = 30%
  affiliate_fee_pct   NUMERIC(5,4) NOT NULL DEFAULT 0.10
);

CREATE TABLE affiliate_codes (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES users(id),
  code        TEXT NOT NULL UNIQUE,
  course_id   UUID REFERENCES courses(id),  -- NULL = platform-wide
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE earnings_ledger (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  payee_id        UUID NOT NULL REFERENCES users(id),
  entry_type      TEXT NOT NULL, -- 'sale', 'affiliate', 'refund', 'payout'
  amount_cents    INT NOT NULL,
  currency        TEXT NOT NULL DEFAULT 'usd',
  stripe_event_id TEXT UNIQUE,
  course_id       UUID REFERENCES courses(id),
  affiliate_code  TEXT,
  status          TEXT NOT NULL DEFAULT 'pending', -- 'pending', 'paid', 'held'
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
) PARTITION BY RANGE (created_at);
CREATE INDEX idx_earnings_payee ON earnings_ledger(payee_id, status, created_at DESC);
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Stripe Connect KYC delays block creator payouts | H | M | Clearly communicate KYC requirement in onboarding; earnings accumulate until KYC complete |
| Affiliate cookie fraud (click stuffing) | M | M | Last-click attribution; rate-limit referral code clicks per IP |
| Tax liability for platform as withholding agent | M | H | Consult tax counsel before GA; document that platform is not responsible for creator taxes |

- **End-to-end** — Playwright (test mode): purchase via affiliate link, verify affiliate commission entry, trigger payout.
- **Security** — Assert creator cannot see another creator's earnings; assert self-referral blocked.
- **Accessibility** — axe on earnings dashboard.
- **Performance / load** — Payout batch for 10,000 creators in <30 min.
- **Manual exploratory** — QA: refund reversal earnings, partial refund, subscription proration, zero-earning month.
