## 1. Problem Statement

Lextures has a RAG-based notebook (`student_notebook_rag_ai.rs`) that answers questions reactively but has no memory of the learner across sessions, no awareness of what they have studied, and no proactive guidance. The 6.9 gap identifies a conversational AI tutor as a major differentiator. For the self-learner market, this is framed as an "AI study buddy" — a persistent companion that knows the learner's goals (from 15.11), tracks their progress, surfaces relevant content, and answers questions in context. Cross-reference: 6.9 covers the underlying tutor infrastructure; this plan is the SL-specific persistent companion experience layered on top.

## 2. Goals

- Extend the RAG notebook into a persistent, session-spanning AI study buddy with memory of the learner's goals, current course, and recent struggles.
- Surface proactive study prompts ("You haven't reviewed Chapter 3 in 5 days — want a quick quiz?").
- Answer learner questions grounded in course materials (RAG) while citing sources.
- Personalize tone and depth based on learner's stated prior knowledge level (from 15.11).
- Comply with AI disclosure requirements (10.17): clearly labeled as AI, with a data-use toggle.

## 3. Non-Goals

- Replacing instructor feedback or grading.
- Chatting about topics completely outside the learner's enrolled courses.
- Storing raw conversation transcripts permanently (short retention window for privacy).
- Human-in-the-loop escalation (deferred to v2 "Ask an Expert" feature).

## 4. Personas & User Stories

- **As a self-learner**, I want an AI companion that remembers I am studying Python for data science so that it gives me relevant examples, not generic ones.
- **As a learner struggling with a concept**, I want to ask a question and get a clear explanation that cites the specific course video or reading I studied.
- **As a learner**, I want the AI to proactively remind me of spaced-repetition reviews so that I retain what I learn.
- **As a learner**, I want to be clearly told when I am talking to an AI and be able to turn off AI data-use for privacy.

## 5. Functional Requirements

- **FR-1.** The system MUST maintain a per-user AI memory context storing: current course, recent module items visited, learner goals (from 15.11), known struggle concepts (from 15.9/quiz wrong answers), and last-session summary.
- **FR-2.** The system MUST ground all study-buddy responses in course materials via RAG (existing `student_notebook_rag_ai.rs`) and always cite the source module item.
- **FR-3.** The system MUST support proactive prompts displayed on the course page and Dashboard when triggered by: content not visited in N days, quiz score below threshold, or upcoming review due (spaced repetition from 1.5).
- **FR-4.** The system MUST display a persistent chat widget accessible from the course page and Dashboard, with conversation history for the current session.
- **FR-5.** The system MUST clearly label the study buddy as AI in all surfaces and provide an in-chat disclosure ("I'm an AI — here's how your data is used").
- **FR-6.** The system MUST respect the learner's AI data-use toggle (10.17): if off, study buddy is disabled with a clear explanation.
- **FR-7.** The system SHOULD store a rolling 7-day conversation summary in the memory context (not full transcripts) for personalization.
- **FR-8.** The system MUST support citation enforcement: responses that cannot be grounded in course materials should say so rather than hallucinate.

## 6. Non-Functional Requirements

- **Performance** — First study-buddy response <3 s; streaming response starts within 1 s.
- **Security** — Memory context contains no raw PII beyond display name; conversation context stored encrypted at rest; AI provider receives only course content + anonymized learner state, not full profile.
- **Privacy & Compliance** — 10.17 AI disclosure required; student can disable AI; conversation summary retained 7 days, full transcripts not retained; GDPR data export includes memory context summary.
- **Accessibility** — Chat widget keyboard-accessible; messages announced via ARIA live region; WCAG 2.1 AA.
- **Scalability** — Memory context is per-user per-course; context window managed to stay within token budget.
- **Reliability** — Fallback: if AI provider unavailable, display "Study buddy is temporarily unavailable" without blocking the course.
- **Observability** — Metric `study_buddy_messages_total`, `study_buddy_citations_per_response` (P50); cost per message; alert if avg cost/msg exceeds $0.05.
- **Maintainability** — Study buddy service extends `server/src/services/student_notebook_rag_ai.rs`; memory store in Redis with 7-day TTL.
- **Internationalization** — Study buddy responds in the learner's locale language (via system prompt instruction); UI labels externalised.
- **Backward compatibility** — Study buddy is additive; existing notebook is not replaced.

## 7. Acceptance Criteria

- **AC-1.** *Given* a learner has studied Python chapter 2 and asks "I don't understand list comprehensions," *when* the study buddy responds, *then* the response cites the specific course module containing list comprehensions and includes an example derived from the course content.
- **AC-2.** *Given* a learner's quiz score on "Functions" was 40%, *when* they open the study buddy the next day, *then* a proactive prompt appears: "You struggled with Functions last time — want to review?"
- **AC-3.** *Given* a learner has the AI data-use toggle set to off, *when* they open the study buddy widget, *then* the widget shows "AI features disabled — enable in settings" with a settings link.
- **AC-4.** *Given* the study buddy is asked a question about a topic not covered in any course material, *when* it responds, *then* it says "I couldn't find information about this in your course materials" rather than making up an answer.
- **AC-5.** *Given* a conversation session lasts 30 minutes, *when* the session ends, *then* a 2–3 sentence summary is stored in the memory context and is used to personalize the next session's greeting.

## 8. Data Model

Migration `102_study_buddy_memory.sql`:
```sql
CREATE TABLE study_buddy_memory (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id           UUID NOT NULL REFERENCES users(id),
  course_id         UUID NOT NULL REFERENCES courses(id),
  goals_summary     TEXT,
  struggle_concepts TEXT[],
  last_session_summary TEXT,
  last_active_at    TIMESTAMPTZ,
  updated_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(user_id, course_id)
);
-- Full conversation transcripts NOT stored in DB (streaming only, max 7-day Redis TTL)
```

## 11. AI / ML Considerations

- Model: OpenRouter-routed LLM (existing infrastructure); default Claude Sonnet; configurable per tenant (19.10).
- Prompt: system prompt includes learner's goal, current course context, struggle concepts, prior conversation summary. Instructs model to cite module items by name and decline ungrounded answers.
- RAG: `student_notebook_rag_ai.rs` provides course-material retrieval; top 5 chunks included in context.
- Eval: golden QA pairs per course for regression testing; citation-grounding rate as eval metric (target: >90% of factual claims cited).
- Fallback: if RAG fails, study buddy still responds but labels response as "general knowledge, not from course materials."
- Cost budget: target <$0.03 per message; context-window management trims memory to fit budget.
- PII: student name included in prompt as "display name"; no email, grade, or sensitive data; 19.11 PII redaction proxy wraps all prompt calls.
- Cross-reference: 6.9 AI tutor infrastructure plan covers the underlying model architecture.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Hallucinated answers presented as course-grounded | M | H | Citation enforcement prompt; fallback label for ungrounded responses |
| Cost overrun from long conversations | M | M | Per-message token budget; context-window trimming; cost alert at $0.05/msg avg |
| Learner over-reliance on AI for assignments (academic integrity) | M | M | Study buddy clearly states "I can explain concepts but should not write your assignments" |
| GDPR data request includes AI conversation | M | M | 7-day rolling Redis TTL; after 7 days, only summary in DB (covered by GDPR export) |

## 15. Rollout Plan

- Feature flag: `ai_study_buddy` (default: off; SL tier).
- Pilot with 20 beta learners; measure engagement and hallucination rate.
- Human eval of 50 responses per pilot week; tune prompt based on findings.
- GA after citation-grounding rate >90% in eval harness.
- Rollback: disable flag; existing notebook unaffected.

## 16. Test Plan

- **Unit** — Memory context serialization; proactive prompt trigger logic; citation extraction; cost budget enforcer.
- **Integration** — End-to-end message → RAG retrieval → LLM → streaming response; memory update after session.
- **End-to-end** — Playwright: learner sends message, response streams with citation link, proactive prompt appears after quiz fail.
- **Security** — Assert no PII in LLM prompt beyond display name; assert AI toggle disables widget; assert user A cannot read user B's memory context.
- **Accessibility** — axe on chat widget; streaming response accessibility.
- **Performance / load** — 100 concurrent study buddy sessions; first response <3 s.
- **AI eval** — 50-question golden set per subject; measure citation rate, factual accuracy, helpfulness (human eval).
