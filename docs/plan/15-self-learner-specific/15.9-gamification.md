## 1. Problem Statement

Self-learner retention is the biggest challenge in the online learning market (Coursera course completion rates average ~15%). Platforms like Duolingo have demonstrated that gamification — streaks, XP, leaderboards, badges — dramatically improves daily engagement and course completion rates. Lextures has no gamification layer whatsoever. Without it, learners have no extrinsic motivation beyond the credential, and the platform is easily abandoned after the initial enthusiasm fades. This is a hard requirement for the self-learner segment.

## 2. Goals

- Implement a learning streak system (consecutive days with at least one completed learning activity).
- Award XP (experience points) for learning activities (completing modules, passing quizzes, finishing courses).
- Build a course-level or platform-level leaderboard showing top learners by XP or streak.
- Award milestone badges for XP thresholds, streak lengths, and course completions.
- Provide a learner profile page showing gamification stats.

## 3. Non-Goals

- Multiplayer/competitive games or head-to-head challenges.
- Instructor leaderboards.
- Blockchain-based token rewards.
- Gamification in HE instructor-led courses (opt-in only in HE context).

## 4. Personas & User Stories

- **As a self-learner**, I want to maintain a daily learning streak so that I build a consistent study habit.
- **As a self-learner**, I want to earn XP for every module I complete so that I feel a tangible sense of progress beyond course percentage.
- **As a competitive learner**, I want to see how I rank against other learners in a course so that I stay motivated to study more.
- **As a learner**, I want streak protection (a "freeze" item I earn) so that missing one day does not destroy my 30-day streak.

## 5. Functional Requirements

- **FR-1.** The system MUST compute a daily learning streak per user: increments when the user completes at least one learning activity on a given calendar day (in their local time zone); resets to 0 if a day is skipped.
- **FR-2.** The system MUST award XP for learning activities: module item viewed (5 XP), quiz passed (20 XP), course completed (100 XP), path completed (300 XP).
- **FR-3.** The system MUST maintain a cumulative XP balance per user and compute an XP-based level (e.g., level = floor(sqrt(xp / 10))).
- **FR-4.** The system MUST support a course-level leaderboard showing top 10 enrolled learners by XP earned in that course.
- **FR-5.** The system MUST award milestone badges: "7-day streak," "30-day streak," "100 XP," "1000 XP," "First Course Complete."
- **FR-6.** The system SHOULD support "streak freeze" items that a learner earns (awarded at 7, 30, 60 streak days) and can spend to protect their streak.
- **FR-7.** The system SHOULD send a push/email reminder when a learner's streak is at risk of ending (15.10 integration).

## 6. Non-Functional Requirements

- **Performance** — XP and streak update <100 ms after activity event; leaderboard query <500 ms.
- **Security** — XP awards are server-generated from activity events, never client-submitted; leaderboard data is pseudonymous (display name, not email).
- **Privacy & Compliance** — Learners can opt out of leaderboard visibility; GDPR: gamification data included in data export.
- **Accessibility** — Streak counter and XP displayed with text labels; leaderboard as sortable `<table>`; WCAG 2.1 AA.
- **Scalability** — XP ledger uses an append-only event table; streak is computed from a daily-activity summary; leaderboard is cached.
- **Reliability** — Activity events idempotent (same item completion does not award XP twice).
- **Observability** — Metric `xp_awarded_total{activity_type}`, `streak_resets_total`; P50/P90 streak length.
- **Maintainability** — Gamification service in `server/src/services/gamification.rs`; XP awards configured in a JSON config file.
- **Internationalization** — "Day Streak" label externalised; streak display uses locale number format.
- **Backward compatibility** — Gamification is opt-in for existing courses; HE courses have it disabled by default.

## 7. Acceptance Criteria

- **AC-1.** *Given* a learner completes their first module item, *when* the completion event fires, *then* 5 XP is added to their balance and the daily streak counter increments to 1.
- **AC-2.** *Given* a learner has a 7-day streak, *when* they complete any activity on day 7, *then* the "7-day Streak" badge is awarded and shown in their profile.
- **AC-3.** *Given* a learner misses a day without a streak freeze, *when* they next log in, *then* their streak counter shows 0 and a "Your streak ended" notification is shown.
- **AC-4.** *Given* a learner views the course leaderboard, *when* they are in the top 10, *then* their display name is highlighted; if they are not in the top 10, their rank is shown below the top 10.
- **AC-5.** *Given* the same item-completion event fires twice (retry scenario), *when* both events are processed, *then* XP is awarded only once (idempotency).

## 8. Data Model

Migration `099_gamification.sql`:
```sql
CREATE TABLE user_gamification (
  user_id           UUID PRIMARY KEY REFERENCES users(id),
  xp_total          INT NOT NULL DEFAULT 0,
  current_streak    INT NOT NULL DEFAULT 0,
  longest_streak    INT NOT NULL DEFAULT 0,
  last_activity_date DATE,
  streak_freezes    INT NOT NULL DEFAULT 0
);

CREATE TABLE xp_events (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       UUID NOT NULL REFERENCES users(id),
  activity_type TEXT NOT NULL,
  source_id     UUID,  -- item_id, quiz_id, course_id, etc.
  xp_awarded    INT NOT NULL,
  idempotency_key TEXT UNIQUE,  -- prevents duplicate awards
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_xp_events_user ON xp_events(user_id, created_at DESC);

CREATE TABLE user_badges (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id    UUID NOT NULL REFERENCES users(id),
  badge_type TEXT NOT NULL,
  awarded_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(user_id, badge_type)
);
```

## 9. API Surface

```
GET  /api/v1/me/gamification           -- streak, XP, level, badges
GET  /api/v1/courses/:id/leaderboard   -- top 10 + current user rank
GET  /api/v1/me/badges                 -- list awarded badges
POST /api/v1/me/gamification/freeze-streak  -- spend a freeze item
```

Activity events trigger XP internally via the same hooks as progress tracking (15.2); no separate client API call.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Time zone handling for streak day boundary | H | M | Use learner's configured time zone (11.4); server computes streak reset in learner's midnight |
| Gamification rewards become more important than learning outcomes | M | M | XP awards favor completion quality (quiz pass) over quantity (item views) |
| Leaderboard discourages slower learners | M | M | Make leaderboard opt-in; default hidden on first run |

## 15. Rollout Plan

- Feature flag: `gamification` (default: off; SL tier).
- Pilot with 100 beta learners.
- Measure 30-day retention improvement before GA.
- GA alongside 15.10 (study reminders complete the retention loop).

## 16. Test Plan

- **Unit** — Streak computation across time zones; XP idempotency; badge award threshold logic.
- **Integration** — Activity event → XP award → streak update → badge check; freeze spend.
- **End-to-end** — Playwright: complete item, verify XP/streak on Dashboard; simulate day skip, verify streak reset.
- **Security** — Assert XP cannot be submitted directly from client; assert leaderboard shows display names only.
- **Accessibility** — axe on Dashboard gamification components; leaderboard table.
- **Performance / load** — 1,000 concurrent XP events; p95 <100 ms.
- **Manual exploratory** — QA: midnight boundary, freeze usage, same-item completion twice, leap-day streak.
