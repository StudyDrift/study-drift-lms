## 1. Problem Statement

Self-learner platforms drive average order value and learner retention by grouping courses into "specializations," "nanodegrees," or "learning paths" (e.g., Coursera Specializations, Udemy Learning Paths). A learner who commits to a Python Data Science path is more likely to complete all three courses and remain subscribed than one who purchases a single course. Lextures currently has no bundling or path concept; every course is an island. Without learning paths, the platform cannot upsell, cannot guide learners through structured curricula, and cannot award path-completion certificates.

## 2. Goals

- Allow creators and admins to create learning paths (ordered collections of courses with a narrative arc).
- Support bundle pricing (purchase a path at a discount vs. buying courses individually).
- Track path-level progress across constituent courses.
- Award a path-completion certificate (15.5) when all courses in the path are complete.
- Surface paths in the public catalog (15.1) with their own landing pages.

## 3. Non-Goals

- AI-generated personalized paths (that is 1.8 recommendations engine).
- K-12 or HE curriculum sequences (those are course sections/modules, not paths).
- Cross-creator paths without creator consent.

## 4. Personas & User Stories

- **As a self-learner**, I want to enroll in a "Machine Learning Engineer" path so that I follow a curated sequence of courses rather than guessing the right order.
- **As a learner**, I want to see my overall path progress in one view across all constituent courses so that I stay motivated.
- **As a creator**, I want to group my courses into a specialization and offer it at a discounted bundle price so that I increase revenue per learner.
- **As a learner**, I want to earn a "Machine Learning Engineer" certificate when I finish all courses in the path.

## 5. Functional Requirements

- **FR-1.** The system MUST support a `learning_paths` model with a title, description, ordered list of constituent courses, a creator/owner, and an optional bundle price.
- **FR-2.** The system MUST allow learners to enroll in a path, which creates individual course enrollments for all constituent courses at once.
- **FR-3.** The system MUST track path-level progress as the fraction of constituent courses completed.
- **FR-4.** The system MUST allow paths to have a bundle price (Stripe product via 15.3) that is distinct from the sum of individual course prices.
- **FR-5.** The system MUST trigger a path-level completion certificate (15.5) when all courses in the path are completed.
- **FR-6.** The system SHOULD support prerequisite ordering: course N in the path is recommended after course N-1 (soft gate, not a hard lock by default).
- **FR-7.** The system SHOULD display path landing pages in the public catalog with path-level metadata (total duration, course count, skills covered).

## 6. Non-Functional Requirements

- **Performance** — Path enrollment (bulk course enrollment) <1 s for a 10-course path.
- **Security** — Bundle purchase entitlement checked before path enrollment; individual course entitlements granted automatically from path entitlement.
- **Privacy & Compliance** — Path enrollment data is subject to the same GDPR policies as course enrollment.
- **Accessibility** — Path progress overview WCAG 2.1 AA; ordered course list uses `<ol>`.
- **Scalability** — Path can contain up to 20 courses; bulk enrollment is a single transaction.
- **Reliability** — Bulk enrollment is atomic; if one course enrollment fails, none are created.
- **Observability** — Metric `path_enrollments_total`, `path_completions_total`.
- **Maintainability** — Path service in `server/src/services/learning_paths.rs`.
- **Internationalization** — Path labels externalised; "X of Y courses complete" formatted.
- **Backward compatibility** — Individual course enrollment and purchase unaffected.

## 7. Acceptance Criteria

- **AC-1.** *Given* a learner purchases a path bundle, *when* payment confirms, *then* they are enrolled in all constituent courses simultaneously.
- **AC-2.** *Given* a learner completes course 3 of a 4-course path, *when* they open their Dashboard, *then* path progress shows "75% — 3 of 4 complete."
- **AC-3.** *Given* a learner completes the final course in a path, *when* completion is recorded, *then* a path-level certificate is issued and a celebration screen appears.
- **AC-4.** *Given* a path is visible in the public catalog, *when* an unauthenticated user views the path landing page, *then* the page lists all constituent courses with their duration and skill tags.
- **AC-5.** *Given* a path bundle price is $99 and individual course total is $147, *when* the learner views the path landing page, *then* both prices are shown with a savings callout.

## 8. Data Model

Migration `095_learning_paths.sql`:
```sql
CREATE TABLE learning_paths (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  creator_id      UUID NOT NULL REFERENCES users(id),
  title           TEXT NOT NULL,
  description     TEXT,
  slug            TEXT UNIQUE,
  bundle_price_cents INT,
  stripe_product_id  TEXT,
  is_public       BOOLEAN NOT NULL DEFAULT FALSE,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE learning_path_courses (
  path_id       UUID NOT NULL REFERENCES learning_paths(id) ON DELETE CASCADE,
  course_id     UUID NOT NULL REFERENCES courses(id),
  position      INT NOT NULL,
  PRIMARY KEY (path_id, course_id)
);

CREATE TABLE path_enrollments (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES users(id),
  path_id     UUID NOT NULL REFERENCES learning_paths(id),
  enrolled_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  completed_at TIMESTAMPTZ,
  UNIQUE(user_id, path_id)
);
```

```
GET  /api/v1/me/paths/:id/progress                -- learner path progress
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Creator removes a course from a path mid-enrollment | M | M | Enrolled learners retain individual course access; path progress excludes removed course |
| Bundle cheaper than subscription incentivizes path-only purchase | L | L | Deliberate — paths are a revenue diversification strategy |

## 15. Rollout Plan

- Feature flag: `learning_paths` (default: off; SL tier).
- Pilot with 2 creators building paths from existing courses.
- GA alongside 15.3 bundle billing.

## 16. Test Plan

- **Unit** — Bulk enrollment atomicity; path progress calculation; bundle entitlement derivation.
- **Integration** — Path enroll API; progress query; completion trigger.
- **End-to-end** — Playwright: learner buys bundle, enrolls in path, completes all courses, receives path certificate.
- **Security** — Assert path enrollment blocked without bundle entitlement.
- **Accessibility** — axe on path landing page and My Paths dashboard.
- **Manual exploratory** — QA: path with one course already purchased individually, remove course from path, path with 1 course.
