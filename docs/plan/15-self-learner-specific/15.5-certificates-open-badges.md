## 1. Problem Statement

Certificates of completion are the primary tangible outcome self-learners seek. Without them, Lextures cannot compete in the SL market. More importantly, Lextures has an opportunity to issue verifiable credentials — Open Badges 3.0 compliant, W3C Verifiable Credential-signed — rather than just a PDF that can be faked. This positions Lextures as a trust infrastructure differentiator and enables the Co-Curricular Transcript (14.13). Certificates are also referenced in `FEATURES.md` §21 as a known gap.

## 2. Goals

- Issue course-completion certificates as IMS Open Badges 3.0 `Achievement` credentials signed as W3C Verifiable Credentials.
- Generate a PDF rendering of the certificate for download and printing.
- Provide a shareable verification URL any third party can use to verify authenticity.
- Support path-level certificates (15.4) using the same infrastructure.
- Feed awarded badges to the Co-Curricular Transcript (14.13).

## 3. Non-Goals

- Badges for partial progress or gamification (those are XP/streak badges in 15.9).
- Integrating with external badge hosting platforms (Badgr, Credly) at GA — native first.
- Physical certificate printing or mailing.
- Accreditation-level transcript replacement (see 14.13 CCR for that).

## 4. Personas & User Stories

- **As a self-learner**, I want to receive a certificate immediately upon completing a course so that I can share it with my employer the same day.
- **As an employer**, I want to verify a candidate's Lextures certificate is authentic without calling the company.
- **As a learner**, I want my certificate to be downloadable as a PDF and shareable to LinkedIn.
- **As a creator**, I want my course to award a branded certificate with my course title and the learner's name so that it has professional value.

## 5. Functional Requirements

- **FR-1.** The system MUST issue an Open Badges 3.0 `AchievementCredential` when a learner completes a course (15.2 completion trigger) or a learning path (15.4).
- **FR-2.** The system MUST sign the credential as a W3C Verifiable Credential using the platform's DID (did:web) with an Ed25519 proof.
- **FR-3.** The system MUST generate a PDF rendering of the certificate using the course's template (or a default Lextures template) that includes the learner's name, course title, completion date, and a QR code linking to the verification URL.
- **FR-4.** The system MUST provide a public verification URL (`/verify/:credential_id`) that validates the VC signature and displays the credential details.
- **FR-5.** The system MUST store issued credentials in a `issued_credentials` table with the full JSON-LD payload and proof.
- **FR-6.** The system SHOULD notify the learner by email with the PDF certificate and verification link upon issuance.
- **FR-7.** The system SHOULD allow course creators to customize the certificate template (logo, background color, credential name).
- **FR-8.** The system MUST surface issued credentials on the learner's Dashboard and profile page.

## 6. Non-Functional Requirements

- **Performance** — Certificate issuance (sign + PDF render) <5 s; async via background job with email delivery.
- **Security** — Ed25519 signing key in secrets vault; private key never exposed via API; verification URL uses credential UUID, not sequential ID.
- **Privacy & Compliance** — Certificate includes learner's display name; learner consents via platform ToS; GDPR: learner may request credential revocation.
- **Accessibility** — PDF certificate has accessible text layer; web verification page WCAG 2.1 AA.
- **Scalability** — Certificate generation runs in background job queue (17.3); scales with queue workers.
- **Reliability** — Idempotent: if completion fires twice, only one credential is issued.
- **Observability** — Metric `credentials_issued_total{type}`; `credential_verifications_total`.
- **Maintainability** — Credential service in `server/src/services/credentials.rs`; PDF renderer in `server/src/services/pdf_render.rs`.
- **Internationalization** — Certificate template text externalised; dates in ISO 8601 in the JSON; locale date format in PDF.
- **Backward compatibility** — No impact on existing course data.

## 7. Acceptance Criteria

- **AC-1.** *Given* a learner completes all items in a self-paced course, *when* the completion event fires, *then* a signed Open Badges 3.0 credential is created and the learner receives an email with the PDF within 2 minutes.
- **AC-2.** *Given* a third party accesses the verification URL for an authentic credential, *when* the page loads, *then* a green checkmark appears with the learner name, course, issuer, and completion date; the W3C VC signature is verified against the platform DID.
- **AC-3.** *Given* a tampered credential JSON is submitted to a verification check, *when* the signature is validated, *then* the response shows "Invalid credential — signature mismatch."
- **AC-4.** *Given* a learner completes a 3-course learning path, *when* the last course completion fires, *then* a path-level credential is also issued in addition to the individual course credential.
- **AC-5.** *Given* a learner views their Dashboard, *when* they have 3 earned credentials, *then* they see a "My Credentials" section with credential cards linking to PDF download and verification URL.

## 8. Data Model

Migration `096_credentials.sql`:
```sql
CREATE TABLE credential_templates (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  course_id       UUID REFERENCES courses(id),  -- NULL = platform default
  path_id         UUID REFERENCES learning_paths(id),
  name            TEXT NOT NULL,
  description     TEXT,
  background_url  TEXT,
  logo_url        TEXT,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE issued_credentials (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  recipient_id    UUID NOT NULL REFERENCES users(id),
  template_id     UUID REFERENCES credential_templates(id),
  source_type     TEXT NOT NULL, -- 'course', 'path', 'ceu'
  source_id       UUID NOT NULL,
  credential_json JSONB NOT NULL,  -- Open Badges 3.0 / W3C VC JSON-LD
  proof           JSONB NOT NULL,
  pdf_key         TEXT,
  issued_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(recipient_id, source_type, source_id)  -- one credential per completion
);
CREATE INDEX idx_issued_credentials_recipient ON issued_credentials(recipient_id, issued_at DESC);
```

## 9. API Surface

```
GET  /api/v1/me/credentials                    -- list issued credentials
GET  /api/v1/credentials/:id/download          -- PDF download (signed URL)
GET  /api/v1/credentials/:id/json              -- raw credential JSON-LD
GET  /verify/:id                               -- public verification page (no auth)
POST /api/v1/internal/credentials/issue        -- internal: triggered by completion event
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| PDF rendering failure leaves learner without certificate | M | M | Retry in background job; admin alert; manual re-issue endpoint |
| Ed25519 key rotation invalidates existing credentials | L | H | Key rotation policy: old key remains published in DID document for verification; new key for new issuances |
| Credential revocation request (GDPR right-to-erasure) | L | M | Add `revoked` flag; verification URL shows "Credential has been revoked" on revoked records |

## 15. Rollout Plan

## 16. Test Plan

- **Unit** — Open Badges 3.0 JSON-LD builder; W3C VC proof construction; PDF template rendering; idempotency check.
- **Integration** — Completion event → credential issuance → email delivery; verification URL validates signature.
- **End-to-end** — Playwright: learner completes course, downloads PDF, verification URL shows valid.
- **Security** — Assert tampered JSON returns invalid; assert private key not in any API response.
- **Accessibility** — axe on verification page and my-credentials page.
- **Performance / load** — 500 concurrent credential issuances; background job handles queue.
- **Manual exploratory** — QA: learner who completes course twice gets only one credential, path completion triggers path credential, CEU credential from 14.17.
