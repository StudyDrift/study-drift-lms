## 1. Problem Statement

Lextures is entirely hidden behind a login wall. Self-learners cannot discover courses without an account, which creates an insurmountable acquisition barrier in a market where Coursera, Udemy, and Khan Academy allow full catalog browsing without registration. There is no public-facing course catalog, no category browse, no search, and no ratings-based sorting. Without this, the self-learner market segment cannot be entered at all — this is the entry point to the entire SL funnel.

## 2. Goals

- Build a publicly accessible course catalog page with full-text search, category filters, level filters, and ratings sort.
- Render catalog pages server-side for SEO discoverability (15.15 dependency).
- Support both free and paid course listings (price shown; enrollment gated behind payment in 15.3).
- Integrate with OpenSearch or Algolia for fast full-text search across title, description, and instructor name.
- Provide Schema.org Course and CourseInstance JSON-LD markup on every course listing for Google Search rich results.

## 3. Non-Goals

- Building the payment or enrollment flow — that is 15.3.
- Course review collection — that is 15.7.
- Instructor profile pages — out of scope for v1 catalog.
- Personalized recommendations on the catalog — deferred to after 1.8 recommendations engine.

## 4. Personas & User Stories

- **As a prospective self-learner**, I want to browse available courses by category without creating an account so that I can explore what Lextures offers before committing.
- **As a prospective learner**, I want to search for "Python programming" and see courses sorted by rating and enrollment count so that I find the most popular options.
- **As a self-learner**, I want to see a course's price, duration, instructor, and description on the catalog listing so that I can make an informed enrollment decision.
- **As a search engine bot**, I want to crawl course pages with structured data so that Lextures courses appear in Google's course-rich results.

## 5. Functional Requirements

- **FR-1.** The system MUST provide a publicly accessible `/courses` catalog page with full-text search over course title, description, and instructor name.
- **FR-2.** The system MUST support filters: category, difficulty level (beginner/intermediate/advanced), price (free/paid/any), duration range, and language.
- **FR-3.** The system MUST support sorting: most popular (enrollment count), highest rated, newest, and relevance (search score).
- **FR-4.** The system MUST render catalog pages and individual course landing pages server-side (SSR via Next.js or Astro) for SEO (15.15 dependency).
- **FR-5.** The system MUST include Schema.org `Course` and `CourseInstance` JSON-LD markup on each course landing page.
- **FR-6.** The system MUST integrate with OpenSearch or Algolia for catalog search with <300 ms latency at p95.
- **FR-7.** The system SHOULD show enrollment count, average rating, instructor name, and estimated duration on catalog cards.
- **FR-8.** The system SHOULD paginate catalog results (cursor-based, 20 per page) and support infinite scroll on the frontend.

## 6. Non-Functional Requirements

- **Performance** — Catalog page initial load <1.5 s (SSR); search results update <300 ms after keystroke debounce.
- **Security** — Public endpoints expose only published, public-visibility courses; draft or institution-private courses excluded.
- **Privacy & Compliance** — No PII in catalog; enrollment counts are aggregate; GDPR-safe.
- **Accessibility** — Catalog search and filter controls WCAG 2.1 AA; cards have descriptive link text; filter panel keyboard-navigable.
- **Scalability** — Catalog index supports 100,000 courses; OpenSearch/Algolia handles index updates on course publish.
- **Reliability** — Catalog served from SSR layer with CDN edge caching; catalog available even if Postgres is slow.
- **Observability** — Metric `catalog_searches_total`, `catalog_page_views_total`; Algolia search analytics.
- **Maintainability** — Catalog indexing service in `server/src/services/catalog_search.rs`; SSR catalog pages in a separate Next.js or Astro app under `clients/web-public/`.
- **Internationalization** — Category names and level labels externalised; search supports multi-language queries.
- **Backward compatibility** — Existing institution-private courses are not indexed; `is_public` flag gates catalog inclusion.

## 7. Acceptance Criteria

- **AC-1.** *Given* an unauthenticated user visits `/courses`, *when* the page loads, *then* they see a catalog of published public courses with search bar and filters — no login prompt.
- **AC-2.** *Given* a user searches "machine learning," *when* results appear, *then* they are sorted by relevance and load within 300 ms of typing (after 300 ms debounce).
- **AC-3.** *Given* a course landing page is rendered, *when* a search engine crawler fetches it, *then* the `<head>` contains a valid Schema.org `Course` JSON-LD script block with name, description, provider, and offers.
- **AC-4.** *Given* a user applies the "Free" filter, *when* the catalog updates, *then* only courses with price = $0 are shown.
- **AC-5.** *Given* a draft course exists, *when* the catalog API is called without authentication, *then* the draft course is not in the results.

## 8. Data Model

Add to `courses` table (migration `092_course_catalog_fields.sql`):
```sql
ALTER TABLE courses
  ADD COLUMN is_public BOOLEAN NOT NULL DEFAULT FALSE,
  ADD COLUMN catalog_category TEXT,
  ADD COLUMN difficulty_level TEXT CHECK (difficulty_level IN ('beginner', 'intermediate', 'advanced')),
  ADD COLUMN language TEXT NOT NULL DEFAULT 'en',
  ADD COLUMN price_cents INT NOT NULL DEFAULT 0,
  ADD COLUMN enrollment_count INT NOT NULL DEFAULT 0,
  ADD COLUMN average_rating NUMERIC(3,2),
  ADD COLUMN search_vector tsvector GENERATED ALWAYS AS (
    to_tsvector('english', title || ' ' || COALESCE(description, ''))
  ) STORED;

CREATE INDEX idx_courses_catalog ON courses(is_public, catalog_category, difficulty_level) WHERE is_public = TRUE;
CREATE INDEX idx_courses_search ON courses USING GIN(search_vector) WHERE is_public = TRUE;
```

## 9. API Surface

```
GET  /api/v1/catalog/courses               -- public catalog list (no auth)
     Params: q, category, level, price_max, sort, cursor, limit
GET  /api/v1/catalog/courses/:slug          -- public course detail (no auth)
GET  /api/v1/catalog/categories             -- category taxonomy list

-- Internal (used by SSR layer)
GET  /api/v1/internal/catalog/courses/:id/json-ld  -- JSON-LD for course
```

All public endpoints have 60-second CDN cache headers.

## 10. UI / UX

New SSR/public app under `clients/web-public/` (Next.js or Astro):
- `/courses` — catalog page with search bar, filter sidebar, course card grid.
- `/courses/:slug` — course landing page with description, curriculum preview, instructor info, CTA.

Course card: thumbnail, title, instructor, rating stars, enrollment count, price/Free badge.
Catalog page: filter sidebar (mobile: bottom drawer), sort dropdown, "X courses found" count.
Landing page: course hero, description, "What you'll learn" bullets, curriculum accordion, instructor bio, enrollment CTA.

States: loading skeleton cards; no-results empty state with search suggestions; error state with retry.
Mobile: single-column card list; filter drawer opens from bottom.
Accessibility: filter checkboxes have group labels; cards are `<article>` elements; "Enroll Now" button has `aria-describedby` pointing to price.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Institution-private courses accidentally indexed | M | H | Catalog API enforces is_public=TRUE; separate public Algolia index |
| Search index staleness after course update | M | M | Webhook-based index update on course publish/update |
| SEO: Google not crawling SSR pages | L | M | Verify with Google Search Console; submit sitemap (15.15) |

## 15. Rollout Plan

- Feature flag: `public_catalog` (default: off; SL tier).
- Internal dogfood with demo courses.
- GA after SSR deployment is validated and JSON-LD tested with Google Rich Results Test.
- Rollback: disable flag; public endpoints return 404.

## 16. Test Plan

- **Unit** — JSON-LD builder; catalog filter query construction; search index event handler.
- **Integration** — Catalog API: filter combinations, pagination, public/private isolation.
- **End-to-end** — Playwright: unauthenticated user browses, searches, filters, views course landing page.
- **Security** — Assert draft and private courses absent from public API without auth token.
- **Accessibility** — axe on catalog page and course landing page; keyboard navigation.
- **Performance / load** — Catalog search p95 <300 ms; catalog page p95 <1.5 s.
- **Manual exploratory** — QA: empty search results, all filters combined, newly published course appears in results.
