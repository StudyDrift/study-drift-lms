## 1. Problem Statement

Instructors migrating to Lextures from Canvas, Blackboard, Moodle, or Schoology have existing question banks and course packages in IMS QTI 2.1/3.0 and IMS Common Cartridge 1.2/1.3 formats. Without an import pipeline, migration requires manual question re-entry — a prohibitive effort for courses with hundreds of questions. This single gap is responsible for significant migration friction and lost institutional deals. Publishers also distribute digital curriculum packages in Common Cartridge format. Supporting QTI import unblocks content ingestion from every major LMS and publisher, making Lextures a viable replacement platform rather than a greenfield-only tool.

## 2. Goals

- Parse and import IMS QTI 2.1 assessment packages (`.zip` containing XML + media) into the Lextures question bank (2.1).
- Parse and import IMS Common Cartridge 1.2 packages (`.imscc` zip) extracting quiz items, assignments, and course structure.
- Map QTI interaction types to Lextures question types (2.2) where possible; flag unsupported types for manual review.
- Preserve question metadata (title, concept tags where present, point values, shuffling flags) from the source package.
- Process imports asynchronously via background job (17.3) and provide a progress/result report.

## 3. Non-Goals

- QTI export from Lextures (v2 roadmap item; this plan is import-only).
- Common Cartridge export.
- Full course shell restoration from Common Cartridge (course structure, content pages, files) — this plan focuses on assessment items; full course import is a broader migration feature.
- QTI 3.0 import in v1 (QTI 3.0 is JSON-based; add as separate parser in v2).
- SCORM import (covered in 2.14).

## 4. Personas & User Stories

- **As a migrating instructor**, I want to upload my Canvas QTI export ZIP and have my 200 quiz questions appear in the Lextures question bank within minutes, not days.
- **As an instructional designer**, I want to import a publisher-provided Common Cartridge package and have the quiz items appear in the question bank, ready to assign.
- **As a course author**, I want unsupported question types (e.g., QTI `uploadInteraction`) to be imported as stubs with a "needs review" flag so I can manually complete them.
- **As an admin**, I want an import history log showing which packages were imported, how many questions succeeded, and how many failed, so I can audit migrations.
- **As an instructor**, I want imported questions to be tagged with the source package name so I can identify their origin in the question bank.

## 5. Functional Requirements

- **FR-1.** The system MUST accept a multipart upload of a QTI 2.1 ZIP or `.imscc` file via `POST /api/v1/admin/imports/qti`.
- **FR-2.** The system MUST parse QTI 2.1 assessment items and map the following interaction types to Lextures question types: `choiceInteraction` → `mc_single`/`mc_multiple`; `textEntryInteraction` → `short_answer`; `extendedTextInteraction` → `short_answer`; `matchInteraction` → `matching`; `orderInteraction` → `ordering`; `hotspotInteraction` → `hotspot`; `numericInteraction` → `numeric`.
- **FR-3.** The system MUST import question stem as rich text (preserving MathML/LaTeX if present, mapped to KaTeX format from 2.3), options, correct answers, and point values.
- **FR-4.** The system MUST flag unsupported interaction types (e.g., `uploadInteraction`, custom interactions) as `status = 'needs_review'` in the question bank rather than skipping them.
- **FR-5.** The system MUST process imports asynchronously via background job and expose progress via `GET /api/v1/imports/{job_id}/status` returning `{ total, processed, succeeded, failed, errors }`.
- **FR-6.** The system MUST create question version 1 for each imported question and set `source = 'qti'`.
- **FR-7.** The system MUST support Common Cartridge 1.2 `.imscc` files: extract the manifest (`imsmanifest.xml`), parse embedded QTI assessment resources, and import those items into the question bank.
- **FR-8.** The system SHOULD extract organisation/module structure from Common Cartridge manifest and create corresponding `course_structure_items` as a draft import (instructor reviews before publishing).
- **FR-9.** The system MAY attempt to extract concept/objective tags from QTI `objectives` or `keywords` metadata and suggest matching Lextures concepts from the concept graph (1.2) for instructor approval.

## 6. Non-Functional Requirements

- **Performance** — A 1 000-question QTI ZIP import MUST complete within 5 minutes via background job. Streaming ZIP extraction avoids loading entire file into memory.
- **Security** — Uploaded ZIP files MUST be virus-scanned (8.6) before extraction. ZIP bomb protection: reject files with compression ratios > 100:1 or extracted size > 500 MB. Path traversal prevention in ZIP extraction (reject `../` entries). Import endpoint restricted to instructor/admin.
- **Privacy & Compliance** — Imported question content may contain student-specific examples from prior institutions; FERPA does not apply to question text (not student records). GDPR: imported content stored on Lextures servers; DPA with source institution may be required.
- **Accessibility** — Import progress page MUST be screen-reader navigable; progress updates via ARIA live region. Error messages MUST include file name and line number where possible.
- **Scalability** — Background job queue (17.3) handles concurrency; up to 10 simultaneous large imports without blocking other jobs (priority queue configuration).
- **Reliability** — Failed individual question imports MUST NOT abort the entire import job; the job continues and records errors per item. Import is idempotent: re-importing the same package skips questions with matching source identifiers.
- **Observability** — Counter `qti_import.questions_imported`, `qti_import.questions_failed`. Log import job start/end with package size. Alert if import job failure rate > 10 %.
- **Maintainability** — QTI parser in `server/src/services/qti_parser.rs`; CC manifest parser in `server/src/services/common_cartridge.rs`. Parsers return an intermediate `ImportItem` struct consumed by the question bank import service.
- **Internationalization** — QTI content is UTF-8; MathML in QTI stems mapped to KaTeX notation. Import error messages use i18n keys.
- **Backward compatibility** — Import creates new question bank rows; no existing content modified.

## 7. Acceptance Criteria

- **AC-1.** *Given* a valid QTI 2.1 ZIP with 50 `choiceInteraction` items, *when* the import completes, *then* 50 `mc_single` questions exist in the question bank with correct stems, options, and correct answers.
- **AC-2.** *Given* a QTI ZIP containing an `uploadInteraction` item, *when* the import processes it, *then* a question row is created with `status = 'needs_review'` and `question_type = 'file_upload'`.
- **AC-3.** *Given* a ZIP bomb (extracted size 600 MB), *when* the upload is received, *then* the server rejects with HTTP 422 "File too large when extracted."
- **AC-4.** *Given* the import is running, *when* `GET /api/v1/imports/{job_id}/status` is called, *then* the response shows `{ total: 50, processed: 23, succeeded: 22, failed: 1, errors: [{item_id, reason}] }`.
- **AC-5.** *Given* the same QTI package is imported twice, *when* the second import runs, *then* no duplicate questions are created; the job reports "50 skipped (already imported)."
- **AC-6.** *Given* a Common Cartridge `.imscc` file with QTI resources, *when* imported, *then* the extracted quiz questions appear in the question bank under the resource titles from the manifest.

## 8. Data Model

```sql
-- Migration: server/migrations/0051_qti_import.sql

CREATE TABLE course.import_jobs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    course_id       UUID REFERENCES course.courses(id),
    import_type     TEXT NOT NULL,    -- 'qti21'|'common_cartridge'
    original_filename TEXT NOT NULL,
    status          TEXT NOT NULL DEFAULT 'pending',  -- 'pending'|'running'|'done'|'failed'
    total_items     INTEGER,
    processed_items INTEGER NOT NULL DEFAULT 0,
    succeeded_items INTEGER NOT NULL DEFAULT 0,
    failed_items    INTEGER NOT NULL DEFAULT 0,
    error_log       JSONB NOT NULL DEFAULT '[]',
    started_at      TIMESTAMPTZ,
    completed_at    TIMESTAMPTZ,
    created_by      UUID NOT NULL REFERENCES "user".users(id),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Track imported source IDs for idempotency
CREATE TABLE course.imported_question_sources (
    question_id     UUID NOT NULL REFERENCES course.questions(id) ON DELETE CASCADE,
    source_type     TEXT NOT NULL,
    source_identifier TEXT NOT NULL,   -- QTI item identifier attribute
    import_job_id   UUID NOT NULL REFERENCES course.import_jobs(id),
    PRIMARY KEY (source_type, source_identifier)
);
```

## 9. API Surface

```
POST /api/v1/imports/qti
  Auth: instructor|admin
  Content-Type: multipart/form-data
  Fields: file (ZIP/IMSCC), course_id
  Response 202: { job_id }

GET  /api/v1/imports/{job_id}/status
  Auth: instructor|admin (must own the job)
  Response: { status, total_items, processed_items, succeeded_items, failed_items, error_log, completed_at? }

GET  /api/v1/imports
  Auth: instructor|admin
  Response: { imports: ImportJobSummary[] }  -- history
```

## 10. UI / UX

**Import page** (`/courses/{id}/import` or `/admin/import`, new page):
- Drag-and-drop upload area accepting `.zip` and `.imscc` files.
- Package type auto-detected from file content; manual override dropdown.
- "Start Import" button.

**Import progress panel**:
- Progress bar: "Processed 45 of 200 questions."
- Live-updating via polling (5-s interval) or WebSocket.
- On completion: summary table showing succeeded/failed counts.
- Expandable error log: item identifier, error type, suggested fix.
- "View imported questions" button → navigates to question bank filtered by `source = 'qti'` and this `import_job_id`.

**Needs-review queue** (in question bank):
- Filter by `status = 'needs_review'` shows flagged questions.
- Per-question "Complete setup" button.

Accessibility: upload progress announced by ARIA live region; error log table keyboard navigable.

## 11. AI / ML Considerations

Optional v2: LLM-assisted concept tag suggestion — given an imported question's stem and the Lextures concept graph, suggest the top 3 matching concepts for instructor approval. Not in v1 scope.

MathML → KaTeX conversion: a deterministic transformation (not ML); rule-based conversion of common MathML structures to equivalent LaTeX.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Malicious ZIP with path traversal | M | H | Sanitise all extracted paths; reject entries with `../`; use a sandboxed extraction directory |
| QTI XML from different LMSs has vendor-specific extensions | H | M | Parse only standard QTI elements; log unrecognised elements; do not fail on unknown extensions |
| MathML → KaTeX conversion loses fidelity | M | M | Flag questions with complex MathML for `needs_review`; log conversion warnings |
| Background job killed mid-import leaves partial data | M | M | Import in a transaction per question; idempotent re-run via source identifier dedup |

## 15. Rollout Plan

- **Feature flag:** `qti_import_enabled` (default: `false`; enable per-course or globally).
- **Migration sequencing:** deploy job queue → deploy parsers → deploy import UI → enable for pilot.
- **Pilot cohort:** one migrating instructor with a 200-question Canvas QTI export.
- **GA criteria:** 95 % of QTI 2.1 `choiceInteraction` items imported correctly; ZIP bomb rejected; import history accessible.
- **Rollback:** disable flag; import endpoint returns 404; imported questions already in bank remain.

## 16. Test Plan

- **Unit** — QTI `choiceInteraction` parser; `matchInteraction` mapper; ZIP bomb detection; path traversal sanitisation; MathML-to-LaTeX converter.
- **Integration** — Import a 10-question QTI 2.1 ZIP → assert questions in bank with correct types and answers; re-import → assert no duplicates; import CC `.imscc` → assert questions extracted.
- **End-to-end** — Playwright: instructor uploads QTI ZIP, watches progress, views imported questions in bank.
- **Security** — ZIP bomb → 422; path traversal entry → 422; virus-infected file → quarantined.
- **Accessibility** — Progress ARIA live region; error log table keyboard navigable.
- **Performance** — 1 000-question import: ≤ 5 min via background job; upload to queue: ≤ 10 s.
- **Manual exploratory** — Canvas QTI export; Blackboard QTI export; publisher CC package; verify `needs_review` flagging for unsupported types.
