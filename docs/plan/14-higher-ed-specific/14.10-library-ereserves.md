## 1. Problem Statement

University courses routinely assign library readings through e-reserves systems like Ex Libris Leganto, Alma Reading Lists, or institution-managed EZproxy-gated journal links. Instructors currently must paste raw URLs or PDF copies into course modules, violating copyright agreements and creating dead links when journal access changes. The absence of library integration means instructors work around the LMS, compliance risks exist from unauthorized PDF copying, and students cannot authenticate to library resources directly from their course page.

## 2. Goals

- Embed Ex Libris Leganto reading lists directly in course modules via LTI 1.3.
- Allow instructors to link EZproxy-authenticated journal and database articles as course resources.
- Surface Alma library catalog search from within the module editor so instructors can add citations without leaving the LMS.
- Ensure student authentication flows through EZproxy seamlessly when clicking library links.

## 3. Non-Goals

- Building a library catalog or discovery layer — use Alma/Leganto APIs only.
- Digitizing physical reserve materials (copyright scanning).
- Inter-library loan (ILL) workflows.
- K-12 library catalog integration (different feature set).

## 4. Personas & User Stories

- **As an instructor**, I want to add a Leganto reading list to my course module so that students access assigned readings through the library's licensed copy, not an uploaded PDF.
- **As a student**, I want to click a library article link in my course and be automatically authenticated via EZproxy so that I am not asked to log in separately.
- **As a librarian**, I want to see which courses have linked to a particular Leganto reading list so that I can maintain the list when journal access changes.
- **As an admin**, I want to configure the institution's EZproxy prefix once so that all library links in all courses are automatically proxied.

## 5. Functional Requirements

- **FR-1.** The system MUST support embedding Leganto reading lists as LTI 1.3 deep-link content items in course modules.
- **FR-2.** The system MUST allow instructors to configure an EZproxy prefix per institution; all external links matching configured domain patterns are automatically proxied through EZproxy.
- **FR-3.** The system SHOULD provide an Alma library catalog search widget in the module content editor, allowing instructors to search and link holdings.
- **FR-4.** The system MUST store library resource links as a dedicated `library_resource` module item type, distinct from generic external links, preserving metadata (title, author, ISSN/ISBN, source).
- **FR-5.** The system SHOULD send a COUNTER-compatible link-resolve event when a student accesses a library item, supporting library usage statistics.
- **FR-6.** The system MAY surface library availability (available/checked out/access restricted) from the Alma API on the linked item display.

## 6. Non-Functional Requirements

- **Performance** — Leganto LTI launch <500 ms; Alma search results <1 s.
- **Security** — EZproxy prefix is admin-only configuration; LTI credentials encrypted at rest.
- **Privacy & Compliance** — Student access events must not include PII when reported to library analytics; FERPA applies to linked enrollment context.
- **Accessibility** — Library resource items in module list have descriptive link text; WCAG 2.1 AA.
- **Scalability** — EZproxy URL rewriting is a string transform; negligible overhead.
- **Reliability** — If Alma API is unavailable, library items still display with cached metadata.
- **Observability** — Metric `library_link_clicks_total{resource_type}`.
- **Maintainability** — Library integration in `server/src/services/library/`; EZproxy rewriting as a utility function.
- **Internationalization** — Catalog search results display title/author as returned by Alma; no locale transforms needed.
- **Backward compatibility** — Existing `external_link` module items continue to work; library resource is a new item type.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor adds a Leganto reading list via LTI deep link, *when* a student clicks the item in their course module, *then* the Leganto reading list launches within the Lextures frame or in a new tab with the student authenticated.
- **AC-2.** *Given* an admin configures `ezproxy.university.edu` as the EZproxy prefix, *when* an instructor links `journals.sage.com/article/123`, *then* the stored URL is `ezproxy.university.edu/login?url=journals.sage.com/article/123`.
- **AC-3.** *Given* an instructor searches "machine learning" in the Alma widget, *when* results load, *then* at least the first 10 matching titles are shown with author and call number, and clicking "Add to Module" creates a library resource item.
- **AC-4.** *Given* a library resource item exists in a module, *when* a developer queries the `module_items` table, *then* the item has `item_type = 'library_resource'` and a non-null `metadata` JSONB with title and source.
- **AC-5.** *Given* a student clicks a proxied library link, *when* the click is recorded, *then* no student PII is included in the COUNTER event log.

## 8. Data Model

Migration `083_library_resources.sql`:
```sql
ALTER TABLE course_structure_items
  ADD COLUMN library_metadata JSONB;
  -- stored when item_type = 'library_resource'
  -- {title, author, isbn_issn, source, alma_mms_id, leganto_list_id, ezproxy_url}

CREATE TABLE library_link_events (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  item_id     UUID NOT NULL REFERENCES course_structure_items(id),
  course_id   UUID NOT NULL REFERENCES courses(id),
  accessed_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  -- no user_id — privacy by design
);
```

## 9. API Surface

```
GET  /api/v1/library/search?q=&org_id=         -- Alma catalog search proxy
POST /api/v1/courses/:id/modules/:mid/items    -- extended to support item_type=library_resource
POST /api/v1/admin/library/config              -- EZproxy prefix + Alma API key config
GET  /api/v1/library/link-events?course_id=    -- COUNTER usage stats (admin/librarian)
```

## 10. UI / UX

Modified: module item editor (`clients/web/src/components/content-page/` and add-item flow)
- New item type "Library Resource" in the Add Item menu.
- Sub-options: "Search Library Catalog" (Alma widget) or "Embed Reading List" (Leganto LTI).
- Alma search widget: search bar, results list with "Add" button per item.
- EZproxy toggle shown for external-link items matching library domains.

Modified: module item display
- Library resource items show book/article icon, author, source badge.
- EZproxy-proxied badge on items that go through proxy.

Admin settings: `clients/web/src/pages/admin/LibraryIntegration.tsx`
- EZproxy prefix field, domain pattern list, Alma API configuration, Leganto LTI registration.

Accessibility: Alma search results list uses `role="listbox"`; "Add" button has contextual `aria-label` per title.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| EZproxy prefix misconfiguration breaks all library links | M | H | Validate URL format; test-link button in admin config |
| Alma API key expires silently | M | M | Monitor API auth errors; admin alert on 401 responses |
| Journal subscriptions change invalidating links | M | M | Alma availability check on item display; show "access may be unavailable" when restricted |

## 15. Rollout Plan

- Feature flag: `library_integration` (default: off).
- Pilot with one institution that has Leganto; validate LTI deep linking.
- GA after EZproxy rewriting is validated with at least 2 different proxy configurations.

## 16. Test Plan

- **Unit** — EZproxy URL rewriting function; Alma response parsing; library item metadata serialization.
- **Integration** — Alma API mock; Leganto LTI deep link flow.
- **End-to-end** — Playwright: instructor adds reading list, student clicks item and Leganto launches.
- **Security** — Assert non-admin cannot modify EZproxy config; assert Alma API key not returned in client API responses.
- **Accessibility** — axe on Alma search widget and module item display.
- **Manual exploratory** — QA: article with no EZproxy match, Leganto list with mixed access items, Alma search returning 0 results.
