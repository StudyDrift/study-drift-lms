## 1. Problem Statement

At the end of every academic term, instructors must submit final grades to the registrar. Lextures currently has no mechanism to roll up computed final grades into a submittable artifact, let alone push them to the SIS. The registrar's office must wait for instructors to manually re-enter grades into the SIS portal, creating delays, transcription errors, and audit headaches. Shipping a final grade export — initially as a standards-compliant CSV and later as AGS passback — closes this loop and is a direct BLOCKER for institutional adoption.

## 2. Goals

- Compute an end-of-term final grade per student per course from the gradebook (weighted categories, drop-lowest rules, etc.).
- Present instructors with a review-and-confirm step before grades are locked.
- Export final grades as a CSV in the institution's SIS import format (Banner, Workday, Colleague).
- Push grades directly via IMS LTI Advantage AGS where the SIS supports it.
- Log every grade submission with actor, timestamp, and grade values for FERPA audit.

## 3. Non-Goals

- Retroactively modifying grades once submitted (requires registrar-level override workflow, out of scope for v1).
- Computing final grades for K-12 standards-based grading (different feature, 3.7).
- Financial aid GPA calculations — SIS responsibility.

## 4. Personas & User Stories

- **As an instructor**, I want a one-click "Submit Final Grades" workflow that computes grades from my gradebook and sends them to the registrar so that I do not re-enter grades manually.
- **As a registrar**, I want final grades delivered as a CSV in our SIS import format at the click of a button so that I can process the term without waiting for manual instructor uploads.
- **As a department chair**, I want to see which courses have and have not submitted final grades at term end so that I can follow up with late instructors.
- **As a student**, I want to know when my instructor has submitted final grades so that I can monitor when my official grade will appear in the SIS.

## 5. Functional Requirements

- **FR-1.** The system MUST compute a final letter grade per student from the current gradebook state (weighted assignment groups, including drop-lowest/replace policies from 3.9 when implemented).
- **FR-2.** The system MUST present a "Final Grade Review" table to the instructor before submission, allowing per-student grade overrides with a required reason.
- **FR-3.** The system MUST export final grades as a CSV conforming to the institution's configured SIS import format (Banner, Workday, or Colleague column mappings configurable via 14.1 adapter).
- **FR-4.** The system MUST support direct AGS passback as the alternative to CSV for SIS-configured institutions (reusing the AGS implementation from 14.1).
- **FR-5.** The system MUST record every submitted grade in a `final_grade_submissions` audit table with instructor ID, timestamp, course ID, student ID, grade, and submission method.
- **FR-6.** The system MUST handle Incomplete (I), Withdrawn (W), and Audit (AU) enrollment states correctly in the export (correct symbol or SIS code).
- **FR-7.** The system SHOULD allow department chairs and admins to view a submission-status dashboard per course per term.
- **FR-8.** The system SHOULD send a confirmation email to the instructor when grades are successfully submitted or when AGS passback fails.
- **FR-9.** The system MAY lock the gradebook from further instructor edits after grade submission, subject to institution configuration.

## 6. Non-Functional Requirements

- **Performance** — Final grade computation for a section of 300 students completes in <3 s; CSV generation for 300 rows in <1 s.
- **Security** — Grade submission is instructor/admin only; audit table is append-only; CSV download links are signed and expire in 1 hour.
- **Privacy & Compliance** — Final grades are FERPA records; CSV files must not be stored unencrypted longer than necessary; signed download URLs expire promptly.
- **Accessibility** — Grade review table WCAG 2.1 AA; override reason fields properly labelled.
- **Scalability** — For large multi-section courses (3,000+ students), CSV generation runs asynchronously in background job with download link delivered via email.
- **Reliability** — AGS passback retries on transient SIS errors; CSV generation is idempotent.
- **Observability** — Metric `final_grades_submitted_total{method}`; alert on AGS passback failure rate >5%.
- **Maintainability** — Grade computation reuses existing `server/src/repos/course_grades.rs` logic; CSV formatting in `server/src/services/grade_export.rs`.
- **Internationalization** — Grade symbols configurable per institution (A/B/C vs. 4.0/3.7/etc.); letter grade scale UI labels externalised.
- **Backward compatibility** — Grade export is additive; does not alter how grades are stored or displayed prior to submission.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor opens "Submit Final Grades," *when* the review table loads, *then* every enrolled active student has a computed final grade shown, with Incomplete and Withdrawn students showing their correct symbols.
- **AC-2.** *Given* an instructor confirms submission to CSV, *when* the download link is clicked, *then* the CSV contains one row per student with SRN, name, grade, and credit hours in the institution's Banner import format.
- **AC-3.** *Given* an institution has AGS passback configured, *when* grades are submitted, *then* all grades appear in the SIS course gradebook within 60 seconds and the Lextures audit log records success.
- **AC-4.** *Given* an instructor overrides a student's computed grade with a reason, *when* the submission is saved, *then* the override and reason appear in the `final_grade_submissions` audit trail.
- **AC-5.** *Given* a department chair opens the grade-submission dashboard, *when* 3 of 10 courses have not yet submitted grades, *then* those courses are highlighted with the instructor's name and contact.

## 8. Data Model

Migration `078_final_grade_submissions.sql`:
```sql
CREATE TABLE final_grade_submissions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  course_id       UUID NOT NULL REFERENCES courses(id),
  enrollment_id   UUID NOT NULL REFERENCES course_enrollments(id),
  submitted_by    UUID NOT NULL REFERENCES users(id),
  computed_grade  TEXT NOT NULL,
  final_grade     TEXT NOT NULL,
  override_reason TEXT,
  submission_method TEXT NOT NULL DEFAULT 'csv', -- 'csv', 'ags'
  submitted_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  sis_ack_at      TIMESTAMPTZ
);
CREATE UNIQUE INDEX idx_final_grade_submission_enrollment
  ON final_grade_submissions(enrollment_id)
  WHERE submission_method = 'ags';
```

## 9. API Surface

```
GET    /api/v1/courses/:id/final-grades/preview
       -- Returns computed grades; auth: instructor

POST   /api/v1/courses/:id/final-grades/submit
       Body: { method: 'csv'|'ags', overrides: [{enrollment_id, grade, reason}] }
       Auth: instructor or admin
       Returns: { download_url?: string, ags_status?: string }

GET    /api/v1/admin/final-grades/status?term_id=
       Auth: admin or department chair
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| AGS passback partial failure leaves inconsistent SIS state | M | H | Idempotent retry; per-student passback result logged; admin alert on partial failure |
| Instructor submits incorrect grades without realizing | M | H | Prominent review step; require confirmation; show computed vs. gradebook basis |
| CSV column format mismatch with SIS import template | M | M | Configurable column mapping in SIS adapter config; test CSV with registrar in pilot |
| Large sections generate CSV slowly blocking the request | L | M | Async generation for >200 students; email download link |

## 15. Rollout Plan

- Feature flag: `final_grade_submission` (default: off).
- Pilot at term end with one institution; validate CSV format with registrar.
- GA after pilot validates AGS passback with Banner.
- Rollback: disable flag; instructors export via existing gradebook CSV (3.11) as fallback.

## 16. Test Plan
