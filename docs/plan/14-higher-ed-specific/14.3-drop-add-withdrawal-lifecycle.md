## 1. Problem Statement

The current `course_enrollments` table tracks a binary enrolled/unenrolled state. University registrars and accreditation requirements mandate a richer enrollment state machine: Withdrawn (W), Audit (AU), Incomplete (I), and No Credit (NC). Without these states, the LMS cannot enforce drop-deadline rules, display the correct grade symbols on transcripts, block withdrawn students from seeing future content, or pass the correct enrollment status to the SIS during grade passback. This is a thin implementation gap — the schema exists but lacks the state machine and business logic.

## 2. Goals

- Extend `course_enrollments` with a formal state machine covering: `active`, `dropped`, `withdrawn`, `audit`, `incomplete`, `no_credit`, `waitlist`.
- Enforce drop-date and withdrawal-date rules from the academic calendar (14.6), blocking state transitions after the deadline.
- Surface correct enrollment status to instructors in the gradebook and course roster.
- Pass correct enrollment status codes to the SIS via AGS passback (14.1).
- Provide an audit trail of every enrollment state change with actor, timestamp, and reason.

## 3. Non-Goals

- Initiating drop/add transactions in the SIS from Lextures (read-only SIS at GA; see 14.2).
- Financial liability calculations (tuition refund schedules) — that lives in the SIS.
- K-12 attendance-based enrollment (covered in 13.2).

## 4. Personas & User Stories

- **As a registrar staff member**, I want to see which students have officially withdrawn versus just stopped attending so that I record the correct grade symbol on the transcript.
- **As an instructor**, I want withdrawn students to appear in a "Former Students" section of the gradebook rather than disappear entirely so that I retain grade history.
- **As a student**, I want my dashboard to reflect my official enrollment status (Audit, Withdrawn) so that I understand which courses count toward my GPA.
- **As a system admin**, I want every enrollment state change logged with the actor and timestamp so that I can audit changes for FERPA compliance.

## 5. Functional Requirements

- **FR-1.** The system MUST support enrollment states: `active`, `dropped`, `withdrawn`, `audit`, `no_credit`, `waitlist`, `incomplete` (incomplete is managed via 14.4).
- **FR-2.** The system MUST enforce that `dropped` transitions are only permitted before the institution's add/drop deadline from the academic calendar (14.6), unless overridden by an admin.
- **FR-3.** The system MUST enforce that `withdrawn` (W) transitions are only permitted between the add/drop deadline and the last withdrawal date, unless overridden.
- **FR-4.** The system MUST record every enrollment state transition in an `enrollment_state_history` table with actor_id, previous_state, new_state, reason, and timestamp.
- **FR-5.** The system MUST accept SIS-driven state changes via the sync adapter (14.1) and apply them as audited transitions.
- **FR-6.** The system MUST display withdrawn students in the gradebook in a collapsed "Former Students" section, retaining their grade history read-only.
- **FR-7.** The system SHOULD allow instructors to manually flag a student as `audit` (AU) subject to admin-level override permissions.
- **FR-8.** The system MUST pass the current enrollment state as an IMS LIS enrollment status code during SIS grade passback.

## 6. Non-Functional Requirements

- **Performance** — State transition endpoint responds in <200 ms; gradebook render with withdrawn students adds <100 ms overhead.
- **Security** — Only admins and instructors with explicit permission may manually change enrollment state; students cannot modify their own state; all changes audit-logged.
- **Privacy & Compliance** — Enrollment status is FERPA-covered; surfaced only to the enrolled student, the instructor of record, and authorized admins.
- **Accessibility** — Gradebook "Former Students" section has WCAG 2.1 AA status indicators with text labels (not color-only).
- **Scalability** — State history table indexes on (enrollment_id, created_at) for efficient per-enrollment history queries.
- **Reliability** — State transitions are atomic DB transactions; no partial state updates.
- **Observability** — Counter `enrollment_state_transitions_total{from, to}`; alert on unexpected bulk transitions.
- **Maintainability** — State machine implemented as an explicit Rust enum with transition guard functions; no ad-hoc string comparisons.
- **Internationalization** — State labels externalised as i18n keys; "Withdrawn" displays in user's locale.
- **Backward compatibility** — Existing `active` enrollments default to `active` state; no backfill required for pre-existing enrolled rows.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor attempts to drop a student after the add/drop deadline, *when* they submit the state change, *then* the API returns a 422 error with a human-readable message referencing the deadline.
- **AC-2.** *Given* a student is marked `withdrawn` by SIS sync, *when* the instructor opens the gradebook, *then* the student appears in the "Former Students" collapsed section with a "W" badge and their existing grades intact.
- **AC-3.** *Given* an admin manually changes an enrollment state, *when* the change is saved, *then* `enrollment_state_history` contains a row with the admin's user ID, old state, new state, and timestamp.
- **AC-4.** *Given* a student with `audit` status, *when* final grades are submitted to the SIS, *then* the passback payload contains the `AU` LIS enrollment status code.
- **AC-5.** *Given* a student has an `active` enrollment, *when* they log in to their dashboard, *then* the course shows "Active" status; if they log in with a `withdrawn` enrollment, the course shows "Withdrawn — [date]".

## 8. Data Model

Migration `076_enrollment_state_machine.sql`:
```sql
CREATE TYPE enrollment_state AS ENUM (
  'waitlist', 'active', 'dropped', 'withdrawn', 'audit', 'no_credit', 'incomplete'
);

ALTER TABLE course_enrollments
  ADD COLUMN state enrollment_state NOT NULL DEFAULT 'active',
  ADD COLUMN state_changed_at TIMESTAMPTZ,
  ADD COLUMN state_reason TEXT;

CREATE TABLE enrollment_state_history (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  enrollment_id   UUID NOT NULL REFERENCES course_enrollments(id),
  actor_id        UUID REFERENCES users(id),
  previous_state  enrollment_state NOT NULL,
  new_state       enrollment_state NOT NULL,
  reason          TEXT,
  source          TEXT NOT NULL DEFAULT 'manual', -- 'manual', 'sis_sync', 'system'
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_enrollment_state_history_enrollment ON enrollment_state_history(enrollment_id, created_at DESC);
```

Backfill: `UPDATE course_enrollments SET state = 'active'` (already the default).

## 9. API Surface

```
PATCH  /api/v1/courses/:id/enrollments/:enrollment_id/state
       Body: { state: EnrollmentState, reason?: string }
       Auth: instructor or admin

GET    /api/v1/courses/:id/enrollments/:enrollment_id/history
       Auth: instructor or admin
```

Gradebook list endpoint (`GET /api/v1/courses/:id/gradebook`) updated to include `state` field and accept `?include_withdrawn=true`.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Instructors accidentally change state causing FERPA issues | M | H | Require reason text; send notification to student and admin on withdrawal |
| SIS sync and manual change race condition | M | M | SIS sync wins on conflict; log both changes; admin alert on override |
| Dropped vs. withdrawn distinction unclear to instructors | H | M | In-product tooltip explaining the legal difference; link to help article |

## 15. Rollout Plan

- Feature flag: `enrollment_state_machine` (default: off; on for HE tier).
- Schema migration runs first; backfills all existing enrollments to `active`.
- Enable flag for pilot HE institution; verify SIS-sync state transitions.
- GA alongside 14.1 SIS integration release.
- Rollback: set flag off; state column remains but UI reverts to simple enrolled/unenrolled.

## 16. Test Plan

- **Unit** — Rust state machine: test all valid and invalid transitions; test deadline enforcement logic.
- **Integration** — API: PATCH state with valid and invalid actors; assert history row created; assert SIS sync creates correct history entry.
- **End-to-end** — Playwright: instructor changes student to withdrawn, verifies gradebook former-students section, student sees status on dashboard.
- **Security** — Assert student cannot PATCH their own enrollment state; assert non-instructor cannot access history endpoint.
- **Accessibility** — axe on gradebook with former-students section visible.
