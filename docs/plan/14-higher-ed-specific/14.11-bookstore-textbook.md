## 1. Problem Statement

Inclusive Access programs (where students are charged a discounted textbook fee as part of tuition) require LMS integration to deliver digital textbook access to enrolled students. Without integration with RedShelf, VitalSource Bridge, or similar platforms, institutions using Inclusive Access cannot deploy Lextures, as students would have no pathway to their required materials from within the LMS. Instructors also lose the ability to create deep links to specific chapters or pages in assigned textbooks.

## 2. Goals

- Support Inclusive Access delivery through VitalSource Bridge LTI 1.3 and RedShelf's BookShelf LTI integration.
- Allow instructors to link specific chapters or pages as module items.
- Display an Inclusive Access opt-out option to students per institution policy.
- Record textbook access events for reporting to the bookstore/publisher.

## 3. Non-Goals

- Processing textbook purchases — handled entirely by the bookstore platform.
- Building a digital reading interface — VitalSource/RedShelf provide the reader.
- Physical textbook tracking or buyback programs.
- K-12 instructional materials (different procurement model).

## 4. Personas & User Stories

- **As a student**, I want to click on "Chapter 3 Reading" in my course module and have the VitalSource reader open with the correct chapter already selected.
- **As an instructor**, I want to deep-link to a specific section of the assigned textbook so that students do not waste time finding the right pages.
- **As a student**, I want to see an opt-out option if my institution offers Inclusive Access so that I can decline if I have a physical copy.
- **As an admin**, I want to configure the institution's bookstore partner (VitalSource or RedShelf) once and have all courses use the correct integration.

## 5. Functional Requirements

- **FR-1.** The system MUST support VitalSource Bridge LTI 1.3 deep linking as a content item type in course modules.
- **FR-2.** The system MUST support RedShelf BookShelf LTI 1.3 deep linking as an alternative provider.
- **FR-3.** The system MUST pass enrolled student identity to the LTI launch context so the bookstore platform can verify access entitlement.
- **FR-4.** The system SHOULD display an Inclusive Access opt-out banner to students in courses with active Inclusive Access titles, linking to the bookstore's opt-out form.
- **FR-5.** The system SHOULD record textbook launch events (anonymized) for COUNTER-compatible reporting.
- **FR-6.** The system MAY surface a "Required Materials" section on the course page listing all linked textbook titles with ISBN and edition.

## 6. Non-Functional Requirements

- **Performance** — Textbook LTI launch <500 ms.
- **Security** — LTI 1.3 signed JWTs; student entitlement check performed by bookstore platform, not Lextures.
- **Privacy & Compliance** — Student enrollment data sent to bookstore only via LTI claims; FERPA disclosure required.
- **Accessibility** — Textbook module items have descriptive link text including title and chapter; WCAG 2.1 AA.
- **Scalability** — Same LTI 1.3 infrastructure as 2.12/14.9; no additional load.
- **Reliability** — Bookstore platform outage shows error message; course content unaffected.
- **Observability** — Metric `textbook_launches_total{provider}`.
- **Maintainability** — Textbook item type uses same LTI 1.3 deep-link mechanism as 14.9 proctoring.
- **Internationalization** — ISBN/title displayed as-is; no locale transforms.
- **Backward compatibility** — Existing external-link module items unaffected.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor adds a VitalSource deep link to a course module, *when* a student clicks it, *then* the VitalSource reader opens at the configured chapter with the student authenticated.
- **AC-2.** *Given* an Inclusive Access course is configured, *when* a student opens the course, *then* an "Inclusive Access — Opt Out" banner is visible with a direct link to the bookstore opt-out page.
- **AC-3.** *Given* a RedShelf tool is registered, *when* an instructor uses deep-link content-item selection, *then* RedShelf's content picker opens and the selected content appears as a module item.
- **AC-4.** *Given* admin registers VitalSource with a client_id, *when* an LTI launch is initiated, *then* the JWT contains the correct `iss`, `aud`, and user sub claims per LTI 1.3 spec.
- **AC-5.** *Given* a textbook launch occurs, *when* the event log is queried, *then* no student PII is stored in the launch event table.

## 8. Data Model

Migration `084_textbook_resources.sql`:
```sql
-- Extends existing course_structure_items with metadata for textbook items
-- item_type = 'textbook_resource'
-- library_metadata JSONB reused: {isbn, title, edition, publisher, chapter, page_range, provider}

CREATE TABLE inclusive_access_courses (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  course_id    UUID NOT NULL UNIQUE REFERENCES courses(id),
  isbn         TEXT NOT NULL,
  title        TEXT NOT NULL,
  opt_out_url  TEXT NOT NULL,
  provider     TEXT NOT NULL DEFAULT 'vitalsource',
  enabled      BOOLEAN NOT NULL DEFAULT TRUE
);
```

## 9. API Surface


## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| VitalSource entitlement check fails for paid students | M | H | Clear error message with bookstore contact; fallback to manual access instructions |
| Opt-out banner fatigue leads to accidental opt-outs | M | M | Require explicit confirmation step in opt-out flow |

## 15. Rollout Plan

- Feature flag: `bookstore_integration` (default: off).
- Pilot with one Inclusive Access campus; validate with bookstore staff.
- GA alongside LTI 1.3 release.

## 16. Test Plan

- **Unit** — LTI JWT claim builder for textbook context; Inclusive Access banner logic.
- **Integration** — VitalSource sandbox launch; RedShelf deep-link selection.
- **End-to-end** — Playwright: instructor adds textbook link, student clicks and reader launches.
- **Security** — Assert student cannot bypass entitlement; assert LTI state single-use.
- **Accessibility** — axe on opt-out banner and textbook module item display.
- **Manual exploratory** — QA: student not in Inclusive Access program, multiple textbook items in one module.
