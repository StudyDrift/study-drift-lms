## 1. Problem Statement

Many university programs — particularly professional degrees, teacher preparation, nursing, and liberal arts — require students to assemble and present evidence of competency in a ePortfolio that spans multiple courses and terms. Lextures currently has no cross-course artifact collection surface. Students cannot curate work samples from multiple courses, instructors cannot evaluate holistic competency evidence, and program directors cannot run outcome-level assessment across cohorts. This is a program-accreditation requirement in fields governed by CAEP, ACEN, ABET, and similar bodies.

## 2. Goals

- Allow students to create a personal ePortfolio containing artifacts (files, text pages, media, assignment submissions) from any enrolled course.
- Support instructor and reviewer annotation and rubric evaluation of portfolio artifacts.
- Enable program-level outcome alignment so artifacts can be tagged to program learning outcomes.
- Provide a shareable, public-facing portfolio page for student professional development.
- Feed artifact data to the Co-Curricular Transcript (14.13).

## 3. Non-Goals

- Full website builder or CMS for student portfolios (simple page/artifact layout only).
- Integration with external ePortfolio platforms (Portfolium, Chalk & Wire, Taskstream) at GA.
- Automated outcome achievement computation from portfolio evidence.
- K-12 digital portfolio (different feature, possible future reuse).

## 4. Personas & User Stories

- **As a student**, I want to add my best assignment submissions to my ePortfolio so that I can share them with future employers and graduate programs.
- **As an instructor**, I want to evaluate a student's capstone portfolio using a rubric and leave feedback on individual artifacts.
- **As a program director**, I want to see aggregated outcome-evidence coverage across the cohort so that I can prepare accreditation reports.
- **As a student**, I want to share a public link to my portfolio so that hiring managers can view it without needing a Lextures account.

## 5. Functional Requirements

- **FR-1.** The system MUST allow students to create one or more portfolios and add artifacts (assignment submission, uploaded file, text page, external URL) from any course.
- **FR-2.** The system MUST allow students to tag each artifact to one or more program or course learning outcomes.
- **FR-3.** The system MUST allow instructors and designated portfolio reviewers to evaluate artifacts using a rubric (from the existing rubric system, `server/migrations/070_assignment_rubrics.sql`).
- **FR-4.** The system MUST generate a shareable, read-only public URL for a portfolio that does not require authentication.
- **FR-5.** The system SHOULD provide a program-level outcome coverage report showing which outcomes have artifact evidence and how many students have provided it.
- **FR-6.** The system MUST store the portfolio and its artifacts in a student-owned namespace; deleting a course enrollment must not delete the artifact from the portfolio.
- **FR-7.** The system SHOULD allow customizable section ordering and a profile/introduction page within the portfolio.
- **FR-8.** The system MAY integrate with 14.13 CCR to include portfolio milestones on the co-curricular transcript.

## 6. Non-Functional Requirements

- **Performance** — Portfolio page loads for public visitor in <2 s; artifacts served via CDN (8.1).
- **Security** — Public portfolio URL is accessible without auth but contains no FERPA-restricted data beyond what the student explicitly made public; private portfolios require authenticated access and correct ownership.
- **Privacy & Compliance** — Student controls what is public; default is private; FERPA: artifacts from graded submissions are education records; student waives FERPA protection for any publicly shared content.
- **Accessibility** — Public portfolio page WCAG 2.1 AA; artifact viewer keyboard-accessible; WCAG conformance statement in portfolio footer.
- **Scalability** — Each student may have up to 200 artifacts; stored in object storage (8.1) with metadata in Postgres.
- **Reliability** — Portfolio read path is independently cacheable; portfolio creation writes go through normal Postgres transactional path.
- **Observability** — Metric `portfolio_artifact_uploads_total`; public portfolio view counter (privacy-safe, no PII).
- **Maintainability** — Portfolio module in `server/src/services/portfolios/`; shared rubric evaluation reuses `server/src/repos/course_grading.rs`.
- **Internationalization** — Portfolio labels and template section names externalised; dates in user locale.
- **Backward compatibility** — No impact on existing course modules or assignment submissions.

## 7. Acceptance Criteria

- **AC-1.** *Given* a student adds an assignment submission to their portfolio, *when* they later unenroll from the course, *then* the artifact remains in their portfolio with the original file accessible.
- **AC-2.** *Given* a student generates a public portfolio link, *when* an unauthenticated visitor opens it, *then* only artifacts the student marked "public" are visible, with no course grades or personal contact information.
- **AC-3.** *Given* an instructor opens a student's portfolio for capstone review, *when* they evaluate an artifact with a rubric, *then* the rubric score is saved and visible to the student in their portfolio.
- **AC-4.** *Given* a program director views the outcome coverage report, *when* there are 50 students in the cohort, *then* the report shows per-outcome artifact submission rates as percentages.
- **AC-5.** *Given* a student tags an artifact to a program outcome, *when* the portfolio is exported to the CCR (14.13), *then* that artifact and its outcome tag appear on the co-curricular transcript.

## 8. Data Model

Migration `085_portfolios.sql`:
```sql
CREATE TABLE portfolios (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  owner_id    UUID NOT NULL REFERENCES users(id),
  title       TEXT NOT NULL,
  intro_text  TEXT,
  is_public   BOOLEAN NOT NULL DEFAULT FALSE,
  public_slug TEXT UNIQUE,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE portfolio_artifacts (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  portfolio_id  UUID NOT NULL REFERENCES portfolios(id) ON DELETE CASCADE,
  artifact_type TEXT NOT NULL, -- 'submission', 'upload', 'text_page', 'url'
  title         TEXT NOT NULL,
  description   TEXT,
  source_submission_id UUID REFERENCES module_assignment_submissions(id),
  file_key      TEXT,
  text_content  TEXT,
  external_url  TEXT,
  outcome_ids   UUID[] NOT NULL DEFAULT '{}',
  is_public     BOOLEAN NOT NULL DEFAULT FALSE,
  sort_order    INT NOT NULL DEFAULT 0,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_portfolio_artifacts_portfolio ON portfolio_artifacts(portfolio_id, sort_order);
```

## 9. API Surface

```
POST/GET/PATCH/DELETE /api/v1/me/portfolios
POST/GET/PATCH/DELETE /api/v1/me/portfolios/:pid/artifacts
GET  /api/v1/portfolios/:slug           -- public read-only view (no auth)
POST /api/v1/portfolios/:pid/artifacts/:aid/evaluate  -- rubric evaluation (reviewer)
GET  /api/v1/admin/programs/:prog_id/portfolio-outcomes-report
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Student accidentally makes FERPA-protected content public | M | H | Explicit opt-in to public per artifact; warning banner when toggling public |
| Large portfolio pages degrade performance | M | M | CDN for artifact files; paginated artifact loading |
| Orphaned file storage from deleted portfolios | M | M | Scheduled cleanup job for file_key artifacts with no portfolio_artifacts row |

## 15. Rollout Plan

- Feature flag: `eportfolio` (default: off).
- Pilot with a teacher-preparation program in a capstone course.
- GA after rubric evaluation and public sharing are validated.

## 16. Test Plan

- **Unit** — Artifact ownership validation; public slug uniqueness; outcome tag serialization.
- **Integration** — Portfolio CRUD; rubric evaluation save; public endpoint returns only public artifacts.
- **End-to-end** — Playwright: student creates portfolio, adds submission, evaluator adds rubric score, student shares public link.
- **Security** — Assert unauthenticated user cannot access private portfolio; assert student A cannot view student B's private portfolio.
- **Accessibility** — axe on portfolio editor and public view; keyboard drag-to-reorder alternative.
- **Performance / load** — Public portfolio with 50 artifacts loads in <2 s.
- **Manual exploratory** — QA: delete course enrollment after adding artifact, public slug conflict, large file artifact.
