## 1. Problem Statement

Universities regularly grant students an "Incomplete" (I) grade when documented medical, family, or extenuating circumstances prevent them from finishing coursework by the end of term. Currently Lextures has no concept of the Incomplete state: no extension date, no record of which assignments remain outstanding, and no workflow to convert the I to a final grade once the student completes work. Without this, registrars and instructors must track Incompletes in spreadsheets, and the grade passed to the SIS cannot reflect the correct symbol.

## 2. Goals

- Add a formal Incomplete grade record tied to an enrollment, capturing the extension deadline and a list of outstanding assignments.
- Extend the LMS course access window for Incomplete students beyond the term end date, up to the extension deadline.
- Notify the student and instructor of the extension deadline and approaching expiry.
- Allow the instructor to finalize the grade (converting I to a letter grade) before or at the extension deadline.
- Export the final resolved grade to the SIS (14.5 / 14.1).

## 3. Non-Goals

- Automatically assigning grades if the student does not complete work (lapse-to-F logic lives in the SIS, not the LMS).
- Managing financial implications of Incompletes (tuition, financial aid) — SIS domain.
- Creating Incomplete workflows for K-12 (not applicable).

## 4. Personas & User Stories

- **As an instructor**, I want to grant a student an Incomplete with a specific deadline so that the student knows exactly when they must submit outstanding work.
- **As a student with an Incomplete**, I want continued access to my course materials and assignment submissions after the term ends so that I can complete the required work.
- **As a registrar**, I want to see all outstanding Incompletes across all courses for the current term so that I can monitor resolution before the lapse deadline.
- **As an instructor**, I want an automated reminder when an Incomplete deadline is approaching so that I do not forget to enter the final grade.

## 5. Functional Requirements

- **FR-1.** The system MUST allow an instructor or admin to grant an Incomplete by creating an `incomplete_grade_records` entry tied to an enrollment, specifying an extension date and a list of outstanding assignment IDs.
- **FR-2.** The system MUST extend LMS course access for the student (read + submission rights) until the extension date, regardless of the term end date.
- **FR-3.** The system MUST send email notifications to the student when an Incomplete is granted, and reminder notifications at 30 days, 7 days, and 1 day before the extension deadline.
- **FR-4.** The system MUST allow the instructor to record the final resolved grade, which transitions the Incomplete to a completed state and marks the grade for SIS passback.
- **FR-5.** The system SHOULD provide a registrar-facing report listing all open Incompletes with student name, course, extension date, and outstanding assignments.
- **FR-6.** The system MUST record who granted the Incomplete and when in the audit log.
- **FR-7.** The system MAY automatically revoke course access and flag the record as lapsed if the extension date passes without resolution, pending admin configuration of lapse behavior.

## 6. Non-Functional Requirements

- **Performance** — Incomplete grant action completes in <500 ms including notification queue dispatch.
- **Security** — Only instructors of record and admins can grant or resolve Incompletes; student cannot self-grant.
- **Privacy & Compliance** — Incomplete records are FERPA-covered; registrar report requires appropriate role access.
- **Accessibility** — Incomplete grant dialog WCAG 2.1 AA; date picker keyboard-accessible.
- **Scalability** — Scheduled reminder job runs nightly; handles up to 10,000 open Incompletes per institution.
- **Reliability** — Reminder job failures retry; at-most-once notification delivery via idempotency key.
- **Observability** — Metric `incomplete_grades_open_total`, `incomplete_grades_resolved_total`; alert on lapse deadline batch failures.
- **Maintainability** — Incomplete logic in `server/src/services/incomplete_grades.rs`; reuses notification service from 6.2.
- **Internationalization** — Deadline dates displayed in student's locale and time zone.
- **Backward compatibility** — No impact on existing enrollments; new table is additive.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor grants a student an Incomplete with an extension date of 90 days, *when* the term end date passes, *then* the student can still access the course and submit assignments until day 90.
- **AC-2.** *Given* an open Incomplete is 7 days from the extension deadline, *when* the nightly reminder job runs, *then* both the student and the instructor receive a reminder email.
- **AC-3.** *Given* an instructor resolves an Incomplete by entering a final letter grade, *when* they submit, *then* the Incomplete record is marked `resolved`, the enrollment state updates, and the grade is queued for SIS passback.
- **AC-4.** *Given* a registrar opens the Incomplete report, *when* there are 15 open Incompletes, *then* the report lists all 15 with extension dates sortable by nearest deadline.
- **AC-5.** *Given* an Incomplete is granted, *when* the audit log is queried, *then* a record exists with the granting instructor's ID, timestamp, and extension date.

## 8. Data Model

Migration `077_incomplete_grades.sql`:
```sql
CREATE TABLE incomplete_grade_records (
  id                   UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  enrollment_id        UUID NOT NULL UNIQUE REFERENCES course_enrollments(id),
  granted_by           UUID NOT NULL REFERENCES users(id),
  extension_deadline   DATE NOT NULL,
  outstanding_item_ids UUID[] NOT NULL DEFAULT '{}',
  notes                TEXT,
  status               TEXT NOT NULL DEFAULT 'open', -- 'open', 'resolved', 'lapsed'
  resolved_grade       TEXT,
  resolved_at          TIMESTAMPTZ,
  resolved_by          UUID REFERENCES users(id),
  created_at           TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_incomplete_open ON incomplete_grade_records(extension_deadline) WHERE status = 'open';
```

## 9. API Surface

```
POST  /api/v1/courses/:id/enrollments/:enrollment_id/incomplete
      Body: { extension_deadline: date, outstanding_item_ids: uuid[], notes?: string }
      Auth: instructor or admin

PATCH /api/v1/courses/:id/enrollments/:enrollment_id/incomplete
      Body: { resolved_grade: string } | { extension_deadline: date }
      Auth: instructor or admin

GET   /api/v1/admin/incompletes?term_id=&status=open
      Auth: registrar or admin
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Extension date expires without notification causing lapse | M | H | Belt-and-suspenders: 30/7/1 day reminders + admin dashboard |
| Instructor forgets to resolve causing SIS passback delay | M | M | Automated lapse-flag after deadline; admin notification |
| Conflicting state between Incomplete and SIS withdrawal | L | H | SIS withdrawal takes precedence; log conflict for admin review |

## 15. Rollout Plan

## 16. Test Plan

- **Unit** — Incomplete service: grant, extend, resolve, lapse transitions.
- **Integration** — API endpoints for grant, resolve, admin report; verify email enqueue.
- **End-to-end** — Playwright: instructor grants Incomplete, student accesses course post-term, instructor resolves with final grade.
- **Security** — Assert student cannot grant own Incomplete; assert non-instructor cannot access endpoints.
- **Accessibility** — axe scan on Incomplete modal.
- **Performance / load** — Reminder job with 10,000 open Incompletes completes in <2 min.
- **Manual exploratory** — Test deadline boundary (day of deadline), test extension date update, test resolution before deadline.

## 18. Open Questions

1. Should the system auto-convert an Incomplete to "F" (lapse-to-F) or just flag it as lapsed pending registrar action?
2. Can an instructor extend the deadline beyond the institution's maximum (e.g., one year), or does the system enforce a max extension?
3. Should students see which specific assignments are outstanding in the Incomplete notice?
