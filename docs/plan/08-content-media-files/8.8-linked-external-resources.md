## 1. Problem Statement

Instructors and students already have large libraries of files in Google Drive, OneDrive, and Dropbox. Having to download and re-upload every file to Lextures wastes time and creates stale copies. There is no way to link an external cloud file as a course resource or as a submission. The `CourseModuleExternalLinkPage.tsx` handles plain URLs, but there is no picker integration that authenticates into a cloud storage provider and selects files with proper metadata. Without this, Lextures competes poorly against Canvas and Google Classroom on common instructor workflows.

## 2. Goals

- Integrate Google Picker API, Microsoft Graph Files API, and Dropbox Chooser SDK as file pickers in the course module authoring and assignment submission flows.
- Store a link reference (not a file copy) with metadata (name, icon, provider, external URL or direct link) in the Lextures DB.
- Optionally copy the file to Lextures object storage for backup (configurable per institution).
- Respect the owning user's OAuth token for access; do not share credentials across users.

## 3. Non-Goals

- Two-way sync of file edits between Lextures and the cloud provider.
- Browsing the file picker without the user being authenticated to the provider (no service-account access to user files).
- Inline editing of Google Docs / Office files within Lextures (render is a view link only).

## 4. Personas & User Stories

- **As an instructor**, I want to attach a Google Slides deck from my Drive to a module without downloading it so that students always see the latest version.
- **As a student**, I want to submit my Google Doc assignment directly from Drive so that I do not need to export it as a PDF.
- **As an instructor**, I want to attach a shared OneDrive folder of readings so that students can browse the folder in-context.
- **As an admin**, I want to restrict which cloud providers are enabled per institution so that we only authorize approved services under our data agreements.

## 5. Functional Requirements

- **FR-1.** The system MUST provide a "Link from cloud" button in the course module editor (`clients/web/src/components/editor/`) that opens a provider picker (Google Drive, OneDrive, Dropbox).
- **FR-2.** The system MUST use the **Google Picker API** (OAuth 2.0 `drive.file` scope) to let users select files from their Google Drive.
- **FR-3.** The system MUST use the **Microsoft Graph Files API** (`Files.Read` scope, MSAL.js) to let users select files from OneDrive / SharePoint.
- **FR-4.** The system MUST use the **Dropbox Chooser** JavaScript SDK to let users select files from Dropbox.
- **FR-5.** The system MUST store a `course.external_links` record with provider, external_id, display_name, view_url, icon_url, and picker_token_encrypted (for optional copy).
- **FR-6.** The system MUST NOT store the user's OAuth refresh token; link access relies on the file being publicly shared or re-authenticated on each access.
- **FR-7.** Admins MUST be able to enable/disable each provider per tenant via the admin settings UI.
- **FR-8.** The system SHOULD offer a "Copy to Lextures" option that downloads the file via the provider API and stores it in object storage (8.1), creating a `storage.objects` record.

## 6. Non-Functional Requirements

- **Performance** — Picker opens in < 3 s; link insertion UI < 500 ms after picker closes.
- **Security** — OAuth flows use PKCE. No provider tokens stored server-side in plaintext. Copy-to-Lextures file passes AV scan (8.6). External links opened in new tab with `rel="noopener noreferrer"`.
- **Privacy & Compliance** — FERPA: student-submitted Drive links accessible only to instructor/grader. GDPR: external links can be deleted (removes DB row; no cloud file deletion). Admin must accept provider DPA before enabling.
- **Accessibility** — Picker buttons keyboard-operable. Post-link insert, focus returns to editor with announcements. WCAG 2.1 AA.
- **Scalability** — Link storage is metadata-only; no storage scalability concern.
- **Reliability** — If the cloud file is deleted or permissions revoked, the link shows a "File unavailable" badge; no broken 404 errors in student view.
- **Observability** — Log provider, action (link/copy), `course_id`, `user_id` to `user_audit`.
- **Maintainability** — Provider integrations in `clients/web/src/services/cloudPicker/` with a common `CloudPickerProvider` interface.
- **Internationalization** — Provider names not translated; UI labels externalised.
- **Backward compatibility** — Existing `CourseModuleExternalLinkPage.tsx` plain-URL flow unaffected.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor with a Google account clicks "Link from Google Drive" *When* they authenticate and select a Slides file *Then* the module shows a Google Slides card with the file name, Google icon, and an "Open in Drive" link.
- **AC-2.** *Given* a student submitting to an assignment clicks "Submit from OneDrive" *When* they select a Word document *Then* the submission record stores the OneDrive view URL and the instructor sees a "Open in OneDrive" link in the grading view.
- **AC-3.** *Given* an admin has disabled Dropbox for the tenant *When* the instructor opens the cloud picker *Then* only Google Drive and OneDrive options are shown.
- **AC-4.** *Given* an instructor selects "Copy to Lextures" for a Drive file *When* the copy completes *Then* a `storage.objects` record exists, the copy passes AV scan, and the link in the module points to the Lextures copy.
- **AC-5.** *Given* a linked Drive file has its permissions revoked *When* a student views the module *Then* the link shows "File unavailable — contact your instructor" rather than a broken link or 404.

## 8. Data Model

```sql
-- server/migrations/080_external_links.sql
-- (existing module_external_links table from migration 060 may be extended)
ALTER TABLE course.module_external_links
  ADD COLUMN IF NOT EXISTS provider TEXT,          -- 'google_drive' | 'onedrive' | 'dropbox' | 'url'
  ADD COLUMN IF NOT EXISTS external_id TEXT,
  ADD COLUMN IF NOT EXISTS icon_url TEXT,
  ADD COLUMN IF NOT EXISTS copied_object_id UUID REFERENCES storage.objects(id);
```

## 9. API Surface

| Method | Path | Auth | Description |
|---|---|---|---|
| `POST` | `/courses/:id/modules/:mid/external-links` | Instructor JWT | Create external link (extended with provider fields) |
| `POST` | `/courses/:id/modules/:mid/external-links/:lid/copy` | Instructor JWT | Trigger copy-to-Lextures job |
| `GET` | `/admin/cloud-providers` | Admin JWT | List enabled providers per tenant |
| `PUT` | `/admin/cloud-providers/:provider` | Admin JWT | Enable or disable a provider |

## 10. UI / UX

- Module editor: "Add resource" menu gains "Link from cloud" sub-menu: Google Drive, OneDrive, Dropbox (only enabled providers shown).
- After picker selection: a card is inserted into the module with file icon, name, and provider badge.
- Assignment submission: add "Attach from cloud" button alongside "Upload file" in the submission widget.
- Accessibility: picker trigger buttons have `aria-haspopup="dialog"`. Provider picker dialog has `role="dialog"` with `aria-label`. Post-selection focus returns to editor.
- Mobile: Google Picker and OneDrive pickers have mobile-responsive modes; Dropbox Chooser uses mobile-optimized view.
- i18n keys: `cloudPicker.linkFromCloud`, `cloudPicker.google`, `cloudPicker.onedrive`, `cloudPicker.dropbox`, `cloudPicker.unavailable`.

## 11. AI / ML Considerations

Not applicable.

## 12. Integration Points

- **Google Picker API** v2 — `https://apis.google.com/js/api.js`, OAuth 2.0 `drive.file` scope.
- **Microsoft Graph Files API** v1.0 — MSAL.js 2.x, `Files.Read` scope.
- **Dropbox Chooser SDK** — `https://www.dropbox.com/static/api/2/dropins.js`.
- Internal: `server/src/routes/courses.rs` (module link endpoints), `clients/web/src/components/editor/` (picker UI), `server/src/workers/` (copy-to-storage job), AV scan (8.6).

## 13. Dependencies & Sequencing

- Must ship after: 8.1 (copy-to-storage path), 4.2 (OIDC for Google/Microsoft user auth — optional but improves UX).
- Can ship in parallel with: 8.3, 8.4, 8.5, 8.6, 8.7.
- Shared infra: OAuth 2.0 PKCE flow in frontend; no new backend auth services needed if no server-side token storage.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Provider OAuth scope rejected by institution IT | M | H | Use minimal scopes (drive.file, not drive); document scope justification |
| Picker popup blocked by browser | M | M | Trigger picker on direct user gesture; document pop-up blocker workaround |
| Linked file silently deleted in provider | M | M | Periodic link-liveness check job; show "unavailable" badge |
| GDPR: provider processes EU student data | M | H | Admin DPA acceptance gate before enabling any provider |

## 15. Rollout Plan

- **Flag:** `FEATURE_CLOUD_PICKER=true` (default false); per-provider sub-flags.
- **Sequencing:** Google Drive first (highest adoption) → OneDrive → Dropbox.
- **GA criteria:** picker functional in top-3 browsers; link insertion < 500 ms.
- **Rollback:** disable provider flags; existing plain-URL links unaffected.

## 16. Test Plan

- **Unit** — Provider adapter interface compliance, link metadata normalization.
- **Integration** — Mock provider APIs: picker returns file metadata, link stored correctly, copy job creates storage object.
- **End-to-end** — Playwright (with mock OAuth popup): select Drive file, verify module card appears; verify OneDrive submission attachment.
- **Security** — Confirm no provider tokens in server DB; confirm cross-tenant link access blocked.
- **Accessibility** — axe scan of picker trigger and post-selection card; keyboard navigation test.
- **Manual exploratory** — Live test with real Google Drive and OneDrive accounts.

## 17. Documentation & Training

## 18. Open Questions

1. Should server-side OAuth be supported (service account access) for institutions using Google Workspace domain-wide delegation?
2. Should we attempt to sync file name updates from the provider, or treat the name as static at link creation time?
3. What happens to copied-to-Lextures files if the institution later disables that cloud provider?
