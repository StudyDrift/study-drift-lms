## 1. Problem Statement

Instructors and self-learners have no way to discover or import free, openly licensed educational content from OER Commons, OpenStax, or MERLOT II without leaving Lextures and copy-pasting URLs. This is a significant differentiator for cost-conscious K-12 districts and self-learners who cannot afford commercial textbooks. Without an integrated OER search and import flow, the content library is limited to what each instructor manually creates or uploads.

## 2. Goals

- Provide an in-product OER search experience drawing from OER Commons API, MERLOT II API, and OpenStax CNX.
- Allow instructors to import OER content directly into a course module with provenance metadata and license attribution.
- Store OER metadata and cache imported assets in object storage (8.1).
- Respect Creative Commons and other open licenses by surfacing license type in the UI.

## 3. Non-Goals

- Hosting a full mirror of OER repositories.
- Automatically tagging OER content with standards alignment (deferred to 1.3).
- Editing or remixing OER content beyond what the source license permits (editorial workflow is a future feature).
- Paying for or indexing commercial content (that is 8.10 DRM).

## 4. Personas & User Stories

- **As an instructor in a title-I school**, I want to find a free, open-licensed chemistry textbook chapter so that my students do not have to buy a textbook.
- **As a self-learner**, I want to search for open courses on machine learning from OER Commons so that I can supplement my study plan without cost.
- **As an instructor**, I want the OER content's license (CC BY, CC BY-SA, etc.) displayed clearly so that I know how I can use and adapt it.
- **As an admin**, I want to disable specific OER sources for our institution so that we only surface age-appropriate content for K-12 students.

## 5. Functional Requirements

- **FR-1.** The system MUST provide an OER search panel in the course module editor accessible via a "Find open resources" button.
- **FR-2.** The system MUST query the **OER Commons API** (v2) by keyword, subject, grade level, and license filter.
- **FR-3.** The system MUST query **MERLOT II REST API** for higher-education materials by discipline and material type.
- **FR-4.** The system MUST surface **OpenStax CNX** book listings (via static catalog or OpenStax API) and allow linking individual chapters.
- **FR-5.** Search results MUST display title, source, license badge (CC icon), grade/level, description, and a preview link.
- **FR-6.** The system MUST allow instructors to "Add to module" which creates a `course.module_external_links` record with provider = `oer_commons` | `merlot` | `openstax` and full attribution metadata.
- **FR-7.** The system SHOULD cache OER search results for 24 hours per query to avoid hammering provider APIs.
- **FR-8.** The system MAY import OER content assets (PDFs, HTML packages) into object storage (8.1) when the instructor chooses "Import copy" instead of "Link."

## 6. Non-Functional Requirements

- **Performance** — OER search results load within 2 s (served from cache for repeated queries). Initial search < 3 s uncached.
- **Security** — OER API keys stored as env secrets. No user PII sent to external OER APIs. External links open with `rel="noopener noreferrer"`.
- **Privacy & Compliance** — License metadata stored accurately; CC attribution displayed per CC license requirements (BY, SA, ND, NC). FERPA: no student data sent to OER providers.
- **Accessibility** — Search results list accessible by keyboard; license badge has `aria-label="Creative Commons BY"` etc. WCAG 2.1 AA.
- **Scalability** — OER search is a thin proxy + cache; low backend load. Cache in Redis (17.5) or Postgres table.
- **Reliability** — If OER provider API is unavailable, return cached results with a "results may be outdated" banner; never fail hard.
- **Observability** — Metrics: `oer_search_requests_total`, `oer_import_total` (by provider). Log provider, query, result_count.
- **Maintainability** — OER proxy in `server/src/services/oer_search.rs`; provider adapters behind a `OerProvider` trait.
- **Internationalization** — License names shown in English (standard CC terminology); UI labels externalised.
- **Backward compatibility** — No changes to existing module external link behavior.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor types "photosynthesis" in the OER search panel *When* results load *Then* at least 5 results from OER Commons appear with title, source, license badge, and grade level within 2 s.
- **AC-2.** *Given* an instructor filters by "CC BY" license *When* results render *Then* no results with NC, ND, or SA-only licenses appear.
- **AC-3.** *Given* an instructor clicks "Add to module" on an OpenStax CNX chapter *When* the add completes *Then* a module external link appears with title "OpenStax: [Chapter Name]", a CC attribution line, and an "Open in OpenStax" link.
- **AC-4.** *Given* the OER Commons API is unreachable *When* a search is performed *Then* the most recent cached results (with an "As of {date}" label) are shown, not an error page.
- **AC-5.** *Given* an admin disables MERLOT for the tenant *When* the OER panel loads *Then* only OER Commons and OpenStax tabs are visible.

## 8. Data Model

```sql
-- server/migrations/081_oer_cache.sql
CREATE TABLE content.oer_search_cache (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  provider    TEXT NOT NULL,
  query_hash  TEXT NOT NULL,
  results_json JSONB NOT NULL,
  fetched_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  expires_at  TIMESTAMPTZ NOT NULL
);
CREATE UNIQUE INDEX ON content.oer_search_cache (provider, query_hash);
CREATE INDEX ON content.oer_search_cache (expires_at);

-- OER attribution stored in existing module_external_links with new columns
ALTER TABLE course.module_external_links
  ADD COLUMN IF NOT EXISTS license_spdx TEXT,
  ADD COLUMN IF NOT EXISTS attribution_text TEXT,
  ADD COLUMN IF NOT EXISTS oer_provider TEXT;
```

## 9. API Surface

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/oer/search?q=&provider=&license=&subject=&level=` | Instructor JWT | Proxy + cached OER search |
| `POST` | `/courses/:id/modules/:mid/oer-import` | Instructor JWT | Add OER link or trigger content import |
| `PUT` | `/admin/oer-providers/:provider` | Admin JWT | Enable / disable provider |

## 10. UI / UX

- Course module editor: "Add resource" → "Find open resources" opens a slide-over panel.
- Panel tabs: OER Commons, MERLOT, OpenStax. Search bar with subject/license/level filters.
- Result card: thumbnail/icon, title, provider logo, CC license icon, grade band, short description, "Preview" (new tab) and "Add to module" buttons.
- Added resource card in module: shows OER provider logo, title, license badge, "Required attribution: [text]."
- Accessibility: panel is a `<dialog>`, tab list uses `role="tablist"`, results list uses `role="list"`. Focus trapped in panel.
- i18n keys: `oer.search`, `oer.addToModule`, `oer.preview`, `oer.license`, `oer.provider`.

## 11. AI / ML Considerations

Optional future enhancement: use embedding similarity to surface OER content relevant to the current module's learning objectives. Not in scope for initial release.

## 12. Integration Points

- **OER Commons API v2** — `https://oercommons.org/api/v2/`.
- **MERLOT II REST API** — `https://www.merlot.org/merlot/viewMaterial.htm` (search API).
- **OpenStax CNX** — `https://openstax.org/api/v2/books` (static catalog + chapter listing).
- Internal: `server/src/services/oer_search.rs` (new), `server/src/routes/courses.rs`, `clients/web/src/components/editor/` (OER panel).

## 13. Dependencies & Sequencing

- Must ship after: 8.1 (for "Import copy" path).
- Can ship in parallel with: most other section 8 features.
- Shared infra: caching (17.5 or Postgres), job queue (17.3) for async content import.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| OER provider API changes / deprecation | M | M | Abstract behind `OerProvider` trait; provider-specific adapters swappable |
| Age-inappropriate content surfaced in K-12 | M | H | Grade-level filter required; admin per-provider enable/disable |
| CC license display error misleads users | L | H | Pull license from structured API field; never infer from description text |
| OER import copies infringe license | L | M | Only allow import for CC licenses that permit reproduction; block NC/ND on "import copy" |

## 15. Rollout Plan

- **Flag:** `FEATURE_OER_LIBRARY=true` (default false).
- **Sequencing:** OER Commons only in beta → add MERLOT → add OpenStax → GA.
- **GA criteria:** search returns results < 3 s; "Add to module" functional; license badge visible.
- **Rollback:** disable flag; no data loss (external links remain if already added).

## 16. Test Plan

- **Unit** — OER provider adapters: mock API responses, cache hit/miss logic, license normalization.
- **Integration** — OER search proxy: cached result returned on second request; provider unavailable returns cache.
- **End-to-end** — Playwright: search "algebra", add first result, confirm module link appears with attribution.
- **Security** — Confirm no student data in OER API request; confirm disabled provider hidden from UI.
- **Accessibility** — axe scan of OER panel; keyboard navigation through results list.
- **Manual exploratory** — Live test against OER Commons and OpenStax sandbox APIs.
