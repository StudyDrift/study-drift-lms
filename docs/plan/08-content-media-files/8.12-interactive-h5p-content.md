## 1. Problem Statement

H5P is the dominant open standard for interactive educational content (interactive videos, drag-and-drop, flashcards, branching scenarios). A large proportion of OER content is published as H5P packages. Lextures cannot render any H5P content, making it incompatible with the OER ecosystem (8.9), unable to embed interactive exercises from OER Commons, and unable to run SCORM/H5P-based publisher content. With no H5P support, instructors must recreate interactive exercises from scratch inside Lextures's native quiz engine when equivalent H5P activities already exist.

## 2. Goals

- Enable instructors to upload and embed H5P packages (`.h5p` zip files) in course modules.
- Render H5P content in an iframe using the H5P standalone library (h5p-standalone) without requiring a server-side H5P hub.
- Capture xAPI statements emitted by H5P activities and forward them to the Lextures event pipeline (9.6).
- Display H5P activity completion status in the gradebook if the activity emits a `passed/failed` xAPI result.

## 3. Non-Goals

- H5P authoring / editing within Lextures (H5P.org editor integration is a future feature).
- Running the full H5P Hub server-side (use client-side standalone renderer only in initial release).
- Grading H5P activities as scored assignments (completion-only in initial release; scored grading deferred).

## 4. Personas & User Stories

- **As an instructor**, I want to upload an interactive drag-and-drop sorting activity I built on H5P.org so that students can practice without any quiz setup on my part.
- **As a student**, I want to interact with a branching-scenario H5P activity embedded in my course so that I can explore decision trees the instructor designed.
- **As an instructor**, I want to see which students completed the H5P activity so that I know who engaged with the interactive exercise.
- **As a self-learner**, I want to consume OER courses with H5P exercises so that I have an interactive learning experience, not just static text.

## 5. Functional Requirements

- **FR-1.** The system MUST accept `.h5p` file uploads (ZIP archive) through the course module file upload flow; store in object storage (8.1) under a `h5p/` prefix.
- **FR-2.** The system MUST extract and store the H5P package manifest (`h5p.json`) as structured metadata in a `content.h5p_packages` table.
- **FR-3.** The system MUST render H5P packages in a sandboxed iframe using **h5p-standalone** (MIT) which loads the H5P JS/CSS assets from the package stored in object storage.
- **FR-4.** The H5P iframe MUST listen for `xAPI` `postMessage` events and forward them to the Lextures xAPI collector endpoint (`POST /xapi/statements` — see 9.6).
- **FR-5.** The system MUST record H5P activity completion per-student (statement verb `http://adlnet.gov/expapi/verbs/completed`) in the `course_grades` or a dedicated `content.h5p_completions` table.
- **FR-6.** Instructors MUST see a completion column in the gradebook for H5P module items showing "Completed" / "Not started" / "In progress" per student.
- **FR-7.** The H5P iframe MUST be served from a separate subdomain or `sandbox` attribute to prevent XSS from malicious H5P packages.
- **FR-8.** The system SHOULD validate the H5P package structure and supported content types before accepting the upload.

## 6. Non-Functional Requirements

- **Performance** — H5P package extracted and ready for rendering within 10 s of upload. Iframe load < 3 s on broadband.
- **Security** — H5P iframe sandboxed: `sandbox="allow-scripts allow-same-origin"` minimum; no `allow-top-navigation`. H5P packages scanned by AV (8.6) before rendering. Malicious JS in H5P limited by sandbox.
- **Privacy & Compliance** — xAPI statements may contain student response data; store per FERPA access rules. No H5P telemetry sent to H5P.org. GDPR: xAPI statements deletable on account erasure.
- **Accessibility** — H5P content accessibility depends on the content type; Lextures must ensure the iframe has a descriptive `title` attribute. Warn instructors if the H5P type is known to have low accessibility ratings. WCAG 2.1 AA wrapper.
- **Scalability** — H5P assets served from CDN via object storage (8.1); iframe rendering is client-side.
- **Reliability** — If H5P package fails to load, display a friendly error with a "Download H5P package" fallback link.
- **Observability** — Metrics: `h5p_packages_uploaded_total`, `h5p_xapi_statements_received_total`. Log package_id, content_type, user_id on render.
- **Maintainability** — H5P extraction worker in `server/src/workers/h5p_extract.rs`; frontend renderer in `clients/web/src/components/H5PPlayer.tsx`.
- **Internationalization** — H5P UI language set from the app locale string and passed to the h5p-standalone `options.l10n` config.
- **Backward compatibility** — No changes to existing module content types.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor uploads a valid `.h5p` Quiz package *When* the upload completes *Then* the module shows an "Interactive Activity" card and students can open and interact with the quiz in an iframe.
- **AC-2.** *Given* a student completes an H5P Drag and Drop activity *When* the activity fires the xAPI `completed` statement *Then* the instructor's gradebook shows "Completed" for that student within 60 seconds.
- **AC-3.** *Given* a malicious H5P package containing JavaScript that attempts `window.top.location = "phishing.example"` *When* the iframe is rendered *Then* the navigation is blocked by the sandbox policy and a CSP violation is logged.
- **AC-4.** *Given* an instructor uploads an H5P package with an unsupported content type *When* the validation runs *Then* the upload is rejected with a clear error listing the unsupported library name.
- **AC-5.** *Given* an H5P package fails to load in the iframe *When* the student opens the module *Then* a "This activity could not be loaded — download the file" message appears with a working download link.
- **AC-6.** *Given* a student's xAPI statements for an H5P course are requested under a GDPR erasure request *When* the deletion job runs *Then* all xAPI statements for that user are deleted from `content.h5p_completions`.

## 8. Data Model

```sql
-- server/migrations/083_h5p_packages.sql
CREATE TABLE content.h5p_packages (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  storage_object_id UUID NOT NULL REFERENCES storage.objects(id),
  course_id       UUID NOT NULL REFERENCES course.courses(id),
  title           TEXT NOT NULL,
  content_type    TEXT NOT NULL,
  h5p_version     TEXT,
  manifest_json   JSONB,
  assets_prefix   TEXT NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE content.h5p_completions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  package_id      UUID NOT NULL REFERENCES content.h5p_packages(id),
  user_id         UUID NOT NULL REFERENCES user_account(id),
  status          TEXT NOT NULL DEFAULT 'not_started',  -- not_started | in_progress | completed | passed | failed
  score_raw       REAL,
  score_max       REAL,
  xapi_statement  JSONB,
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (package_id, user_id)
);
CREATE INDEX ON content.h5p_completions (package_id, status);
```

## 9. API Surface

| Method | Path | Auth | Description |
|---|---|---|---|
| `POST` | `/courses/:id/modules/:mid/h5p` | Instructor JWT | Upload H5P package; returns package_id |
| `GET` | `/courses/:id/h5p/:package_id/render` | JWT (enrolled) | Returns HTML page bootstrapping h5p-standalone |
| `POST` | `/xapi/statements` | JWT | Receive xAPI statement from H5P iframe (also used by 9.6) |
| `GET` | `/courses/:id/h5p/:package_id/completions` | Instructor JWT | List per-student completion status |

## 10. UI / UX

- Module item type: "Interactive H5P Activity" — icon, title, completion badge.
- Student view: full-width iframe (responsive height from H5P `resize` postMessage). Loading spinner while package loads.
- Instructor view: "Upload H5P (.h5p file)" button in the module editor Add menu.
- Gradebook: "H5P" column type showing Completed / In Progress / Not Started badge.
- Error state: friendly message + download fallback.
- Accessibility: iframe `title="Interactive activity: {package title}"`. After iframe interaction, focus management via `postMessage` coordination.
- i18n keys: `h5p.loading`, `h5p.error`, `h5p.completed`, `h5p.inProgress`, `h5p.notStarted`.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| H5P package contains malicious JS | M | H | AV scan + iframe sandbox + CSP headers |
| H5P library version incompatibility | M | M | Pin h5p-standalone version; test against 10 most common H5P content types |
| Large H5P packages slow module load | M | M | Lazy-load H5P iframe only when module item is visible (IntersectionObserver) |
| xAPI statement flood from auto-looping H5P | L | M | Rate-limit xAPI ingestion per user per package at 100 statements/min |

## 15. Rollout Plan

- **Flag:** `FEATURE_H5P=true` (default false).
- **Sequencing:** upload + extract pipeline → iframe renderer → xAPI collector → gradebook completion column → enable flag.
- **GA criteria:** 5 common H5P content types render correctly; xAPI completion captured; gradebook column populated.
- **Rollback:** disable flag; uploaded .h5p files remain in storage but are not rendered.

## 16. Test Plan

- **Unit** — H5P manifest parsing, xAPI statement normalization, completion status derivation from verb.
- **Integration** — Worker: extract real H5P package to object storage prefix; verify assets reachable.
- **End-to-end** — Playwright: upload H5P True/False quiz, student opens it, clicks answer, verify completion status in gradebook.
- **Security** — Sandbox escape test (malicious H5P attempting top-navigation); CSP violation logged.
- **Accessibility** — iframe title present; post-completion focus test.
- **Performance** — Large H5P package (10 MB) load time < 3 s on throttled broadband.
- **Manual exploratory** — Test 10 H5P content types from H5P.org; test xAPI statement shapes for each.
