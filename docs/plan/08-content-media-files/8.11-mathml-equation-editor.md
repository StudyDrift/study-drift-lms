## 1. Problem Statement

The TipTap-based content editor in `clients/web/src/components/editor/` has no capability for inserting or rendering mathematical notation. STEM instructors cannot create calculus lectures, physics problem sets, or chemistry equations. Students cannot type formula responses. This is a hard blocker for any K-12 math course or HE STEM department. The ecosystem solution is a MathML 4-compatible rendering engine (MathJax or KaTeX) paired with a visual equation editor.

## 2. Goals

- Integrate KaTeX (fast rendering) as the math renderer with MathJax as the accessibility fallback.
- Add a TipTap extension for inline math (`$...$`) and display math (`$$...$$`) stored as LaTeX source with MathML output.
- Provide a visual equation editor dialog (keyboard math input palette + LaTeX input field) for users unfamiliar with LaTeX syntax.
- Render equations in all content surfaces: module content pages, quiz questions, assignment descriptions, and student responses.

## 3. Non-Goals

- A full CAS (computer algebra system) for symbolic computation.
- Accepting handwritten equation photos (deferred to 19.7).
- Real-time collaborative equation editing.

## 4. Personas & User Stories

- **As a calculus instructor**, I want to write `∫₀^π sin(x) dx` in my lecture notes and have it rendered beautifully so that students see properly typeset math.
- **As a student answering a math question**, I want a clickable equation keyboard so that I can enter a fraction or square root without knowing LaTeX syntax.
- **As a physics instructor**, I want to author equations that remain accessible to screen readers through MathML markup so that my blind students can follow the math.
- **As an instructor**, I want to paste LaTeX I already have and see it rendered immediately so that I do not need to reformat existing materials.

## 5. Functional Requirements

- **FR-1.** The system MUST render LaTeX-delimited math (`$inline$` and `$$display$$`) in all TipTap-rendered HTML surfaces using **KaTeX** ≥ 0.16.
- **FR-2.** All KaTeX-rendered expressions MUST include MathML output (`output: "mathml"`) for screen reader compatibility per **MathML 4** and WCAG 2.1 SC 1.3.1.
- **FR-3.** The TipTap editor MUST provide a custom `MathNode` extension that stores the LaTeX source as node data and renders the KaTeX HTML client-side.
- **FR-4.** The system MUST provide an equation editor dialog accessible via toolbar button or `/` command (`/equation`) with two modes: LaTeX text input with live preview, and a symbol palette of common math symbols organized by category (Greek letters, operators, fractions, integrals, etc.).
- **FR-5.** Equations MUST be editable in-place: double-clicking a rendered equation re-opens the editor dialog pre-populated with the LaTeX source.
- **FR-6.** The system MUST render math in student response text areas (short-answer quiz responses, assignment text submissions) using the same KaTeX renderer.
- **FR-7.** For instructors without LaTeX knowledge, the system SHOULD expose a subset of 50+ common STEM symbols via the palette (Greek, arithmetic, calculus, logic, set theory).

## 6. Non-Functional Requirements

- **Performance** — KaTeX renders a typical equation in < 5 ms; renders up to 50 equations per page without perceptible jank. Initial KaTeX bundle loaded lazily (import only when a math node is present).
- **Security** — KaTeX output is a controlled HTML subset; do not use `dangerouslySetInnerHTML` with unsanitized user input. Validate LaTeX input against KaTeX's supported command set before rendering.
- **Privacy & Compliance** — No equation data leaves the browser for rendering; entirely client-side. MathML ensures screen-reader accessibility (ADA, WCAG 2.1 SC 1.3.1).
- **Accessibility** — MathML markup must be present in the DOM (not just visual SVG/HTML). KaTeX MathML output mode provides this. Equation editor dialog keyboard-navigable (Tab through symbol palette, Enter to insert). ARIA: `role="math"` and `aria-label` with a text description on each equation block.
- **Scalability** — Client-side only; no server load impact.
- **Reliability** — If KaTeX throws on invalid LaTeX, display the raw LaTeX string in a `<code>` block with an "Equation error" badge, never crash the page.
- **Observability** — Log equation editor opens and insert events to `user_audit` (action=`equation_inserted`, course_id, user_id).
- **Maintainability** — TipTap extension in `clients/web/src/components/editor/MathExtension.ts`; equation editor dialog in `clients/web/src/components/editor/EquationEditorDialog.tsx`.
- **Internationalization** — Symbol palette labels externalised. Equation editor dialog labels in i18n.
- **Backward compatibility** — Existing TipTap content without math nodes unaffected.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor types `$$\frac{d}{dx}\sin(x) = \cos(x)$$` in the editor *When* the content is rendered on the module page *Then* a properly typeset display equation appears and the DOM contains `<math>` MathML markup.
- **AC-2.** *Given* a student uses NVDA screen reader and navigates to an equation *When* focus reaches the math element *Then* NVDA announces the MathML text description of the equation.
- **AC-3.** *Given* an instructor double-clicks a rendered equation *When* the editor dialog opens *Then* the LaTeX source is pre-populated and live preview renders the updated equation as they type.
- **AC-4.** *Given* an instructor clicks "Greek letters" in the symbol palette and selects `θ` *When* the symbol is inserted *Then* `\theta` appears at the cursor position in the LaTeX input and the preview updates.
- **AC-5.** *Given* an instructor enters invalid LaTeX `\frac{1}{}` *When* the dialog renders the preview *Then* a "Equation syntax error" message appears next to the preview; saving is not blocked but the error is noted.
- **AC-6.** *Given* a page contains 40 inline equations *When* the page loads *Then* all 40 equations render within 1 s on a modern laptop (KaTeX batch render).

## 8. Data Model

No schema changes. Equations stored as LaTeX strings within TipTap's JSON document blob in existing `course.module_content_pages` and `course.module_quizzes` JSONB columns.

New TipTap node schema (stored in JSON):
```json
{ "type": "math", "attrs": { "latex": "\\frac{a}{b}", "display": true } }
```

## 9. API Surface

No new HTTP endpoints. Equation content saved through existing content update endpoints in `server/src/routes/courses.rs`.

## 10. UI / UX

- Editor toolbar: new `Σ` (sigma) button opens equation editor dialog.
- Slash command: `/equation` triggers inline math insertion.
- Equation editor dialog: left pane = LaTeX text input (monospace); right pane = live KaTeX preview. Tabbed symbol palette below input: General, Greek, Calculus, Logic, Chemistry.
- Inline math in content: rendering is inline-block, vertically centered with surrounding text.
- Display math: full-width centered block with generous vertical margin.
- Error state: red border on equation block, tooltip "Equation cannot be rendered — check LaTeX syntax."
- Mobile: equation editor dialog scrollable; symbol palette wraps.
- Accessibility: dialog `role="dialog"`, `aria-labelledby="equation-editor-title"`. Symbol buttons have `aria-label="Insert theta"`. KaTeX math elements have `aria-label` derived from a text description function.
- i18n keys: `equation.editor`, `equation.preview`, `equation.insert`, `equation.syntax`, `equation.symbols`.

## 11. AI / ML Considerations

Future: AI-assisted LaTeX generation from natural-language description ("the integral of sine x from 0 to pi"). Not in initial scope.

## 12. Integration Points

- **KaTeX** ≥ 0.16 (`katex` npm package, MIT license) — client-side LaTeX rendering to HTML + MathML.
- **MathJax** 3.x (optional A11y fallback) — full MathML rendering for assistive technologies if KaTeX MathML is insufficient.
- **MathML 4** — W3C specification for math markup.
- **TipTap** 2.x — editor framework; custom `Node` extension API.
- Internal: `clients/web/src/components/editor/MathExtension.ts` (new), `clients/web/src/components/editor/EquationEditorDialog.tsx` (new), `clients/web/src/components/editor/` (existing editor infrastructure).

## 13. Dependencies & Sequencing

- No hard backend dependencies; pure frontend.
- Should ship before: 2.3 (math in quiz questions depends on this renderer) so 8.11 is the shared foundation.
- Can ship in parallel with all other section 8 features.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| KaTeX bundle increases page load noticeably | M | M | Lazy-load KaTeX only when math node present in document |
| MathML screen reader support inconsistent across AT | M | H | Test with NVDA, JAWS, VoiceOver; keep MathJax as fallback option |
| Instructors unfamiliar with LaTeX | H | M | Symbol palette covers the most common 50 symbols; add "need help?" link to LaTeX guide |
| TipTap schema migration for existing documents | L | L | New node type is additive; old docs without math nodes unaffected |

## 15. Rollout Plan

- **Flag:** `FEATURE_EQUATION_EDITOR=true` (default false initially; low risk to enable broadly).
- **Sequencing:** integrate KaTeX renderer → add TipTap MathNode → build equation editor dialog → expose toolbar button → enable flag.
- **GA criteria:** equations render in Chrome, Firefox, Safari; MathML present in DOM; symbol palette covers 50+ symbols.
- **Rollback:** disable flag; TipTap math nodes render as raw LaTeX text.

## 16. Test Plan

- **Unit** — MathNode extension: latex attribute storage, display mode toggle, serialization to HTML.
- **Integration** — KaTeX render test: known equations render expected HTML; invalid LaTeX triggers error state.
- **End-to-end** — Playwright: instructor inserts equation via toolbar, saves page, student views page, equation rendered correctly.
- **Accessibility** — axe scan of page with 10 equations; NVDA script verifying `<math>` element announcement.
- **Performance** — Render 50 equations on one page; measure render time < 1 s.
- **Manual exploratory** — Test every symbol palette category; test copy-paste of LaTeX from external sources.

## 17. Documentation & Training
