## 1. Problem Statement

Any institution subject to ADA (Section 508), WCAG 2.1, or the UK Equality Act must provide captions on video content. Lextures serves every uploaded video as a raw file with no captions and no transcript. This is a procurement BLOCKER for any US higher-education or K-12 customer and a liability risk for self-learner videos. Auto-captioning via OpenAI Whisper or a cloud speech API, delivered as WebVTT files attached to HLS streams, closes the compliance gap and simultaneously provides searchable transcripts that benefit all learners.

**Cross-link:** 8.4 captions ↔ 12.4 accessibility.

## 2. Goals

- Auto-generate WebVTT caption files for every uploaded video using OpenAI Whisper (self-hosted or API) with a cloud fallback.
- Attach captions to HLS streams via `#EXT-X-MEDIA TYPE=SUBTITLES` entries in the master playlist.
- Provide a transcript view alongside the video player in the course UI.
- Allow instructors to edit, replace, or disable auto-generated captions.
- Surface caption quality score and flag low-confidence segments for instructor review.

## 3. Non-Goals

- Real-time live-stream captioning (requires live-stream feature, deferred).
- Translation of captions into multiple languages (deferred to 19.5 AI summarization/translation).
- Captioning of audio-only uploads (initial scope: video only; audio extends naturally later).

## 4. Personas & User Stories

- **As a Deaf student**, I want accurate captions on every lecture video so that I can follow the course content without relying on an interpreter.
- **As an instructor**, I want captions generated automatically so that I do not need to spend hours transcribing each video.
- **As an instructor**, I want to edit the auto-generated transcript to correct technical terms before students see it so that the captions are accurate.
- **As a student with English as a second language**, I want to read the transcript alongside the video so that I can pause and look up unfamiliar words.
- **As an accessibility coordinator**, I want a report showing which videos still lack reviewed captions so that I can prioritize remediation.

## 5. Functional Requirements

- **FR-1.** The system MUST trigger a caption job within 60 seconds of a transcode job (8.3) reaching `done` status.
- **FR-2.** The system MUST use OpenAI Whisper (via `WHISPER_BACKEND`: `whisper-api`, `whisper-local`, `azure-speech`, `google-speech`) to transcribe the audio track.
- **FR-3.** The system MUST produce a WebVTT file conforming to W3C WebVTT specification stored at `{tenant_id}/{course_id}/captions/{video_id}/{lang}.vtt` in object storage.
- **FR-4.** The system MUST inject `#EXT-X-MEDIA TYPE=SUBTITLES,GROUP-ID="subs"` entries into the HLS master playlist referencing the generated VTT.
- **FR-5.** The system MUST store the full transcript as plaintext in the `storage.captions` table for search indexing.
- **FR-6.** The system SHOULD compute a per-segment confidence score; segments below 0.70 confidence MUST be flagged for instructor review.
- **FR-7.** Instructors MUST be able to edit the transcript via a text editor UI and re-export to VTT.
- **FR-8.** The system SHOULD detect the spoken language automatically (Whisper language detection) and record it in the caption record.
- **FR-9.** Caption generation MUST be skippable per-video (instructor opt-out toggle) while still being required by default.

## 6. Non-Functional Requirements

- **Performance** — Caption generation completes within 2× the video duration for Whisper-local; within 5 min for Whisper API for a 1-hour video. UI shows progress status.
- **Security** — Audio data sent to external ASR API only if `WHISPER_BACKEND != whisper-local`. If sending externally, data processing agreement required; admin must explicitly opt in.
- **Privacy & Compliance** — ADA Section 508 compliance: captions required. FERPA: transcripts tied to course enrollment access. GDPR: transcript deletion on video deletion. No PII sent to Whisper API beyond what is in the audio content. Cross-link: §12.4.
- **Accessibility** — VTT files must conform to WebVTT spec; `<track kind="subtitles">` injected in `<video>` element; captions toggleable from player controls. WCAG 2.1 SC 1.2.2.
- **Scalability** — Caption worker scales alongside transcode worker; up to 20 concurrent ASR jobs.
- **Reliability** — Fallback: if Whisper API is unavailable, retry with local Whisper model (if `whisper-local` available), else mark `caption_status=api_unavailable` and alert.
- **Observability** — Metrics: `caption_job_created_total`, `caption_job_duration_seconds`, `caption_low_confidence_segments_total`. Log `job_id`, `video_id`, `lang`, `confidence_avg`, `backend`.
- **Maintainability** — Caption worker in `server/src/workers/captioning.rs`; VTT formatter in `server/src/services/vtt_formatter.rs`.
- **Internationalization** — Whisper detects language; `lang` stored in caption record. Player caption menu shows detected language label.
- **Backward compatibility** — Videos uploaded before 8.4 may be retroactively captioned via admin bulk-trigger job.

## 7. Acceptance Criteria

- **AC-1.** *Given* a video upload completes transcoding *When* 60 seconds have elapsed *Then* a caption job is in `processing` state and instructor sees "Captions: Generating…" in the video settings panel.
- **AC-2.** *Given* a caption job completes *When* a student opens the video player *Then* the player shows a "CC" button and loading the VTT track renders captions synchronized with speech.
- **AC-3.** *Given* a caption segment has confidence < 0.70 *When* the instructor opens the transcript editor *Then* the low-confidence segment is highlighted in amber with a "Review" badge.
- **AC-4.** *Given* an instructor edits and saves the transcript *When* the save completes *Then* the updated VTT is uploaded to object storage, the HLS manifest updated, and all subsequent player sessions load the revised captions.
- **AC-5.** *Given* `WHISPER_BACKEND=whisper-api` and the API is unreachable *When* a caption job runs *Then* the system retries 3 times, marks `caption_status=failed`, and sends an admin alert; no exception propagates to the student UI.
- **AC-6.** *Given* a course video with captions *When* the course is deleted *Then* all VTT files are removed from object storage and the `storage.captions` row is deleted.

## 8. Data Model

```sql
-- server/migrations/077_captions.sql
CREATE TYPE storage.caption_status AS ENUM
  ('queued', 'processing', 'done', 'failed', 'api_unavailable', 'instructor_reviewed');

CREATE TABLE storage.captions (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  storage_object_id UUID NOT NULL REFERENCES storage.objects(id) ON DELETE CASCADE,
  lang              TEXT NOT NULL DEFAULT 'en',
  vtt_key           TEXT,
  transcript_text   TEXT,
  confidence_avg    REAL,
  backend           TEXT NOT NULL,
  status            storage.caption_status NOT NULL DEFAULT 'queued',
  has_low_confidence BOOLEAN NOT NULL DEFAULT false,
  created_at        TIMESTAMPTZ NOT NULL DEFAULT now(),
  reviewed_at       TIMESTAMPTZ,
  reviewed_by       UUID REFERENCES user_account(id)
);
CREATE INDEX ON storage.captions (storage_object_id, lang);
CREATE INDEX ON storage.captions (status) WHERE status NOT IN ('done', 'instructor_reviewed');
```

## 10. UI / UX

- Video player (`VideoPlayer.tsx`): "CC" button in control bar; clicking opens a subtitle track selector. Captions render as white text on black semi-transparent background (WCAG SC 1.4.3 compliant contrast).
- Transcript panel: collapsible drawer below video showing full transcript with timestamps. Clicking a timestamp seeks the video to that cue.
- Instructor caption editor: TipTap-style inline editor per VTT cue with low-confidence highlights. "Save & Publish" button updates the stored VTT.
- Status badge on module item: "Captions: Generating" (amber), "Captions: Ready" (green), "Review needed" (orange), "Captions: Failed" (red).
- Accessibility: `<track kind="subtitles" src="..." srclang="en" label="English">` injected; caption toggle keyboard shortcut `C`.
- i18n keys: `captions.generating`, `captions.ready`, `captions.reviewNeeded`, `captions.ccButton`, `captions.transcript`.

## 11. AI / ML Considerations

- **Models:** OpenAI Whisper `large-v3` (local) or OpenAI Whisper API (`whisper-1`). Fallback: Google Cloud Speech-to-Text v2 or Azure Cognitive Services Speech.
- **Prompts:** Whisper takes audio input; prompt seeding with course title improves domain vocabulary accuracy.
- **Eval metric:** Word Error Rate (WER) target < 10 % on English academic speech.
- **Fallback path:** `whisper-local` → `whisper-api` → `azure-speech` → `failed`.
- **PII redaction:** Transcript text scanned for patterns matching student PII (name, email) before storage when `CAPTION_PII_REDACT=true`.
- **Cost budget:** Whisper API is $0.006/min audio; cap at 10 h/day per tenant with alert at 80 %.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| WER too high for technical/STEM content | H | M | Prompt seed with course vocabulary; allow instructor editing |
| External ASR sends student audio to third-party | M | H | Default to local Whisper; require admin opt-in for external API |
| Caption job backlog during course start | M | M | Priority queue; batch off-peak; alert at queue depth > 100 |
| VTT timing drift after instructor edits | L | M | Re-validate cue timestamps on save; reject overlapping cues |

## 15. Rollout Plan

- **Flag:** `FEATURE_AUTO_CAPTIONING=true` (default false).
- **Sequencing:** deploy caption worker → deploy schema → deploy API endpoints → deploy UI components → enable flag for pilot courses → monitor WER via sampling → GA.
- **Dogfood:** internal Lextures video content.
- **GA criteria:** WER < 10 % on 20-video sample; caption ready < 5 min for 1-hour video; instructor editor functional.
- **Rollback:** disable flag; video player hides CC button; VTT files remain in storage.

## 16. Test Plan

- **Unit** — VTT formatter: cue boundary calculation, confidence masking, timestamp serialization.
- **Integration** — Caption worker with mock Whisper API: queue → process → VTT stored → manifest updated.
- **End-to-end** — Playwright: upload test video with known transcript, verify CC button active, verify cue text matches expected at 10-second mark.
- **Security** — Confirm external ASR not called when `WHISPER_BACKEND=whisper-local`; confirm VTT access restricted to enrolled users.
- **Accessibility** — Playwright + axe: caption toggle button labeled; captions visible on video with 3:1 contrast vs. background; screen-reader announces "Captions enabled."
- **Performance / load** — 10 concurrent caption jobs; confirm queue drains within SLA.
- **Manual exploratory** — Verify low-confidence highlight in editor; test edit-save-reload VTT cycle.

## 17. Documentation & Training

- End-user docs: "Accessibility features — captions on videos" with instructions for students to enable captions.
- Instructor docs: "Reviewing and editing auto-generated captions — walkthrough, expected accuracy, how to request recaptioning."
- Admin docs: choosing Whisper backend, configuring API keys, monitoring caption queue health.
- Internal runbook: `docs/runbooks/captioning-worker.md` — restart worker, clear stuck jobs, evaluate WER.

## 18. Open Questions

1. Should we self-host Whisper `large-v3` on a GPU worker or rely on the API exclusively? GPU hosting lowers per-unit cost but adds infra complexity.
2. Should we support multi-language captions in the initial release (detect + produce VTT in the detected language only) or plan for instructor-specified target languages?
3. What is the maximum audio duration we process in one job (current thinking: 3 hours with chunked processing)?
4. Should the transcript be indexed for full-text search within a course?
