## 1. Problem Statement

The California Consumer Privacy Act (CCPA, Cal. Civ. Code § 1798.100 et seq.) and its amendment the California Privacy Rights Act (CPRA, effective January 1 2023) grant California residents the right to know, delete, correct, and opt out of the sale or sharing of personal information. Lextures has no "Do Not Sell or Share My Personal Information" link, no privacy-rights request handler, and no opt-out signal processing (Global Privacy Control / GPC). Serving California residents — which is true for any consumer-facing SL product and any US K-12/HE deployment — without these controls exposes Lextures to CPRA enforcement by the California Privacy Protection Agency (CPPA) with fines up to $7,500 per intentional violation.

## 2. Goals

- Implement a "Do Not Sell or Share" opt-out mechanism compliant with CPRA § 1798.120 and GPC signal processing.
- Build a unified California Privacy Rights request handler (know, delete, correct, limit sensitive PI processing).
- Provide a 45-day response workflow for rights requests.
- Disclose the categories of personal information collected, sold/shared, and disclosed for business purposes.
- Update the privacy policy and add the required homepage link.

## 3. Non-Goals

- GDPR DSAR tooling (see 10.3) — the workflows are adjacent but separate legal instruments.
- Cookie consent / opt-in for non-California residents.
- Building a full Consent Management Platform (CMP) — v1 covers statutory CCPA/CPRA requirements only.

## 4. Personas & User Stories

- **As a California resident using Lextures**, I want to opt out of the sale or sharing of my personal information so that my data is not monetized without my consent.
- **As a California resident**, I want to request a copy of all personal information Lextures has collected about me in the past 12 months so that I can verify data practices.
- **As a California resident**, I want to request deletion of my personal information so that I can exercise my right to be forgotten under CPRA.
- **As a California resident**, I want to limit the use of my sensitive personal information (e.g., browsing history, AI interaction logs) to what is necessary for the primary service.
- **As a compliance admin**, I want to process and track rights requests within 45 days and log all actions.

## 5. Functional Requirements

- **FR-1.** The system MUST display a "Do Not Sell or Share My Personal Information" link in the footer and on the privacy settings page, linking to the opt-out mechanism (CPRA § 1798.120(a)).
- **FR-2.** The system MUST process the Global Privacy Control (GPC) browser signal as an automatic opt-out of sale/sharing without requiring additional user action (CPPA regulations § 7025).
- **FR-3.** The system MUST provide a California Privacy Rights request form accepting: Right to Know (categories + specific pieces), Right to Delete, Right to Correct, Right to Limit Sensitive PI.
- **FR-4.** The system MUST verify the requestor's identity before fulfilling a Right to Know or Delete request (CPRA § 1798.130(a)(3)); acceptable methods: account login (for registered users) or email-with-signed-link (for unregistered).
- **FR-5.** The system MUST respond to rights requests within 45 calendar days with a one-time 45-day extension allowed if notified (CPRA § 1798.130(a)(2)).
- **FR-6.** The system MUST NOT discriminate against users who exercise CPRA rights (§ 1798.125).
- **FR-7.** The system SHOULD publish a privacy notice listing: categories of PI collected, business/commercial purposes, categories of third parties PI is disclosed to, and categories of sensitive PI (CPRA § 1798.100(a)).
- **FR-8.** The system MUST log all rights-request events and their resolution in `admin_audit_log`.

## 6. Non-Functional Requirements

- **Performance** — Rights-request queue response page MUST load in < 1 second; export generation reuses GDPR DSAR job infrastructure.
- **Security** — Identity verification tokens MUST expire in 24 hours. Sensitive PI access requires step-up auth per `server/src/http_auth.rs`.
- **Privacy & Compliance** — Conformance with CCPA Cal. Civ. Code §§ 1798.100–1798.199, CPRA amendments, and CPPA regulations 11 CCR § 7000 et seq. OWASP ASVS 4.0 §8. NIST 800-53: IP-1 (Consent), IP-4 (Complaint Management). GPC signal processing per IAB GPC specification.
- **Accessibility** — All new UI pages MUST conform to WCAG 2.1 AA (see 10.7).
- **Scalability** — GPC signal check MUST be evaluated at request middleware level with < 1 ms overhead; opt-out flag cached in Redis.
- **Reliability** — Rights request processing MUST be idempotent; duplicate requests within 12 months return existing response.
- **Observability** — Counter `ccpa_optouts_total`, gauge `ccpa_requests_pending`, alert if > 45-day SLA breached.
- **Maintainability** — CCPA logic in `server/src/services/ccpa.rs`, reusing DSR infrastructure from `server/src/services/gdpr.rs`.
- **Internationalization** — Privacy rights request form MUST be available in English and Spanish per California requirements (Cal. Civ. Code § 1798.130(a)(5)).
- **Backward compatibility** — `ccpa_do_not_sell` flag defaults `false`; no impact on existing users unless they opt out.

## 7. Acceptance Criteria

- **AC-1.** *Given* a visitor sends an HTTP request with `Sec-GPC: 1` header, *when* the middleware processes the request, *then* the user's session is flagged as `do_not_sell = true` and no personal information is sent to ad-tech or analytics third parties.
- **AC-2.** *Given* a California resident submits a Right to Know request, *when* the identity is verified and the request is processed, *then* a response listing all PI categories collected in the past 12 months is delivered within 45 days.
- **AC-3.** *Given* a registered user submits a Right to Delete request, *when* the deletion is approved, *then* personal information is deleted consistent with the CPRA obligations (excluding exempted categories) and the user receives written confirmation.
- **AC-4.** *Given* a user has opted out of sale/sharing, *when* they submit a subsequent request that would constitute sale/sharing, *then* the request is blocked and the opt-out flag is preserved.
- **AC-5.** *Given* an admin reviews the rights-request queue, *when* a request is within 5 days of the 45-day deadline, *then* an escalation alert is triggered in the observability stack.

## 8. Data Model

```sql
-- Migration: 076_ccpa.sql

ALTER TABLE user.users
  ADD COLUMN ccpa_do_not_sell BOOLEAN NOT NULL DEFAULT FALSE,
  ADD COLUMN ccpa_limit_sensitive_pi BOOLEAN NOT NULL DEFAULT FALSE;

CREATE TABLE IF NOT EXISTS compliance.ccpa_requests (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID REFERENCES user.users(id),  -- NULL for unregistered requester
  requester_email TEXT NOT NULL,
  request_type    TEXT NOT NULL CHECK (request_type IN ('know_categories','know_specific','delete','correct','limit_sensitive')),
  status          TEXT NOT NULL DEFAULT 'pending' CHECK (status IN ('pending','verified','in_progress','completed','denied')),
  verification_token_hash TEXT,
  response_payload TEXT,     -- JSON summary or archive path
  requested_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  due_at          TIMESTAMPTZ GENERATED ALWAYS AS (requested_at + INTERVAL '45 days') STORED,
  completed_at    TIMESTAMPTZ,
  extended        BOOLEAN NOT NULL DEFAULT FALSE
);

CREATE INDEX idx_ccpa_requests_due ON compliance.ccpa_requests(due_at) WHERE status != 'completed';
```

## 10. UI / UX

- **Footer link** — "Do Not Sell or Share My Personal Information" visible on all public pages.
- **Privacy Settings page** — Do Not Sell toggle, Limit Sensitive PI toggle.
- **California Privacy Rights form** (`/privacy-rights/california`) — request type selector, identity verification.
- **Status page** (`/privacy-rights/california/status/:id`) — request progress with deadline.
- i18n keys prefixed `ccpa.*`; English + Spanish.
- WCAG 2.1 AA; semantic HTML form with labels; no CAPTCHA barriers per CPRA guidance.

## 11. AI / ML Considerations

When `ccpa_limit_sensitive_pi = true`, AI features must exclude the user's browsing history, inferences drawn from PI, and biometric data from any prompt context. Reuses AI consent gate from `gdpr_consents` (10.3) with an additional CCPA-specific check.
