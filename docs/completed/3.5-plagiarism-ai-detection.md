## 1. Problem Statement

Lextures markets itself as an AI-first LMS yet has no capability to detect AI-generated or plagiarised content in student submissions. This is an existential gap: every HE RFP requires a plagiarism detection integration (typically Turnitin), and the rise of LLM-assisted writing has created a separate AI-authorship detection requirement. As an AI-native platform, Lextures has a unique opportunity to offer an internal AI-authorship signal (using the same AI infrastructure) alongside external provider integrations, turning a compliance necessity into a product differentiator.

## 2. Goals

- Integrate with at least one external plagiarism detection provider (Turnitin, Copyleaks, or GPTZero) via their REST API.
- Provide an optional internal AI-authorship detection signal computed using existing AI infrastructure.
- Store originality report metadata with the submission (3.14).
- Surface similarity scores and AI-probability scores in the grader panel alongside the submission.
- Allow instructors to configure detection per assignment (disabled / plagiarism-only / AI-only / both).
- Comply with data residency constraints when sending student text to third-party APIs.

## 3. Non-Goals

- Building a full plagiarism corpus internally (use external providers for corpus comparison).
- Automated grade reduction based on similarity score (human-in-the-loop only).
- Real-time detection during typing (post-submit only for MVP).
- Detection on file types other than text, PDF, DOCX, and plain-text code submissions.

## 4. Personas & User Stories

- **As an instructor**, I want each text submission automatically checked against Turnitin's corpus so that I receive a similarity score before I begin grading.
- **As an instructor**, I want a one-click view of the full Turnitin originality report embedded in the grader panel so that I do not need to leave Lextures.
- **As a department chair**, I want to configure institution-wide plagiarism detection as the default for all new assignments so that every instructor benefits automatically.
- **As a student**, I want to see my similarity score after submission so that I understand my standing before grades are returned.
- **As an admin**, I want to configure our Turnitin API credentials at the institution level so that all courses use a single agreement without per-instructor keys.

## 5. Functional Requirements

- **FR-1.** The system MUST support integration with Turnitin Similarity API v2 and/or Copyleaks API v3 as external providers, configurable per institution.
- **FR-2.** The system MUST offer an internal AI-authorship probability score computed via the existing OpenRouter AI infrastructure as a supplementary signal.
- **FR-3.** After a student submits an assignment, the system MUST enqueue a detection job via the background job queue (17.3); the job submits the text/file to the configured provider and polls or handles a webhook callback.
- **FR-4.** The system MUST store the originality report result (similarity percentage, AI probability, report URL / embedded iframe token) linked to the submission record (see 3.14).
- **FR-5.** The system MUST surface the similarity score and AI probability in the grader panel with colour-coded indicators (green / amber / red bands configurable by institution).
- **FR-6.** The system MUST allow instructors to open the full originality report in an embedded iframe or new tab.
- **FR-7.** The system SHOULD allow students to view their own similarity score (configurable by instructor: show / hide / show-after-grading).
- **FR-8.** The system MUST allow detection to be disabled per assignment (opt-out) to handle cases like personal reflective writing.
- **FR-9.** The system MUST NOT send student content to external APIs unless the institution has accepted the provider's DPA and the instructor has not disabled detection for the assignment.
- **FR-10.** The system SHOULD support GPTZero API as an additional AI-content detection provider option.

## 6. Non-Functional Requirements

- **Performance** — Detection jobs are async; submission acknowledgement ≤ 500 ms regardless of detection. Report available within the provider's SLA (Turnitin: typically 2–10 min).
- **Security** — API keys for external providers stored in secrets management (17.17), not in environment variables in plaintext. Student text transmitted over TLS 1.3. Provider responses validated (HMAC webhook signatures).
- **Privacy & Compliance** — Student submission content is PII/FERPA-protected. Third-party data processing agreements (DPA) must be signed with each provider before enabling. GDPR Article 28 processor requirements apply. Data residency: EU institutions must use provider EU endpoints. Students must be notified (per FERPA/institution policy) that submissions are checked.
- **Accessibility** — Originality report indicators keyboard accessible; colour coding supplemented with text labels and icons (WCAG 2.1 AA).
- **Scalability** — Detection jobs fan-out with concurrency limit per provider API rate limit. Queue backed by 17.3 job system; retry with exponential backoff.
- **Reliability** — If provider API is unavailable, submission still succeeds; detection status = 'pending'. Retry for up to 48 h before marking 'failed'. Instructor alerted of failed detection.
- **Observability** — Metrics: `detection.job_enqueued`, `detection.job_completed`, `detection.provider_error`. Log provider name, similarity score bucket, assignment_id.
- **Maintainability** — Provider integrations behind a trait `OriginalityProvider` in `server/src/services/originality/`; each provider is a separate module (turnitin.rs, copyleaks.rs, gpt_zero.rs, internal.rs).
- **Internationalization** — Score labels and status messages i18n-keyed. Score thresholds configurable, not hardcoded.
- **Backward compatibility** — Existing submissions have no detection records; detection status shown as 'not checked'.

## 7. Acceptance Criteria

- **AC-1.** *Given* Turnitin is configured for the institution and detection is enabled on assignment A, *when* a student submits a text document, *then* a detection job is enqueued within 1 s of submission confirmation.
- **AC-2.** *Given* a detection job completes with a 45% similarity score, *when* the instructor opens the grader panel, *then* an amber indicator showing "45% similarity" is visible adjacent to the submission.
- **AC-3.** *Given* an instructor disables detection for assignment B, *when* a student submits to B, *then* no detection job is enqueued and no request is made to any external provider.
- **AC-4.** *Given* the external provider API returns a webhook callback with the report URL, *when* the instructor clicks "View Report," *then* the originality report opens in an embedded iframe inside the grader panel.
- **AC-5.** *Given* an institution has not signed a provider DPA (provider not configured), *when* an instructor attempts to enable detection on an assignment, *then* the system shows an error: "Plagiarism detection requires institutional configuration."
- **AC-6.** *Given* the internal AI-authorship detection is enabled, *when* a submission is processed, *then* an AI probability score (0–100%) is stored and displayed alongside the external similarity score.
- **AC-7.** *Given* the provider API is unavailable for 2 h, *when* the retry limit is not yet reached, *then* the submission's detection status remains 'pending' and the instructor sees "Detection in progress."

## 8. Data Model

```sql
-- Migration: server/migrations/077_originality_detection.sql

CREATE TABLE originality_reports (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    submission_id   UUID NOT NULL REFERENCES module_assignments(id) ON DELETE CASCADE,
    provider        TEXT NOT NULL CHECK (provider IN ('turnitin','copyleaks','gptzero','internal')),
    status          TEXT NOT NULL DEFAULT 'pending'
                    CHECK (status IN ('pending','processing','done','failed','skipped')),
    similarity_pct  NUMERIC(5,2),
    ai_probability  NUMERIC(5,2),
    report_url      TEXT,
    report_token    TEXT,              -- embed token (Turnitin EULA viewer)
    provider_report_id TEXT,
    error_message   TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

ALTER TABLE module_assignments
    ADD COLUMN originality_detection TEXT NOT NULL DEFAULT 'disabled'
        CHECK (originality_detection IN ('disabled','plagiarism','ai','both'));

CREATE INDEX idx_originality_submission ON originality_reports(submission_id);
```

## 11. AI / ML Considerations

- **Internal AI detection**: use OpenRouter with a classifier prompt on the submission text. Output: probability 0–1 that text is AI-generated. Cache result; do not re-run on subsequent views.
- **Fallback**: if OpenRouter fails, internal AI score not shown; external provider result still displayed.
- **PII redaction**: strip student name from text before sending to internal AI model (19.11).
- **Cost**: internal detection ~1000 tokens per 500-word submission; budget $0.002 per check.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Provider API pricing changes or service outage | M | H | Abstract behind trait; fallback gracefully; document alternative providers |
| Student PII sent to third party without DPA | H | H | Hard block at enqueue: provider_configured AND dpa_accepted check before any API call |
| GDPR: EU student data leaves EU endpoint | M | H | Per-institution endpoint config; default to provider's EU region |
| False positives cause student disputes | H | M | Score is advisory only; UI copy "Discuss with your instructor"; no automated grade penalty |

## 15. Rollout Plan

- Feature flag: `ORIGINALITY_DETECTION_ENABLED` (default: `false`).
- Phase 1: internal AI detection only (no external provider required).
- Phase 2: Copyleaks integration (simpler DPA than Turnitin) for pilot.
- Phase 3: Turnitin integration for HE pilot customers.
- GA criteria: detection for 100 submissions in pilot with zero PII leaks; webhook reliability > 99%.
- Rollback: disable flag; no student data sent.
