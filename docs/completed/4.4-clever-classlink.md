# 4.4 — Clever / ClassLink

> Implementation plan. Source: [docs/MISSING_FEATURES.md](../MISSING_FEATURES.md) §4.4.
>
> **Shipped in repo (phase 1 — SSO + JIT):** Clever OAuth 2.0 + PKCE Instant Login at `/auth/clever/login` and `/auth/clever/callback` (`internal/service/cleverauth`), Clever `/v3.0/me` + user profile, JIT with `clever_id` and `is_minor`; optional OIDC routes `/auth/oidc/clever|classlink/*` for parity and account linking; ClassLink OIDC with tenant issuer env `CLASSLINK_OIDC_ISSUER`. DB migrations include `120_clever_classlink.sql`, `121_oidc_providers_clever_classlink.sql`; `GET /api/v1/auth/oidc/status` exposes `cleverEnabled`, `classlinkEnabled`, and legacy `clever` / `classlink` booleans for the login UI. Clever Secure Sync and ClassLink OneRoster roster consumption remain future work.

## Deferred from this iteration

- Clever Secure Sync (scheduled `/v3.0/students`, `/teachers`, `/sections` jobs), admin sync/config HTTP APIs, and webhook verification.
- ClassLink Roster Server as a distinct consumer (beyond existing OneRoster 4.3 when the district uses that path).
- Account profile “Connected via Clever / ClassLink” copy (data model supports `settings.user_oidc_identities` rows for `clever` and `classlink`).

## Configuration (environment)

- Clever PKCE flow: `CLEVER_SSO_ENABLED=1`, `CLEVER_CLIENT_ID`, `CLEVER_CLIENT_SECRET`, optional `CLEVER_DISTRICT_ID`; redirect base `OIDC_PUBLIC_BASE_URL` (or `LTI_API_BASE_URL`). Legacy names `CLEVER_OIDC_CLIENT_ID` / `CLEVER_OIDC_CLIENT_SECRET` are accepted as aliases.
- ClassLink OIDC: `CLASSLINK_SSO_ENABLED=1`, `CLASSLINK_OIDC_ISSUER`, `CLASSLINK_OIDC_CLIENT_ID`, `CLASSLINK_OIDC_CLIENT_SECRET`.

## Problem statement (full plan archive)

Clever and ClassLink are the dominant SSO and rostering middleware platforms used by US K-12 school districts. Teachers and students expect one-click access from district portals. Without integration, Lextures cannot appear in many district app libraries.

Goals include Clever Instant Login, Clever Secure Sync (roster), ClassLink SSO, and ClassLink Roster Server consumption; phased rollout with feature flags `CLEVER_SSO_ENABLED`, `CLASSLINK_SSO_ENABLED`. See historical sections in git history for full FR/NFR/acceptance criteria and test plans.
