# 4.4 — Clever / ClassLink

> Implementation plan. Source: [docs/MISSING_FEATURES.md](../MISSING_FEATURES.md) §4.4. **Completed (phase 1 — SSO + JIT):** Clever OAuth 2.0 + PKCE Instant Login, `/v3.0/me` + user profile fetch, JIT user provisioning with `clever_id` and `is_minor` (Clever `is_under_13`); ClassLink OIDC via `/auth/oidc/classlink/*` with `classlink_id` on users; DB tables `settings.clever_district_configurations`, `settings.clever_sso_flow_state`; OIDC identity providers extended for `clever` and `classlink`.

## Deferred from this iteration

- Clever Secure Sync (scheduled `/v3.0/students`, `/teachers`, `/sections` jobs), admin sync/config HTTP APIs, and webhook verification.
- ClassLink Roster Server as a distinct consumer (beyond existing OneRoster 4.3 when the district uses that path).
- Account profile “Connected via Clever / ClassLink” copy (data model supports `settings.user_oidc_identities` rows for `clever` and `classlink`).

## Configuration (environment)

- Clever: `CLEVER_SSO_ENABLED=1`, `CLEVER_CLIENT_ID`, `CLEVER_CLIENT_SECRET`, optional `CLEVER_DISTRICT_ID`; redirect base `OIDC_PUBLIC_BASE_URL` (or `LTI_API_BASE_URL`).
- ClassLink OIDC: `CLASSLINK_SSO_ENABLED=1`, `CLASSLINK_OIDC_ISSUER`, `CLASSLINK_OIDC_CLIENT_ID`, `CLASSLINK_OIDC_CLIENT_SECRET`.
