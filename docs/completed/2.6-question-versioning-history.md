# 2.6 — Question Versioning & History

Status: completed.

Implemented work:
- Added DB support for question versions (`course.question_versions`) and delivered attempt version tracking (`attempt_question_selections.version_number`) in `server/migrations/080_question_versioning.sql`.
- Extended question bank entities and APIs to persist and expose version metadata (`versionNumber`, `isPublished`).
- Added instructor endpoints for version history and restore:
  - `GET /api/v1/courses/{course_code}/questions/{question_id}/versions`
  - `GET /api/v1/courses/{course_code}/questions/{question_id}/versions/{version_number}`
  - `POST /api/v1/courses/{course_code}/questions/{question_id}/versions/{version_number}/restore`
- Updated question update flow to create immutable version snapshots for published questions, with optional `changeNote`.
- Updated quiz attempt materialization and delivery resolution to store and respect the delivered question version.
- Added question bank UI support for version history viewing and restoring, plus change-note capture on edits.
