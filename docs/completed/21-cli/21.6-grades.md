## 1. Problem Statement

Instructors grading large cohorts — especially in institutions with strict reporting deadlines — need to export and update grades in bulk without using the web UI for each submission. External systems (SIS, Banner, Canvas) also need grade exports in CSV for import. This story covers `grades list`, `grades update`, and `grades export`.

## 2. Goals

- `lextures grades list --course <id>` shows the gradebook for a course.
- `lextures grades update` grades a single submission with a score and optional comment.
- `lextures grades export --course <id> --format csv` downloads grades as a CSV file.
- All commands support `--json`.

## 3. Non-Goals

- Bulk grading from a CSV import (follow-on; can be scripted via `update` in a loop).
- Rubric-based grading UI (web only).
- Grade appeal workflows.

## 4. Personas & User Stories

- **As an instructor**, I want `lextures grades export --course CS101 --format csv > grades.csv` so that I can import grades into my institution's SIS at the end of term.
- **As a TA**, I want `lextures grades update --submission abc123 --score 88 --comment "Good work"` so that I can grade submissions from a script that reads a local annotation file.
- **As an admin generating compliance reports**, I want `lextures grades list --course <id> --json` so that I can feed grade data into our analytics pipeline.

## 5. Functional Requirements

- **FR-1.** `lextures grades list --course <id>` MUST return all grades for the course; supports `--user <id>` to filter to a single student; supports `--limit`, `--page`.
- **FR-2.** `lextures grades update` MUST accept `--submission <id>` (required), `--score <n>` (required), `--comment <text>` (optional).
- **FR-3.** `lextures grades export --course <id> --format csv` MUST write a CSV to stdout (redirect to file) or to `--output <path>`.
- **FR-4.** CSV format MUST include columns: `student_id`, `student_name`, `student_email`, `assignment_title`, `score`, `max_points`, `percentage`, `submitted_at`, `graded_at`, `comment`.
- **FR-5.** All commands MUST support `--json`.
- **FR-6.** `lextures grades update` MUST be idempotent: re-grading an already-graded submission MUST update the score (not error).

## 6. Non-Functional Requirements

- **Performance** — Export for a 500-student course MUST complete in < 30 s; stream rows rather than buffering.
- **Privacy** — Grade data is sensitive; CLI MUST warn if output is being written to a world-readable file.
- **Security** — Only instructors and admins MUST be permitted to `update`; server enforces. Export MUST require instructor or admin role.

## 7. Acceptance Criteria

- **AC-1.** *Given* `lextures grades list --course <id> --json`, *When* it runs, *Then* stdout is a valid JSON array with score and student fields.
- **AC-2.** *Given* `lextures grades update --submission <id> --score 95`, *When* it runs, *Then* the server returns the updated grade and the CLI prints confirmation.
- **AC-3.** *Given* `lextures grades export --course <id> --format csv > out.csv`, *When* it runs, *Then* `out.csv` is valid CSV with the required columns.
- **AC-4.** *Given* `--user <student-id>` is passed to `grades list`, *When* it runs, *Then* only that student's grades appear.
- **AC-5.** *Given* a TA calls `grades update` for a student not in their section, *When* it runs, *Then* the server returns 403 and the CLI exits `2` with a clear permission error.

## 8. Data Model

No new database changes. Leverages existing grade records. See `server/internal/repos/coursegrades/bulk_put.go` and `server/internal/httpserver/gradebook_grades_put.go`.

## 9. API Surface

Uses existing endpoints:

```
GET    /api/courses/:course_id/grades         ?user_id=&limit=&page=
PUT    /api/submissions/:id/grade             { score, comment }
GET    /api/courses/:course_id/grades/export  ?format=csv        (streams CSV)
```

## 10. UI / UX

Non-JSON `grades list`: tabular with columns `Student`, `Assignment`, `Score`, `Max`, `%`, `Status`. Export streams to stdout with no extra UI output (pipe-friendly).
