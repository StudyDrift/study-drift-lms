## 1. Problem Statement

Admins and instructors frequently need to create, list, and manage courses in bulk — for example, provisioning dozens of courses at semester start or auditing active courses across an org. The web UI requires navigating each course individually. A CLI `courses` resource lets these workflows be scripted, templated, and automated via CI/CD pipelines.

## 2. Goals

- `lextures courses list` returns all courses visible to the authenticated user, with optional org and term filters.
- `lextures courses create` creates a course from flags or a JSON/YAML input file.
- `lextures courses get <id>` fetches full course details.
- `lextures courses delete <id>` deletes a course with a confirmation prompt.
- All commands support `--json` for scripting.

## 3. Non-Goals

- Course content or module management (future stories).
- Bulk import from CSV (future; use scripted `create` loop for now).
- Course archiving (separate `archive` subcommand may be added later).

## 4. Personas & User Stories

- **As an admin**, I want to run `lextures courses list --org my-org --json | jq '.[].id'` to get all course IDs for scripting downstream operations.
- **As an instructor**, I want `lextures courses create --name "Spring 2027 CS101" --code CS101` to provision a course from a CI job without touching the web UI.
- **As a developer writing an integration test**, I want `lextures courses delete <id>` to tear down test courses programmatically.

## 5. Functional Requirements

- **FR-1.** `lextures courses list` MUST return all courses the authenticated user can see; `--org <id>` and `--term <term>` MUST filter results.
- **FR-2.** `lextures courses create` MUST accept `--name` (required) and `--code` (required) flags; `--org <id>` is optional (defaults to user's org).
- **FR-3.** `lextures courses get <id>` MUST print full course details including enrollment counts, term, and status.
- **FR-4.** `lextures courses delete <id>` MUST prompt "Are you sure? (y/N)" unless `--force` is passed; MUST exit non-zero if the course is not found.
- **FR-5.** All commands MUST support `--json` for machine-readable output.
- **FR-6.** `lextures courses list` MUST support `--limit <n>` and `--page <n>` for pagination.

## 6. Non-Functional Requirements

- **Performance** — List MUST stream/paginate; default page size 50.
- **Security** — Delete MUST verify the user has the `course:delete` permission server-side; CLI enforces the confirmation prompt as a UX guard only.
- **Reliability** — Network errors MUST be retried once with exponential backoff before surfacing to the user.

## 7. Acceptance Criteria

- **AC-1.** *Given* authenticated, *When* `lextures courses list --json` is run, *Then* stdout is a valid JSON array of course objects.
- **AC-2.** *Given* `--org unknown-id`, *When* `lextures courses list` runs, *Then* the CLI prints a human-readable error from the server and exits `2`.
- **AC-3.** *Given* `lextures courses create --name "Test" --code T101`, *When* the command succeeds, *Then* the new course ID is printed (human) or `{"id": "…"}` (JSON mode).
- **AC-4.** *Given* `lextures courses delete <id>` with no `--force`, *When* the user types `n`, *Then* the course is not deleted and exit code is `0`.
- **AC-5.** *Given* `lextures courses delete <id> --force`, *When* the command runs, *Then* no prompt is shown and the course is deleted.

```
GET    /api/courses/:id
DELETE /api/courses/:id
```

## 10. UI / UX

CLI table output (non-JSON): aligned columns for `ID`, `Name`, `Code`, `Term`, `Enrolled`. `--json` outputs the raw API response array.
