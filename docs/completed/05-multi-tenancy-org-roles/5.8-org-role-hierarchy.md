## 1. Problem Statement

The current RBAC in `server/src/authz.rs` is either course-scoped (owner/instructor/student) or global (global_admin). There is no middle tier: an org admin who should be able to see all courses in their district, or a principal who can manage only their school's resources. Every institutional deployment of Lextures requires at least an org-admin role to onboard users, create courses, and manage settings without the vendor (super-admin) being involved. Without org-level roles the product cannot be self-served by institutions.

## 2. Goals

- Introduce first-class `org_admin` and `org_unit_admin` roles in the RBAC model.
- Extend `server/src/authz.rs` to evaluate org-scoped and unit-scoped role grants.
- Provide UI and API for org admins to assign/revoke org roles.
- Ensure backward compatibility with existing course-scoped grants.
- Document the full role permission matrix.

## 3. Non-Goals

- Fine-grained permission bits (individual capabilities) — V1 uses coarse-grained roles only.
- Custom role definitions per org — deferred.
- Cross-org role delegation — not supported.
- Designer / TA / Observer roles — covered in 5.9.

## 4. Personas & User Stories

- **As a district IT director**, I want an `org_admin` role so that I can add teachers, create courses, and manage settings for my district without needing vendor support.
- **As a school principal**, I want an `org_unit_admin` scoped to my school so that I can manage my building's courses and teachers without seeing other schools.
- **As a global admin**, I want to assign and revoke org-admin roles so that I can bootstrap new organizations.
- **As a developer**, I want the authz layer to evaluate role grants in a deterministic, well-tested way so that adding new roles does not break existing logic.

## 5. Functional Requirements

- **FR-1.** The system MUST define an `org_roles` enum with values: `org_admin`, `org_unit_admin`, `org_viewer` (read-only auditor at org scope).
- **FR-2.** The system MUST store org role grants in an `org_role_grants` table: `user_id`, `org_id`, `org_unit_id` (nullable), `role`, `granted_by`, `granted_at`, `expires_at` (nullable).
- **FR-3.** The system MUST update `server/src/authz.rs` to check org role grants on every request, with a role hierarchy: `global_admin` > `org_admin` > `org_unit_admin` > course-scoped roles.
- **FR-4.** The system MUST provide APIs for org admins to list, assign, and revoke org role grants for users in their org.
- **FR-5.** The system MUST emit audit log events for every org role grant and revocation.
- **FR-6.** The system SHOULD support time-limited role grants (non-null `expires_at`) and a scheduled job that revokes expired grants.
- **FR-7.** The system SHOULD display an org-roles summary table in the admin UI showing which users hold which org roles.
- **FR-8.** The system MAY support delegation: an org_admin can grant org_unit_admin roles but cannot grant org_admin roles (that requires global_admin or another org_admin).

## 6. Non-Functional Requirements

- **Performance** — role lookup must add ≤2 ms overhead; resolved role must be cached in the authenticated request context (not re-queried per authz check).
- **Security** — privilege escalation must be blocked: org_unit_admin cannot grant org_admin; validated by authz tests. Role grant API must be behind org_admin or global_admin guard.
- **Privacy & Compliance** — role grants are sensitive admin records; included in audit log (10.11); accessible to FERPA data-access reviewers.
- **Accessibility** — role management UI must be keyboard and screen-reader accessible.
- **Scalability** — an org may have ≤1,000 org-level role grants; index on `(org_id, user_id)`.
- **Reliability** — expired grant revocation job must be idempotent.
- **Observability** — `org_role` logged on every admin action; alert if grant revocation job fails.
- **Maintainability** — role resolution centralized in `server/src/authz.rs`; no ad-hoc role checks in route handlers.
- **Internationalization** — role names and descriptions externalized for UI display.
- **Backward compatibility** — existing `user_course_grants` and global_admin role (`server/migrations/061_global_admin_role.sql`) remain unchanged.

## 7. Acceptance Criteria

- **AC-1.** *Given* a user is granted `org_admin` for Org-A, *When* they call `GET /api/orgs/org-a/courses`, *Then* all courses in Org-A are returned.
- **AC-2.** *Given* a user is granted `org_unit_admin` scoped to "Lincoln High", *When* they call `GET /api/orgs/org-a/courses`, *Then* only courses in "Lincoln High" or its subtree are returned.
- **AC-3.** *Given* an `org_unit_admin` user, *When* they attempt to grant `org_admin` to another user, *Then* HTTP 403 is returned.
- **AC-4.** *Given* a role grant with `expires_at` in the past, *When* the revocation job runs, *Then* the grant is removed and the user loses the elevated access.
- **AC-5.** *Given* an org admin revokes a role grant, *When* the audit log is queried, *Then* a record exists with `action=role_revoked`, `target_user_id`, `role`, and `actor_id`.
- **AC-6.** *Given* a user with no org role, *When* they attempt to call any `/api/orgs/:id/` admin endpoint, *Then* HTTP 403 is returned.

## 8. Data Model

```sql
-- server/migrations/080_org_role_grants.sql
CREATE TABLE org_role_grants (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id       UUID NOT NULL REFERENCES organizations(id),
  user_id      UUID NOT NULL REFERENCES users(id),
  org_unit_id  UUID REFERENCES org_units(id),
  role         TEXT NOT NULL CHECK (role IN ('org_admin','org_unit_admin','org_viewer')),
  granted_by   UUID NOT NULL REFERENCES users(id),
  granted_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  expires_at   TIMESTAMPTZ,
  UNIQUE (org_id, user_id, role, org_unit_id)
);

CREATE INDEX idx_org_role_grants_user  ON org_role_grants(user_id, org_id);
CREATE INDEX idx_org_role_grants_expiry ON org_role_grants(expires_at)
  WHERE expires_at IS NOT NULL;
```

## 9. API Surface

| Verb | Path | Auth scope | Description |
|---|---|---|---|
| GET | `/api/orgs/:id/role-grants` | `org_admin` | List org role grants |
| POST | `/api/orgs/:id/role-grants` | `org_admin` | Assign org role |
| DELETE | `/api/orgs/:id/role-grants/:gid` | `org_admin` | Revoke org role grant |

Request body for POST: `{ user_id: string; role: "org_admin"|"org_unit_admin"|"org_viewer"; org_unit_id?: string; expires_at?: string }`

## 10. UI / UX

New tab in admin console: "Roles & Permissions" — table of users with their org roles; add/remove role modal.
Role assignment modal: user search autocomplete + role dropdown + optional expiry date.

Empty state: "No org role grants yet. Assign an org admin to allow self-service management."
Mobile: role table collapses to card list.
Accessibility: role assignment modal traps focus; role dropdown is a labeled `<select>`; revoke action requires confirmation.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Privilege escalation via grant API | M | H | Server-side role-hierarchy check; automated authz matrix test |
| Role resolution adds noticeable latency | M | M | Cache resolved role in request extension; benchmark |
| Expired grants not cleaned up promptly | M | M | Revocation job runs every 15 min; also check `expires_at` at authz evaluation time |

## 15. Rollout Plan

- Feature flag: `org_roles_enabled` (default: `false`).
- Sequence: DB migration → authz update → API → UI → staging → production.
- Dogfood: assign internal team as org_admin of dev org.
- GA criteria: role hierarchy tests pass; privilege escalation test fails as expected.
- Rollback: disable flag; grants table is additive.

## 16. Test Plan

- **Unit** — role hierarchy resolution; expiry check.
- **Integration** — grant/revoke API; authz guard with each role type.
- **End-to-end** — Playwright: assign org_admin, log in as that user, verify access.
- **Security** — authz matrix: escalation attempt returns 403; unit-admin limited to subtree.
- **Accessibility** — role modal accessible; role table keyboard navigable.
- **Performance / load** — authz resolution with 1,000 concurrent requests; measure latency overhead.
- **Manual exploratory** — expired grant revocation; org_viewer read-only verification.
