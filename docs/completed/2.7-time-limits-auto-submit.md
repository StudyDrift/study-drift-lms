# 2.7 — Time Limits & Auto-Submit on Timeout

Status: completed.

## Delivered

- Added timeout-specific attempt failure semantics via `ATTEMPT_TIME_EXPIRED` (HTTP 403).
- Added database support for timeout auto-submit tracking:
  - `course.quiz_attempts.auto_submitted BOOLEAN NOT NULL DEFAULT FALSE`
  - `idx_quiz_attempts_deadline_open` partial index for open timed attempts.
- Added backend auto-submit sweep service (`server/src/services/quiz_auto_submit.rs`) that:
  - scans expired in-progress attempts
  - computes earned/possible from saved responses
  - finalizes attempts as submitted with `auto_submitted = true`
- Wired a 30-second background sweep loop in server startup (`server/src/lib.rs`).
- Upgraded learner timer UX in quiz taking:
  - HH:MM:SS countdown
  - 10-minute and 1-minute warning announcements
  - assertive ARIA announcement near timeout
  - automatic submit trigger when timer reaches zero
  - color transitions for normal/warn/critical remaining time

## Notes

- Server deadline enforcement remains authoritative.
- Auto-submit uses currently saved responses at sweep time.
