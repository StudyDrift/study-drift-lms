## 1. Problem Statement

Students may miss an assignment due to illness, emergency, or an approved accommodation. Today Lextures has no way to mark an assignment as "excused" — a distinct state from "missing" (0 score) or "not submitted." In the absence of an excused state, instructors must enter a placeholder score (often 100% to avoid grade damage) or leave the field blank, both of which distort the gradebook and misrepresent the student's record. Excused assignments must be excluded from averages and drop calculations, and must appear distinctly in transcripts and progress reports.

## 2. Goals

- Add an `excused` boolean flag to individual grade rows in `course_grades`.
- Exclude excused assignments from all grade calculations (group averages, final course grade, drop-lowest).
- Display the excused state distinctly in the gradebook and My Grades (not as a zero or blank).
- Log every excusal and un-excusal in the grade-change audit trail (3.10).
- Support bulk-excusing via CSV import (3.11 "excuse" column).

## 3. Non-Goals

- Automated excusal workflows triggered by attendance (deferred to K-12 attendance, 13.2).
- Accommodation-based automatic extensions (deferred to 2.11 Accommodations engine).
- Excusal of entire courses or terms (registrar-level incomplete grade — 14.4).

## 4. Personas & User Stories

- **As an instructor**, I want to mark a missed quiz as "Excused" for a student who was ill so that their course grade is not affected by one missed assessment.
- **As a student**, I want to see "Excused" in My Grades rather than a zero so that I understand my status accurately.
- **As a TA doing bulk grading**, I want to mark excused status in a CSV upload so that I can process a batch of accommodation requests at once.
- **As an admin reviewing a grade dispute**, I want to see the audit record of when an excusal was granted and by whom.

## 5. Functional Requirements

- **FR-1.** The system MUST add an `excused` boolean column (default `false`) to `course_grades`.
- **FR-2.** When `excused = true`, the assignment score MUST be excluded from all grade average computations, group calculations, and drop-lowest rules.
- **FR-3.** The gradebook and My Grades MUST display excused assignments with an "EX" indicator rather than a score or blank.
- **FR-4.** The system MUST allow instructors to toggle the excused state via the gradebook cell context menu.
- **FR-5.** Every excusal toggle (excused → unexcused, or unexcused → excused) MUST create a `grade_audit_events` record with `action = 'excused'` or `'unexcused'` (3.10).
- **FR-6.** The system MUST support an `excuse` column in bulk CSV import (3.11), accepting values "EX," "excused," or "1."
- **FR-7.** The system MAY allow instructors to provide a reason when marking a grade excused (stored in audit event `reason` field).

## 6. Non-Functional Requirements

- **Performance** — Excused flag checked during grade computation (no additional query; already fetched with grade row). No performance impact.
- **Security** — Excusal toggle requires instructor/owner role. Students cannot self-excuse.
- **Privacy & Compliance** — Excused state is part of the FERPA education record. Included in DSAR exports and grade-change audit trail.
- **Accessibility** — "EX" cell has `aria-label="Excused — not counted toward grade"`. Context menu keyboard accessible. WCAG 2.1 AA.
- **Scalability** — Single boolean column; negligible storage impact.
- **Reliability** — Excusal is an atomic DB update + audit insert in one transaction.
- **Observability** — Metric: `grade_excusals_total` counter. Log `excused` events with `assignment_id`, `student_id`.
- **Maintainability** — Excusal logic in `server/src/services/grading/excusal.rs` (trivial); mostly wired into existing grade computation.
- **Internationalization** — "EX" label i18n-keyed; full label "Excused" for tooltip.
- **Backward compatibility** — `excused = false` for all existing rows; no change in behaviour.

## 7. Acceptance Criteria

- **AC-1.** *Given* a student with quiz scores [60, 70, 80] where the 60 is excused, *when* the group average is computed, *then* the average is (70+80)/2 = 75.
- **AC-2.** *Given* an excused grade, *when* the student opens My Grades, *then* the cell shows "EX" and no numeric score.
- **AC-3.** *Given* the instructor marks a grade as excused, *then* a `grade_audit_events` row with `action = 'excused'` and `changed_by` = instructor is written.
- **AC-4.** *Given* a CSV import with `excuse = EX` for student S on assignment A, *when* import is confirmed, *then* the grade row for S/A has `excused = true` and an audit event is written.
- **AC-5.** *Given* a drop-lowest rule with N=1 and one excused assignment among [50, EX, 80, 90], *when* the computation runs, *then* the excused score is excluded from both the drop calculation and the average; the dropped score is 50.
- **AC-6.** *Given* a student calls the My Grades API, *when* their grade is excused, *then* the API returns `{ score: null, status: "excused" }`, not `{ score: 0 }`.
