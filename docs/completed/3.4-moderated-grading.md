## 1. Problem Statement

Many higher-education institutions require that high-stakes assignments (capstone papers, written exams, dissertation chapters) be graded by two independent graders, with a third moderator resolving disagreements above a configurable threshold. Lextures today has no concept of multiple simultaneous graders on a single assignment, no provisional score workflow, and no reconciliation step. This prevents adoption by programs with inter-rater reliability requirements, external examiner workflows (UK/AU), or accreditation demands for standardised marking.

## 2. Goals

- Allow an assignment to have one or more provisional graders whose scores are hidden from each other (double-blind) until a final moderator reviews.
- Provide a reconciliation interface where the moderator sees all provisional scores and can accept one, average them, or enter a final override score.
- Integrate with blind grading (3.3) so grader identities are also hidden where required.
- Emit grade-change audit events (3.10) for every provisional and final score.
- Gate final grade visibility behind grade posting policy (3.8).

## 3. Non-Goals

- Peer grading / student-to-student grading (separate feature).
- Workflow automation beyond assignment-level configuration.
- External examiner portal (institutional SSO required first — 4.1/4.2).

## 4. Personas & User Stories

- **As a course coordinator**, I want to assign two TAs as independent graders on the final essay so that each scores submissions without seeing the other's marks.
- **As a moderator / instructor-of-record**, I want to see all provisional scores side by side and click "Accept grader 1's score" or type a final override so that I can resolve disagreements efficiently.
- **As a grader (TA)**, I want to see only my assigned submissions and not the other TA's scores so that my marking is not influenced by theirs.
- **As a student**, I want to receive only the final reconciled grade, not the provisional scores, so that the grading process remains fair.

## 5. Functional Requirements

- **FR-1.** The system MUST allow an assignment to be configured with a list of provisional graders (1–N users with grader role) and a single moderator.
- **FR-2.** Provisional graders MUST each see their own score entry for each submission and MUST NOT see other graders' provisional scores until the moderator reveals them.
- **FR-3.** The system MUST support a configurable agreement threshold (e.g., scores differ by > 15%); submissions outside this threshold MUST be flagged for mandatory moderator review.
- **FR-4.** The moderator MUST see a reconciliation dashboard showing all provisional scores per submission, flag status, and options: "Accept Grader A," "Accept Grader B," "Use Average," "Override."
- **FR-5.** The system MUST prevent the final grade from being posted (visible to students) until the moderator has reconciled all flagged submissions.
- **FR-6.** The system MUST record a `grade_change` audit event (3.10) for every provisional score and every final reconciliation decision.
- **FR-7.** The system SHOULD support double-blind mode where student identities are hidden from provisional graders (combining 3.3).
- **FR-8.** The system MAY notify the moderator when all provisional graders have finished their batch.

## 6. Non-Functional Requirements

- **Performance** — Reconciliation dashboard loads all provisional scores for an assignment in ≤ 1 s for ≤ 500 submissions. Score submission p95 ≤ 200 ms.
- **Security** — Provisional scores stored with a `grader_id`; route handler enforces that grader A cannot read grader B's rows. Moderator role enforced at `server/src/authz.rs`.
- **Privacy & Compliance** — All provisional and final scores are FERPA education records. Stored under the same retention policy as `course_grades`. Accessible in DSAR exports.
- **Accessibility** — Reconciliation dashboard table keyboard navigable; WCAG 2.1 AA.
- **Scalability** — Provisional scores table partitioned by `assignment_id`; supports N graders per submission (N ≤ 10 in practice).
- **Reliability** — Score submissions idempotent (upsert by `(submission_id, grader_id)`). Concurrent grader writes handled via DB upsert, no optimistic locking needed.
- **Observability** — Metrics: `moderated_grading.reconciliation_completed`, `moderated_grading.flagged_submissions_count`. Log grader_id on each provisional score event.
- **Maintainability** — New `moderated_grading` module in `server/src/services/`; routes in `server/src/routes/grading.rs`.
- **Internationalization** — All UI labels i18n-keyed.
- **Backward compatibility** — Non-moderated assignments behave exactly as today; new columns default to NULL.

## 7. Acceptance Criteria

- **AC-1.** *Given* two provisional graders assigned to assignment A, *when* Grader 1 submits a provisional score, *then* Grader 2's API call to list scores for that submission returns only Grader 2's entry, not Grader 1's.
- **AC-2.** *Given* Grader 1 scores 70 and Grader 2 scores 90 on the same submission with a 15% threshold, *when* the moderator opens the reconciliation dashboard, *then* that submission is flagged as "Needs review."
- **AC-3.** *Given* the moderator selects "Use Average" for a flagged submission, *when* the action is saved, *then* the final grade is set to 80 and an audit event is recorded.
- **AC-4.** *Given* moderated grading is configured and some submissions are unreconciled, *when* the instructor attempts to post grades, *then* the system blocks the action and shows a count of unreconciled items.
- **AC-5.** *Given* a student with no moderator or grader role, *when* they call the provisional scores API endpoint, *then* the server returns HTTP 403.
- **AC-6.** *Given* the moderator has reconciled all submissions, *when* grades are posted, *then* students see only the final reconciled score and not the individual provisional scores.

## 8. Data Model

```sql
-- Migration: server/migrations/076_moderated_grading.sql

ALTER TABLE module_assignments
    ADD COLUMN moderated_grading BOOLEAN NOT NULL DEFAULT FALSE,
    ADD COLUMN moderation_threshold_pct INT DEFAULT 15,
    ADD COLUMN moderator_id UUID REFERENCES users(id);

CREATE TABLE provisional_grades (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    submission_id   UUID NOT NULL REFERENCES module_assignments(id) ON DELETE CASCADE,
    grader_id       UUID NOT NULL REFERENCES users(id),
    score           NUMERIC(8,2),
    rubric_data     JSONB,
    submitted_at    TIMESTAMPTZ,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (submission_id, grader_id)
);

ALTER TABLE course_grades
    ADD COLUMN reconciliation_source TEXT
        CHECK (reconciliation_source IN ('grader_a','grader_b','average','override','single')),
    ADD COLUMN reconciled_by UUID REFERENCES users(id),
    ADD COLUMN reconciled_at TIMESTAMPTZ;

CREATE INDEX idx_provisional_submission ON provisional_grades(submission_id);
```

| POST | `/api/courses/:cid/assignments/:aid/submissions/:sid/reconcile` | moderator | Reconcile final grade |

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Grader accidentally sees other grader's score via API | M | H | Server-side filter by grader_id; integration test the isolation |
| Moderator misses flagged items and posts anyway | M | H | Block posting until reconciliation complete; counter badge in UI |
| Complex RBAC when same user is both grader and moderator | L | M | Validate at assignment save: moderator_id NOT IN graders list |

## 15. Rollout Plan

- Feature flag: `MODERATED_GRADING_ENABLED` (default: `false`).
- Phase 1: schema + API; Phase 2: grader panel changes; Phase 3: moderation dashboard.
- Pilot: one department running external examiner workflow.
- GA criteria: zero cross-grader score leaks in security test; reconciliation flow tested end-to-end.
- Rollback: disable flag; provisional_grades table retained.

## 16. Test Plan

- **Unit** — Score isolation logic, threshold flag computation, reconciliation source derivation.
- **Integration** — Grader isolation: grader A cannot read grader B rows. Moderator can read all. Reconcile updates `course_grades`.
- **End-to-end** — Playwright: two grader sessions → score differently → moderator reconciles → student sees final only.
- **Security** — Authz matrix: student, unassigned user, wrong-course grader all get 403.
- **Accessibility** — axe-core on reconciliation dashboard; keyboard navigation.
- **Manual exploratory** — Test threshold edge cases; test single-grader moderated assignment.
