## 1. Problem Statement

Standards-based grading (SBG) replaces aggregate point totals with proficiency scores on discrete academic standards (e.g., "CCSS.MATH.3.NBT.A.2 — Fluently add within 1000"). Many K-12 districts — and a growing number of competency-based higher-education programs — have adopted SBG, making it a hard requirement for K-12 institutional sales. Lextures has a `course_learning_outcomes` table (migration 072) but no mechanism to link assignments or rubric criteria to specific standards, aggregate proficiency levels, or generate mastery transcripts. Without SBG, Lextures cannot enter the K-12 district market where most adoption decisions are driven by principals and curriculum directors who require standards reporting.

## 2. Goals

- Allow a course to import or define academic standards (from Common Core, NGSS, state-specific sets, or custom).
- Link assignment rubric criteria and quiz questions to one or more standards.
- Compute per-student, per-standard proficiency levels using a configurable scoring algorithm.
- Display standards mastery in a student-facing proficiency view and instructor-facing standards gradebook.
- Generate a mastery transcript (PDF or structured data) showing each student's proficiency per standard.

## 3. Non-Goals

- Building a full national-standards database (import from CSV or well-known standard APIs).
- LTI-based standards passback to district SIS (deferred to 14.1 SIS integration).
- Adaptive path branching based on standards mastery (deferred to 1.4).
- Common Core or NGSS licensing / redistribution — instructors import their own standard sets.

## 4. Personas & User Stories

- **As a K-12 instructor**, I want to tag each quiz question and rubric criterion with a Common Core standard so that the system tracks proficiency per standard automatically.
- **As a district curriculum director**, I want to see a class-level heatmap of standards mastery so that I know which standards need reteaching.
- **As a student**, I want to see a list of the standards I am "Meeting," "Approaching," or "Not yet meeting" so that I know what to focus on.
- **As a parent**, I want to see my child's mastery transcript at the end of the term so that I understand their progress beyond letter grades.
- **As a competency-based HE program coordinator**, I want assignments aligned to program competencies so that accreditation reports can be auto-generated from gradebook data.

## 5. Functional Requirements

- **FR-1.** The system MUST allow importing a set of academic standards into a course from a CSV file with columns (standard_id, description, subject, grade_level).
- **FR-2.** The system MUST allow rubric criteria and quiz question types to be tagged with one or more standards.
- **FR-3.** The system MUST define a configurable proficiency scale per course (e.g., 4=Exceeds, 3=Meets, 2=Approaching, 1=Not Yet, or percentage bands).
- **FR-4.** The system MUST compute each student's current proficiency level per standard by aggregating linked scores, using the instructor's chosen aggregation rule (most-recent, highest, mean, decaying average).
- **FR-5.** The system MUST provide a standards-based gradebook view distinct from the traditional points-based gradebook, showing standards as columns and students as rows.
- **FR-6.** The system MUST generate a printable/PDF mastery transcript per student listing each standard and its current proficiency level.
- **FR-7.** The system SHOULD display a per-student, per-standard proficiency timeline (how proficiency changed over multiple assessments).
- **FR-8.** The system SHOULD export class-level standards mastery as CSV or JSON for import into district SIS or reporting tools.

## 6. Non-Functional Requirements

- **Performance** — Proficiency computation for a class of 30 students × 50 standards completes in ≤ 2 s (batch job on grade submission, not on-demand recalculation). Standards gradebook loads ≤ 1 s.
- **Security** — Standards data accessible to enrolled users; proficiency records scoped by `course_id` and guarded by `server/src/authz.rs`. Mastery transcript download restricted to instructor, student-self, and parent/guardian.
- **Privacy & Compliance** — Per-standard proficiency records are FERPA education records. Mastery transcripts included in DSAR exports. COPPA: no K-12 minor's transcript sent outside platform without parental consent.
- **Accessibility** — Standards gradebook keyboard navigable; heatmap colour coding supplemented with text labels (WCAG 2.1 AA 1.4.1 Use of Color).
- **Scalability** — Aggregation runs as a background job after each grade submission; results cached in `student_standard_proficiencies` table. Expect 200 standards × 150 students per course.
- **Reliability** — Proficiency aggregation job is idempotent; safe to re-run. Failed jobs retried via 17.3 job queue.
- **Observability** — Log `standard_proficiency_computed` events. Metric: proficiency_job_duration_seconds.
- **Maintainability** — Aggregation logic in `server/src/services/grading/standards.rs`; strategy pattern for aggregation rules.
- **Internationalization** — Proficiency label names configurable per course; i18n keys for default labels.
- **Backward compatibility** — Non-SBG courses completely unaffected. `course_learning_outcomes` (072) retained and extended rather than replaced.

## 7. Acceptance Criteria

- **AC-1.** *Given* a course with standards imported and a rubric criterion tagged to standard S1, *when* a student receives a score of 4/4 on that criterion, *then* the student's proficiency for S1 updates to "Exceeds."
- **AC-2.** *Given* a student has three assessments linked to standard S2 with scores [2, 3, 3], *when* the aggregation rule is "most-recent," *then* proficiency = 3 (Meets).
- **AC-3.** *Given* the standards gradebook view, *when* the instructor opens it, *then* each cell shows the proficiency label (not a raw score) for the corresponding student-standard pair.
- **AC-4.** *Given* the instructor clicks "Generate mastery transcript" for a student, *when* the PDF is downloaded, *then* it lists all standards in the course, each student's current proficiency level, and the course/term name.
- **AC-5.** *Given* a standards CSV import with 200 rows, *when* the import completes, *then* all 200 standards appear in the course standards list within 5 s.
- **AC-6.** *Given* a non-SBG course, *when* the instructor opens the gradebook, *then* the traditional points view is shown unchanged and no SBG controls are present.

## 8. Data Model

```sql
-- Migration: server/migrations/079_standards_based_grading.sql

CREATE TABLE course_standards (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    course_id       UUID NOT NULL REFERENCES courses(id) ON DELETE CASCADE,
    external_id     TEXT,             -- e.g. "CCSS.MATH.3.NBT.A.2"
    description     TEXT NOT NULL,
    subject         TEXT,
    grade_level     TEXT,
    position        INT NOT NULL DEFAULT 0,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE standard_alignments (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    standard_id     UUID NOT NULL REFERENCES course_standards(id) ON DELETE CASCADE,
    alignable_type  TEXT NOT NULL CHECK (alignable_type IN ('rubric_criterion','quiz_question')),
    alignable_id    UUID NOT NULL,
    weight          NUMERIC(4,2) NOT NULL DEFAULT 1.0
);

CREATE TABLE student_standard_proficiencies (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    course_id       UUID NOT NULL REFERENCES courses(id),
    student_id      UUID NOT NULL REFERENCES users(id),
    standard_id     UUID NOT NULL REFERENCES course_standards(id),
    proficiency     NUMERIC(4,2),
    level_label     TEXT,
    last_assessed   TIMESTAMPTZ,
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (course_id, student_id, standard_id)
);

ALTER TABLE courses
    ADD COLUMN sbg_enabled BOOLEAN NOT NULL DEFAULT FALSE,
    ADD COLUMN sbg_proficiency_scale_json JSONB;
-- scale_json: [{level: 4, label: "Exceeds", min_score: 3.5}, ...]
-- aggregation_rule stored in course settings JSON
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |

| Standards import CSV format varies by district | H | M | Provide template CSV; document required column schema |
| Proficiency aggregation performance with large cohorts | M | M | Background job with indexed lookup; batch updates |
