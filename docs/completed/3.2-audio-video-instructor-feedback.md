## 1. Problem Statement

Instructors today can only return typed comments on assignments. Audio and video feedback — shown by research to significantly increase student engagement and comprehension of feedback — is entirely absent. Competitors like Canvas Studio and Turnitin allow instructors to record a short voice or screen-capture comment directly in the grader. Without this, Lextures loses high-touch courses (language learning, performance arts, studio courses) and instructors who prefer conversational feedback over written comments.

## 2. Goals

- Allow instructors to record an audio clip (microphone) or video clip (camera or screen + camera) directly in the grader panel and attach it to a submission as feedback.
- Store recorded media in object storage (8.1) and link to the submission grade record.
- Allow instructors to upload a pre-recorded file as an alternative to live capture.
- Students can play feedback media inline when they view the returned submission.
- Auto-caption audio/video feedback for accessibility (ADA, WCAG 2.1 AA).

## 3. Non-Goals

- Student audio/video submission types (separate from feedback — that is a question-type feature in 2.2).
- Live real-time video sessions with students (deferred to 6.4 Synchronous video / virtual classroom).
- Video editing beyond trim start/end.
- Transcription storage or search of feedback content.

## 4. Personas & User Stories

- **As an instructor**, I want to click "Record voice feedback," speak for 60 seconds, and have that recording attached to the student's submission so that my nuanced verbal comments are preserved.
- **As an instructor teaching a language course**, I want to record a video showing my face while commenting on a student's pronunciation essay so that students receive human, personalised feedback.
- **As a student**, I want to open my graded assignment and see a "Play feedback" button so that I can listen to my instructor's comments without downloading a file.
- **As a student with hearing impairment**, I want auto-generated captions on the instructor's video feedback so that I can read the content in full compliance with my accommodations.

## 5. Functional Requirements

- **FR-1.** The system MUST provide an in-browser recorder widget (audio-only and video+audio modes) using the browser MediaRecorder API.
- **FR-2.** The system MUST allow instructors to upload a pre-recorded file (MP3, MP4, MOV, WEBM, M4A) as an alternative to live capture.
- **FR-3.** The system MUST store recorded/uploaded feedback media in object storage (8.1), keyed by `(submission_id, feedback_media_id)`.
- **FR-4.** The system MUST allow a maximum recording duration of 10 minutes; the UI MUST display a countdown timer.
- **FR-5.** The system MUST link media feedback records to the submission's grade record and expose them via the submissions API.
- **FR-6.** The system MUST allow playback in-browser via an HTML5 `<video>` / `<audio>` element with seek, pause, and volume controls.
- **FR-7.** The system SHOULD trigger auto-captioning via an AI speech-to-text service (Whisper or equivalent) as a background job after upload.
- **FR-8.** The system SHOULD allow the instructor to trim the recording (set start/end trim points) before saving.
- **FR-9.** The system MAY notify the student via the notification system when media feedback is posted.

## 6. Non-Functional Requirements

- **Performance** — Media upload MUST use resumable/chunked upload (8.2); playback start latency ≤ 2 s via pre-signed CDN URL. Captioning job completes within 2× the recording duration.
- **Security** — Media files served via time-limited pre-signed object storage URLs (15 min TTL); direct URL enumeration not possible. Upload endpoint validates MIME type and file extension before storage. Max upload size 500 MB.
- **Privacy & Compliance** — Instructor voice/video recordings are associated with a student's educational record under FERPA. Captioning processing MUST route through a PII-aware pipeline (see 19.11). Recordings deleted when submission is deleted.
- **Accessibility** — Player controls keyboard-accessible (Space=play/pause, arrow keys=seek, M=mute). Auto-captions displayed as `<track kind="captions">`. Caption edit UI for instructor correction. WCAG 2.1 AA.
- **Scalability** — Async captioning via job queue (17.3); no synchronous AI calls on the upload path. Expect up to 1 k new media objects per day at scale.
- **Reliability** — Upload uses multipart with server-side reassembly; partial uploads resumable within 24 h. Caption job retried up to 3× on failure.
- **Observability** — Metrics: `feedback_media.uploaded_count`, `feedback_media.caption_job_duration_seconds`. Log `submission_id`, `instructor_id`, media duration on each upload.
- **Maintainability** — Media feedback service isolated in `server/src/services/feedback_media.rs`; routes in `server/src/routes/feedback_media.rs`.
- **Internationalization** — Auto-captions stored per locale; UI strings externalised.
- **Backward compatibility** — Submissions without media feedback render unchanged.

## 7. Acceptance Criteria

- **AC-1.** *Given* an instructor is in the grader panel, *when* they click "Record audio," grant microphone permission, speak for 30 s, and click "Stop & Save," *then* the audio appears as an inline player on the submission within 5 s.
- **AC-2.** *Given* a saved audio feedback recording, *when* the student opens the returned submission, *then* an inline audio player is visible and plays the correct recording.
- **AC-3.** *Given* a 2-minute video feedback recording, *when* the background caption job completes, *then* a caption track appears on the player and accurately transcribes at least 90% of words.
- **AC-4.** *Given* an instructor uploads a 200 MB MP4 file, *when* the upload is interrupted at 50%, *then* resuming the upload continues from the 50% checkpoint without re-uploading previous bytes.
- **AC-5.** *Given* a student without enrollment in the course, *when* they attempt to fetch the pre-signed media URL, *then* the server returns HTTP 403 and the object storage URL remains unexposed.
- **AC-6.** *Given* a recording exceeds 10 minutes, *when* the timer reaches 10:00, *then* recording automatically stops and the instructor is shown a save prompt.

## 8. Data Model

```sql
-- Migration: server/migrations/074_submission_feedback_media.sql

CREATE TABLE submission_feedback_media (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    submission_id   UUID NOT NULL REFERENCES module_assignments(id) ON DELETE CASCADE,
    uploader_id     UUID NOT NULL REFERENCES users(id),
    media_type      TEXT NOT NULL CHECK (media_type IN ('audio', 'video')),
    storage_key     TEXT NOT NULL,          -- object storage key
    duration_secs   INT,
    caption_status  TEXT NOT NULL DEFAULT 'pending'
                    CHECK (caption_status IN ('pending','processing','done','failed')),
    caption_key     TEXT,                   -- storage key for VTT file
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ
);

CREATE INDEX idx_feedback_media_submission ON submission_feedback_media(submission_id)
    WHERE deleted_at IS NULL;
```

## 9. API Surface

| Verb | Path | Auth | Description |
|---|---|---|---|
| POST | `/api/courses/:cid/assignments/:aid/submissions/:sid/feedback-media/initiate` | instructor | Initiate multipart upload, returns upload URL |
| POST | `/api/courses/:cid/assignments/:aid/submissions/:sid/feedback-media/complete` | instructor | Finalise multipart upload |
| GET | `/api/courses/:cid/assignments/:aid/submissions/:sid/feedback-media` | instructor/student-self | List media feedback items |
| GET | `/api/courses/:cid/assignments/:aid/submissions/:sid/feedback-media/:mid/url` | instructor/student-self | Get pre-signed playback URL |
| DELETE | `/api/courses/:cid/assignments/:aid/submissions/:sid/feedback-media/:mid` | uploader or instructor | Delete media feedback |

Rate limit: 10 upload initiations per minute per user.

Key flows:
1. Instructor clicks "Add Media Feedback" → modal opens with two tabs: "Record" and "Upload."
2. Record tab: click microphone/camera icon → browser permission prompt → recording starts → waveform visualizer shown → stop → trim handles shown → save.
3. Upload tab: drag-and-drop or file picker → progress bar (chunked) → save.
4. Student submission view: "Feedback" section shows text comments and media players side-by-side.

ARIA: `role="timer"` on countdown; `aria-live="polite"` on upload progress.

## 11. AI / ML Considerations

- **Model**: OpenAI Whisper (via OpenRouter or direct) for captioning.
- **Prompt**: transcription only, no summarisation.
- **PII redaction**: captions processed through redaction layer (19.11) before storage.
- **Fallback**: if captioning fails after 3 retries, caption_status = 'failed'; instructor shown "Add captions manually" prompt.
- **Cost budget**: ~$0.006 / minute of audio; cap at 10 min = ~$0.06 per recording.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Browser MediaRecorder API inconsistencies (Safari) | H | M | Polyfill with RecordRTC; test on Safari 17+ |
| Caption accuracy low for non-English instructors | M | M | Expose manual caption edit; set accuracy expectation in docs |
| Large video files cause slow upload experience | M | H | Enforce 10-min limit; recommend compression; chunked upload with progress |
| Whisper API cost spike | L | M | Per-course and per-tenant token budget (see 19.14) |

## 15. Rollout Plan

- Feature flag: `FEEDBACK_MEDIA_ENABLED` (default: `false`).
- Phase 1: storage schema + upload/playback API, no recorder UI.
- Phase 2: recorder widget + upload UI behind flag.
- Phase 3: captioning job integration.
- GA criteria: 50 recordings in pilot with no data loss, caption accuracy ≥ 85% on English, WCAG caption test pass.
- Rollback: disable flag; DB rows and storage objects retained.

## 16. Test Plan

- **Unit** — upload initiation/completion service, captioning job retry logic, authz checks.
- **Integration** — multipart upload flow with S3-compatible mock; caption job enqueue/dequeue.
- **End-to-end** — Playwright (mocked MediaRecorder): record → save → student view → play.
- **Security** — Verify pre-signed URL expiry; verify MIME type rejection for disallowed types.
- **Accessibility** — axe-core on player; keyboard play/pause/seek; VTT captions display in browser player.
- **Performance / load** — k6: 20 concurrent uploads of 100 MB files; monitor S3 multipart throughput.
